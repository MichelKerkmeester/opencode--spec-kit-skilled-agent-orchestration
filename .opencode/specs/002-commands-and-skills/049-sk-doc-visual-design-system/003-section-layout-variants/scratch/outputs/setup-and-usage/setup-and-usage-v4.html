07 // SETUP AND USAGE

Failure-Aware Cookbook

Execute initialization and usage commands chronologically. Each operational step is paired with an immediate diagnosis matrix to help you resolve environment and configuration failures without breaking flow.

01

Bootstrap Storage & Context

Create the isolated workspace directory and baseline configuration payload. This ensures state is retained between workflow executions.

bash 
mkdir -p .opencode/memory
touch .opencode/config.json
echo '{"schema": "v2", "strict": true}' > .opencode/config.json

Diagnosis Matrix

  • EACCES_DENIED Lacking write permissions in the current directory. Elevate permissions or run chmod +w .
  • WARN_OVERWRITE If config.json already exists, using > will overwrite custom rules. Use >> to append safely.
02

Initialize Provider Connection

Authenticate the CLI tool against the preferred backend runtime. This step maps your environment variables to the active profile.

bash 
export PROVIDER_API_KEY="your_secure_token"
tool run setup:provider --verify --profile default

Diagnosis Matrix

  • ERR_UNAUTHORIZED API key rejected. Ensure the token is structurally valid and the session hasn't expired.
  • ERR_TIMEOUT Handshake failed. Check if an active VPN or strict proxy is blocking outbound TCP traffic.
03

Execute Primary Workflow

With the baseline active and the provider verified, initiate a strict workflow plan. Ensure your target directory matches the expected operational scope.

bash 
tool run workflow:plan "update UI routing" --strict

Diagnosis Matrix

  • VAL_SCHEMA_FAIL Strict mode rejected the plan due to malformed targets. Check --help for required arguments.
  • STATE_LOCKED A prior run crashed. Delete the hidden lockfile inside .opencode/memory/.lock and try again.
