07 // Setup and Usage

Mode Matrix Setup

Choose a runtime mode first, then apply the matching bootstrap, validation, execution, and fallback steps. The matrix keeps setup decisions explicit so teams can onboard without guessing environment behavior.

Mode Bootstrap Validation Gate Execution Pattern Fallback Rule

Local Sandbox

Single developer, rapid feedback.

 Required

Create local profile and cache path

tool setup:init --profile local --storage ./.cache/context

 Default

Run non-blocking checks

tool setup:verify --strict false

 Required

Plan then apply in same session

tool workflow:plan "change" && tool workflow:apply

 Default

Persist context snapshot only

tool workflow:save --mode local

CI Pipeline

Headless verification and artifact checks.

 Required

Inject token and enable non-interactive mode

tool setup:init --profile ci --no-prompt

 Required

Fail on warnings

tool setup:verify --strict true --fail-on-warning

 Default

Run plan-only workflow for review

tool workflow:plan "change" --ci --emit-report

 Required

Publish logs and abort merge

tool workflow:save --report ci-artifacts.json

Hardened Runtime

Audited operations and controlled rollout.

 Required

Load signed profile and locked storage

tool setup:init --profile hardened --storage /var/lib/toolctx

 Required

Run immutable policy checks

tool setup:verify --policy immutable --strict true

 Required

Plan, approve, then apply

tool workflow:plan "change" --approval required

 Required

Rollback to signed snapshot

tool workflow:rollback --snapshot latest-signed

How to use this matrix

Read each row left to right and execute commands in sequence. Do not mix columns across different rows.

Audit signal

Capture the exact mode, validation gate, and fallback command in tickets so incidents remain reproducible.

Safe default

If mode is unknown, use Local Sandbox first, validate output, then promote to CI with strict checks.
