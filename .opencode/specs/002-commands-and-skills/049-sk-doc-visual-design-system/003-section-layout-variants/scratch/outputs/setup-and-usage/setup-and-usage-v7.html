07 // Setup and Usage

Progressive Sandbox Flow

Move from a safe sandbox to production readiness through explicit promotion gates. Each phase has one command path and one acceptance signal to keep rollout predictable.

Phase 01

Create isolated workspace

Start with disposable storage and local credentials.

Gate

Workspace can be created and reset without side effects.

tool setup:init --profile sandbox --storage ./sandbox/context

Phase 02

Run guided validation

Verify config, schema, and required dependencies.

Gate

Validation output returns no errors and known warnings only.

tool setup:verify --profile sandbox --strict true

Phase 03

Simulate real workflow

Execute plan, apply, and persistence against test payloads.

Gate

Generated artifacts pass review and can be reproduced twice.

tool workflow:plan "sandbox-run" && tool workflow:apply && tool workflow:save

Phase 04

Promote to controlled mode

Switch profile and enforce stricter runtime checks.

Gate

Promotion checklist approved by reviewer and CI baseline is green.

tool setup:promote --from sandbox --to ci --require-review
