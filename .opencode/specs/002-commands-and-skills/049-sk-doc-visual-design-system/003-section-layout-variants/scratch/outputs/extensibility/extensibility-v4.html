Section 08 // Extensibility

Extension Model

The system architecture supports controlled adaptation. By separating conceptual intent from implementation details, you can extend capabilities predictably without compromising core platform stability.

Design Philosophy

Convention Over Configuration

Avoid reinventing structural paradigms. Pre-built templates provide the necessary scaffolding so you can focus entirely on mapping custom logic and domain-specific content.

Isolated Capabilities

Extensions must not pollute the global namespace. Wrap new tools, agents, and behaviors in bounded contexts (Skills) that are explicitly invoked when required by the orchestrator.
