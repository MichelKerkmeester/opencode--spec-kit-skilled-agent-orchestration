Contextual Command Flow

Commands are grouped by execution context instead of by chronology only. This keeps environment setup, source acquisition, and runtime verification distinct for faster troubleshooting.

Workspace context Lane 01

Validate tooling

Confirm command surface availability before any clone action.

opencode --version
then

Validate runtime

Ensure package scripts execute on supported runtime versions.

node --version
then

Validate package manager

Keep lockfile resolution behavior consistent across contributors.

npm --version
Repository context Lane 02

Acquire source

Clone canonical repository to preserve onboarding parity.

git clone https://github.com/example/repo.git
then

Enter project root

Scope all subsequent commands to the repository context.

cd repo
then

Resolve dependencies

Install required packages and capture install output signals.

npm install
Runtime context Lane 03

Initialize app process

Start local service using project-standard runtime script.

npm run dev
verify

Confirm endpoint

Check for local URL and successful module compilation logs.

http://localhost:3000
handoff

Proceed to feature docs

Continue with section-level implementation walkthroughs.

open docs/README.md
