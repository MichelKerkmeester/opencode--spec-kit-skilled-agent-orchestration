Initialization Phase 2 Hooks
onPluginInit
Fires immediately after the plugin is registered in memory.
context, config
void | Promise<void>

Implementation Example

export default {
  name: 'my-custom-plugin',
  onPluginInit: async (context, config) => {
    // Setup initial state or connect to external DB
    console.log(`Initializing with config: ${config.apiKey}`);
    await db.connect(config.dbUrl);
  }
}
beforeServerStart
Fires just before the HTTP server binds to the port.

void

Implementation Example

export default {
  beforeServerStart: (app) => {
    // Inject custom Express middleware globally
    app.use(myCustomAuthMiddleware);
  }
}
Runtime / Data Phase 4 Hooks
beforeRecordSave
Intercepts DB save operations. Can mutate data or throw to abort.
collection, record
Record

Implementation Example

export default {
  beforeRecordSave: (collection, record) => {
    if (collection === 'users') {
      // Mutate record before it hits the database
      record.email = record.email.toLowerCase();
    }
    return record; // Must return the mutated record
  }
}
afterRecordCreate
Fires after a new record is successfully persisted.
collection, record
void

Implementation Example

export default {
  afterRecordCreate: async (collection, record) => {
    if (collection === 'orders') {
      // Trigger side effect (e.g. email)
      await mailer.sendReceipt(record.userId, record.id);
    }
  }
}
