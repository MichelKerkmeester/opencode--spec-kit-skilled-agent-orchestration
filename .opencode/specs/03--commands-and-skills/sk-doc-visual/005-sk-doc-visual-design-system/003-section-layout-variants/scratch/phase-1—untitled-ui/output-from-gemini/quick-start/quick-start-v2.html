Get started in seconds

Select your primary development environment below to see tailored installation instructions and prerequisites.

macOS Recommended
System Prerequisites
  • macOS 12.0 (Monterey) or later
  • Homebrew package manager
  • Xcode Command Line Tools
1

Add the Homebrew tap

First, add our official tap to your local Homebrew installation. This tells your system where to find our signed binaries.

$brew tap untitled/cli
2

Install the CLI

Run the install command to download the latest stable release. Homebrew will automatically handle dependencies.

$brew install untitled-cli
3

Verify installation

Check that the CLI was installed correctly and is accessible from your path.

$untitled --version
Untitled CLI version 2.4.0 (darwin-arm64)
Running on Apple Silicon? Native ARM64 is supported automatically. Read macOS advanced docs
Linux
System Prerequisites
  • Debian/Ubuntu or RHEL based distro
  • glibc 2.28 or higher
  • cURL installed
1

Run the installation script

Download and execute our bash script which automatically detects your architecture and installs the correct binary.

$curl -fsSL https://untitled.ui/install.sh | bash
2

Add to your PATH

Ensure the installation directory is in your shell's PATH. Add this to your ~/.bashrc or ~/.zshrc.

$export PATH=$PATH:~/.untitled/bin
Supports AMD64 and ARM64 architectures. View manual APT/YUM instructions
Windows via PowerShell
System Prerequisites
  • Windows 10 / 11 (64-bit)
  • PowerShell 5.1 or later
  • Administrator execution policy
1

Enable script execution

Open PowerShell as Administrator and allow local scripts to execute to enable the installer.

PS>Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
2

Install module

Pull the module directly from the PowerShell Gallery.

PS>Install-Module -Name UntitledCLI -Force
WSL2 users should follow the Linux instructions. Read Windows advanced docs
Docker Isolated
System Prerequisites
  • Docker Engine 20.10+
  • Docker Compose v2
1

Pull the latest image

Grab the minimal Alpine-based container from Docker Hub. Image size is ~45MB.

$docker pull untitled/cli:latest
2

Run with volume mounts

To allow the CLI to modify your local project files, mount your current working directory when running the container.

$docker run -v $(pwd):/app -w /app untitled/cli init
Ideal for CI/CD pipelines and sandboxed environments. View Docker Compose examples
