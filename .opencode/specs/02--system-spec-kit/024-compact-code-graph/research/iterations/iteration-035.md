## Novelty Justification

This iteration did more than restate the phase specs. It produced a cross-phase readiness scorecard, separated confirmed dependencies from assumed ones, quantified total and MVP LOC, identified the actual minimum viable critical path, and verified live repo evidence showing which prerequisites already exist versus which hook/runtime surfaces are still missing. The most important new synthesis is the packet-identity mismatch: `research/research.md` is still a code-graph study, while the packet spec/plan/phases now define a hook-based context-preservation system.

## Recommendations for Implementation

1. Reconcile the packet before coding: decide the authoritative scope for `024-compact-code-graph` and either retitle/replace `research/research.md` or explicitly mark it as superseded background research.
2. Merge Phases 1 and 2 into one implementation PR/phase group with a single authoritative contract for file layout, cache state, SessionStart routing, and compiled output paths.
3. Ship the `/spec_kit:resume` `profile:"resume"` fix immediately as a low-risk precursor or first commit in the main implementation branch.
4. Keep Phase 4 v1 narrow: runtime detection plus instruction-file fallback only. Defer MCP-level time-gap compaction detection until after the base hook path is proven.
5. Treat Phase 3 as release-hardening, not MVP. Implement it after compaction/startup recovery is working.
6. Defer the broad Phase 5 agent sweep and the Phase 6 doc sweep until the actual runtime contract has landed, otherwise those edits will churn.
7. Start Phase 7 with a slim regression suite around compact/startup/resume behavior, then expand to the full 7-scenario matrix once the runtime contract and Stop-hook schema are stable.
