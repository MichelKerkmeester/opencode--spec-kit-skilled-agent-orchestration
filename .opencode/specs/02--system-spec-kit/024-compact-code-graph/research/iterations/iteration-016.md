8. Error handling is intentionally degrade-not-fail. ## Novelty Justification
Most of the high-value detail here is implementation-specific rather than obvious from the hook name: the non-exported result type, newest-first constitutional ordering, triggered-first truncation policy, the distinction between hook-local 4000-token enforcement and later envelope-level response budgeting, the fact that compaction uses `matchTriggerPhrases(..., 5)` without signal boosts, and the non-fatal degradation behavior when DB/cache startup is not ready.

## Recommendations for Implementation
1. Treat `autoSurfaceAtCompaction()` as a thin public hook and `autoSurfaceMemories()` as the reusable core; if Claude hooks need parity across compaction and other lifecycle events, target the core shape, not just the wrapper.
2. Export `AutoSurfaceResult` and `ConstitutionalMemory` from a shared public type surface before wiring external hook callers, otherwise consumers will have to duplicate the contract by hand.
3. Preserve the existing truncation semantics if you reimplement externally: drop triggered matches first, then oldest constitutional entries, and return `null` rather than a malformed oversized payload.
4. If Claude hook consumers need stronger trigger relevance, note that the current compaction path does not use `matchTriggerPhrasesWithStats()` or signal boosts; adopting the exact existing behavior means sticking with plain `matchTriggerPhrases(contextHint, 5)`.
5. Plan for graceful degradation: when the server is up but memory backing services are not ready, the effective contract is “continue without surfaced context,” not “fail the request.”
