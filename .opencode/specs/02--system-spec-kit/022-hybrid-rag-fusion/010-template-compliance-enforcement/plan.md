## 1. SUMMARY

### Technical Context

| Aspect | Value |
|--------|-------|
| **Language/Stack** | Markdown (agent definitions), Shell (pre-commit hook), YAML (config), TOML (Codex agent) |
| **Framework** | system-spec-kit validation pipeline (`validate.sh`, `template-structure.js`) |
| **Storage** | File-based (agent definitions, reference file, git hooks) |
| **Testing** | Manual validation via `validate.sh --strict`; pre-commit hook trigger test |

### Overview
This plan implements a 3-layer defense-in-depth architecture to enforce structural template compliance for all spec folder documents generated by AI agents. Layer 1 injects compact structural contracts into all @speckit agent definitions so agents know the exact required headers and anchors before writing. Layer 2 replaces 3 conflicting validation timing directives with a single unambiguous "validate after each write" protocol. Layer 3 installs an already-written pre-commit hook that hard-blocks non-compliant commits. The approach targets the highest-impact, lowest-risk changes first (plan.md scaffold injection addresses 42.9% of all failures).

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE

### Pattern
3-Layer Defense-in-Depth (Prompt Injection + Validation Loop + Commit Gate)

### Key Components
- **Shared Reference File** (references/template-compliance-contract.md): Canonical single-source-of-truth containing full structural contracts for Levels 1-3, phase addenda rules, and sync protocol. Approximately 140 lines.
- **Compact Inline Contract** (49 lines in each @speckit agent def): Anchor-to-H2 mapping tables for all 5 Level 2 document types plus Level 3 decision-record.md. Embedded directly in agent prompts to eliminate runtime dependency on skill folder loading.
- **Post-Write Validation Protocol** (new section in each agent def): Single directive replacing 3 conflicting timing instructions. Includes exit code parsing, fix loop (max 3 attempts), and stop conditions.
- **Pre-Commit Hook** (`pre-commit-spec-validate.sh`): Existing 6-rule fast validation subset. Configured via `.speckit-enforce.yaml` with progressive rollout (`created_after` date filtering).

### Data Flow
```
Agent receives task -> Reads inline contract (Layer 1: headers + anchors per doc type)
    -> Writes .md file following contract structure
    -> Runs validate.sh --strict (Layer 2: post-write check)
    -> Fixes errors if any (max 3 attempts per file)
    -> Proceeds to next file
    -> git add + git commit
    -> Pre-commit hook fires (Layer 3: 6-rule fast check)
    -> Commit accepted or blocked
```
