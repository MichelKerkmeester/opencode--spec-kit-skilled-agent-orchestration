### 2. Verify "89 Flags" Claim
**The count of 89 is inaccurate.** The claim likely stems from a naive regex search (e.g., counting instances of the string `SPECKIT_`) across the entire repository.
- There are exactly **22 boolean flags** exported in `search-flags.ts`.
- There are roughly **35 valid boolean process.env toggles** active in source code outside of `search-flags.ts`.
- The remaining ~15 matches are configuration values (limits, models), mock flags exclusive to `*.vitest.ts` files, dead flags, and text markers (like `<!-- SPECKIT_LEVEL: N -->`). 
The true count of actionable boolean feature flags is **around 57**.

### 3. `isFeatureEnabled` Divergence Check

There is massive architectural divergence regarding how feature flags are evaluated.

**1. Does rollout-policy.ts share implementation with search-flags.ts?**
`search-flags.ts` imports `isFeatureEnabled` from `lib/cache/cognitive/rollout-policy.ts`. However, this file is an identical duplicate of `lib/cognitive/rollout-policy.ts`. Furthermore, ~60% of the codebase bypasses `search-flags.ts` entirely and evaluates `process.env` directly inline.

**2. What happens with `isFeatureEnabled("1")`?**
It evaluates to **FALSE (disabled)**. 
The logic in `rollout-policy.ts` explicitly asserts:
```typescript
const flagEnabled = rawFlag === undefined || rawFlag.trim().length === 0 || rawFlag === 'true';
```
Since `"1"` is neither undefined, empty, nor `"true"`, `flagEnabled` becomes `false`. 

**3. Are there inconsistencies between the two files / approaches?**
**Yes, severe logical inconsistencies exist.**
Because `isFeatureEnabled` strictly requires `'true'` or `undefined` to evaluate as enabled, it will reject `"1"`, `"yes"`, or `"TRUE"`. Meanwhile, dozens of files in the codebase (like `lib/scoring/mpab-aggregation.ts`) check for graduation using `process.env.SPECKIT_DOCSCORE_AGGREGATION?.toLowerCase() !== 'false'`.
- Using direct `!== 'false'` means `"1"` evaluates to **TRUE**.
- Using `search-flags.ts` (which delegates to `isFeatureEnabled`) means `"1"` evaluates to **FALSE**.
*(Note: Interestingly, `isDocscoreAggregationEnabled` is exported in `search-flags.ts`, but `mpab-aggregation.ts` directly evaluates `process.env` anyway, ignoring the centralized flag function!)*
