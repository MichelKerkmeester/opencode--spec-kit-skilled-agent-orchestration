# TASK B — Reindex API Design

## 1. Current State

`scripts/memory/reindex-embeddings.ts` (132 LOC) already imports from the stable `api/indexing.ts` facade — **not** deep internals. This was addressed in a prior refactor. It uses:

```typescript
import { initializeIndexingRuntime, warmEmbeddingModel, runMemoryIndexScan, closeIndexingRuntime }
  from '@spec-kit/mcp-server/api/indexing';
```

However, the **lifecycle orchestration** (init → warm → scan → close + error handling) is still bespoke in the script. Every consumer must know the 4-step bootstrap sequence. A `reindex()` API wraps this into a single call.

### 3. Proposed Module Code (~45 LOC)

```typescript
// mcp_server/api/reindex.ts
// @public — Single-call reindex API for scripts and CLI tools.

import type { MCPResponse } from '@spec-kit/shared/types';
import {
  initializeIndexingRuntime,
  warmEmbeddingModel,
  runMemoryIndexScan,
  closeIndexingRuntime,
} from './indexing';

export interface ReindexOptions {
  specFolder?: string | null;
  force?: boolean;
  includeConstitutional?: boolean;
  includeSpecDocs?: boolean;
  incremental?: boolean;
  onProgress?: (step: { phase: string; message: string; detail?: Record<string, unknown> }) => void;
}

export interface ReindexResult {
  status: string;
  scanned: number;
  indexed: number;
  updated: number;
  unchanged: number;
  failed: number;
  embeddingProfile: { provider: string; model: string; dim: number } | null;
  constitutional?: { found: number; indexed: number; alreadyIndexed: number };
  files?: { status: string; file: string; isConstitutional?: boolean }[];
}

export async function reindex(options: ReindexOptions = {}): Promise<ReindexResult> {
  const { force = true, includeConstitutional = true, onProgress, ...scanArgs } = options;
  const emit = onProgress ?? (() => {});

  try {
    emit({ phase: 'init', message: 'Initializing database and search modules' });
    initializeIndexingRuntime();

    emit({ phase: 'warmup', message: 'Warming embedding model' });
    const profile = await warmEmbeddingModel('warmup test');

    emit({ phase: 'scan', message: 'Scanning and indexing memory files' });
    const response: MCPResponse = await runMemoryIndexScan({
      force,
      includeConstitutional,
      ...scanArgs,
    });

    const data = JSON.parse(response.content[0].text);

    return {
      status: data.status,
      scanned: data.scanned,
      indexed: data.indexed,
      updated: data.updated,
      unchanged: data.unchanged,
      failed: data.failed,
      embeddingProfile: profile ? { provider: profile.provider, model: profile.model, dim: profile.dim } : null,
      ...(data.constitutional ? { constitutional: data.constitutional } : {}),
      ...(data.files ? { files: data.files } : {}),
    };
  } finally {
    emit({ phase: 'close', message: 'Closing database' });
    closeIndexingRuntime();
  }
}
```

```typescript
  console.log(`Updated:    ${result.updated}`);
  console.log(`Unchanged:  ${result.unchanged}`);
  console.log(`Failed:     ${result.failed}`);
  if (result.embeddingProfile) {
    console.log(`Provider:   ${result.embeddingProfile.provider}`);
    console.log(`Model:      ${result.embeddingProfile.model}`);
  }
  if (result.constitutional) {
    console.log(`Constitutional: ${result.constitutional.indexed} indexed`);
  }
  console.log('STATUS=OK');
}

main().catch((err) => { console.error('FATAL:', err); process.exit(1); });
```

**Key benefits**: Script drops from 132 → ~50 LOC. Lifecycle errors are impossible (init/close always paired). The `onProgress` callback enables both CLI and programmatic use.
