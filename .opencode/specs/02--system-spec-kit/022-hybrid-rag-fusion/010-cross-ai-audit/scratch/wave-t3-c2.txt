**Task B — Stemmer Double-Consonant Dedup Bug**

1. **Other words affected**
- The unconditional dedup impacts many normal trailing-double-consonant words, e.g.:
- `all -> al`, `add -> ad`, `off -> of`, `bell -> bel`, `bill -> bil`, `cliff -> clif`, `staff -> staf`, `shell -> shel`, `spill -> spil`, `well -> wel`, `will -> wil`, `call -> cal`, `tall -> tal`, `ball -> bal`, `buzz -> buz`, `jazz -> jaz`.
- Quick dictionary sweep (`/usr/share/dict/words`) found about **1,064 lowercase words** where dedup fires **without any suffix removal** (the exact bug class).

2. **Search quality impact**
- Primary impact is **over-stemming collisions** (precision loss): unrelated terms collapse to same stem.
- This also flattens BM25 term specificity (IDF distortion), so relevant docs can rank lower (practical recall loss in top-k).
- Important nuance: your examples `bass`/`mass` are mainly broken by the separate broad `-s` stripping rule, not this dedup block. So dedup gating alone will not fix those two.

3. **Proposed fix (as requested)**
```ts
function simpleStem(word: string): string {
  let stem = word.toLowerCase();
  let suffixRemoved = false;

  if (stem.endsWith('ing') && stem.length > 5) { stem = stem.slice(0, -3); suffixRemoved = true; }
  else if (stem.endsWith('tion') && stem.length > 6) { stem = stem.slice(0, -4); suffixRemoved = true; }
  else if (stem.endsWith('ed') && stem.length > 4) { stem = stem.slice(0, -2); suffixRemoved = true; }
  else if (stem.endsWith('ly') && stem.length > 4) { stem = stem.slice(0, -2); suffixRemoved = true; }
  else if (stem.endsWith('es') && stem.length > 4) { stem = stem.slice(0, -2); suffixRemoved = true; }
  else if (stem.endsWith('s') && stem.length > 3) { stem = stem.slice(0, -1); suffixRemoved = true; }

  if (suffixRemoved && stem.length >= 3) {
    const last = stem[stem.length - 1];
    if (last === stem[stem.length - 2] && !/[aeiou]/.test(last)) {
      stem = stem.slice(0, -1);
    }
  }

  return stem;
}
```

- If you also want to fix `bass`/`mass`/`class`/`glass`, add a second guard to the `-s` rule (`!stem.endsWith('ss')`).
