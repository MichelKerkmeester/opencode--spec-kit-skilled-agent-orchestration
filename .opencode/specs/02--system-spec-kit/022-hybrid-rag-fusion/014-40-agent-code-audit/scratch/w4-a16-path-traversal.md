### FINDING-01: Symlinked subdirectories bypass `writeFilesAtomically()` containment
- **File:** `.opencode/skill/system-spec-kit/scripts/core/file-writer.ts:15-25,109-188`
- **Severity:** HIGH
- **Category:** SECURITY
- **Current Behavior:** The function blocks `..` lexically, then checks `path.resolve(filePath)` against `path.resolve(contextDir)`, but it does not canonicalize the target before opening temp/backup files. If a caller passes a nested filename like `subdir/doc.md` and `contextDir/subdir` is a symlink to an external directory, the temp write and rename happen in the symlink target first; `verifyResolvedWriteTarget()` only detects the escape after the external write already occurred, and rollback is best-effort.
- **Expected Behavior:** The real target directory should be proven to stay under the real `contextDir` before any open/write/rename/backup happens, and symlinked path segments should be rejected.
- **Root Cause:** Containment is enforced with lexical `path.resolve(...).startsWith(...)` checks, while `realpath` verification is deferred until after the rename.
- **Suggested Fix:** Canonicalize `contextDir` once, canonicalize or `lstat` every target parent path before creating temp/backup files, reject symlinked descendants, and derive temp/backup paths from the validated real parent directory.

### FINDING-02: `SPEC_FOLDER` containment ignores symlinks, enabling specs-directory escape

A symlinked spec folder (for example `specs/999-link -> /tmp/outside`) or a symlinked specs root still passes that lexical check, so downstream doc probing runs against the external directory.

- **Root Cause:** The guard treats normalized strings as trustworthy paths; it never compares canonical filesystem locations.
- **Suggested Fix:** Make `findActiveSpecsDir()` return a canonical real path (or reject roots whose real path escapes `CONFIG.PROJECT_ROOT`), then compare a canonicalized candidate path against that canonical root before calling any filesystem helpers.

### FINDING-03: `toRelativePath()` accepts sibling-prefix paths outside the project root
