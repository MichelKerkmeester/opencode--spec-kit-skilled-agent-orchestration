### FINDING-01: Alias normalization stops at three top-level fields
- **File:** `.opencode/skill/system-spec-kit/scripts/extractors/opencode-capture.ts:79-98`; `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:106-125,415-426`

- **Category:** DESIGN
- **Current Behavior:** `ConversationCapture` carries both `session_id/sessionId`, `session_title/sessionTitle`, and `captured_at/capturedAt`, but `transformOpencodeCapture()` only normalizes those three top-level aliases via a raw object cast. Nested snake_case fields such as `metadata.total_messages`, `metadata.session_created`, `user_message_id`, `assistant_message_id`, and `call_id` never get a canonical internal shape.
- **Expected Behavior:** The capture should be normalized once into a single canonical schema, including nested fields, before any downstream transformation.
- **Root Cause:** `opencode-capture.ts` and `input-normalizer.ts` define different DTOs, and the bridge between them is an ad hoc top-level patch instead of a full schema transform.
- **Suggested Fix:** Introduce a shared `ConversationCapture`/`NormalizedConversationCapture` model and normalize all legacy aliases centrally before building `userPrompts`, `observations`, and `FILES`.

### FINDING-04: Timestamp handling is inconsistent and silently fabricates “now”

- **Suggested Fix:** Carry canonical ISO timestamps through the whole pipeline, add a `timestampSource`/`isSyntheticTimestamp` marker, and derive human-readable formats only in the final rendering layer.
