**Findings**
- High: Deep-mode multi-query loses cross-variant score coherence. The original query is always first in the variant list, then Stage 1 runs a full hybrid search per variant and deduplicates by first-seen `id`, keeping baseline order instead of the highest-scoring duplicate. Inside `hybrid-search`, fusion is score-aware and fallback merges keep the better score, but that logic is bypassed across variants, and Stage 2 intentionally does not reweight hybrid results later. The current test suite codifies this baseline-first behavior. - Medium: Much of `DOMAIN_VOCABULARY_MAP` is effectively dead data. The map stores two synonyms for many terms, but `expandQuery()` breaks after the first synonym that creates a new variant, so the second synonym is normally unreachable.
