**Audit Report**

**Bottom Line**

The graph subsystem is not dead overall. The Stage 1 graph retrieval channel, Stage 2 causal boost, and Stage 2 graph-signal scoring are all wired into the live search pipeline. But community detection is only partially wired, and two graph features depend on side data that I could not find being populated in runtime code: `community_assignments` and `degree_snapshots`. That makes community injection effectively `NULL` by default, and graph momentum likely `0` unless an external job/manual path fills snapshots. 1. **`graph-search-fn.ts` is functional, not a stub.**
It defines a real `GraphSearchFn` factory that returns a callable search function, and that function is passed into `hybridSearch.init(...)` at server startup when unified graph is enabled. The live search path executes it in the hybrid pipeline and fuses its results with other channels. Refs: [search-types.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/search-types.ts#L4), [graph-search-fn.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/graph-search-fn.ts#L453), [context-server.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/context-server.ts#L890), [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L646)

It returns `Array<Record<string, unknown>>`, sorted by descending `score`, with graph-flavored result rows keyed by memory `id` plus fields like `source: 'graph'`, `relation`, `sourceId`, `targetId`, `title`, and sometimes `specFolder`. The underlying retrieval is real: FTS5-backed causal-edge matching with a LIKE fallback, plus hierarchy augmentation for `specFolder` searches. Refs: [graph-search-fn.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/graph-search-fn.ts#L73), [graph-search-fn.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/graph-search-fn.ts#L133), [graph-search-fn.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/graph-search-fn.ts#L198), [graph-search-fn.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/graph-search-fn.ts#L465)

5. **Actual graph contribution to search is mixed: non-NULL overall, but partly hollow.**
The Stage 1 graph channel is only routed for `complex` queries by default; `simple` and `moderate` skip it. When it does run, its results enter fusion and receive a graph weight. Refs: [query-router.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/query-router.ts#L51), [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L561), [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L646), [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L732)

Stage 2 graph contributions are counted and surfaced in metadata, so the pipeline expects live graph effects. Refs: [stage2-fusion.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/pipeline/stage2-fusion.ts#L559), [stage2-fusion.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/pipeline/stage2-fusion.ts#L798), [memory-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/handlers/memory-search.ts#L1003)

- `community` is effectively `NULL` unless `community_assignments` is populated elsewhere. **Verdict:** graph contribution is not NULL overall, but “community detection” looks NULL in practice, and “momentum” is probably minimal/zero without an external snapshot population path.
