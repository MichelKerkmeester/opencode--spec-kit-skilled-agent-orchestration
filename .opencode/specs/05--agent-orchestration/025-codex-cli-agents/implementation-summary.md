## What Was Built

The Codex CLI agent system now has all 9 agent definition files that `config.toml` was already referencing. Before this work, the `.codex/agents/` directory was empty and every agent dispatch would fail to load its specialized instructions.

### Automated Conversion Pipeline

A Python script reads each of the 9 ChatGPT agent markdown files, strips the YAML frontmatter, applies text substitutions (model references, path conventions, profile names), and wraps the full system prompt in a Codex CLI config overlay TOML structure. Each output file contains three fields: `sandbox_mode`, `model_reasoning_effort`, and `developer_instructions`.

### Agent Sandbox and Reasoning Configuration

Read-only agents (context, orchestrate, review, ultra-think) get `sandbox_mode = "read-only"`. Write-capable agents (debug, handover, research, speckit, write) get `sandbox_mode = "workspace-write"`. Reasoning effort ranges from `medium` (documentation-focused agents) through `high` (analysis agents) to `xhigh` (debug, review, ultra-think).

### Config Registry Completion

The missing `[agents.ultra-think]` entry was added to `config.toml`, bringing the total to 9 registered agents. All `config_file` references now resolve to existing TOML files.

<!-- ANCHOR:how-delivered -->
## How It Was Delivered

All 9 files were generated by a single Python script run, then verified in a second script that parsed every TOML file with `tomli`, checked config references, spot-checked key content sections, and scanned for unreplaced references. The old `toml` 0.10.2 library had a false-positive parse failure on multi-line literal strings; switching to `tomli` (spec-compliant) confirmed all files are valid TOML.
<!-- /ANCHOR:how-delivered -->

<!-- ANCHOR:decisions -->
## Key Decisions

| Decision | Why |
|----------|-----|
| Used `'''` (literal strings) instead of `"""` (basic strings) | Avoids escape sequence processing for markdown content containing backslashes like `\[PLACEHOLDER\]` |
| Kept `model_reasoning_effort = "xhigh"` as-is | Plan specified it; Codex CLI treats it as a string passthrough to the model API |
| Preserved full system prompts verbatim | Agent behavior depends on the complete workflow instructions, not just metadata |
| Used Python script for batch conversion | More reliable than manual per-file conversion for 9 files with consistent substitutions |
| Verified with `tomli` instead of `toml` | The `toml` 0.10.2 library has known issues with multi-line literal strings; `tomli` is TOML 1.0 spec-compliant |

<!-- ANCHOR:limitations -->
## Known Limitations

1. **`model_reasoning_effort = "xhigh"` is non-standard.** The Codex CLI may not pass this value to all model providers. If agents using `xhigh` don't behave as expected, change to `high`.
2. **MCP servers are inherited from parent config.** The TOML files don't redeclare MCP servers since all 3 (sequential_thinking, spec_kit_memory, code_mode) are already global in config.toml.
3. **Old `toml` Python library fails on these files.** The `toml` 0.10.2 package incorrectly rejects valid multi-line literal strings. Use `tomli` (Python 3.11+) or `tomli` backport for verification.
<!-- /ANCHOR:limitations -->
