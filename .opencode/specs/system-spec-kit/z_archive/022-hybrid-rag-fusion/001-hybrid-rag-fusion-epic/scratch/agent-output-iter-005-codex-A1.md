**Fusion Report**

1. `fuseResultsMulti()` is weighted RRF with a convergence bonus.
- The per-list contribution is `weight * (1 / (k + rank))`, where `rank` is 1-based (`i + 1`) and `k` defaults to `60`. Caller `k` may be `0`; env override `SPECKIT_RRF_K` is only accepted if it parses to a finite positive number. See [rrf-fusion.ts#L223](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L223), [rrf-fusion.ts#L129](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L129), [rrf-fusion.ts#L36](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L36).
- After summing per-source reciprocal-rank terms, it adds `0.10 * (uniqueSourceCount - 1)` for items seen in 2+ distinct sources; source count is deduped before bonus calculation. Scores are then optionally min-max normalized to `[0,1]` unless `SPECKIT_SCORE_NORMALIZATION=false`. See [rrf-fusion.ts#L266](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L266), [rrf-fusion.ts#L281](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L281), [rrf-fusion.ts#L464](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L464).
- Important live-path nuance: `GRAPH_WEIGHT_BOOST=1.5` only applies when a list omits `weight`. In `hybrid-search.ts`, every live channel is pushed with an explicit weight, so that default graph boost is not used there. See [rrf-fusion.ts#L243](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/rrf-fusion.ts#L243), [hybrid-search.ts#L620](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L620), [hybrid-search.ts#L658](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L658).

2. `hybridAdaptiveFuse()` switches between adaptive RRF and standard RRF, not RSF.
- It computes intent-based weights, checks `SPECKIT_ADAPTIVE_FUSION` plus rollout bucketing, then returns either `standardFuse()` (equal-weight RRF) or `adaptiveFuse()` (weighted RRF + optional recency boost). Dark-run computes both and returns a diff. See [adaptive-fusion.ts#L344](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/adaptive-fusion.ts#L344), [adaptive-fusion.ts#L191](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/adaptive-fusion.ts#L191), [adaptive-fusion.ts#L281](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/algorithms/adaptive-fusion.ts#L281).
- RSF is dormant. The RSF module explicitly says it is “not actively used,” and the feature doc says RRF remains live while RSF is shadow/offline only. See [rsf-fusion.ts#L7](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/rsf-fusion.ts#L7), [02-relative-score-fusion-in-shadow-mode.md#L18](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/feature_catalog/12--query-intelligence/02-relative-score-fusion-in-shadow-mode.md#L18).
- Live hybrid search uses `hybridAdaptiveFuse()` mainly as a weight oracle. It reads `adaptiveResult.weights`, rewrites list weights, and then still calls `fuseResultsMulti(lists)` for the actual ranking. See [hybrid-search.ts#L732](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L732), [hybrid-search.ts#L745](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L745).

4. Is there meaningful diversity versus single-channel results?
- Yes, structurally. Live ranking combines separate vector, FTS, BM25, graph, and degree lists, and rewards cross-channel agreement via summed reciprocal-rank terms plus convergence bonus. That is materially different from taking any one channel alone. FTS and BM25 are separate lists, but they are same-family lexical channels and both receive the same adaptive `keywordWeight`, so their diversity is weaker than vector-vs-graph or vector-vs-lexical diversity. 6. Would a pre-fusion dedup stage help?
- Inference: not as a blanket step. Cross-channel duplicate IDs are the whole point of fusion here, because both RRF and RSF reward agreement across sources. Removing those duplicates before fusion would destroy convergence signal. - The pipeline already does targeted dedup where it makes sense: query-variant merging, R12 baseline+expanded merge, and summary-hit append all dedup by ID before continuing. See [stage1-candidate-gen.ts#L292](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/pipeline/stage1-candidate-gen.ts#L292), [stage1-candidate-gen.ts#L378](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/pipeline/stage1-candidate-gen.ts#L378), [stage1-candidate-gen.ts#L600](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/pipeline/stage1-candidate-gen.ts#L600).

**Bottom Line**
Live search is still firmly RRF-based. The real runtime pattern is: `routeQuery()` chooses 2/3/5 candidate channels, each active channel may or may not actually contribute results, `hybridAdaptiveFuse()` supplies intent-aware weights, and final ranking still comes from `fuseResultsMulti()`. The only “skip” behavior that is definitely real is channel routing and optional stage-level extras like query expansion and summary search.
