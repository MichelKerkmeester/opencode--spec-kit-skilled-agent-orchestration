**Recommendation Synthesis**
1. Pick one authoritative hybrid ranking boundary. The cleanest fix is to make Stage 1 collect raw channel candidates directly and remove ranking logic from the Stage 1 hybrid dependency path, rather than nesting `hybridSearchEnhanced()` inside the pipeline.
2. Thread `config.detectedIntent` through every Stage 1 `searchWithFallback()` call so hybrid adaptive fusion uses the same intent decision as the handler.
3. Replace Stage 4’s custom extractor with `resolveEffectiveScore()` to remove ordering and normalization drift.
4. Normalize BM25 fallback IDs to numbers before they enter `PipelineRow`, then add one end-to-end test for the BM25-only fallback path.
