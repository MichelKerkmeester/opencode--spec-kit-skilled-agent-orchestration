## Focus

## Novelty Justification
This iteration produced mostly net-new structure rather than repetition: it narrowed true recovery surfaces to four agent families, separated the false-positive `compact skeptic/referee` wording from actual compaction handling, established that `autoSurface` is absent across all runtime agent files, confirmed that `.codex/agents/` is currently empty, and identified the pre-existing Context Package branch as the smallest reusable insertion point for hook-aware behavior.

## Recommendations for Implementation
1. Patch `.opencode/agent/context.md`, `.opencode/agent/orchestrate.md`, `.opencode/agent/handover.md`, and `.opencode/agent/speckit.md` first, then mirror the same wording into `.claude/agents/` and `.gemini/agents/`. No `.codex/agents/` patch is possible until that directory has actual agent files.
2. Reuse the existing Context Package branch instead of inventing a new workflow. Recommended pattern:

```md
If dispatched with a compaction-hook recovery bundle or Context Package:
- If the bundle contains a next safe action plus the required supporting signals, consume it first.
- Skip Layer 1 memory checks and broad recovery scans.
- Fall back to `memory_match_triggers` / `memory_context` only for missing fields.

If no hook bundle is present:
- Run the current memory-first flow unchanged.
```

3. In `@orchestrate`, treat "context compaction occurred" as an invalidation event for the "agent file already loaded" exception, and forward the hook-derived recovery bundle to downstream agents as part of the Context Package contract. [SOURCE: .opencode/agent/orchestrate.md:155-162]
