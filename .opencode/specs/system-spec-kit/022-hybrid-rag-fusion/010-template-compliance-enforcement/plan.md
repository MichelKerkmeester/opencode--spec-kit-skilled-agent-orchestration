## 1. SUMMARY

### Technical Context

| Aspect | Value |
|--------|-------|
| **Language/Stack** | Markdown (agent definitions), Shell (validation workflow), TOML (Codex agent) |
| **Framework** | system-spec-kit validation pipeline (`validate.sh`, `template-structure.js`) |
| **Storage** | File-based (agent definitions, reference file) |
| **Testing** | Manual validation via `validate.sh --strict` plus agent-log audit for per-file validation execution |

### Overview
This plan implements a 2-layer defense-in-depth architecture to enforce structural template compliance for all spec folder documents generated by AI agents. Layer 1 injects compact structural contracts into all @speckit agent definitions so agents know the exact required headers and anchors before writing. Layer 2 replaces conflicting validation timing directives with a single unambiguous "validate after each write" protocol. Current state: implementation work is delivered for Phases A-C, while Phase D verification evidence is still pending.

## 3. ARCHITECTURE

### Pattern
2-Layer Defense-in-Depth (Prompt Injection + Validation Loop)

### Key Components
- **Shared Reference File**: canonical contract artifact in the system-spec-kit references area covering Levels 1-3 structural contracts, phase addenda rules, and sync protocol.
- **Compact Inline Contract** (49 lines in each @speckit agent definition): Anchor-to-H2 mapping tables for all 5 Level 2 document types plus Level 3 decision-record structure support. Embedded directly in agent prompts to eliminate runtime dependency on skill folder loading.
- **Post-Write Validation Protocol** (new section in each agent def): Single directive replacing 3 conflicting timing instructions. Includes exit code parsing, fix loop (max 3 attempts), and stop conditions.

### Data Flow
```
Agent receives task -> Reads inline contract (Layer 1: headers + anchors per doc type)
    -> Writes .md file following contract structure
    -> Runs validate.sh --strict (Layer 2: post-write check)
    -> Fixes errors if any (max 3 attempts per file)
    -> Proceeds to next file
```
