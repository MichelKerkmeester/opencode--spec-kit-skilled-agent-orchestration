**Findings**

In practice, sampled/runtime code still uses ad hoc checks in [context-server.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/context-server.ts#L229), [context-server.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/context-server.ts#L1095), [memory-context.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/handlers/memory-context.ts#L999), [dynamic-token-budget.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/dynamic-token-budget.ts#L57), [confidence-truncation.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/confidence-truncation.ts#L47), [channel-representation.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/channel-representation.ts#L70), and [save-quality-gate.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/validation/save-quality-gate.ts#L228). That means `FLAG=0` can stay enabled and rollout behavior is no longer centrally enforced.

- `Low` Module export style is mostly named-export based, but not fully consistent across the server code. [memory-parser.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/parsing/memory-parser.ts#L124) uses normal TypeScript named exports throughout, then adds a parallel CommonJS surface via [memory-parser.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/parsing/memory-parser.ts#L924). Since the package already exposes a clean package export map in [package.json](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/package.json#L6), this mixed surface is standards drift and an extra maintenance burden.

- `Low` Public API JSDoc is not complete in the sampled lib module. `collectRawCandidates` is exported from [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L2095) but its definition at [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L1370) has no JSDoc block, unlike adjacent exported APIs such as [hybrid-search.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/lib/search/hybrid-search.ts#L1448).

- `Low` Error handling is generally typed well, but the sampled handler still has at least one silent bare catch. In [memory-save.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/handlers/memory-save.ts#L220), spec-doc-health failures are swallowed with `catch {}` instead of the prevailing `catch (error: unknown)` pattern used elsewhere in the same file. That weakens observability and breaks the otherwise consistent typed-error style.

- `Low` Test naming is uniform inside `mcp_server/tests`, but not across the broader Spec Kit Memory codebase. The server package expects Vitest in [package.json](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/mcp_server/package.json#L22), and the MCP server tests follow `*.vitest.ts`. But shared parsing still has [quality-extractors.test.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/parsing/quality-extractors.test.ts#L1) and [spec-doc-health.test.ts](/Users/michelkerkmeester/MEGA/Development/Opencode%20Env/Public/.opencode/skill/system-spec-kit/shared/parsing/spec-doc-health.test.ts#L1), so the convention is not repo-wide if `shared/` is in scope.
