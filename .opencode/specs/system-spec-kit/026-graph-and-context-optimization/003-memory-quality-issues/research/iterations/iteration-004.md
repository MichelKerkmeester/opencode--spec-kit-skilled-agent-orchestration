# Iteration 4: D3 Root Cause — Garbage Trigger Phrases

## Focus
Trace defect D3 to the exact code paths that generate garbage phrase outputs, with special attention to the observed path-token leaks (`kit/026`, `optimization/001`, `systems/004`, `graph`, `and`) and the sliding-window bigram fragments (`across prioritization`, `prioritization tiers`, `tiers full`, `full level`, `level spec`, `spec set`). The key clarification from this pass is that the broken phrases come from two sibling pipelines: frontmatter `trigger_phrases` and rendered `Key Topics`. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1198-1299`, `.opencode/skill/system-spec-kit/templates/context_template.md:273-275`, `.opencode/skill/system-spec-kit/templates/context_template.md:831-833`]

## Hypothesis
Iteration 1 already narrowed D3 to the workflow trigger build and the indexer fallback, so the working hypothesis was that path fragments were entering through `specFolderName` handling and that adjacent-word garbage was coming from an n-gram extractor operating over normalized text. [CITATION: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/003-memory-quality-issues/research/iterations/iteration-001.md:118-119`]

[CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1209-1231`]

That means `graph` and `and` survive, and slash-bearing tokens can be reintroduced after the filter has already run. - The shared tokenizer used by the trigger/topic extractor lowercases text and splits on whitespace and punctuation, but not on `/` or `-`. It also removes digits-only tokens before n-gram counting. This preserves slash-bearing tokens like `kit/026` as single tokens and collapses content around removed digits/stopwords before bigrams are counted. [CITATION: `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:116-129`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:176-183`]

[CITATION: `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`, `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:291-351`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:116-149`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:176-183`]

## Root Cause
D3 is a mixed defect class with two concrete roots:

1. **Frontmatter trigger phrase pollution** is caused by workflow-side reinsertion of folder-derived tokens after quality filtering. The exact defect owner is `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1271-1295`. Trigger condition: any `specFolderName` whose whitespace-split normalized form contains slash-bearing or generic 3+ character tokens. Because `folderNameForTriggers` preserves `/` and the append loop runs after `filterTriggerPhrases()`, phrases like `kit/026`, `optimization/001`, `systems/004`, `graph`, and `and` are reintroduced directly into `preExtractedTriggers`. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1228-1295`]

2. **Sliding-window fragment topics** are caused by topic extraction over a stopword-collapsed token stream. The exact owner is the `extractKeyTopics()` -> `SemanticSignalExtractor.extractTopicTerms()` -> `buildTopicTerms()` -> `buildRankedNgrams()` chain in `.opencode/skill/system-spec-kit/scripts/core/topic-extractor.ts:24-50` and `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`. Trigger condition: summaries or decision rationales that contain meaningful nouns separated by stopwords, determiners, or numeric markers. Once those tokens are removed by `tokenize()` + aggressive stopword filtering, bigrams are counted over the collapsed stream and produce fragments like `tiers full`, `full level`, and `level spec`. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/topic-extractor.ts:24-50`, `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`, `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:291-351`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:116-149`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:176-183`]

[CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1226-1231`, `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1271-1295`, `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-graph-context-systems/004-graphify/memory/06-04-26_15-37__deep-research-on-graphify-external-repo-python.md:656-661`]

No separate path tokenizer is needed for those phrases; they are the direct output of the topic extractor’s stopword-collapsed bigram logic. [CITATION: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-graph-context-systems/001-claude-optimization-settings/memory/06-04-26_12-05__deep-research-run-8-iterations-via-cli-copilot.md:163-163`, `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-graph-context-systems/001-claude-optimization-settings/memory/06-04-26_12-05__deep-research-run-8-iterations-via-cli-copilot.md:174-174`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:116-149`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:176-183`, `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`]

The workflow merge is the path that contaminates saved markdown; the indexer merge can further contaminate indexed metadata because it does not re-filter manual phrases. ## Defect Chain
Structured JSON save -> `runWorkflow()` builds summary/decision/file/folder trigger source -> workflow extracts auto trigger phrases -> workflow merges manual phrases -> `filterTriggerPhrases()` runs too early -> workflow appends raw folder tokens and reintroduces slash fragments and generic single words into `trigger_phrases` -> independently, workflow computes `Key Topics` from summary/decisions -> aggressive stopword filtering collapses the token stream -> bigram ranking over filtered tokens emits phrase fragments like `tiers full` and `level spec` -> template renders both polluted lists into the saved memory file. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1198-1299`, `.opencode/skill/system-spec-kit/scripts/core/topic-extractor.ts:24-50`, `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`, `.opencode/skill/system-spec-kit/templates/context_template.md:273-275`, `.opencode/skill/system-spec-kit/templates/context_template.md:831-833`]

## Findings
1. The `trigger_phrases` path leak is owned by workflow, not the renderer: folder-derived tokens are appended after filtering in `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1271-1295`. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1232-1295`]
2. The exact path-tokenizer responsible for `kit/026`, `optimization/001`, and `systems/004` is the workflow pair `specFolderName.replace(/^\d{1,3}-/, '').replace(/-/g, ' ')` plus `split(/\s+/)`, not a dedicated path parser elsewhere. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1226-1231`, `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1271-1272`]
3. The shared tokenizer preserves slash-bearing tokens because its split regex does not include `/`, and it removes numeric-only tokens before n-gram counting. [CITATION: `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:116-129`]
4. The `Key Topics` garbage bigrams come from the topic extractor path, not the frontmatter trigger extractor path. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1198-1199`, `.opencode/skill/system-spec-kit/scripts/core/topic-extractor.ts:24-50`]
5. The fragment phrases are created because bigrams are counted after aggressive stopword filtering, which makes non-adjacent content words artificially adjacent in the filtered stream. [CITATION: `.opencode/skill/system-spec-kit/scripts/lib/semantic-signal-extractor.ts:260-284`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:144-149`, `.opencode/skill/system-spec-kit/shared/trigger-extractor.ts:176-183`]
6. Existing validation is insufficient: `filterTriggerPhrases()` misses 3-5 letter generic singletons like `graph` and `and`, `ensureMinSemanticTopics()` does not quality-filter existing topics at all, and the indexer merge does not re-filter manual phrases before indexing. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:122-156`, `.opencode/skill/system-spec-kit/scripts/core/frontmatter-editor.ts:134-151`, `.opencode/skill/system-spec-kit/scripts/core/memory-indexer.ts:112-122`]

## Question Status
Q3 RESOLVED: D3 is not one bug but a paired garbage-phrase defect. Slash/path and single-word pollution in saved `trigger_phrases` come from workflow-side folder-token reinsertion after filtering, while fragment bigrams like `tiers full` come from `Key Topics` extraction over a stopword-collapsed token stream.
