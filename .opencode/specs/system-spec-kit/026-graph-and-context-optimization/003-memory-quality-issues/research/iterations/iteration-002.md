# Iteration 2: D1 Root Cause — Truncated OVERVIEW

## Focus
This iteration traced defect D1 end-to-end through the JSON-mode save pipeline to identify the first concrete truncation point for the `## 1. OVERVIEW` section. The main question was whether the cut happens during normalization, semantic-summary generation, template rendering, or in the session-data extractor that feeds `{{SUMMARY}}` into the context template.

## Hypothesis (from iteration 1)
Iteration 1 left three candidate paths for D1: `collectSessionData()` summary construction in `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts`, Step 7.5 semantic-summary generation via `generateImplementationSummary()` in `.opencode/skill/system-spec-kit/scripts/core/workflow.ts`, and a template/render-time contraction around `populateTemplate('context', ...)` plus the `context_template.md` `OVERVIEW` block. [CITATION: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/003-memory-quality-issues/research/iterations/iteration-001.md:124-130`]

## Evidence collected
- `normalizeInputData()` does not shorten the raw `sessionSummary` field on the fast path. The workflow merges normalized fields back onto `preloadedData` with explicit projection and does not overwrite `sessionSummary`, so the original JSON value survives into `collectedData`. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:617-637`]
- Inside `normalizeInputData()`, the raw `sessionSummary` is read verbatim and reused for `recentContext.learning` and the fallback user prompt. The only clipping in this module is in `buildSessionSummaryObservation()`, which truncates the derived observation narrative to 500 characters plus ellipsis, not the source field itself. [CITATION: `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:274-283`, `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:668-745`]
- `collectSessionData()` is the first place that assigns the `SUMMARY` field used by the template. When `data.sessionSummary` is present and longer than 20 characters, it is hard-clamped with `data.sessionSummary.substring(0, 500)`. [CITATION: `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts:875-881`]
- The `OVERVIEW` section template is a direct `{{SUMMARY}}` substitution. There is no additional trimming in the template; the `## ... OVERVIEW` block renders whatever `SUMMARY` already contains. [CITATION: `.opencode/skill/system-spec-kit/templates/context_template.md:330-351`]
- The renderer layer does not truncate content. `populateTemplate()` only reads `context_template.md`, calls `renderTemplate(template, data)`, and returns the rendered string. [CITATION: `.opencode/skill/system-spec-kit/scripts/renderers/template-renderer.ts:176-205`]
- Step 7.5 `generateImplementationSummary()` does not feed the `OVERVIEW` block. It computes semantic metadata (`task`, `solution`, file/decision/outcome summaries), and the workflow only stores a 100-character `semanticSummary.task` in metadata, not in `{{SUMMARY}}`. [CITATION: `.opencode/skill/system-spec-kit/scripts/lib/semantic-summarizer.ts:468-613`, `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1113-1128`, `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1421-1427`]
- The broken `DETAILED CHANGES` example is a sibling manifestation of the same source text, but through a different clamp. The saved line at `001-claude-optimization-settings/...md:202` is 492 characters and ends with `...`, which matches `buildSessionSummaryObservation()` trimming the observation narrative to 500 characters, removing the final partial word, then appending ellipsis. [CITATION: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-graph-context-systems/001-claude-optimization-settings/memory/06-04-26_12-05__deep-research-run-8-iterations-via-cli-copilot.md:199-205`, `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:279-283`, `.opencode/skill/system-spec-kit/templates/context_template.md:375-389`]

## Root cause identified
The exact D1 `OVERVIEW` truncation originates in `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts:876-877`:

```ts
const SUMMARY: string = (typeof data.sessionSummary === 'string' && data.sessionSummary.length > 20)
  ? data.sessionSummary.substring(0, 500)
```

That hard cap becomes `sessionData.SUMMARY`, and the `OVERVIEW` block in `.opencode/skill/system-spec-kit/templates/context_template.md:333-335` renders `{{SUMMARY}}` directly. Because the extractor uses a raw `substring(0, 500)` with no sentence-boundary handling, any longer `sessionSummary` is cut exactly at character 500, even if that splits a sentence. Renderer/template code does not introduce the defect; it only propagates the already-truncated value. [CITATION: `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts:875-881`, `.opencode/skill/system-spec-kit/templates/context_template.md:333-335`, `.opencode/skill/system-spec-kit/scripts/renderers/template-renderer.ts:194-205`]

## Defect chain
D1 origin: JSON payload provides a long `sessionSummary` -> `normalizeInputData()` preserves the raw field but also creates a clipped observation narrative for the feature observation path -> `collectSessionData()` hard-clamps `SUMMARY` to `sessionSummary.substring(0, 500)` -> `runWorkflow()` passes `sessionData` into `populateTemplate('context', ...)` -> `context_template.md` renders `{{SUMMARY}}` into `## 1. OVERVIEW` -> saved memory file shows a shortened overview, while the matching feature observation in `DETAILED CHANGES` is clipped separately with ellipsis. ## Findings (count and list)
1. `normalizeInputData()` preserves the raw JSON `sessionSummary`; it does not truncate that source field during the workflow merge path. [CITATION: `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:617-637`, `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:668-745`]
2. `buildSessionSummaryObservation()` applies a separate 500-character narrative clamp with word-boundary trimming and ellipsis, affecting the `DETAILED CHANGES` feature entry rather than the `OVERVIEW` block. [CITATION: `.opencode/skill/system-spec-kit/scripts/utils/input-normalizer.ts:274-283`, `.opencode/skill/system-spec-kit/templates/context_template.md:380-385`]
3. The exact `OVERVIEW` defect owner is `collectSessionData()` at `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts:876-877`, where `SUMMARY` is assigned with `data.sessionSummary.substring(0, 500)`. [CITATION: `.opencode/skill/system-spec-kit/scripts/extractors/collect-session-data.ts:875-881`]
4. `context_template.md` renders the overview with a direct `{{SUMMARY}}` placeholder and adds no additional clipping logic. [CITATION: `.opencode/skill/system-spec-kit/templates/context_template.md:330-351`]
5. `populateTemplate()` and the renderer layer do not truncate; they pass through already-prepared values. [CITATION: `.opencode/skill/system-spec-kit/scripts/renderers/template-renderer.ts:176-205`]
6. `generateImplementationSummary()` is not part of the overview truncation path; its only nearby truncation is the metadata-only `implSummary.task.substring(0, 100)` in workflow metadata. [CITATION: `.opencode/skill/system-spec-kit/scripts/lib/semantic-summarizer.ts:468-613`, `.opencode/skill/system-spec-kit/scripts/core/workflow.ts:1421-1427`]
