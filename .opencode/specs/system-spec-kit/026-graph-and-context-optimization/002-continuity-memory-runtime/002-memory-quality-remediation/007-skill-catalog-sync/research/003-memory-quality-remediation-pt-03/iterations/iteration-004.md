## Findings

### P0 — must update before Sub-PR-14 ships
- **F004.1** — Template guide still tells operators to use `generate-context.ts` for memory saves
  - **Location**: `.opencode/skill/system-spec-kit/references/templates/template_guide.md:600-626`
  - **Stale content**: The guide says memory files are auto-generated by `generate-context.ts` and repeats "always use `generate-context.ts`" as the Memory Save Rule.
  - **Why stale**: The current runtime contract uses `scripts/dist/memory/generate-context.js` as the canonical executable. Referring operators to the TypeScript source file conflicts with the canonical command surface defined in AGENTS, CLAUDE, and the skill itself.
  - **Required update**: Replace the TypeScript-script wording with `generate-context.js` runtime wording while keeping the JSON-file / `--json` / `--stdin` options intact.

### P1 — should update; defer only with rationale
- **F004.2** — Level specifications table still labels `memory/` creation with `generate-context.ts`
  - **Location**: `.opencode/skill/system-spec-kit/references/templates/level_specifications.md:762-769`
  - **Stale content**: ``| `memory/` | Session context preservation | `generate-context.ts` script via `/memory:save` |``
  - **Why stale**: The same section now shows the correct `generate-context.js --json` command immediately below, so the creation-method table contradicts the canonical runtime usage.
  - **Required update**: Rename the creation method to `generate-context.js` or "generate-context runtime script" so the table and example agree.

- **F004.3** — Memory system reference inventory still lists `generate-context.ts` as the memory-file generator
  - **Location**: `.opencode/skill/system-spec-kit/references/memory/memory_system.md:23-26,687-688`
  - **Stale content**: The component table and the scripts list both identify `scripts/memory/generate-context.ts` as the save workflow.
  - **Why stale**: This is the reference page that explains the live memory architecture. Leaving the inventory on the source file blurs the distinction between implementation source and the shipped runtime entrypoint.
  - **Required update**: Point both inventory surfaces at `scripts/dist/memory/generate-context.js` and optionally note the TypeScript source separately.

- **F004.4** — Execution methods section heading still presents the save workflow as `generate-context.ts`
  - **Location**: `.opencode/skill/system-spec-kit/references/workflows/execution_methods.md:77-90`
  - **Stale content**: The heading is `### generate-context.ts` even though every usage example under it invokes `generate-context.js`.
  - **Why stale**: Readers scanning headings get the wrong executable name before they reach the examples.
  - **Required update**: Rename the section to `generate-context.js` or "generate-context runtime script" so the heading matches the commands.
