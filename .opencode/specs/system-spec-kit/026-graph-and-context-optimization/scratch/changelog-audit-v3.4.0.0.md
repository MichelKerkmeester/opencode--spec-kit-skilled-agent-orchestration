## Fundamental gaps (P0)

### 1. Missing packet 015: `/spec_kit:start` removal and `/spec_kit:plan --intake-only`

**What is missing:** The changelog does not describe the new command-surface merger at all. Current packet 015 says the shipped state is: shared intake contract at `references/intake-contract.md`, `/spec_kit:plan --intake-only`, `/spec_kit:complete` refactored onto the shared module, `/spec_kit:start` hard-deleted, two YAML assets deleted, Gemini CLI routing deleted, and the skill-registry/`COMMAND_BOOSTS` entry removed [`.opencode/specs/system-spec-kit/026-graph-and-context-optimization/015-start-into-plan-merger/implementation-summary.md:57-84`](../015-start-into-plan-merger/implementation-summary.md). The current changelog still summarizes the tail as only **three** follow-on cutovers and never mentions this fourth command-surface change [`.opencode/changelog/01--system-spec-kit/v3.4.0.0.md:5`](../../../../changelog/01--system-spec-kit/v3.4.0.0.md).

**Recommended changelog update:** Insert a new section, e.g. `## Start-into-Plan Merger`, after `## Middleware Deprecation` or immediately after the save-flow material. Minimum content should mention:

1. `/spec_kit:start` is deleted.
2. `/spec_kit:plan --intake-only` is the replacement.
3. `intake-contract.md` is the shared canonical module.
4. `complete.md` and `resume.md` were rewired to the new intake contract.

**What is missing/stale:** The changelog's Upgrade section still says **"No migration required"** and only documents save-flow/env-flag migration notes [`.opencode/changelog/01--system-spec-kit/v3.4.0.0.md:407-415`](../../../../changelog/01--system-spec-kit/v3.4.0.0.md). **Recommended changelog update:** Add an Upgrade bullet under the current `## Upgrade` section:

> `Start-into-plan merger: /spec_kit:start was removed. Migrate to /spec_kit:plan --intake-only. External scripts or docs that still call /spec_kit:start will fail until updated.`
