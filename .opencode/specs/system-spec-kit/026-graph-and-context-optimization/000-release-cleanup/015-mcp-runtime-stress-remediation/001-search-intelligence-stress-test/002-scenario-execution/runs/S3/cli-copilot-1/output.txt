**Decision rationale:** in `026/003/004`, the packet decided `/memory:save` should be **planner-first by default** because the old default path was doing too much mutation and follow-up work for every save, while the handler already had enough information to return a safe plan without mutating. The broader rationale was **trim-targeted**: keep the proven writer core, but remove optional/default-path cost from the hot path.  
**Source anchors:** `decision-record.md#adr-001`, `spec.md#problem`, `implementation-summary.md#what-built`

| Mode / decision | Rationale | Source anchor |
|---|---|---|
| `plan-only` = default | Default mutation was too heavy and included optional subsystems; planner output gives route + legality + follow-up actions without changing files, making the default path reviewable and non-mutating. | `spec.md#problem`, `implementation-summary.md#what-built` |
| `full-auto` = explicit fallback | They kept the canonical atomic writer because it was still a proven, load-bearing core for operators/workflows that still need automatic mutation; making it explicit avoids paying that cost on every save. | `decision-record.md#adr-001`, `implementation-summary.md#what-built` |
| `hybrid` = reserved, currently behaves like `plan-only` | Mixed-flow behavior was not defined yet, so the value was kept reserved and documented honestly rather than pretending it had separate live behavior. | `spec.md#scope`, `implementation-summary.md#limitations` |
