## Analysis
The sanitizer to reuse is already well-defined and tested. The renderer's `sanitizeSkillLabel()` canonical-folds input, rejects multiline and instruction-shaped labels, strips control characters, and collapses whitespace before anything becomes model-visible (`render.ts:50`-`render.ts:66`). The renderer is explicitly documented as the prompt-boundary guard that ignores free-form prompt-adjacent fields (`render.ts:93`-`render.ts:99`), and the shared-payload layer has already turned that guard into a reusable transport contract for advisor labels and source refs (`shared-payload.ts:491`-`shared-payload.ts:509`, `shared-payload.ts:555`-`shared-payload.ts:587`).

The current re-index path is not equivalent to that boundary. `parseSkillMetadata()` verifies basic JSON shape and allowed enum/range fields, but it carries raw metadata strings and string arrays into the parsed node (`skill-graph-db.ts:347`-`skill-graph-db.ts:379`), accepts edge `context` as a required string (`skill-graph-db.ts:401`-`skill-graph-db.ts:407`), and persists node metadata directly into SQLite (`skill-graph-db.ts:540`-`skill-graph-db.ts:548`). Query helpers later select and return those fields (`skill-graph-queries.ts:89`-`skill-graph-queries.ts:106`), while the query handler strips only `sourcePath` and `contentHash` (`query.ts:182`-`query.ts:190`).

That makes A7 an adopt-now guardrail, not a prototype. A6 already deferred broad live watching for all `SKILL.md` support content (`iteration-006.md:35`-`iteration-006.md:42`), but A7 is about the ingestion boundary whenever re-index does run. Since the advisor runtime currently extracts raw `SKILL.md` frontmatter name/description/keywords into scoring records (`skill_advisor_runtime.py:53`-`skill_advisor_runtime.py:105`, `skill_advisor_runtime.py:180`-`skill_advisor_runtime.py:189`), Phase 027 should make any new TypeScript re-index/advisor-unification path sanitize repository-authored labels, refs, derived trigger text, and graph strings before durable storage or shared-payload publication.

The implementation rule should be fail-closed for model-visible identity fields and conservative for auxiliary graph fields. Labels/source refs should use the exact `sanitizeSkillLabel()` contract already covered by renderer and shared-payload tests (`advisor-renderer.vitest.ts:60`-`advisor-renderer.vitest.ts:84`, `shared-payload-advisor.vitest.ts:113`-`shared-payload-advisor.vitest.ts:160`). Longer descriptive fields such as edge `context`, derived keywords, descriptions, and future auto-derived triggers should be canonical-folded, control-character stripped, line-bounded or dropped before advisor exposure; B7 can still decide keyword-stuffing defenses, but A7 should establish the sanitizer-at-re-index invariant now.

## Verdict
- **Call:** adopt now
- **Confidence:** high
- **Rationale:** The renderer/shared-payload sanitizer is already a reusable, tested boundary, while the current re-index path stores raw metadata strings that can surface through graph/advisor outputs. Phase 027 should apply the same sanitizer at re-index before storing or publishing advisor-visible skill metadata.
