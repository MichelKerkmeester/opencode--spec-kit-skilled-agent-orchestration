## Analysis
Track B and Track C are conceptually compatible: B puts auto-derived phrases in `graph-metadata.json.derived` and the SQLite skill graph, while C wants an advisor-specific projection with explicit signal, lexical, semantic, and graph lanes rather than memory-table reuse. The live schema also gives a concrete feed-through path because `skill_nodes` stores both `intent_signals` and `derived`, and the scan transaction persists changed derived JSON before advisor code reads it.

The coherence problem is the current scoring boundary. Existing advisor loading merges `intent_signals` and `derived.trigger_phrases` into one `signals` map, `_apply_signal_boosts()` records all of them as `(signal)`, and downstream confidence treats any positive signal boost as intent evidence. That contradicts the B-track safeguards, which require derived/generated terms to remain a separate lower-trust lane with caps, corpus validation, freshness provenance, and anti-stuffing demotions.

C4 already provides the architectural fix: put auto-derived keywords into the fusion scorer as a distinct `derived_generated` or `derived_keyword` channel, not as raw direct intent. The derived channel should consume B5 freshness and B6 corpus generation, apply B4/B7 caps and haircuts, and then flow through C's centralized analytical fusion alongside explicit author signals, lexical/name evidence, graph/causal priors, conflict penalties, and optional shadow semantic features.

So the cross-track answer is not a hard conflict; it is a required adjustment. B can feed C's fusion scorer immediately if the implementation preserves evidence provenance and prevents double-counting. Without that adjustment, derived keywords would improve recall but inflate confidence and widen the keyword-stuffing attack surface before C7/C8 measurement can prove the gain.

## Verdict
- **Call:** needs-adjustment
- **Confidence:** high
- **Rationale:** B's derived keyword set has a valid storage and freshness path into C's scorer, but it must enter as a separate capped/provenance-aware derived channel rather than the current direct `(signal)` intent boost lane. This adjustment aligns B4/B7 safeguards with C4's centralized analytical fusion and C6's no-embedding hot path.
