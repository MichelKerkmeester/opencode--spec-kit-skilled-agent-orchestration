## Boundary Stress Results

- SUPERFICIALLY PORTABLE BUT REJECT: one shared Public graph for code, memory, and skills. External Project intentionally unifies code symbols, communities, processes, routes, tools, relations, and embeddings in one graph, but Public's owners encode different truth units: Code Graph owns structural code nodes/edges, Memory owns causal memory relations, and Skill Advisor owns skill metadata edges [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/ARCHITECTURE.md:348] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/ARCHITECTURE.md:460] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code_graph/lib/code-graph-db.ts:68] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/lib/storage/causal-edges.ts:18] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/skill_advisor/scripts/skill_graph_compiler.py:42].
- SUPERFICIALLY PORTABLE BUT ADAPT ONLY: default group-member behavior. External Project documents that `@group` defaults to the lexicographically first repo member for query/context, which is acceptable as a External Project group convenience but dangerous for Public routing because Skill Advisor and Code Graph should surface ambiguity instead of silently choosing a member [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/tools.ts:63] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/tools.ts:183] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code_graph/handlers/query.ts:73].
- SUPERFICIALLY PORTABLE BUT REJECT MUTATION: low-confidence rename application. External Project correctly labels graph versus text-search edits, but apply mode does not apply the previewed line patches; it rewrites every word-boundary match in each file, so Public should stop at an advisory plan until exact ranges and apply semantics are proven identical [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/tools.ts:258] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/local/local-backend.ts:2291] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/local/local-backend.ts:2319].
- SUPERFICIALLY PORTABLE BUT DEFER: route/shape intelligence. External Project's `shape_check` requires response keys and consumer property accesses, and its implementation parses keys/counts from FETCHES relation reasons; Public Code Graph lacks that indexed route-consumer lane today, so adding the tool surface first would create false precision [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/tools.ts:443] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/local/local-backend.ts:3227] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/mcp/local/local-backend.ts:3332].
