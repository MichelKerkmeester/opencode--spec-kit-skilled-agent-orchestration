## Runner Contract Portability

- External Project has a real static phase runner, not just architectural labeling: it validates duplicate phase names, verifies every dependency exists, topologically sorts with Kahn's algorithm, and fails on cycles with a concrete cycle path [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:22] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:31] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:40] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:70].
- The most portable part of the External Project DAG is dependency hygiene: each phase receives only declared dependency outputs, which prevents hidden coupling to undeclared phases [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:182] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:184].
- External Project also emits terminal progress events for graph-validation failures and per-phase failures before rethrowing, which is portable to Public because Public already has readiness and scan surfaces that need observable failure states [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:155] [SOURCE: .opencode/specs/system-spec-kit/026-graph-and-context-optimization/001-research-and-baseline/007-external-project/external/src/core/ingestion/pipeline-phases/runner.ts:198] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code-graph/lib/ensure-ready.ts:210].

- The actionable Public adaptation is a small scan-stage contract around existing parse, persist, enrichment, and readiness steps, not a wholesale in-memory graph accumulator, because Public's current write path already treats persistence failure as retryable by leaving the file stale [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code-graph/lib/ensure-ready.ts:227] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code-graph/lib/ensure-ready.ts:230] [SOURCE: .opencode/skill/system-spec-kit/mcp_server/code-graph/lib/ensure-ready.ts:242].
