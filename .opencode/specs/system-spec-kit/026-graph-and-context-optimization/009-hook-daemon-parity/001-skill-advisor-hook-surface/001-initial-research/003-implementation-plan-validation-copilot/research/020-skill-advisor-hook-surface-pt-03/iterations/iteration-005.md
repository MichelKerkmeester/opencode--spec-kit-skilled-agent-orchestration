## Findings

### P1

1. **Gemini schema-version mismatch is acknowledged but not operationalized.** `007` says Gemini stdin shape varies by version and should fail open on unknown payloads, but that is only a prose-level mitigation. There is no requirement or edge-case entry that forces a versioned fixture matrix, no explicit schema discriminator, and no acceptance scenario for "known old payload", "known new payload", and "unknown-but-forward-compatible payload". Because plain fail-open would silently disable advisor injection on version drift, this should be patched before implementation starts. 2. **Copilot's SDK availability matrix is still under-specified.** `007` correctly treats the checked-in repo surface as notification-only and leaves the real SDK-path proof as a validation item, but it still defaults to "yes on recent versions" without pinning a minimum supported SDK/package family or a release-contract floor. Wave-2 already proved the local wrapper is not enough, and the current checked-in test only asserts `{}` notification behavior, so the child needs an explicit compatibility matrix before implementation can safely choose the SDK path as default. External docs currently point at the public-preview Copilot SDK hook surface (`onUserPromptSubmitted` with `additionalContext`) rather than the checked-in wrapper path, which makes the missing version floor a real release-contract gap rather than a theoretical one. 3. **Codex does not define precedence when both stdin and argv are present with different payloads.** `008` already covers the empty/empty case and the single-channel stdin-or-argv contract, but real wrappers can accidentally pass both. This is the main parser edge case still missing from `008`'s runtime contract. ### P2

1. **Claude reconnect/replay after a mid-hook interruption is not called out.** `006` handles malformed stdin, big stdin, missing `cwd`, and subprocess death during read, but it does not cover the real reconnect case where a prompt-submit hook is interrupted, the session resumes, and the same prompt may be replayed or re-associated using folder-level rather than session-level state. Wave-2's concurrency/migration work already established that session identity must beat broader folder recency; if Claude reconnects mid-hook, the adapter should prove that duplicate delivery only yields duplicate-safe diagnostics and never leaks another session's continuity. Because the adapter is advice-only and fail-open, this is a patch-before-implementation concern rather than a blocker.
