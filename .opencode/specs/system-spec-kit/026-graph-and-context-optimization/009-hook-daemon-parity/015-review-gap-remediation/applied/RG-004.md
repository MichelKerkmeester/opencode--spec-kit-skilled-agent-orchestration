# Applied: RG-004

## Source
- Severity: P1
- Bucket: `013-impl-defects`
- Source audit: `013/review:F-003`
- Finding summary: The `partialOutput.omittedAnchors` contract only counted the first post-deadline anchor, so a timeout during one anchor expansion under-reported how many later anchors were skipped.

## Target Files Modified
| Path | Change Summary | Lines Touched |
|---|---|---|
| `.opencode/skill/system-spec-kit/mcp_server/code-graph/lib/code-graph-context.ts` | Updated both deadline exit paths to count every unresolved anchor left in the build loop and stop once deadline pressure makes the rest unreachable. | 118-121, 145-149 |
| `.opencode/skill/system-spec-kit/mcp_server/code-graph/tests/code-graph-context-handler.vitest.ts` | Added real-builder regression coverage plus the supporting graph/seed mocks needed to drive a deterministic multi-anchor timeout path inside the existing handler suite. | 5-54, 59-92, 328-413 |

## Evidence of Change

### `.opencode/skill/system-spec-kit/mcp_server/code-graph/lib/code-graph-context.ts`

Before:
```ts
    if (performance.now() - contextStart > deadlineMs) {
      partialReasons.add('deadline');
      omittedAnchors += 1;
      break;
    }
```

After:
```ts
    if (performance.now() - contextStart > deadlineMs) {
      partialReasons.add('deadline');
      omittedAnchors += resolvedAnchors.length - sections.length;
      break;
    }
```

Added the same remaining-anchor accounting when `expandAnchor(...)` returns `deadlineExceeded`, so a partial current section still counts every anchor left behind:

```ts
    if (expansion.deadlineExceeded) {
      partialReasons.add('deadline');
      omittedAnchors += resolvedAnchors.length - sections.length;
      break;
    }
```
