# Applied: RG-001

## Source
- Severity: P0
- Bucket: `014-impl-defects`
- Source audit: `014/review:F-003`
- Finding summary: The public `advisor_recommend` cache key ignored explicit `workspaceRoot`, which allowed one process to reuse cached recommendations across repositories when freshness matched.

## Target Files Modified
| Path | Change Summary | Lines Touched |
|---|---|---|
| `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/handlers/advisor-recommend.ts` | Threaded the resolved `workspaceRoot` into the public recommendation cache key so cached results stay repository-scoped. | 177 |
| `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/lib/prompt-cache.ts` | Extended the exact prompt-cache key contract to include an optional `workspaceRoot` segment without widening unrelated callers. | 25, 75 |
| `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/tests/handlers/advisor-recommend.vitest.ts` | Added a multi-workspace regression that proves same-prompt, same-freshness requests do not bleed recommendations across repositories and still hit cache within one workspace. | 284-323 |

## Evidence of Change

### `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/handlers/advisor-recommend.ts`

Before:
```ts
  const key = advisorPromptCache.makeKey({
    canonicalPrompt: input.prompt.trim(),
    sourceSignature,
    runtime: 'mcp',
```

After:
```ts
  const key = advisorPromptCache.makeKey({
    canonicalPrompt: input.prompt.trim(),
    sourceSignature,
    workspaceRoot,
    runtime: 'mcp',
```

### `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/lib/prompt-cache.ts`

Before:
```ts
export interface AdvisorPromptCacheKeyParts {
  readonly canonicalPrompt: string;
  readonly sourceSignature: string;
  readonly runtime: string;
```

After:
```ts
export interface AdvisorPromptCacheKeyParts {
  readonly canonicalPrompt: string;
  readonly sourceSignature: string;
  readonly workspaceRoot?: string;
  readonly runtime: string;
```

Before:
```ts
  const payload = [
    parts.canonicalPrompt,
    parts.sourceSignature,
    parts.runtime,
```

After:
```ts
  const payload = [
    parts.canonicalPrompt,
    parts.sourceSignature,
    parts.workspaceRoot ?? '',
    parts.runtime,
```

### `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/tests/handlers/advisor-recommend.vitest.ts`

Added regression:
```ts
  it('isolates cache entries by resolved workspace root', async () => {
    mockReadAdvisorStatus.mockReturnValue(status('live'));
    mockScoreAdvisorPrompt
      .mockReturnValueOnce(scoreResult([recommendation({ skill: 'repo-a-skill' })]))
      .mockReturnValueOnce(scoreResult([recommendation({ skill: 'repo-b-skill' })]));
```

The new assertions verify:
- repo A gets `repo-a-skill`
- repo B gets `repo-b-skill`
- a repeat repo A request becomes a cache hit
- the scorer runs exactly twice, proving workspace isolation plus in-workspace reuse
