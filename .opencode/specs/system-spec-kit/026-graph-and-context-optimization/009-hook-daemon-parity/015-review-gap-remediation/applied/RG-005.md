# Applied: RG-005

## Source
- Severity: P1
- Bucket: `013-impl-defects`
- Source audit: `013/review:F-004`
- Finding summary: No-op incremental scans could report `filesSkipped: 0` and clear the persisted edge-enrichment summary even when fresh files were skipped before parsing and no new graph state was written.

## Target Files Modified
| Path | Change Summary | Lines Touched |
|---|---|---|
| `.opencode/skill/system-spec-kit/mcp_server/code-graph/lib/structural-indexer.ts` | Extended the `indexFiles(...)` return array with a `preParseSkippedCount` field and incremented it when incremental mode skips fresh files before parsing. | 43-45, 1339-1385 |
| `.opencode/skill/system-spec-kit/mcp_server/code-graph/handlers/scan.ts` | Folded the pre-parse skip count into `filesSkipped` and guarded enrichment-summary replacement/clearing behind actual persisted graph writes. | 186-245 |
| `.opencode/skill/system-spec-kit/mcp_server/code-graph/tests/code-graph-scan.vitest.ts` | Added a regression for no-op incremental scans preserving the persisted enrichment summary while still reporting pre-parse skips, and tightened the existing replacement-state regression. | 27-32, 83-95, 154-234 |

## Evidence of Change

### `.opencode/skill/system-spec-kit/mcp_server/code-graph/lib/structural-indexer.ts`

Before:
```ts
export async function indexFiles(config: IndexerConfig, options: IndexFilesOptions = {}): Promise<ParseResult[]> {
  const results: ParseResult[] = [];
```

After:
```ts
export interface IndexFilesResult extends Array<ParseResult> {
  preParseSkippedCount: number;
}

export async function indexFiles(config: IndexerConfig, options: IndexFilesOptions = {}): Promise<IndexFilesResult> {
  const results = [] as IndexFilesResult;
  results.preParseSkippedCount = 0;
```

Before:
```ts
    if (skipFreshFiles && !isFileStale(file)) continue;
```

After:
```ts
    if (skipFreshFiles && !isFileStale(file)) {
      results.preParseSkippedCount++;
      continue;
    }
```

After:

```ts
  const graphEdgeEnrichmentSummary = summarizeGraphEdgeEnrichment(results);
```

Before:
```ts
  if (graphEdgeEnrichmentSummary) {
```

After:
```ts
  if (filesIndexed > 0 && graphEdgeEnrichmentSummary) {
    graphDb.setLastGraphEdgeEnrichmentSummary(graphEdgeEnrichmentSummary);

  }
```

### `.opencode/skill/system-spec-kit/mcp_server/code-graph/tests/code-graph-scan.vitest.ts`

- `filesSkipped` includes the three files skipped before parse
- `filesIndexed` remains `0` for the no-op incremental pass
- `graphEdgeEnrichmentSummary` stays `null` in the response payload
- neither `setLastGraphEdgeEnrichmentSummary(...)` nor `clearLastGraphEdgeEnrichmentSummary()` fires without a replacement write
