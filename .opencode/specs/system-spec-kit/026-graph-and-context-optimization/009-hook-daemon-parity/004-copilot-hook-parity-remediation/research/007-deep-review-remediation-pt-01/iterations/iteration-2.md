# Iteration 2: Custom agents and ACP server surface
## Focus
This iteration narrowed the gap around Q2 and Q6 by checking whether Copilot CLI custom agents or ACP server mode can act like Claude-style prompt/session hooks. The goal was to separate documented subagent/session transport features from any real text-injection channel.

## Actions Taken
1. Read the official Copilot CLI custom-agent concept page to verify what an agent profile can define and whether GitHub describes it as a main-session or subagent surface. Sources: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
2. Read the official custom-agent creation guide and CLI programmatic reference to confirm how `/agent` and `--agent` are loaded and invoked in practice. Sources: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli ; https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-programmatic-reference
3. Read the custom-agent configuration reference to inspect the schema for any dynamic startup-context or per-prompt injection fields beyond prompt/tools/MCP configuration. Sources: https://docs.github.com/en/copilot/reference/custom-agents-configuration ; https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents
4. Inspected the public `github/copilot-cli` changelog and ACP-labeled public issues to map what ACP currently exposes and which missing capabilities users are still requesting. Sources: https://github.com/github/copilot-cli/blob/65944372230c09dc862de673ff6eeb869b9787f3/changelog.md ; https://github.com/github/copilot-cli/issues/2555 ; https://github.com/github/copilot-cli/issues/1245 ; https://github.com/github/copilot-cli/issues/2044

## Findings
- **`/agent` and `--agent` target delegated subagents, not the main session prompt.** GitHub's docs say custom-agent work is done by a subagent with its own context window, and the CLI guide/programmatic reference describe `--agent` as selecting that custom agent for a task. Sources: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli ; https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-programmatic-reference
- **The documented custom-agent schema is static prompt/tools/MCP configuration; no hook-like field for runtime startup/advisor payload injection was found.** The schema covers prompt text, tool filters, and `mcp-servers`, but nothing like `sessionStart`, `userPromptSubmitted`, or a dynamic context callback inside the agent profile. Sources: https://docs.github.com/en/copilot/reference/custom-agents-configuration ; https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents
- **ACP is evolving as a session/transport surface, not as a documented prompt-hook API.** The public repo changelog adds `--acp`, agent/plan session modes, ACP session loading, and client-supplied MCP servers; an open issue asking for all slash commands to be advertised over ACP indicates ACP is still a narrower surface than the full interactive CLI. Sources: https://github.com/github/copilot-cli/blob/65944372230c09dc862de673ff6eeb869b9787f3/changelog.md ; https://github.com/github/copilot-cli/issues/2555
- **Public ACP issue traffic points the wrong way for a hidden context-injection channel.** One open feature request asks for an `ask_user`/`ask_question` extension because ACP currently only covers permission prompts, and another reports that arbitrary custom JSON-RPC methods are not exposed to the agent. That makes ACP look like a constrained host protocol, not an existing channel for pushing arbitrary startup/advisor text into model context. Sources: https://github.com/github/copilot-cli/issues/1245 ; https://github.com/github/copilot-cli/issues/2044

## Questions Answered
- **Q2:** Strongly advanced toward "no" for system-prompt-level startup injection via custom agents. Evidence: custom agents are documented as subagents with their own context window, selected by `/agent` or `--agent`, and configured through static `.agent.md` prompt/tools/MCP metadata rather than runtime callbacks. Sources: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli ; https://docs.github.com/en/copilot/reference/custom-agents-configuration
- **Q6:** Strongly advanced but not fully closed. Evidence now shows ACP is a real server/session surface with MCP/session features, but public release notes and open ACP issues suggest missing custom extension methods and missing richer host interaction, which argues against an already-supported bidirectional context-injection path. Sources: https://github.com/github/copilot-cli/blob/65944372230c09dc862de673ff6eeb869b9787f3/changelog.md ; https://github.com/github/copilot-cli/issues/2555 ; https://github.com/github/copilot-cli/issues/1245 ; https://github.com/github/copilot-cli/issues/2044

## Questions Remaining
- **Q1:** Does Copilot CLI have any documented pre-prompt or lifecycle hook that can mutate prompt/session content, rather than just run side-effectful commands?
- **Q3:** Are `/skills` comparable to Claude Code skills in a way that could carry advisor text into the active model context?
- **Q4:** Can MCP do anything beyond callable tools/data sources in practice, especially from ACP or plugin packaging?
- **Q5:** Is there any startup-loaded config/instruction surface beyond the already-known instruction files that could carry generated session context?
- **Q6:** Is there any actual ACP protocol/spec documentation or SDK contract that explicitly confirms or disproves arbitrary host-to-model context injection?
- **Q7:** Are there any startup env vars beyond documented directory/model/auth settings that could carry payload text?
- **Q8:** What is the least-bad wrapper strategy if only static instructions plus `-p` prepending are available?

## Next Focus
Push on Q4, Q5, and Q7 by mapping instruction-file loading, hook return semantics, and env/config surfaces against the desired startup/advisor payload shape, then compare that with wrapper-based fallback options.
