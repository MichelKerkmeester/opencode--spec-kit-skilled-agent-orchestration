# Iteration 9: agent, skills, MCP, and ACP injection boundaries
## Focus
Resolve whether Copilot CLI's documented agent-facing extension points can inject startup or per-prompt text the way Claude hooks do, with emphasis on `--agent`, `/agent`, ACP, and the real `~/.copilot` customization surfaces.

## Actions Taken
1. Re-read the active deep-research state and strategy packet to keep this pass aligned to Q1/Q2 and avoid repeating exhausted directions.
2. Fetched the current official Copilot CLI docs for the main CLI surface, custom agents, custom instructions, hooks, skills, MCP, ACP, and the `~/.copilot` config directory to map all documented injection points ([about Copilot CLI](https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli), [create custom agents for CLI](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli), [invoke custom agents](https://docs.github.com/en/copilot/how-tos/copilot-cli/use-copilot-cli-agents/invoke-custom-agents), [add custom instructions](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions), [about hooks](https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-hooks), [about agent skills](https://docs.github.com/en/copilot/concepts/agents/about-agent-skills), [add skills](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-skills), [add MCP servers](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-mcp-servers), [ACP server](https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server), [CLI config dir reference](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-config-dir-reference)).
3. Empirically probed the local installation with `copilot help environment`, `copilot help config`, `copilot --help`, and `~/.copilot/*` inspection to compare the documented surfaces with the actual 1.0.34 profile on disk.
4. Cross-compared the docs to distinguish three different classes of extension: always-on custom instructions, delegated subagents/skills, and external-client ACP control.

## Findings
- **Copilot CLI has a documented hook API, but the documented contract is command execution rather than prompt mutation.** Official docs expose `sessionStart`, `userPromptSubmitted`, `preToolUse`, and related hook events for running shell commands, with a schema limited to `bash`/`powershell`, `cwd`, `env`, and `timeoutSec`; the CLI config reference mirrors that by describing repo/user hook definitions, not a "return prompt text" channel. ([about hooks](https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-hooks), [CLI config dir reference](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-config-dir-reference))
- **`--agent` / `/agent` custom agents are delegated subagents with their own context windows, not an always-on startup injector for the main session.** GitHub explicitly says custom-agent work is carried out by a temporary subagent and that agents are selected through `/agent`, prompt inference, or `--agent`; that is compatible with task-specific delegation, not with silently prepending startup context to every main-session turn. ([create custom agents for CLI](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli), [invoke custom agents](https://docs.github.com/en/copilot/how-tos/copilot-cli/use-copilot-cli-agents/invoke-custom-agents))
- **Skills are on-demand instruction packs, not a Claude-style global hook surface.** Skills live in `.github/skills` or `~/.copilot/skills`, and when Copilot decides a skill is relevant its `SKILL.md` is injected into the agent context; GitHub separately recommends using custom instructions for nearly-every-task guidance and skills for more detailed task-specific guidance. ([add skills](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-skills), [about agent skills](https://docs.github.com/en/copilot/concepts/agents/about-agent-skills))
- **Copilot CLI does provide native always-on instruction-loading surfaces.** The documented automatic instruction inputs are repo `.github/copilot-instructions.md`, path-specific `.github/instructions/**/*.instructions.md`, root/current-directory `AGENTS.md`, `$HOME/.copilot/copilot-instructions.md`, and extra directories named by `COPILOT_CUSTOM_INSTRUCTIONS_DIRS`; the main CLI docs also expose `--no-custom-instructions`, confirming this is a first-class startup prompt surface. ([add custom instructions](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli))
- **The real user-level customization root is broader than the sparse current local profile suggested.** GitHub documents `~/.copilot/agents`, `~/.copilot/skills`, `~/.copilot/hooks`, and `~/.copilot/mcp-config.json` as supported user-level surfaces and notes that many of these paths are created on demand, which explains why the current local 1.0.34 profile still only shows a minimal `config.json` plus runtime-managed files until those features are used. ([CLI config dir reference](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-config-dir-reference), [add custom instructions](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions))
- **ACP is the cleanest documented way to inject startup/advisor payloads if Spec Kit becomes the client wrapper instead of relying on an in-process hook.** GitHub's ACP reference shows the external client starting `copilot --acp`, creating a session with `newSession({ cwd, mcpServers: [] })`, then sending prompts via `prompt(...)`; that means a wrapper client could prepend a startup brief or advisor line before each prompt, but the injection would happen outside Copilot CLI rather than through a native passive hook. ([ACP server](https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli))
- **MCP remains documented as pull-based tool/data access, not a server-push prompt channel.** The MCP docs describe servers as sources of external tools, data, and services configured in `mcp-config.json`, after which Copilot may use those tools when relevant; there is no documented mechanism for an MCP server to unilaterally inject text into every session start or prompt submission. ([add MCP servers](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-mcp-servers), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli))

## Questions Answered
- **Q1:** Yes, Copilot CLI has a documented hook/event system (`sessionStart`, `userPromptSubmitted`, etc.), but the documented interface is shell-command execution and hook config, not prompt-text injection into the model context. Evidence: [about hooks](https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-hooks), [CLI config dir reference](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-config-dir-reference).
- **Q2:** Custom agents do **not** look like a system-prompt-at-startup surface; they are delegated subagents with separate context windows, invoked via `/agent`, prompt inference, or `--agent`. Evidence: [create custom agents for CLI](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli), [invoke custom agents](https://docs.github.com/en/copilot/how-tos/copilot-cli/use-copilot-cli-agents/invoke-custom-agents).
- **Q3:** Skills are comparable to reusable instruction packs, but they are loaded when relevant rather than acting as an always-on session hook. Evidence: [add skills](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-skills), [about agent skills](https://docs.github.com/en/copilot/concepts/agents/about-agent-skills).
- **Q4:** MCP is documented as configurable external tools/data access, not as a push mechanism for injecting prompt text. Evidence: [add MCP servers](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-mcp-servers), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli).
- **Q5:** Yes, Copilot CLI has documented automatic instruction files and directories (`copilot-instructions.md`, `AGENTS.md`, `.instructions.md`, home-level instruction file), which are the nearest native startup-context surface. Evidence: [add custom instructions](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli).
- **Q6:** `--acp` does expose a bidirectional protocol that an external client could exploit to prepend context before `prompt(...)`, but only if Spec Kit owns the client side. Evidence: [ACP server](https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server), [use Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli).
- **Q7:** Copilot documents env vars for instruction-directory discovery, home/config relocation, model/provider selection, permissions, and telemetry; none in this pass are documented as arbitrary startup/prompt payload carriers. Evidence: [add custom instructions](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions), [CLI config dir reference](https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-config-dir-reference).
