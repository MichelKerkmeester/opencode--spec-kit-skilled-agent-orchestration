## Findings
- **Hooks exist, but the documented hook contract blocks prompt/session injection.** Copilot CLI documents `sessionStart` and `userPromptSubmitted` hooks, yet the hook reference says `sessionStart` output is ignored and `userPromptSubmitted` output is ignored with “prompt modification not currently supported in customer hooks,” so these hooks cannot inject a model-visible startup payload or advisor brief. Sources: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/use-hooks ; https://docs.github.com/en/copilot/reference/hooks-configuration
- **Custom agents are documented as specialized agent profiles/subagents, not as a hook-like startup extension for the main session.** Agent profiles define prompt/tools/MCP servers and are instantiated for tasks; the custom-agent doc explicitly says the main Copilot agent can run built-in and custom agents as subagents with separate context windows, which is weaker than Claude-style main-session prompt injection. Sources: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **The documented native startup-text channel is file-based custom instructions, including local home-directory scope, not generated per-session hook output.** Copilot CLI automatically adds instructions from `.github/copilot-instructions.md`, `AGENTS.md`, and `$HOME/.copilot/copilot-instructions.md`, and can scan extra directories from `COPILOT_CUSTOM_INSTRUCTIONS_DIRS`; this is the closest documented mechanism to startup context injection, but it is file-backed/static unless an external wrapper rewrites those files between prompts. Sources: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli

## Questions Answered
- **Q1:** Yes, Copilot CLI has a documented hook system (`sessionStart`, `userPromptSubmitted`, `preToolUse`, etc.), but the official hook contract says startup/prompt hook outputs are ignored, so there is no documented pre-prompt mutation API for customer hooks. Evidence: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/use-hooks ; https://docs.github.com/en/copilot/reference/hooks-configuration
- **Q2:** The `--agent` / `/agent` surface maps to custom agent profiles plus built-in agents that the main agent can run as subagents with separate context windows; this narrows the answer toward “no Claude-style startup injection surface,” though a direct CLI-focused `create custom agents` page would still help close the remaining ambiguity around explicit main-session selection semantics. Evidence: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **Q5:** Yes, there is a documented file-based startup-instruction surface: repo `.github/copilot-instructions.md`, `AGENTS.md`, and local `$HOME/.copilot/copilot-instructions.md`, with optional directory expansion via `COPILOT_CUSTOM_INSTRUCTIONS_DIRS`. Evidence: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
