Acceptance gate: `resume` and `compact` surfaces gain richer cached summaries while `startup` and `clear` remain bounded and `memory_context(...resume...)` stays canonical. Acceptance gate: proposal generation runs before mutation, post-save review can reject noisy output, and final writes remain scoped to the current save workflow. Acceptance gate: Public emits documented JSON contracts (`trends`, `skill_usage`, `agent_delegation`, `hook_performance`, `findings`, `recommendations`) with cache pricing and surfaced token counts, while presentation remains a separate concern.
