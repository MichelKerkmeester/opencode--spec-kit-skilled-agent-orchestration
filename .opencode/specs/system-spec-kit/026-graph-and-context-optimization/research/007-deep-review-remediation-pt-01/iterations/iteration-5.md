# Iteration 5: Hook mutation limits and custom agent loading
## Focus
This iteration focused on Q1 and Q2: whether Copilot CLI's documented lifecycle hooks can inject startup or per-prompt payloads, and whether `/agent` / `--agent` can act as a main-session system-prompt extension point. The goal was to separate true injection surfaces from delegated-subagent surfaces.

## Findings
- **Copilot CLI does expose documented lifecycle hooks for `sessionStart` and `userPromptSubmitted`, but those hooks are not a prompt-mutation API.** The concept page says hooks run shell commands at lifecycle moments, and the reference page states that `sessionStart` output is ignored and `userPromptSubmitted` output is ignored because prompt modification is not currently supported in customer hooks. That means hooks can log, validate, or side-effect, but they cannot directly prepend a `SessionStart:startup` payload or an `Advisor: ...` brief into the model input. [SOURCE: https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-hooks] [SOURCE: https://docs.github.com/en/copilot/reference/hooks-configuration]
- **`/agent` and `--agent` are documented as custom-agent profile selection for delegated subagents, not as a way to replace or augment the main session's system prompt.** GitHub says custom agents are instantiated when assigned to a task, and the how-to says their work is carried out using a temporary subagent with its own context window; the customization comparison page likewise defines custom agents as specialized abilities delegated by the main CLI agent. [SOURCE: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents] [SOURCE: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli] [SOURCE: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/comparing-cli-features]
- **The documented custom-agent schema supports prompts, tools, and MCP servers, but not an "always load at session start" or "inject into every user prompt" property.** The concept page describes agent profiles as Markdown with YAML frontmatter for behavior, tools, and optional MCP servers, while the configuration reference details properties such as `tools` and `mcp-servers` plus processing/versioning rules; none of the documented properties describe main-agent startup injection. [SOURCE: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents] [SOURCE: https://docs.github.com/en/copilot/reference/custom-agents-configuration]
- **GitHub's own CLI customization model keeps persistent context, lifecycle automation, and delegated specialization as separate mechanisms.** The comparison page says custom instructions are loaded at session start, hooks run shell commands at lifecycle moments, and custom agents are delegated specialist definitions; the CLI overview uses the same split. This makes `/agent` a poor parity target for the Claude hook behavior and pushes the remaining viable paths toward custom instructions, ACP, or an external wrapper. [SOURCE: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/comparing-cli-features] [SOURCE: https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli]

## Questions Answered
- **Q1 moved forward substantially:** Copilot CLI has a documented lifecycle hook system with `sessionStart` and `userPromptSubmitted`, but the hook reference explicitly says their outputs are ignored for session/prompt mutation, so this is not a first-party pre-prompt injection API. Evidence: https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-hooks ; https://docs.github.com/en/copilot/reference/hooks-configuration
- **Q2 moved forward substantially:** `/agent` / `--agent` selects a custom-agent profile that runs as a delegated subagent with its own context window and task-scoped instantiation, not as an always-on system-prompt extension for the main CLI session. Evidence: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/create-custom-agents-for-cli ; https://docs.github.com/en/copilot/reference/custom-agents-configuration

- **Q6:** Whether `--acp` provides a practical bidirectional context-injection path when Copilot CLI is run behind an external client.
- **Q7:** Whether any additional Copilot-specific startup environment variables besides the already-found instructions-directory variable can influence session context.
