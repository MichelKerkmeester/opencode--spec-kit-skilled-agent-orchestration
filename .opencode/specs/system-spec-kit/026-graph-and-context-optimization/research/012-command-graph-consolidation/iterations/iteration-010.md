## 3. Final Findings
1. The deep-research side is implementation-ready only if `REQ-001`, `REQ-003`, `REQ-004`, and `REQ-010` are rewritten as one shared contract: acquire the lock first, classify `folder_state`, append pre-init context only inside exact anchors, and replace post-synthesis findings only inside one generated fence under a stable host anchor. The converged recommendation from iterations 005, 006, and 007 is consistent: keep detection late in INIT so the JSONL sink already exists, keep write-back in SYNTHESIS rather than SAVE, and fail closed on anchor or marker drift rather than guessing. 2. `REQ-005` and `NFR-R02` are the packet's biggest unresolved contradiction. The current spec simultaneously says `/start` writes the three canonical artifacts as one transactional unit and says that if `generate-description.js` crashes after `spec.md` is written, the `spec.md` remains for later retry. Runtime evidence confirms why the contradiction exists: `generate-description.js` writes only `description.json`, and nothing in the current helper surfaces provides all-or-nothing publication for the trio. The fix is not to weaken the packet into vague recovery language; it is to make the command-layer contract explicit: canonical artifact emission uses staged temp siblings plus final commit, while pre-existing canonical files remain untouched on failure and the command surfaces the exact repair step. 3. The `graph-metadata.json.manual.*` schema is no longer an open design question. Iterations 004, 005, and 009 all converge on the same answer: keep the existing `manual.depends_on`, `manual.related_to`, and `manual.supersedes` arrays, but require object entries rather than strings. The safe minimum contract is `[{ packet_id, reason, source, spec_folder?, title? }]`, with `packet_id` as the authoritative edge target, `reason` plus `source` required for provenance, and `spec_folder` or `title` treated as optional denormalized hints. Dedupe should be by `packet_id` within each relation type so repeated repair or delegated reruns do not multiply edges. 4. Placeholder-upgrade must move from heuristic text search to an explicit seed-marker contract. The converged pattern is: deep-research seed-create writes Level 1 placeholder blocks only inside known anchors and wraps each placeholder-bearing region in exact marker pairs such as `<!-- BEGIN SPECKIT SEED:deep-research/scope --> ... <!-- END SPECKIT SEED:deep-research/scope -->` and `<!-- BEGIN SPECKIT SEED:deep-research/requirements --> ... <!-- END SPECKIT SEED:deep-research/requirements -->`. `/start` detects `repair_mode=upgrade_placeholders` only when those markers remain or when the paired sentinel text is still present; it exits successfully only when every tracked seed block is replaced with user-confirmed content or the literal fallback `N/A - insufficient source context`, and then removes the seed markers. A generic `"contains TODO"` test is explicitly too broad. 5. `/start` needs two distinct completion paths: canonical artifact emission and optional memory save. Runtime evidence makes the split unavoidable. `generate-context.js` is a JSON-primary memory-save workflow with memory-quality gates and `INSUFFICIENT_CONTEXT_ABORT` behavior for thin inputs, which is appropriate for session narrative capture but wrong for empty-folder creation or metadata-only repair. Therefore `/start` should treat `spec.md` + `description.json` + `graph-metadata.json` as the baseline success boundary, then offer an optional memory-save mode only when direct `/start` has structured session evidence worth saving. Metadata-only repair and placeholder-upgrade must skip `generate-context.js`. ## 5. NEW REQ SUGGESTIONS
1. `REQ-011 - Intake resume and re-entry contract`
   Acceptance criteria:
   - `/start`, delegated `/plan`, and delegated `/complete` normalize `start_state` into `empty-folder`, `partial-folder`, `repair-mode`, `placeholder-upgrade`, or `populated-folder`.
   - Re-entry persists and returns `resume_question_id`, `repair_mode`, and `reentry_reason` so reruns resume at the first unresolved question or seed marker rather than replaying already accepted answers.
   - Success is blocked until the canonical trio validates and any tracked seed markers are cleared or explicitly replaced with `N/A - insufficient source context`.
2. Companion non-REQ delta: `NFR-S04 - Structured intake audit contract`
   Acceptance criteria:
   - Typed events exist for `start_intake_delegated`, `start_mode_resolved`, `start_reentry_detected`, `manual_relationships_recorded` or `manual_relationships_skipped`, `start_artifacts_committed`, and `start_memory_save_requested` or `start_memory_save_skipped`.
   - The audit payloads include `parent_command`, `spec_path`, `start_state`, `repair_mode`, `resume_question_id`, and counts or targets for recorded manual relationships.

## 8. EXECUTIVE SUMMARY AND FINAL RECOMMENDATIONS
Executive summary: The packet direction is stable, but the current `spec.md` is still one edit pass away from implementation readiness. The deep-research side is converged: late INIT detection, single-writer locking, seeded Level 1 placeholder blocks, and fenced post-synthesis findings sync. The `/start` side is also converged at the schema level: canonical trio first, manual relationship objects under `manual.*`, placeholder-upgrade driven by explicit seed markers, and a strict split between metadata repair and optional memory save. The only remaining blockers are packet-local wording contradictions, especially around transactionality and save behavior. `/spec_kit:plan` should treat this iteration as the authoritative edit list, not as another research branch.

Final recommendations for synthesis and planning:
- Rewrite `REQ-001` through `REQ-010` and `NFR-R02` before starting implementation work.
- Add `REQ-011` and `NFR-S04` in the same edit pass so re-entry and audit behavior are not left implicit.
- Remove memory-indexing from baseline `/start` success; keep it as an explicit optional mode.
- Replace generic TODO placeholder language with exact deep-research seed markers and clear conditions.
- Close the now-resolved open questions so the packet no longer advertises already-decided design branches.
