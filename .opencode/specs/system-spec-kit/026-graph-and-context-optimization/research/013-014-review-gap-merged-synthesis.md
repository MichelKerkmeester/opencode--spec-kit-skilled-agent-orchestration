## The One P0

`RG-001` is the only `P0`, and it is the highest-risk issue in the merged set because it can return recommendations from the wrong repository without any crash or obvious operator signal.

- Source: `014/review:F-003`
- Bucket: `014-impl-defects`
- Summary: `advisor_recommend` accepts explicit `workspaceRoot`, but the cache key ignores that root. If freshness state matches, one process can reuse cached recommendations across repositories.
- Why it matters: this is a live correctness/isolation failure on a public MCP surface. It breaks repository scoping and can surface the wrong skill recommendations for the wrong workspace.
- Evidence:
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/handlers/advisor-recommend.ts:163`
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/handlers/advisor-recommend.ts:174`
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/lib/prompt-cache.ts:22`
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/lib/prompt-cache.ts:70`
- Target files:
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/handlers/advisor-recommend.ts`
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/lib/prompt-cache.ts`
  - `.opencode/skill/system-spec-kit/mcp_server/skill-advisor/tests/handlers/advisor-recommend.vitest.ts`
- Recommended fix: include resolved `workspaceRoot` in the cache key, or key from a workspace-scoped freshness signature, then add a multi-workspace regression that proves recommendations cannot bleed across repositories.
