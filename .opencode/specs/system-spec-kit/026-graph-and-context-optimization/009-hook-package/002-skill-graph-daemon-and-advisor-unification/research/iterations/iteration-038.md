## Analysis
The risk register should mirror the staged architecture rather than introduce new scope. The highest operational risks sit in `027/001`: watcher scope, WAL writer coordination, stale/unavailable prompt behavior, and rebuild-from-source recovery. A2 deliberately avoids broad all-files watching, A4 sets the SQLite transaction as the consistency boundary, and A8 requires scan leases plus fail-open recovery, so the mitigation is not more watcher breadth; it is bounded scope, post-commit generation publication, writer leases, and explicit stale/unavailable diagnostics (`iteration-002.md:43`-`iteration-002.md:48`; `iteration-004.md:43`-`iteration-004.md:55`; `iteration-008.md:46`-`iteration-008.md:53`; `iteration-037.md:44`).

The highest routing-quality risks sit in `027/002` and `027/003`. B4 and B7 show that derived triggers can improve recall while also widening over-fire and ranking-poisoning surfaces if generated phrases enter the current direct `(signal)` lane. X3 confirms this as the only real cross-track adjustment: derived keywords are allowed, but only as a separate capped/provenance-aware lane with freshness and corpus gates. C4 then gives the scorer-level mitigation: centralized analytical fusion, normalized channels, caps, confidence separation, and shadow-only learned scoring (`iteration-012.md:32`-`iteration-012.md:44`; `iteration-015.md:44`-`iteration-015.md:56`; `iteration-019.md:45`-`iteration-019.md:57`; `iteration-034.md:35`-`iteration-034.md:50`; `iteration-037.md:45`-`iteration-037.md:46`).

The highest migration risks sit in `027/004` and `027/005`: moving from Python subprocess behavior to MCP-native TypeScript while preserving hook, plugin, CLI, privacy, and manual testing contracts. D2/D3/D4 define a clear module and MCP-tool target, but D4's confidence is only medium because the Python file is large and parity must be staged. D7 and D8 mitigate by keeping `skill_advisor.py` and the OpenCode bridge as compatibility surfaces until parity and documentation migration are complete (`iteration-025.md:55`-`iteration-025.md:67`; `iteration-026.md:44`-`iteration-026.md:90`; `iteration-027.md:48`-`iteration-027.md:60`; `iteration-030.md:65`-`iteration-030.md:78`; `iteration-031.md:32`-`iteration-031.md:44`).
