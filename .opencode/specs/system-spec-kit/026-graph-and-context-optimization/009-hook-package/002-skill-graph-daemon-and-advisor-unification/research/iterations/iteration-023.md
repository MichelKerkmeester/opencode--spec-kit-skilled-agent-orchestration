## Analysis
The accuracy target should be a release gate with two tiers. The current baseline is 112/200 exact top-1 matches, so a 70% full-corpus target means 140/200 correct: +28 correct prompts, +14 percentage points absolute, and roughly +25% relative improvement over 56%. That is ambitious enough to justify adding deterministic trigger derivation, analytical fusion, skill-graph priors, and ambiguity calibration, yet it is not fantasy math: the report has 37 `UNKNOWN` outputs alone, so eliminating a large share of stale/missing-index failures can plausibly reach much of the required lift before prompt-time embedding search is considered.

The frozen holdout should use the C7 20% split. With 200 prompts and the default holdout selector, that is 40 rows; the minimum adoption gate should be >=70% top-1 on holdout, or at least 28/40 correct. Because a 40-row holdout is coarse, it should not stand alone. The full 200-row score, holdout score, per-skill slices, `none` false-fire count, `UNKNOWN` count, ambiguity behavior, and latency gates must move together. In particular, C5's ambiguity work needs a concrete target: reduce false confident single-skill recommendations on close top-two prompts, and avoid turning genuinely multi-skill prompts into a brittle single winner.

Hybrid-search literature supports expecting meaningful gains when complementary rankers are fused, but it also argues for caution on cost and domain variance. RRF's original claim is consistent improvement over individual rankers, BEIR shows stronger methods often cost more, and C6 already found that memory embedding search sits in a different latency class than the advisor hook. Therefore, lexical + skill-graph + deterministic analytical fusion can use the 70% gate, but prompt-time embedding/hybrid retrieval should need a stricter gate: >=75% full-corpus top-1 (150/200), >=72.5% holdout top-1 (29/40), no latency regression beyond C6's hook budget, and at least two shadow-evaluation cycles before promotion.

The final C8 target set should therefore be: adopt-now complexity is justified at >=70% exact top-1 on both the full corpus and frozen holdout, `UNKNOWN` reduced from 37 to <=10, no increase in gold-`none` false fires, no regression for explicit skill-name prompts, and secondary ranking metrics such as Recall@3/MRR@5/Kendall tau tracked because the system may correctly surface multi-skill ambiguity without changing exact top-1. A 75% stretch gate should be required for any learned or prompt-time semantic channel, with C4/C7 keeping that path shadow-only until it clears both accuracy and performance gates.

## Verdict
- **Call:** adopt now
- **Confidence:** high
- **Rationale:** Adopt concrete C8 release gates now: 70% full-corpus and holdout top-1 for deterministic C1-C7 improvements, with stricter 75% full-corpus and 72.5% holdout gates before prompt-time embedding or learned scoring leaves shadow mode.
