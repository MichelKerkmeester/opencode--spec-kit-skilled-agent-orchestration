## Analysis
The repo does not need a brand-new idea for ablation; it already has the core procedure. `eval_run_ablation` establishes the correct experimental shape for G2: freeze a baseline, disable one lane at a time, measure paired deltas per query, classify helpful vs harmful contributions, and store the run so later tuning decisions are auditable rather than anecdotal. The important adaptation is scope: G2 should reuse that control pattern for advisor lanes, not reuse the retrieval tool's default Recall@20 metric as-is. `.opencode/skill/system-spec-kit/mcp_server/lib/eval/ablation-framework.ts:6-17,537-545,588-675,723-845`, `.opencode/skill/system-spec-kit/feature_catalog/07--evaluation/01-ablation-studies-evalrunablation.md:18-26`

For advisor routing, the fixture matrix is already mostly specified by r01. Every lane-off run should execute against the 200-prompt corpus, the frozen 20% holdout, the explicit-skill slice, the gold-`none` safety slice, the derived-only / derived-dominant slice, the ambiguity / `UNKNOWN` slice, adversarial stuffing cases, and latency plus parity checks. Those fixtures directly map to the risks r01 identified: derived phrases must not create threshold-clearing false fires, semantic-shadow must remain non-blocking until promotion gates pass, and the TypeScript port must preserve Python outcomes before the shim is retired. `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/009-hook-package/002-skill-graph-daemon-and-advisor-unification/research/027-skill-graph-daemon-and-advisor-unification-pt-01/research.md:135-139,147-159`, `.opencode/skill/skill-advisor/feature_catalog/feature_catalog.md:439-448,631-635`, `.opencode/skill/skill-advisor/scripts/skill_advisor_regression.py:190-197,223-229`

That leads to an adopt-now protocol with three mandatory phases. First, run the fully enabled baseline using G1's initial weights and record top-1, pass-threshold, abstain/`UNKNOWN`, false-fire, and latency outcomes across the fixture slices. Second, perform one-lane-off ablations for `explicit_author`, `derived_generated`, `lexical`, and `graph/causal`; keep `semantic-shadow` in shadow-only ablations until G5/G6 promotion gates are satisfied, because r01 still forbids it from influencing live routing. Third, reject any weight move or lane promotion that improves aggregate top-1 while failing a protected slice such as gold-`none`, explicit-skill stability, parity, or latency. `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/009-hook-package/002-skill-graph-daemon-and-advisor-unification/research/027-skill-graph-daemon-and-advisor-unification-pt-01/iterations/iteration-050.md:23-35`, `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/009-hook-package/002-skill-graph-daemon-and-advisor-unification/research/027-skill-graph-daemon-and-advisor-unification-pt-01/research.md:147-159`, `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/009-hook-package/002-skill-graph-daemon-and-advisor-unification/research/027-skill-graph-daemon-and-advisor-unification-pt-01/next-research-paths.md:72-76,121-127`
