# Iteration 4: Custom instructions, skills, and ACP transport
## Focus
This iteration targeted the remaining main-session injection surfaces around Copilot CLI custom instructions, skills, and `--acp`. The goal was to determine whether any of them can carry Spec Kit Memory startup/advisor payloads into the active session without patching Copilot itself.

## Actions Taken
1. Read the active packet state, strategy, and prior iteration so this pass stayed aligned with the loop and avoided repeating already-proven hook/agent findings. Sources: `deep-research-state.jsonl`; `deep-research-strategy.md`; `iterations/iteration-3.md`.
2. Fetched the official Copilot CLI overview plus the custom-instructions guide to verify which files and environment variables are documented as automatic prompt context inputs. Sources: https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli ; https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions
3. Fetched the official agent-skills concept page to determine whether `/skills` is a dynamic prompt-injection surface or a reusable instructions/resources mechanism. Sources: https://docs.github.com/en/copilot/concepts/agents/about-agent-skills ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
4. Fetched the official ACP server reference and GitHub changelog entry for ACP public preview to map what `--acp` actually exposes to an external client. Sources: https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server ; https://github.blog/changelog/2026-01-28-acp-support-in-copilot-cli-is-now-in-public-preview/
5. Empirically probed the live `~/.copilot` directory and `config.json` keys to check whether this installation already exposes local agents/skills/hooks/instruction config surfaces. Sources: local `find ~/.copilot -maxdepth 2`; local `~/.copilot/config.json` key inspection.

## Findings
- **Copilot CLI does have documented main-session instruction files, and they are auto-added to requests.** GitHub documents repository-wide `.github/copilot-instructions.md`, path-specific `.github/instructions/**/*.instructions.md`, root/CWD `AGENTS.md`, and local `$HOME/.copilot/copilot-instructions.md`; it also documents `COPILOT_CUSTOM_INSTRUCTIONS_DIRS` as a directory-discovery env var, not an inline payload env var. Sources: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **Skills are capability packs, not a documented fixed-format advisor-brief injection channel.** GitHub describes skills as folders of instructions, scripts, and resources loaded when relevant from project or personal skill directories such as `.github/skills` and `~/.copilot/skills`; this is conceptually closer to reusable task augmentation than to a guaranteed `Advisor: ...` line prepended on every prompt. Sources: https://docs.github.com/en/copilot/concepts/agents/about-agent-skills ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **ACP is the first documented surface that can carry host-supplied context objects, but only when Copilot CLI is run as a server behind another client.** GitHub's ACP docs say clients can initialize sessions, set working directories, and send prompts with text, images, and context resources; the changelog frames this as a way for IDEs, custom frontends, CI/CD systems, and multi-agent systems to integrate with Copilot. That suggests wrapper-level context injection is plausible through an ACP client, not through the stock `copilot` interactive prompt loop by itself. Sources: https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server ; https://github.blog/changelog/2026-01-28-acp-support-in-copilot-cli-is-now-in-public-preview/
- **Custom agents still look like delegated subagents, while instruction files remain the documented main-session context path.** The custom-agent docs say an agent profile is instantiated when assigned to a task and defines prompts/tools/MCP servers, whereas the custom-instructions docs explicitly say instruction files are automatically added to submitted requests. Sources: https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-custom-agents ; https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions

## Questions Answered
- **Q3 moved forward:** skills are documented as directories of instructions/scripts/resources loaded when relevant, which makes them more like reusable augmentation bundles than a Claude-style per-prompt hook brief. Evidence: https://docs.github.com/en/copilot/concepts/agents/about-agent-skills ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **Q5 moved forward substantially:** Copilot CLI does have documented startup-readable instruction files (`.github/copilot-instructions.md`, `AGENTS.md`, `$HOME/.copilot/copilot-instructions.md`) and a directory-discovery env var, but this iteration found no documented `config.json` key or separate hook/plugin file that injects arbitrary startup payload text. Evidence: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions ; https://docs.github.com/en/copilot/concepts/agents/about-copilot-cli
- **Q6 moved forward substantially:** `--acp` is not just a hidden local flag; GitHub documents it as a real server mode where external clients create sessions and send prompt/context payloads. That opens a possible parity workaround only through an external ACP client layer, not by normal CLI self-injection. Evidence: https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server ; https://github.blog/changelog/2026-01-28-acp-support-in-copilot-cli-is-now-in-public-preview/
- **Q7 moved forward:** the only clearly documented environment variable discovered in this pass is `COPILOT_CUSTOM_INSTRUCTIONS_DIRS`, which changes where Copilot looks for instruction files rather than carrying the advisor/startup payload directly. Evidence: https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/add-custom-instructions ; https://docs.github.com/en/copilot/reference/copilot-cli-reference/acp-server

## Questions Remaining
- **Q4:** Whether `/mcp` / `--additional-mcp-config` can do anything beyond tool/data exposure in the main CLI session, especially when combined with instructions or hooks.
- **Q6:** Whether the ACP protocol surface includes a practical, durable way to inject both startup context and a per-prompt advisor brief without rebuilding too much CLI UX.
- **Q7:** Whether any additional Copilot-specific startup env vars besides `COPILOT_CUSTOM_INSTRUCTIONS_DIRS` exist and are documented.
- **Q8:** What the least-friction wrapper looks like if normal CLI mode still cannot prepend dynamic per-prompt text.

## Next Focus
Iteration 5 should target Q4 and the remaining Q6/Q8 gap by inspecting MCP configuration docs plus ACP client semantics to decide whether the best viable path is MCP+instructions, ACP wrapper, or a plain shell wrapper.
