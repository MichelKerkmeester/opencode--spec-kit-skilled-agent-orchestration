## Analysis
The memory MCP retrieval stack strongly favors analytical fusion for live ranking. RRF and adaptive fusion provide deterministic channel combination, intent-specific semantic/keyword/recency/graph weights, bounded overlap bonuses, graph caps, score normalization, channel representation enforcement, and confidence truncation. Stage 2 then acts as a single scoring authority so later boosts do not double-count intent or leave score aliases stale. That is exactly the shape the advisor needs: multiple independent evidence channels, explicit caps, and debug-friendly attribution.

The current advisor is already a hand-tuned analytical scorer, not a learned ranker. It has fixed graph multipliers for `enhances`, `siblings`, and `depends_on`, explicit phrase and keyword boosts, confidence/uncertainty formulas, margin calibration, and conflict penalties. C3's recommendation to keep skill relations in `skill_edges` means C4 should not import memory rows or memory causal scoring directly; instead, it should adapt the same fusion architecture to advisor signals: keyword/phrase evidence, semantic skill hits, skill-graph priors, explicit skill mentions, conflict penalties, and confidence calibration.

Learned scoring exists in the memory stack, but the live contract is deliberately conservative. The learned combiner has a useful eight-feature shape and ridge-regression implementation, yet Stage 2 only runs it as shadow scoring and logs manual-vs-learned deltas when a model is available. The environment reference reinforces that shadow learning is opt-in and not a live ranking authority. For Phase 027, that argues against replacing advisor scoring with learned weights now, especially before C7 evaluates the labeled prompt corpus and C8 sets target accuracy thresholds.

Therefore the immediate adoption should be an analytical advisor fusion layer modeled on memory MCP: centralize score mutation, normalize all channel outputs, cap graph/causal contributions, keep confidence separate from structural trust, and expose explainability for why a skill was recommended. Learned fusion should be wired only as a shadow/prototype path using the existing learned-combiner feature vocabulary once advisor-specific labels and holdout metrics exist.

## Verdict
- **Call:** adopt now
- **Confidence:** high
- **Rationale:** Adopt analytical fusion now because the repo already has robust deterministic fusion, capping, confidence, attribution, and ablation patterns, while learned weighting is explicitly shadow-only and lacks C7/C8 calibration for advisor routing.
