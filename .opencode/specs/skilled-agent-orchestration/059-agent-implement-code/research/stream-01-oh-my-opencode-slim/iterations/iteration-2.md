## Questions Answered

- Q2 answered: the plugin does not implement stack-specific detection for Node/Go/Rust/etc. Its reusable stack-agnostic pattern is a codemap skill that asks the agent to inspect repository structure, infer include/exclude globs, then maintain hash-based folder maps.
- Q4 mostly answered: write safety combines `apply_patch` preflight/rewrite/verification, prompt-level write/delegation nudges, fixer prompt constraints, and per-agent MCP permission maps. Direct `edit`/`write` rewriting is explicitly out of scope.
- Q3 close-out: `SUBAGENT_DELEGATION_RULES` appears dead/unused after exact search; runtime caller restriction is instead OpenCode agent `mode` classification plus task-tool validation and retry guidance.
- Q5 partially advanced: task dispatch shape, `task_id` resume semantics, and `parentID` event semantics are now captured.
