# Focus

Q3 — caller-restriction enforcement in `opencode-swarm-main`, with priority on the concrete harness-level mechanism that prevents workers from invoking other workers directly.

# Actions Taken

1. Inspected the adversarial permission test to identify the expected boundary and edge cases.
2. Inspected agent registry/config generation in `src/agents/index.ts` to locate the OpenCode SDK configuration that grants or withholds task delegation.
3. Inspected the architect prompt and coder prompt to separate hard harness enforcement from prompt-level role discipline.
4. Checked tool whitelist and delegation-envelope code for adjacent dispatch-contract reinforcement.

# Findings

1. The concrete caller-restriction mechanism is generated OpenCode agent config: only agents whose generated name is exactly `architect` or ends with `_architect` receive `mode = "primary"` and `permission = { task: "allow" }`; all others receive `mode = "subagent"` and no task permission. This is the harness-level boundary for architect-only worker dispatch. Citation: `src/agents/index.ts:651`.

2. The adversarial test confirms the default swarm grants task permission to `architect` and denies it to non-architect agents including `coder`, `reviewer`, `explorer`, `sme`, `critic`, `test_engineer`, and `docs`. Citation: `src/agents/architect-permission.adversarial.test.ts:43`.

3. The same test confirms prefixed multi-swarm architects such as `cloud_architect`, `local_architect`, `mega_architect`, and `paid_architect` are legitimate dispatch-capable primary agents, while prefixed workers such as `cloud_coder` and `cloud_reviewer` remain without task permission. Citations: `src/agents/architect-permission.adversarial.test.ts:55`, `src/agents/architect-permission.adversarial.test.ts:104`.

4. The permission classifier is suffix-based, not registry-backed: `name === "architect" || name.endsWith("_architect")`. The adversarial matrix explicitly labels `not_an_architect`, `_architect`, and `__architect` as matching cases, which means a malicious or careless generated agent name ending in `_architect` would become primary and task-capable. Citation: `src/agents/architect-permission.adversarial.test.ts:123`.

5. Tool filtering reinforces but does not replace caller restriction. `AGENT_TOOL_MAP` gives the architect the broad orchestration/tool set, while `coder` has a narrower implementation/search/build set and no task dispatch capability in the generated SDK config. Citations: `src/config/constants.ts:48`, `src/config/constants.ts:124`, `src/agents/index.ts:753`.

6. The architect prompt encodes an execution protocol that requires `declare_scope` before coder delegation and states that skipping it causes coder writes to be blocked by scope-guard. This is write-safety enforcement around architect-to-coder dispatch, not the primary caller-restriction mechanism. Citation: `src/agents/architect.ts:1186`.

7. The coder prompt adds prompt-level role discipline: the coder must not use the Task tool or delegate, and if it sees references to other agents it must treat them as orchestrator context. The prompt also contains a write-blocked protocol forbidding bypass through shell redirection or interpreter writes. Citations: `src/agents/coder.ts:3`, `src/agents/coder.ts:111`.

8. Dispatch payloads are validated through a `DelegationEnvelope` shape requiring fields like `taskId`, `targetAgent`, `action`, `commandType`, `files`, and `acceptanceCriteria`, and validation blocks `slash_command` delegation plus invalid target agents. This supports dispatch integrity, but it does not itself decide who is allowed to invoke workers. Citations: `src/types/delegation.ts:6`, `src/hooks/delegation-gate.ts:238`, `src/hooks/delegation-gate.ts:254`, `src/hooks/delegation-gate.ts:265`.

# Questions Answered

Q3 answered. The concrete enforcement is OpenCode SDK agent configuration generated by `getAgentConfigs`: architect agents are `primary` with `permission.task = "allow"`; worker agents are `subagent` without task permission. The highest-risk caveat is the suffix-based architect classifier, which can overmatch any generated name ending in `_architect`.
