### D3: Reducer Family Fix

**What**: `inferFamily()` at `reduce-state.cjs:66` hardcodes `session-handover` as the default.

**Fix**: Derive family from the record's `profileId`. Use a simple mapping:
- `handover` → `session-handover`
- `context-prime` → `session-bootstrap`
- Everything else → use the profileId itself as the family name (e.g., `agent-improver` → `agent-improver`)

This way the dashboard shows the actual profile identity instead of a misleading label.

**File**: `.opencode/skill/sk-improve-agent/scripts/reduce-state.cjs` line 66

### D5: Fix Accepted Counting

**What**: Line 207 of `reduce-state.cjs` only counts `record.type === 'accepted'`. Normal scored records have a `recommendation` field (`candidate-acceptable`, `candidate-better`) but no `type: 'accepted'`.

**Fix**: Also count records where `recommendation` is `candidate-acceptable` or `candidate-better` as accepted. Rejected = `candidate-worse` or `candidate-rejected`. This makes the dashboard counters meaningful for normal scoring runs.

**File**: `.opencode/skill/sk-improve-agent/scripts/reduce-state.cjs` lines 207-215
