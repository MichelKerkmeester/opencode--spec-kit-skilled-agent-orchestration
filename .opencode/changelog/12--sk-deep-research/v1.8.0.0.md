This release adds CLI executor selection to `sk-deep-research`. The loop's iteration dispatch now branches on `config.executor.kind` ‚Äî `native` (default, preserves current `@deep-research` + Opus dispatch bit-identically) or `cli-codex` (invokes `codex exec` with configurable model, reasoning effort, and service tier). YAML keeps owning state, convergence, lifecycle, and the reducer. The CLI is a tool inside the workflow, not a replacement for it.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/018-deep-loop-cli-executor/` (Level 3)

---

## What Changed

#### Two-branch dispatch in the YAML workflow

`step_dispatch_iteration` in both auto and confirm YAMLs was refactored. The old hardcoded `agent: deep-research, model: opus` block is now the `if_native` branch of a selector keyed on `config.executor.kind`. Users who never touch the new config see identical behavior to previous releases ‚Äî same agent, same model, same iteration file, same JSONL record.

The new `if_cli_codex` branch invokes:

```bash
codex exec \
  --model <config.executor.model> \
  -c model_reasoning_effort="<config.executor.reasoningEffort>" \
  -c service_tier="<config.executor.serviceTier>" \
  -c approval_policy=never \
  --sandbox workspace-write \
  - < {spec_folder}/research/prompts/iteration-{n}.md
```

Both branches share a pre-dispatch `render_prompt_pack` step that renders the new template to a prompt file, a post-dispatch `validateIterationOutputs` step that asserts iteration file + JSONL delta + required fields, and a `record_executor_audit` step that appends `executor: {kind, model, reasoningEffort, serviceTier}` to the iteration's JSONL record on non-native runs.

#### Executor-agnostic prompt pack

A new template at `.opencode/skill/sk-deep-research/assets/prompt_pack_iteration.md.tmpl` replaces the inline `context: |` block inside the YAML. The template is rendered identically whether consumed by the native agent's context or piped into `codex exec` stdin. LEAF constraint, 12 tool-call budget, `graphEvents` optional field, and the explicit iteration-file + JSONL delta output contract are all preserved in the template.

#### Config schema additions

`assets/deep_research_config.json` gains an `executor` block with six fields:

```json
"executor": {
  "kind": "native",
  "model": null,
  "reasoningEffort": null,
  "serviceTier": null,
  "sandboxMode": null,
  "timeoutSeconds": 900
}
```

Validation lives in the new shared `executor-config.ts` module. Invalid combinations (e.g. `cli-codex` without `model`, or reserved-but-not-wired kinds like `cli-copilot`) throw typed errors with spec-pointer messages.

#### Setup flag parsing

The command doc at `.opencode/command/spec_kit/deep-research.md` now accepts five new flags during setup: `--executor`, `--model`, `--reasoning-effort`, `--service-tier`, and `--executor-timeout`. Precedence is CLI > config file > schema default. The consolidated setup prompt adds a conditional executor question when nothing is specified in flags or topic text.

#### Contract documentation

`SKILL.md` replaces the forward-looking executor-CLI note with a full **Executor Selection Contract** section documenting supported kinds, invariants, failure modes, JSONL audit field format, and the template path. `references/loop_protocol.md` ¬ß3 adds a new **Executor Resolution** subsection explaining how branch resolution interacts with the existing `schema_mismatch` ‚Üí `stuck_recovery` path.

#### New shared modules

Four TypeScript modules land under `.opencode/skill/system-spec-kit/mcp_server/lib/deep-loop/`:

- `executor-config.ts` ‚Äî Zod schema, `parseExecutorConfig`, `resolveExecutorConfig`, two typed error classes
- `prompt-pack.ts` ‚Äî `renderPromptPack` token substitution with unbound-variable detection
- `post-dispatch-validate.ts` ‚Äî iteration file + JSONL delta invariant checker
- `executor-audit.ts` ‚Äî appends the executor audit block to the last JSONL record

All four modules are shared with `sk-deep-review` (see that skill's v1.5.0.0 entry).

## Upgrade

No migration needed for existing users. `executor.kind` defaults to `"native"`, which preserves pre-release behavior exactly ‚Äî same agent dispatch, same JSONL records, same iteration files.

To opt into `cli-codex` dispatch, invoke:

```
/spec_kit:deep-research :auto "topic" \
  --executor=cli-codex \
  --model=gpt-5.4 \
  --reasoning-effort=high \
  --service-tier=fast
```

Requires `codex` on PATH. Unwired kinds (`cli-copilot`, `cli-gemini`, `cli-claude-code`) are reserved in the enum but throw `ExecutorNotWiredError` at config load time with a pointer to future specs.
