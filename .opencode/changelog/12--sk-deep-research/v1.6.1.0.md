### Lifecycle Retraction (Lane 3)

- `fork` and `completed-continue` lineage branches are **deferred** and no longer runtime-supported. Only `new`, `resume`, and `restart` are honest to the shipped workflow.
- `spec_kit_deep-research_confirm.yaml` `step_classify_session.on_resume` now asks for `[resume, restart, cancel]` instead of the previous four-option menu. `on_restart` writes a typed `restarted` JSONL event that carries the full persisted contract (`sessionId`, `parentSessionId`, `lineageMode`, `continuedFromRun`, `generation`, `archivedPath`, `timestamp`). `resumed` writes the same field set with `archivedPath: null`.
- `references/loop_protocol.md §Lifecycle Branch Contract` rewrites the runtime model, gives the canonical JSONL event payload, and explicitly marks `fork` and `completed-continue` as reserved for a future release.
- `references/state_format.md §Event Records` table annotates `forked` and `completed_continue` as deferred (no emitted fields).

Commit: `86c7fbc9d`.

### Graph Events Namespace Documentation (Lane 2)

- `references/state_format.md` adds a typed `graphEvents` field to the iteration record table and a dedicated §"Graph Events" section. The section documents the payload shape (`type`, `id`, `label`, optional `relation`/`source`/`target`/`metadata`) and the namespace rules the coverage-graph SQLite store now enforces: every entry is scoped by `(specFolder, loopType, sessionId, id)`, and producers MUST NOT salt ids with the session id to "avoid collisions" because the DB layer handles namespace isolation.
- The shared coverage-graph SQLite projection was bumped to `SCHEMA_VERSION = 2` with a composite primary key of `(spec_folder, loop_type, session_id, id)` on both `coverage_nodes` and `coverage_edges`. Migration is drop-and-recreate on first open of the v2 schema (dev-only cache, no durable production state). This directly closes F004 / F005 / F006 / F013 from the closing audit — two sessions reusing the same logical node or edge id now get independent rows, and the collision regression in `scripts/tests/session-isolation.vitest.ts` ("shared-ID collisions" describe block) pins the contract.

Commit: `be79ed0a0`.
