## [**1.2.2.0**] - 2026-03-28

**Canonical research packet contract and wrapper parity** -- Moves research-mode durable state into a dedicated `research/` packet, normalizes review iterations into `review/iterations/`, keeps `review-report.md` at the `review/` root, and syncs the `.agents` command and workflow wrappers so they route through the renamed deep-research surfaces consistently.

> Spec folder: `.opencode/specs/03--commands-and-skills/035-sk-deep-research-path-migration/` (Level 3)

---

## Highlights

### Category: Research Packet Contract

- **Dedicated `research/` packet root** -- Research mode now writes `research/research.md`, `research/iterations/iteration-NNN.md`, and the related config, state, strategy, dashboard, and pause files under a single canonical packet instead of mixing root files with `scratch/`.
- **Review iteration normalization** -- Review mode keeps the final verdict at `review/review-report.md` while moving iteration artifacts into `review/iterations/` for a cleaner durable layout.
- **Iteration folders excluded from canonical spec indexing** -- Helper and routing logic now treat `research/iterations/` and `review/iterations/` as working artifacts rather than canonical spec documents.

### Category: Runtime and Wrapper Parity

- **Runtime guidance synchronized** -- Deep-research and deep-review guidance across OpenCode, Claude, Codex, Gemini, and `.agents` now describe the same packet layout.
- **`.agents` command wrappers aligned** -- `plan.toml`, `complete.toml`, and `phase.toml` no longer refer to stale `/spec_kit:research` or `@research` surfaces.
- **Workflow shim retargeted** -- `.agents/workflows/spec_kit_research.md` now points at `.opencode/command/spec_kit/deep-research.md`.

### Category: Migration Support

- **Packet migration utility updated** -- The `system-spec-kit` migration support now rehomes the old deep-research packet layout into the new `research/` and `review/iterations/` structure.
- **Bounded legacy tolerance retained** -- Helper logic still recognizes some legacy packet shapes during transition, but first-party docs and tracked packet corpus now advertise only the new contract.

## Upgrade

No manual migration is required for the tracked repo corpus. The in-repo migration utility has already moved first-party packets to the new layout. If you maintain custom or older external spec folders that still use root `research.md`, `scratch/iteration-*`, or direct `review/iteration-*` paths, migrate them to the canonical `research/` and `review/iterations/` structure before relying on the updated contract.
