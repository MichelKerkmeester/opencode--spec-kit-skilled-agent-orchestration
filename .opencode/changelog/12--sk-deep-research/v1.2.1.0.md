## [**1.2.1.0**] - 2026-03-27

**Review mode polish and durable review packet contract** -- Simplified the review mode taxonomy from the initial 11-dimension model to a streamlined 7-dimension set, added canonical contract manifest, and aligned the full review-mode runtime around a dedicated `review/` subfolder so durable state no longer lives in `scratch/`.

---

## Highlights

### Category: Review Mode Refinement

- **Simplified review taxonomy** -- Consolidated from 11 dimensions to 7, with canonical contract manifest defining the authoritative dimension set and scoring rules.
- **Smart router integration** -- Added review mode to intent signals and phase detection in SKILL.md, enabling the skill router to correctly identify review-mode invocations.
- **Testing playbook alignment** -- Updated 6 playbook scenario files to match simplified review taxonomy terminology and scoring.
- **Cross-runtime agent sync** -- Updated `@deep-review` agent definitions across all 4 runtimes (OpenCode, Claude, Codex, Agents) with the simplified taxonomy.

### Category: Review Folder Contract

- **Durable review packet moved into `review/`** -- Review-mode config, JSONL state, strategy, dashboard, iteration artifacts, pause sentinel, and final review report now live under `{spec_folder}/review/` instead of being treated as disposable `scratch/` data.
- **Legacy scratch session migration** -- Review-mode startup now detects legacy scratch-based review packets and rehomes only the review-specific artifacts before resume classification, protecting existing sessions from being stranded.
- **Workflow and runtime parity** -- Review auto and confirm YAML workflows, the shared deep-research command entrypoint, and deep-review agent contracts across `.opencode`, `.claude`, `.codex`, and `.gemini` were synchronized around the new `review/` contract.
- **Docs and playbook truth-sync** -- `review_mode_contract.yaml`, review templates, references, READMEs, and review playbook scenarios now all describe the same canonical review packet location.

### Category: Documentation

- **Spec folder updates** -- Updated success criteria and changelog in spec 030 with post-release fix evidence.
- **YAML filename fix** -- Corrected YAML filename typo in DR-040 playbook scenario.
