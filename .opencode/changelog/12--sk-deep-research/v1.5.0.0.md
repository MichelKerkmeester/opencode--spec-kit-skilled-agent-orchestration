## [**1.5.0.0**] - 2026-04-11

This release lands the 042 Deep Research & Review Runtime Improvement Bundle for `sk-deep-research`. The research loop now has a frozen stop-reason taxonomy with legal-stop gates, a first-class blocked-stop event, reducer-owned audit journal + dashboard refresh, semantic coverage graph convergence signals, and advisory tuning via the offline optimizer. Every change is guarded by new vitest coverage and a fully exercised 32/32 manual playbook.

---

### Runtime Truth Foundation (Phase 1)

- Stop-reason enum is now frozen: `converged`, `maxIterationsReached`, `blockedStop`, `stuckRecovery`, `error`, `manualStop`, `userPaused`. Legacy strings are mapped through a single migration path.
- Legal-stop decision tree now requires convergence math, key-question coverage, evidence density, and hotspot saturation gates to all pass before STOP is accepted.
- Blocked-stop events are now persisted as first-class JSONL records with `blockedBy`, `gateResults`, and a `recoveryStrategy` hint so resume flows can replay the decision trace.
- Resume semantics are now explicit: `resume`, `restart`, `fork`, and `completed-continue` all carry consistent lineage fields across config, JSONL, strategy, and command docs.
- The reducer now owns the audit journal rollup, timing metrics, coverage depth, evidence-quality summary, and claim ledger verification — iteration agents no longer author these sections directly.

---

### Semantic Coverage Graph (Phase 2)

- Deep-research iterations now emit optional `graphEvents` arrays (`{type, id, label, relation?, source?, target?}`) on each JSONL record so coverage graph nodes and edges can be reconstructed by the reducer and MCP surfaces.
- Convergence now consults the coverage graph: contradiction density, graph momentum, and depth/degree signals can block STOP even when ratio-based gates already pass.
- Novelty justification now includes a semantic novelty lens, and the 3-signal convergence weights are published in `quick_reference.md` and the strategy asset so operators can reason about why the loop converged.
- A `sourceStrength` field and the `originalKeyQuestions` focus-alignment guard close the last remaining drift paths flagged by deep review.

---

### Offline Optimizer Integration (Phase 4a)

- `deep_research_config.json` now declares the tunable surface the offline optimizer is allowed to touch (`convergenceThreshold`, `stuckThreshold`, `maxIterations`) via `_optimizerManaged`, with locked fields protected from drift.
- The optimizer manifest, replay corpus, rubric, and advisory promotion gate are wired into command docs so operators know which thresholds are machine-suggested vs. hand-tuned.
- Promotion stays advisory-only: the loop never auto-applies optimizer recommendations.

---

### Reducer, Reliability, and State Hygiene

- `scripts/reduce-state.cjs` now excludes `thought` and `insight` iterations from the stuck count and rolling-average trend, fixing a false "stuck" signal for analytical-only iterations.
- Timing, coverage-depth, evidence-quality, and claim-ledger rollups are now all part of the reducer's idempotent output, so dashboard refresh stays deterministic across re-runs.
- A `stuck_recovery` JSONL event plus a least-covered-dimension pivot fall-back give the loop a deterministic escape from plateaus before it trips the legal-stop gates.

---

### Playbook and Operator Guidance

- Added `029-graph-events-emission.md` and `031-graph-convergence-signals.md` to the manual testing playbook to exercise graph emission and graph-aware convergence end to end.
- Every playbook scenario was executed against the updated runtime — 32/32 PASS with zero PARTIALs, matching the quality bar set by the 042 deep-review iterations.
- `references/convergence.md`, `references/state_format.md`, and `references/quick_reference.md` all reflect the legal-stop and blocked-stop model plus the graph-convergence lens.

---

### Verification

- Vitest infrastructure was widened to include `scripts/tests/**/*.vitest.ts` alongside the existing `mcp_server/tests/**/*.vitest.ts`, unblocking execution of 6k+ lines of previously dark scripts-side test code. The full suite is green (10,335 passing, 0 failing after this release's test fixes).
- Manual playbook run through GPT-5.4 high fast against every scenario: 32/32 PASS.
- Ten iterations of `/spec_kit:deep-review` against the 042 phases closed every review finding (no open P0/P1).

## Upgrade

No migration is required for existing research packets. Legacy stop-reason strings are mapped through the taxonomy on the fly. Resume and completed-continue flows carry the same lineage contract as v1.4.0.0, extended with legal-stop gate results and optional graph events.

---

> **Update (post-release) — 2026-04-11:**
>
> The "resume / restart / fork / completed-continue" lifecycle contract promised in the Contract Truth section of this release was only partially shipped. The 042 closing-audit remediation (10-iteration Codex `spec_kit:deep-review` session `rvw-2026-04-11T13-50-06Z`) found that `fork` and `completed-continue` were advertised as live runtime branches but had no workflow wiring, no reducer ancestry output, and no persisted lineage-transition events. Only `new`, `resume`, and `restart` were actually runtime-supported.
>
> **Release `v1.6.1.0` retracts** `fork` and `completed-continue` to deferred/reserved status. The canonical one-session model lives in `references/loop_protocol.md §Lifecycle Branches (current release)`, and the persisted `resumed` / `restarted` JSONL events carry the full lineage contract (`sessionId`, `parentSessionId`, `lineageMode`, `continuedFromRun`, `generation`, `archivedPath`, `timestamp`). `fork` and `completed-continue` remain reserved for a future release.
>
> See `.opencode/changelog/12--sk-deep-research/v1.6.1.0.md` for the full retraction and the Lane 1–5 commit references (`17b5f28ed`, `be79ed0a0`, `86c7fbc9d`, `2c4f74b9f`, `f99739742`).
