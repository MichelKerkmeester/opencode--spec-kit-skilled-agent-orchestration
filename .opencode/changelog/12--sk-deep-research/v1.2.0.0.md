## 1. OVERVIEW

This release adds a review mode to sk-deep-research, upgrading from v1.1.0 to v1.2.0. Review mode repurposes the existing iterative loop for code quality auditing instead of external research. It introduces a new `@deep-review` LEAF agent (hybrid of @review rubric + @deep-research state protocol), separate YAML workflows, adapted convergence algorithm, 7 review dimensions, 6 cross-reference verification protocols, and an 11-section review-report.md with release readiness verdicts. All existing research mode behavior is untouched.

---

## 2. HIGHLIGHTS

### Review Mode (`/spec_kit:deep-research:review`)

- **`:review` suffix** on existing command — not a separate skill or command
- **`:review:auto`** (default) and **`:review:confirm`** execution modes with 5 approval gates
- **5 review target types**: spec-folder, skill, agent, track, arbitrary files
- **Scope discovery** resolves target to concrete file list with cross-reference targets

### @deep-review LEAF Agent

- **Hybrid design**: @review quality rubric (5-dimension scoring, P0/P1/P2, Hunter/Skeptic/Referee) + @deep-research state protocol (JSONL, strategy, iteration lifecycle)
- **Deployed across 5 runtimes**: .opencode, .opencode/chatgpt, .claude, .codex, .agents
- **Tool budget**: 9-12 target, max 13 (no WebFetch — Read/Grep/Glob for code analysis)
- **Review target is strictly read-only** — never modifies code under review

### 7 Review Dimensions

Ordered by risk priority (inventory pass first, then):
1. Correctness — logic, edge cases, error handling
2. Security — auth, injection, data exposure
3. Spec Alignment — spec claims vs actual code
4. Completeness — required files, tests, checklist items
5. Cross-Ref Integrity — IDs, links, runtime anchors
6. Patterns — coding standards, template compliance
7. Documentation Quality — accuracy, clarity

### 6 Cross-Reference Verification Protocols

- Spec claims vs code implementation
- Checklist [x] items vs actual evidence
- SKILL.md contracts vs agent files
- Agent cross-runtime consistency
- Feature catalog vs implementation
- Testing playbook vs capability

### Severity-Weighted Convergence

- **`newFindingsRatio`** replaces `newInfoRatio` with severity weights (P0=10, P1=5, P2=1)
- **P0 override rule**: single new P0 floors ratio at 0.50, blocking convergence
- **Adapted signals**: rolling avg (weight 0.30, window 2), MAD (weight 0.25), dimension coverage (weight 0.45, threshold 100%)
- **5 quality guards**: evidence completeness, scope alignment, no inference-only, severity coverage, cross-reference
- **Stuck threshold**: 2 (vs 3 for research)

### 11-Section review-report.md

1. Executive Summary (verdict, score, P0/P1/P2 counts)
2. Score Breakdown (5 dimensions)
3. P0 Findings (Blockers)
4. P1 Findings (Required)
5. P2 Findings (Suggestions)
6. Cross-Reference Results
7. Coverage Map
8. Positive Observations
9. Convergence Report
10. Remediation Priority
11. Release Readiness Verdict (FAIL / CONDITIONAL / PASS WITH NOTES / PASS)

### Post-Review Workflow

| Verdict | Condition | Next Command |
|---------|-----------|-------------|
| FAIL | Active P0 or score <70 | `/spec_kit:plan` for remediation |
| CONDITIONAL | No P0, active P1 | `/spec_kit:plan` for fixes |
| PASS WITH NOTES | Only P2 | `/create:changelog` |
| PASS | No findings | `/create:changelog` |

### Testing Playbook

- **11 new scenarios** (DR-031 through DR-041) in category `07--review-mode/`
- Covers: kickoff, scope discovery, dimension ordering, iteration format, convergence, quality guards, cross-references, adversarial self-check, synthesis, verdicts, dashboard
- All 11 scenarios PASS after consistency sweep

## 5. UPGRADE

No migration required. Review mode is purely additive — new files and new sections in existing files. Existing research mode behavior, state files, and convergence algorithm are untouched. The `mode` field in config defaults to `"research"` for backward compatibility.

To use review mode: `/spec_kit:deep-research:review "target"`
