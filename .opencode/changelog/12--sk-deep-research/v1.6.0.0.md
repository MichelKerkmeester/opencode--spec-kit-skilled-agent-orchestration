This release lands Phase 008 of the 042 Deep Research & Review Runtime Improvement Bundle - closing the 12 P0/P1/P2 recommendations surfaced by the 20-iteration deep-research audit. `sk-deep-research` now actively utilizes the coverage graph on the live loop path: `deep_loop_graph_upsert` and `deep_loop_graph_convergence` are called from the auto/confirm YAML workflows, reducer-owned surfaces expose `graphConvergenceScore` / `graphDecision` / `graphBlockers`, and blocked-stop JSONL events surface in the findings registry, dashboard, and strategy `next-focus` anchor. The goal is to make the workflow easier to follow and easier to trust.

---

#### Contract Truth (Part A)

- `step_emit_blocked_stop` wires a first-class `blocked_stop` JSONL event into both research workflows. Each event carries `blockedBy`, `gateResults`, `recoveryStrategy`, and full session run history so downstream reducers can surface the exact blocker.
- `step_normalize_pause_events` rewrites raw `paused` to `userPaused` and raw `stuck_recovery` to `stuckRecovery` at emission time using the frozen `STOP_REASONS` enum. The live workflow no longer leaks free-form strings into `synthesis_complete.stopReason`.
- The reducer no longer filters non-iteration JSONL rows. `blocked_stop`, `userPaused`, and `stuckRecovery` events are preserved in the `events` stream and consumed by the blocked stop history builder / the graph convergence rollup builder.

---

#### Graph Wiring (Part B, ADR-001)

- ADR-001 accepted: **MCP handler main**. The CJS helper at `scripts/lib/coverage-graph-convergence.cjs` is now a thin adapter to the authoritative `mcp_server/lib/coverage-graph/coverage-graph-signals.ts` implementation. `sourceDiversity`, `contradictionDensity`, `questionCoverage`, and `evidenceDepth` are all harmonized; `graph-convergence-parity.vitest.ts` guards the cross-layer expected behavior (3/3 passing).
- Both `spec_kit_deep-research_auto.yaml` and `spec_kit_deep-research_confirm.yaml` now call `deep_loop_graph_upsert` after `step_reduce_state` (persisting iteration `graphEvents` to the SQLite store) and `deep_loop_graph_convergence` before the inline 3-signal vote in `step_check_convergence`. The combined stop rule: STOP requires BOTH the inline vote and graph decision to agree; `STOP_BLOCKED` routes to the `blocked_stop` emission path.
- The reducer's `buildGraphConvergenceRollup(events)` consumes `graph_convergence` JSONL events and exposes `graphConvergenceScore`, `graphDecision`, `graphBlockers` as new registry fields alongside `convergenceScore`.

---

#### Reducer Surfacing (Part C)

- the render dashboard helper emits new `BLOCKED STOPS` and `GRAPH CONVERGENCE` sections backed by the reducer-collected event data.
- the update strategy content helper overrides the `next-focus` anchor with the latest blocked-stop `recoveryStrategy` when blocked-stop is the most recent loop event, so operators see the blocker and recovery hint before choosing the next iteration direction.
- `state_format.md` §5 documents all new registry fields, the `blocked_stop` and `graph_convergence` event schemas, and the combined-stop rule.

---

#### Session Scoping (REQ-012)

- Shared coverage-graph reads (`coverage-graph-query.ts`, `coverage-graph/convergence.ts`, `coverage-graph/query.ts`, `coverage-graph/status.ts`) now accept `sessionId` and scope aggregation by `specFolder + loopType + sessionId`. Backward-compat preserved when `sessionId` is absent. `session-isolation.vitest.ts` validates that two concurrent research sessions in the same packet cannot see each other's graph nodes (3/3 passing).

---

#### Tool Routing Parity (ADR-003)

- `code_graph_query` and `code_graph_context` are now provisioned in the LEAF-tool budget of `.opencode/command/spec_kit/deep-research.md` and in the runtime agent permissions at `.opencode/agent/deep-research.md` + `.claude/agents/deep-research.md`. Runtime `.codex/` and `.gemini/` mirrors already reference the tools in routing prose; tool availability is guaranteed at the MCP server registration layer.

---

#### Fixtures and Regression Coverage

- New fixture: `scripts/tests/fixtures/interrupted-session/` - a deep-research packet with two complete iterations plus a third iteration truncated mid-write (truncated markdown + malformed JSONL tail). Demonstrates reducer recovery under `--lenient`.
- New vitest suite: `scripts/tests/graph-aware-stop.vitest.ts` - 3 test cases covering `STOP_BLOCKED` propagation, `STOP_ALLOWED` resolution, and default values when no graph events exist. **The suite fails if graph wiring is not active**, providing a regression guard against future drift.
- New playbook scenarios under `manual_testing_playbook/04--convergence-and-recovery/`: `032-blocked-stop-reducer-surfacing.md` and `033-graph-aware-stop-gate.md`.

---

#### Verification

- Phase 008 vitest run (4 suites, 12 tests): **all passing**. `graph-convergence-parity` (3/3), `session-isolation` (3/3), `review-reducer-stop with an error` (3/3), `graph-aware-stop` (3/3).
- Reducer smoke tests on the new fixtures: `iterationsCompleted: 2` recovery, `blockedStopHistory` surfaces correctly, `graphConvergenceScore` defaults to 0 when no events exist.
- Idempotency preserved: re-running the reducer against the same state produces byte-identical output.

## Upgrade

No migration is required for existing research packets. New registry fields (`blockedStopHistory`, `graphConvergenceScore`, `graphDecision`, `graphBlockers`) are additive only. Existing v1.5.0.0 packets resume cleanly under the new reducer logic. Raw JSONL events like `paused` and `stuck_recovery` are rewritten to their normalized enum values at emission time - packets created under v1.5.0.0 remain readable, but new emissions will use the normalized forms.
