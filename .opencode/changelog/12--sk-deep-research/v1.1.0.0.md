## 1. OVERVIEW

This release upgrades sk-deep-research from v1.0.0 to v1.1.0 based on findings from a 10-iteration deep research session analyzing 7 open-source auto-research repos (ResearcherSkill, pjhoberman/autoresearch, gpt-researcher, AutoResearchClaw, AI-Research-SKILLs, ralph, ARIS). All changes are surgical additions to existing files -- no architectural rewrites. The skill remains a single monolithic skill per sk-doc standards (no modular split).

---

## 2. HIGHLIGHTS

### Category: Observability (P0)

- **Persistent dashboard** -- Auto-generated `deep-research-dashboard.md` after every iteration showing iteration table, question status, trend, dead ends, next focus, and active risks. New asset template, new loop protocol step, new YAML workflow step.
- **Resume decision gate** -- In confirm mode, dashboard is shown on resume with choice to resume, restart (archive), or fork. Replaces silent auto-resume.

### Category: Quality Protection (P0)

- **Quality guards** -- Three binary pass/fail checks that block convergence STOP: source diversity (>=2 sources per answered question), focus alignment (answers match original key questions), no single-weak-source (no tentative-only answers). Guard violations logged as JSONL events.
- **Negative knowledge** -- Ruled-out directions and dead ends are now required iteration output sections. Consolidated into mandatory "Eliminated Alternatives" section in final research.md synthesis. Optional dead-end-coverage convergence signal (reference-only).

### Category: Iteration Enrichment (P1)

- **Extended iteration statuses** -- Status enum expanded from `complete|timeout|error|stuck` to include `insight` (conceptual breakthrough, low ratio) and `thought` (analytical-only, no evidence gathering).
- **Novelty justification** -- Required `noveltyJustification` field in JSONL iteration records: 1-sentence explanation alongside newInfoRatio for auditability.
- **Focus track labels** -- Optional `focusTrack` field for post-hoc iteration grouping by research direction.
- **Per-iteration budget** -- New `maxToolCallsPerIteration` (default 12) and `maxMinutesPerIteration` (default 10) config fields with orchestrator enforcement.

### Category: Research Quality (P1)

- **Diagnosis-led recovery** -- Failure mode classification (shallow sources, contradictory evidence, topic too broad, repetitive findings, source exhaustion) before recovery strategy selection. Targeted recovery prompts instead of generic "try something different."
- **Research charter** -- Non-Goals and Stop Conditions sections added to strategy template, validated at initialization (Step 5a).
- **Source-hygiene confirmation** -- Optional `sourceStrength` field (primary/secondary/tentative). Tentative findings don't count toward question coverage in convergence.

### Category: Template & Standards Compliance

- **sk-doc asset alignment** -- Strategy and dashboard templates restructured with numbered sections, retrieval anchors, Overview sections (Purpose + Usage), and horizontal rules per `skill_asset_template.md`.
- **Config updated** -- Dashboard added to fileProtection map with `auto-generated` protection level.
- **5 runtime agent files updated** -- All v1.1.0 features (statuses, ruled-out, justification, source diversity rule, dashboard awareness, track labels) applied consistently across Claude, OpenCode, ChatGPT, Codex, and Gemini agents.
- **Manual testing playbook expanded** -- 11 new scenarios (DR-020 through DR-030) covering quality guards, dashboard, novelty justification, ruled-out synthesis, charter validation, track labels, and insight/thought statuses. Total: 30 scenarios.
