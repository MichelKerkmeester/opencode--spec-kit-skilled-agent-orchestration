This patch release tightens the research reducer and the surrounding reference docs so they describe the same behavior. It fixes state handling, restores visible run-history data, and closes a few cases where the workflow could silently drift from the docs.

## Bug Fixes

These fixes correct behavior that could confuse operators or produce the wrong result. The goal is to make the workflow more predictable in normal use.

### Reducer now consumes terminal stop metadata

**Problem:** The research reducer dropped `synthesis_complete` events and derived dashboard status from stale config.

**Fix:** `reduce-state.cjs` now parses the latest `synthesis_complete` event to derive dashboard status.

---

### Resume/restart lineage now persists

**Problem:** Lineage fields were emitted to JSONL but never propagated to config-visible state. `continuedFromRun` was never surfaced in the dashboard.

**Fix:** The reducer detects lifecycle events and updates run history fields in the registry and dashboard.

---

### Graph convergence scoring no longer collapses to zero

**Problem:** When `blendedScore` was absent from `graph_convergence` events, the scoring collapsed to `0` instead of using a fallback.

**Fix:** Added the same numeric fallback pattern used on the review side, so missing `blendedScore` values no longer collapse the graph score to zero.

---

### Reducer is now fail-closed on corrupt JSONL

**Problem:** The research reducer returned silently on corruption instead of throwing.

**Fix:** `reduceResearchState()` now throws a structured corruption error instead of returning quietly, matching the deep-review side of the system.

---

### Completed sessions no longer re-entered for synthesis

**Problem:** Both workflow YAMLs routed completed sessions directly to the synthesize step despite the expected behavior saying this was unsupported.

**Fix:** Completed packets are now a hard stop requiring explicit operator choice (restart, archive, or exit).

## Documentation

The supporting guidance was updated so the written instructions match the shipped behavior. That makes the workflow easier to follow without reading the code first.

### deep_research_config.json cleaned of phantom lineage modes

Removed `fork` and `completed-continue` from the config template. Changed `archiveRoot` to `research_archive`. Only `new`, `resume`, `restart` remain.

### loop_protocol.md corrected

Updated the wave section to describe the shipped planner behavior instead of the older question-count model, and corrected the replay description from delta-based to full-history.

### Agent definition updated

`deep-research.md` now lists `research/deep-research-config.json` as an explicit Step 1 input.

### Graph-events playbook updated

`029-graph-events-emission.md` now references the current flat `graphEvents` schema and the `handleCoverageGraphUpsert()` interface.
