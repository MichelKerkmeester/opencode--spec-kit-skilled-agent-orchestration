This release explains the correct way to set Codex reasoning depth and removes guesswork around the available options. It helps operators avoid using the wrong command pattern. The goal is to make the workflow easier to follow and easier to trust.

---

#### Clearer reasoning-depth setup

People were trying to use the wrong command style for reasoning depth. The docs now list the supported levels and explain the correct configuration-based setup path. Automated runs are less likely to fail because of an avoidable syntax mistake.

---

#### Consistent guidance across the package

The flag and model guidance needed to say the same thing everywhere. The main guide, reference docs, comparison notes, and templates were all updated together. Users can move between documents without getting mixed messages.
