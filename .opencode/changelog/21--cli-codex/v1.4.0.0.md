This release gives the Codex skill a single, predictable default and a simple way to tune it. Every delegation now runs on `gpt-5.5` at `medium` reasoning through the `fast` service tier, and users can change only the reasoning depth when they need more (or less) thinking. The guide and all templates were rewritten to match this one-model story.

---

#### One model, one default

The skill used to split delegations across two models and ask the caller to pick. That split made every script choose between `gpt-5.4` and `gpt-5.3-codex` up front, even for ordinary work. The default dispatch is now `--model gpt-5.5 -c model_reasoning_effort="medium" -c service_tier="fast"` for every task. Zero-input calls go through this path and stay reproducible across runs.

---

#### User override through reasoning only

Callers sometimes need deeper analysis (architecture, security audits) or something lighter (trivial lookups). Instead of swapping models, the skill now honors explicit effort phrasing like `Use gpt 5.5 high fast` or `Use gpt 5.5 low`, resolving to the matching `model_reasoning_effort` value. The model and service tier stay fixed, so overrides are narrow and predictable.

---

#### Guide and templates rewritten to match

The skill documentation still referred to two models in tables, comparisons, and per-task examples. Every model table, selection strategy, agent definition, and bash snippet across `SKILL.md`, `README.md`, and all four reference files now lists `gpt-5.5` as the only model and uses reasoning effort for task tuning. Readers get one consistent story from the overview through the deepest template.

## Upgrade

No migration is required. Any existing script that passed `--model gpt-5.4` or `--model gpt-5.3-codex` should be updated to `--model gpt-5.5` and, if desired, given an explicit `-c model_reasoning_effort="..."` value. Scripts that relied on the old default will automatically pick up the new `gpt-5.5` · `medium` · `fast` dispatch when they regenerate their command lines from the skill.
