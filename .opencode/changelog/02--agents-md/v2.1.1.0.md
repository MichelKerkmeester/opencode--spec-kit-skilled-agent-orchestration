This release adds an explicit adversarial review method to the core review, debug, and planning agents. It also cleans up Gemini runtime naming so the mirrored agent copies describe the same tools and model choices as the shipped runtime.

---

### @review: Adversarial Self-check

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** New Section 9.1 with 3-pass Hunter/Skeptic/Referee protocol for P0/P1 findings. Adds sycophancy warning, summary table template, Fast Path skip. Self-validation expanded to 6 questions. New anti-pattern: "Never let sycophancy bias findings". This makes the workflow easier to discover and use.

---

### @debug: Adversarial Hypothesis Validation

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** New Phase 3.1 between Hypothesize and Fix with Counter-Evidence Search, Alternative Explanation Check, Anchoring Check, Prior Attempt Echo Check, and Post-Challenge Re-Ranking Table. New anti-pattern: "Never skip adversarial validation". This makes the workflow easier to discover and use.

---

### @ultra-think: Adversarial Cross-critique

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** New Section 6.1 with cross-strategy Hunter/Skeptic/Referee challenge (triggered when strategies within 15 points). Convergence Check flags artificial agreement. New anti-pattern: "Convergence Sycophancy". This makes the workflow easier to discover and use.

---

### Gemini runtime mirrors

**Problem:** This part of the documentation needed to match the related files more closely.

**Fix:** Fixed a pre-existing mismatch where the body text still used Copilot tool names while the Gemini metadata used Gemini-native ones. The mirrored files now consistently use `grep_search`, `list_directory`, `read_file`, and `run_shell_command`, and the model reference now points to `gemini-3.1-pro-preview`.

---

### Gemini edit-path cleanup

**Problem:** This area needed a clearer and safer way to work.

**Fix:** The same cleanup was applied to the remaining Gemini runtime copy so edit operations also use the correct Gemini-native tool naming.
