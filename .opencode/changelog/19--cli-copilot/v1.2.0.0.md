## [**1.2.0.0**] - 2026-03-09

**Documented the non-interactive reasoning effort mechanism via `~/.copilot/config.json`, added per-model support tables with defaults, and streamlined the model catalog from 18 to 5 recommended models.**

---

## Highlights

### Updated

- **Reasoning Effort Config Key** — Discovered and documented the `"reasoning_effort"` config key in `~/.copilot/config.json` as the non-interactive mechanism for setting reasoning depth. Replaces the previous workaround (interactive mode persistence) with a direct, scriptable approach
- **Per-Model Support Table** — Added detailed per-model reasoning effort support matrix showing which levels each model supports and their defaults (e.g., GPT-5.4 supports `xhigh`, defaults to `high`; GPT-5.1 supports up to `high`, defaults to `medium`)
- **Non-Interactive Code Example** — Added Python snippet for programmatically setting `reasoning_effort` in config before `-p` invocations, enabling fully autonomous xhigh reasoning without interactive setup
- **Internal Behavior Documentation** — Documented how Copilot CLI reads config, validates against model capabilities, and passes `reasoning_effort` in API requests
- **Streamlined Model Catalog** — Reduced from 18 models to 5 recommended models (GPT-5.4, GPT-5.3-Codex, Claude Opus 4.6, Claude Sonnet 4.6, Gemini 3.1 Pro Preview) for clearer guidance
- **Removed Open Issue Reference** — Removed link to `copilot-cli#1048` since the config file mechanism provides a working non-interactive solution
