This release lands Phase 008 of the 042 Deep Research & Review Runtime Improvement Bundle. `sk-deep-review` now actively utilizes the coverage graph on the live review loop, persists blocked-stop bundles through reducer-owned surfaces, fails loud (instead of silent) on malformed JSONL and missing machine-owned anchors, and splits `repeatedFindings` into semantically distinct `persistentSameSeverity` + `severityChanged` buckets for precise triage. The goal is to make the workflow easier to follow and easier to trust.

---

### Contract Truth (Part A)

- `step_emit_blocked_stop` wires a first-class `blocked_stop` JSONL event into both review workflows with review-specific gate names: `convergenceGate`, `dimensionCoverageGate`, `p0ResolutionGate`, `evidenceDensityGate`, `hotspotSaturationGate`. Each event carries the full `blockedBy` / `gateResults` / `recoveryStrategy` payload.
- `step_normalize_pause_events` rewrites raw `paused` to `userPaused` and `stuck_recovery` to `stuckRecovery` at emission time using the frozen `STOP_REASONS` enum.
- The reducer no longer drops non-iteration events. the blocked stop history builder scans `blocked_stop` rows (with backward-compat for the legacy nested `legalStop` payload shape) and exposes `blockedStopHistory` as a registry field.

---

### Graph Wiring (Part B)

- `spec_kit_deep-review_{auto,confirm}.yaml` now call `deep_loop_graph_upsert` after `step_reduce_review_state` and `deep_loop_graph_convergence` before the inline review convergence vote in `step_check_convergence`. Combined stop rule: STOP requires BOTH inline vote and graph decision to agree. `STOP_BLOCKED` routes to the blocked-stop emission path.
- `buildGraphConvergenceRollup(records)` scans `graph_convergence` JSONL events and exposes `graphConvergenceScore`, `graphDecision`, `graphBlockers` as new registry fields alongside `convergenceScore`.
- Shared coverage-graph reads are now session-scoped: `coverage-graph-query.ts`, `convergence.ts`, `query.ts`, `status.ts` all accept and propagate `sessionId`. Review packets with multiple concurrent sessions no longer bleed across each other's graph state.

---

### Reducer Fail-Closed Hardening (Part C)

- the parse jsonl helper is now backed by the parse jsonl detailed helper which collects malformed lines into a `corruptionWarnings` array instead of silently dropping them. `parseJsonl` still returns the plain records array for backward compatibility.
- the reduce review state helper now emits a warning to stderr when `corruptionWarnings.length > 0` and exits with code `2`. Pass `--lenient` (CLI) or `lenient:true` (function option) to preserve the v1.2.0.0 fail-open behavior for legacy packets.
- the replace anchor section helper now throws `Error('Missing machine-owned anchor "<id>" in deep-review strategy file...')` when a required anchor is absent. Pass `--create-missing-anchors` (or `createMissingAnchors:true`) to bootstrap missing anchor blocks from scratch.
- `review-reducer-stop with an error.vitest.ts` validates all three paths: corruption detection, missing anchor throw, and `--lenient` escape hatch (3/3 passing).

---

### repeatedFindings Split (REQ-018)

- The `repeatedFindings` registry field is now **deprecated**. It's retained as a backward-compat union of the two new arrays:
- `persistentSameSeverity` - findings observed in ≥2 iterations with no severity transitions (only the initial discovery entry).
- `severityChanged` - findings that went through ≥1 severity transition (P0↔P1↔P2) in their `transitions` history.
- Dashboard renders both new counters in the TREND section, making triage at scale (50+ findings) possible without conflating "same finding reappeared" with "finding's severity was upgraded".

---

### Reducer Surfacing (Part C)

- Dashboard gained new sections: `BLOCKED STOPS`, `GRAPH CONVERGENCE`, `CORRUPTION WARNINGS`. Each entry is rendered with timestamps, gate result summaries, and recovery strategy hints.
- Strategy `next-focus` anchor prefers the latest blocked-stop `recoveryStrategy` when blocked-stop is the most recent loop event - operators see the blocker before choosing next-iteration direction.

## Upgrade

**Backward-compatible for existing packets** as long as JSONL files are clean and strategy files have all machine-owned anchors. If a packet was authored under v1.2.0.0 and contains:
- Malformed JSONL lines → the v1.3.0.0 reducer will exit non-zero. Pass `--lenient` to preserve the prior fail-open behavior while still reporting corruption in the registry.
- Missing machine-owned strategy anchors → the v1.3.0.0 reducer will throw. Pass `--create-missing-anchors` (or create the missing anchor blocks manually) to bootstrap.
- `repeatedFindings` consumers → the field is retained as a deprecated union of `persistentSameSeverity` + `severityChanged`. New code should read the split arrays directly.

No migration is required for packets created under v1.3.0.0 or later.
