## [**1.1.0.0**] - 2026-04-03

This release hardens `sk-deep-review` into a fully explicit review-packet contract. The skill now has canonical `deep-review-*` naming across all surfaces, an unambiguous lifecycle and reducer model, synchronized runtime mirrors, and focused Vitest guards that keep the contract from drifting.

---

### Review Contract Alignment

- Canonical review artifacts are now frozen across the skill, references, workflow assets, and runtime mirrors: `deep-review-config.json`, `deep-review-state.jsonl`, `deep-review-findings-registry.json`, `.deep-review-pause`, and `deep-review-dashboard.md`.
- Lifecycle branches are now explicit and consistent across the review packet: `resume`, `restart`, `fork`, and `completed-continue`.
- Release-readiness states are now treated as first-class contract terms instead of scattered prose.
- Reducer inputs, outputs, metrics, and machine-owned boundaries are now documented directly in the contract surfaces.

---

### Runtime and Workflow Parity

- OpenCode, Claude, Gemini, and Codex runtime mirrors now all point at the same review packet model.
- The auto and confirm workflow assets now both encode migration logic, reducer refresh behavior, synthesis boundaries, and release-readiness propagation.
- Legacy `deep-research-*` names are now limited to the intentional scratch migration path instead of leaking into live review surfaces.

---

### Playbook and Operator Guidance

- The review references now describe one canonical lifecycle and one canonical state model.
- The manual testing playbook no longer teaches stale `deep-research-*` review-mode names.
- The config asset now exposes the reducer metrics expected by the contract tests, making future drift easier to catch.

---

### Verification

- Added `deep-review-contract-parity.vitest.ts` to guard naming, lifecycle, runtime parity, and migration boundaries.
- Added `deep-review-reducer-schema.vitest.ts` to lock reducer inputs, outputs, severity weights, and release-readiness schemas.
- Phase validation passed under the strict Spec Kit validator for `040-sk-auto-deep-research-review-improvement/002-sk-deep-review-improvements`.
