## [**1.3.1.0**] - 2026-04-11

Patch release closing the 042 closing-audit remediation as it affects `sk-deep-review`. Ten-iteration Codex `spec_kit:deep-review` session `rvw-2026-04-11T13-50-06Z` (verdict CONDITIONAL, 0 P0 / 10 P1 / 6 P2) routed its findings into a five-lane remediation pass on branch `system-speckit/026-graph-and-context-optimization`. This release lands the review-side changes from that pass. All test suites stay green: 908 passed / 55 skipped / 0 failed on `scripts/tests/`, `tsc --noEmit` clean on `mcp_server`.

### Claim-Adjudication as a Live STOP Gate (Lane 1)

- `step_check_convergence` in both `spec_kit_deep-review_auto.yaml` and `spec_kit_deep-review_confirm.yaml` now runs a universal **claim-adjudication pre-check (step 0)** before hard stops, composite convergence, and the legal-stop decision tree. A missing or failing typed claim-adjudication packet sets `candidate_decision = BLOCKED`, skips the remaining evaluation, and routes through `step_emit_blocked_stop` with `blockedBy: ["claimAdjudicationGate"]`.
- The legal-stop decision tree gains a new gate `f) claimAdjudicationGate` that records `last_claim_adjudication_passed` and `claim_adjudication_active_count`. The `blocked_stop` event `gateResults` map now surfaces `claimAdjudicationGate: {pass, activeP0P1}` alongside the five original gates.
- `step_post_iteration_claim_adjudication` now persists its result as a typed `claim_adjudication` JSONL event with `{passed, missingPackets, activeP0P1, reason?}`, so the next iteration's pre-check can read it from state.jsonl.
- `references/state_format.md §9 CLAIM ADJUDICATION` replaces the old prose markdown template with the canonical typed packet schema: `findingId`, `claim`, `evidenceRefs`, `counterevidenceSought`, `alternativeExplanation`, `finalSeverity`, `confidence`, `downgradeTrigger`, optional `transitions`. `references/loop_protocol.md §Step 4a` mirrors the schema and documents the blocked_stop emission path when a packet is missing or incomplete.

REQ-026, REQ-027. Commit: `17b5f28ed`.

### Coverage-Graph Session Isolation (Lane 2)

- `deep-loop-graph.sqlite` was bumped to `SCHEMA_VERSION = 2`. `coverage_nodes` and `coverage_edges` now share a composite primary key of `(spec_folder, loop_type, session_id, id)`, so two review sessions reusing the same logical id get independent rows instead of one row that the later upsert overwrites.
- Public API signatures tightened: `upsertNode`, `upsertEdge`, `getNode(ns, id)`, `getEdge(ns, id)`, `getEdgesFrom(ns, sourceId)`, `getEdgesTo(ns, targetId)`, `updateEdge(ns, id, updates)`, `deleteNode(ns, id)`, `deleteEdge(ns, id)` all require a `Namespace`. `initDb` migrates by dropping the v1 tables and indexes before creating the v2 schema (dev-only cache, safe to drop).
- `scripts/tests/session-isolation.vitest.ts` gains a "shared-ID collisions" describe block that upserts identical node and edge ids across two sessions and asserts both rows survive independently. Covers the update-in-place path and namespace-scoped traversal.
- `references/state_format.md` adds a typed `graphEvents` field to the iteration record table and a dedicated §Graph Events section. The section documents the payload shape (`dimension`/`file`/`finding`/`evidence`/`remediation`/`edge` with `type`, `id`, `label`, optional `relation`/`source`/`target`/`metadata`) and the namespace rules producers must follow.

REQ-028, REQ-029. Commit: `be79ed0a0`.

### Lifecycle Retraction (Lane 3)

- `fork` and `completed-continue` lineage branches are **deferred** and no longer runtime-supported. Only `new`, `resume`, and `restart` are honest to the shipped workflow.
- `spec_kit_deep-review_confirm.yaml` `step_classify_session.on_resume` asks for `[resume, restart, cancel]` and appends a typed `resumed` or `restarted` JSONL event with the full persisted contract (`sessionId`, `parentSessionId`, `lineageMode`, `continuedFromRun`, `generation`, `archivedPath`, `timestamp`). `spec_kit_deep-review_auto.yaml` mirrors the retraction in `user_inputs.lineage_mode` and `step_classify_session.on_resume`/`on_restart` branches, and `note_deferred_branches` documents the one-session contract.
- `references/loop_protocol.md §Lifecycle Branches (current release)` replaces the four-option table with the canonical three-mode contract and the full persisted event schema. `references/state_format.md §3 CONFIG` updates the `lineageMode` row to mark `fork`/`completed-continue` as deferred.
- `scripts/tests/deep-review-contract-parity.vitest.ts` now asserts `on_restart:` plus `note_deferred_branches:` on the auto path, asserts `on_fork:` is absent, and swaps the legacy four-option confirm list for the new `[resume, restart, cancel]` plus the deferred note.

REQ-030, REQ-031. Commit: `86c7fbc9d`.

### Canonical Review Schema and Convergence Docs (Lane 4)

- `.opencode/agent/deep-review.md §Step 5 Write Findings` was rewritten so every passage matches the reducer parser at `scripts/reduce-state.cjs:186`: `# Iteration N:` heading, `## Focus` / `## Scorecard` / `## Findings` / `## Cross-Reference Results` / `## Assessment` / `## Ruled Out` / `## Dead Ends` / `## Recommended Next Focus` sections, `### P0 — Blocker` / `### P1 — Required` / `### P2 — Suggestion` subsections, `- **FNNN**: Title — file:line — Description` bullets. The earlier free-form template silently dropped findings; the new template round-trips cleanly.
- Both review YAMLs pre-serialize `review_dimensions_json` into a proper JSON string array before writing the config JSONL row. The old shape `["{review_dimensions}"]` collapsed `correctness,security` into a single string element and weakened replay tooling. The `pre_serialize` directive spells out the acceptance rules for the three input shapes (`"all"`, comma-separated, array).
- `references/convergence.md §Shared Stop Contract` replaces the phantom `legalStop` record wrapper with the canonical persisted `blocked_stop` JSONL event shape, including the current six-gate `gateResults` map (`convergenceGate`, `dimensionCoverageGate`, `p0ResolutionGate`, `evidenceDensityGate`, `hotspotSaturationGate`, `claimAdjudicationGate`) plus the top-level `recoveryStrategy` and `graph_blocker_detail_json` fields. Signal 3 of the composite convergence vote is corrected to **Dimension Coverage** (was mis-labelled "Novelty Ratio" in one spot); the weighted vote math is unchanged.

### Upgrade

No data migration is required for existing review packets. The coverage-graph SQLite cache auto-migrates by dropping and recreating on first open of the v2 schema. Existing v1.3.0.0 state logs remain readable; new emissions use the typed `claim_adjudication` event and the canonical `blocked_stop` shape.

Operators who relied on the `fork` or `completed-continue` lifecycle branches should archive the prior `review/` tree manually or use `restart` to do both in one step. The two deferred branches remain reserved for a future release.

Reviewers MUST now emit a typed claim-adjudication packet for every new P0/P1 finding. A missing or malformed packet vetoes STOP via the `claimAdjudicationGate` pre-check on the next iteration. See `references/state_format.md §9` for the canonical schema.
