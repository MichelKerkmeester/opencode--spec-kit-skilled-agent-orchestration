## [**1.2.0.0**] - 2026-04-11

This release lands the 042 Deep Research & Review Runtime Improvement Bundle for `sk-deep-review` and closes the long-standing architectural gap between the two deep-loop skills. `sk-deep-review` now has a frozen stop-reason taxonomy, review-specific legal-stop gates, first-class blocked-stop events, a semantic coverage graph integration, and — new this release — the same reducer script and runtime-capability resolver that `sk-deep-research` has always had, unifying the two packet contracts at the code level.

---

### Runtime Truth Foundation (Phase 1)

- Stop-reason enum is now frozen: `converged`, `maxIterationsReached`, `blockedStop`, `stuckRecovery`, `error`, `manualStop`, `userPaused`. Legacy review stop strings are mapped through a single migration path.
- Legal-stop decision tree is now review-specific: STOP requires convergence, dimension coverage, P0 resolution, evidence density, and hotspot saturation gates to all pass together before the loop accepts a STOP verdict.
- Blocked-stop events are now persisted as first-class JSONL records with `blockedBy` (for example `["dimensionCoverageGate", "p0ResolutionGate"]`), full `gateResults`, and a review-specific `recoveryStrategy` hint so resume flows can replay the decision.
- Resume semantics (`resume`, `restart`, `fork`, `completed-continue`) and lineage fields are now consistent across config, JSONL, strategy, references, and command docs.
- The review reducer now owns the findings registry, dashboard refresh, and strategy anchor updates — iteration agents no longer write these surfaces directly.

---

### Semantic Coverage Graph (Phase 2)

- Review iterations now emit optional `graphEvents` arrays on each JSONL record so the coverage graph can reconstruct review scope, dimensions covered, and cross-reference protocols.
- Convergence for review now consults the coverage graph: dimension coverage gate severity was raised from warning to **blocking**, so an uncovered required dimension can no longer silently allow STOP.
- 3-signal convergence weights (rollingAvg, madScore, dimensionCoverage) are now published in `quick_reference.md` and the strategy asset so operators can reason about why the review loop converged.
- New playbook scenarios exercise graph event emission and graph-aware convergence in review mode.

---

### Script Parity With sk-deep-research

- Added `scripts/reduce-state.cjs`: review-specific state reducer that parses the JSONL iteration log, the per-iteration `iteration-NNN.md` files, and the strategy file, then regenerates `deep-review-findings-registry.json`, `deep-review-dashboard.md`, and the machine-owned strategy anchors. Idempotent: re-running with identical inputs produces byte-identical output.
- The reducer tracks finding lifecycle directly: `firstSeen`, `lastSeen`, severity transitions, and automatic detection of repeated findings across iterations — no more manual registry bookkeeping.
- Dimension coverage is derived from JSONL `dimensions` fields plus strategy anchors, and the provisional verdict (PASS/CONDITIONAL/FAIL) is computed from active P0/P1 counts and surfaced directly on the dashboard alongside `hasAdvisories`.
- Added `scripts/runtime-capabilities.cjs` and `assets/runtime_capabilities.json` so review runtimes (OpenCode, Claude, Codex, Gemini, legacy `.agents` wrapper) resolve from one canonical matrix, matching the long-standing deep-research contract.
- `deep_review_config.json` now declares `reducerScriptPath` and `runtimeCapabilityResolver`, and both `spec_kit_deep-review_auto.yaml` and `spec_kit_deep-review_confirm.yaml` invoke the reducer via `node .opencode/skill/sk-deep-review/scripts/reduce-state.cjs {spec_folder}` during `step_reduce_review_state` with an idempotency guarantee.

---

### Offline Optimizer Integration (Phase 4a)

- `deep_review_config.json` now declares the tunable surface (`convergenceThreshold`, `stuckThreshold`, `maxIterations`) via `_optimizerManaged`, with the reducer wiring, file protection, session identity, and review dimensions all locked against optimizer drift.
- The optimizer manifest, replay corpus, and advisory promotion gate are wired into command docs so review operators know which thresholds are machine-suggested vs. hand-tuned.
- Promotion stays advisory-only: the loop never auto-applies optimizer recommendations.

---

### Reliability Fixes

- Dimension coverage gate severity changed from `warning` to `blocking` for uncovered required dimensions in review mode — the earlier severity silently allowed STOP even with a required dimension untouched.
- Graph events are now a first-class optional field in both iteration record and coordination board contracts, closing a drift path flagged by deep review.
- Stuck-recovery JSONL event plus least-covered-dimension pivot give the loop a deterministic escape from plateaus before it hits the legal-stop gates.

---

### Playbook and Operator Guidance

- Added `015-graph-events-review.md` and `021-graph-convergence-review.md` to the manual testing playbook to exercise graph emission and graph-aware convergence end to end in review mode.
- Every playbook scenario was executed against the updated runtime — 30/30 PASS with zero PARTIALs.
- `references/convergence.md`, `references/state_format.md`, `references/quick_reference.md`, and `assets/review_mode_contract.yaml` all reflect the legal-stop and blocked-stop model plus the graph-convergence lens.
- The review reducer smoke test (2 iterations, 4 findings, F001 flagged as repeated across iterations 1 and 2) confirms the lifecycle tracking and idempotency guarantees.

### Verification

- `scripts/tests/nested-changelog.vitest.ts` and `scripts/tests/architecture-boundary-enforcement.vitest.ts` path-resolution bugs (macOS `/var` ↔ `/private/var` symlink and `process.cwd()`-rooted script resolution) are fixed.

No migration is required for existing review packets. Legacy stop-reason strings map through the taxonomy on the fly. Operators invoking `/spec_kit:deep-review` now get reducer-owned registry, dashboard, and strategy refreshes via the new script, idempotent across reruns. The new `scripts/` directory mirrors the `sk-deep-research` layout exactly.

---

> **Update (post-release) — 2026-04-11:**
>
> A 10-iteration closing audit of this release found two promises it did not deliver:
>
> - The four lifecycle modes this release advertised for re-entering a review session (`resume`, `restart`, `fork`, `completed-continue`) were only half-shipped. `fork` and `completed-continue` had no workflow wiring — picking them silently did the same thing as `resume`.
> - The claim-adjudication packet this release described as a hard stop gate was never actually consulted by the stop-checking logic. A reviewer could skip the packet entirely and the loop would still finish as if everything was fine.
>
> Release `v1.3.1.0` retracts the two unwired lifecycle modes, wires the claim-adjudication gate into every stop path (including a universal pre-check that runs before the hard stops), makes the shared graph database safe for concurrent sessions, rewrites the canonical agent template so the reducer can actually parse it, and replaces the phantom stop-contract shape in the convergence reference with the real `blocked_stop` event. See `v1.3.1.0.md` for plain-English Problem/Fix pairs on each one.
