## Bug Fixes

### Reducer now consumes terminal stop metadata

**Problem:** The reducer ignored `synthesis_complete` events and derived dashboard status from stale config, so the dashboard never showed the actual stop reason or legal-stop outcome after a loop finished.

**Fix:** `reduce-state.cjs` now parses the latest `synthesis_complete` JSONL record and derives `stopReason` and terminal legal-stop bundle from it.

---

### Resume/restart lineage now persists to reducer-visible state

**Problem:** Lifecycle events (`resumed`, `restarted`) were emitted to JSONL but never propagated to config-visible fields. The dashboard always showed lineage mode as `new` even after a resume.

**Fix:** The reducer detects lifecycle events and updates `parentSessionId`, `continuedFromRun`, and `lineageMode` in the registry and dashboard.

---

### Claim-adjudication finalSeverity is now canonical

**Problem:** The reducer ignored `finalSeverity` from claim-adjudication packets, so the findings registry used the original severity even after adjudication changed it.

**Fix:** The reducer now parses `claim_adjudication` events by `findingId`, uses `finalSeverity` as the canonical registry value, and seeds transitions from the packet.

---

### Stale claim-adjudication STOP veto cleared when findings resolve

**Problem:** The dashboard could show a STOP veto from a failed adjudication even after all active P0/P1 findings were resolved.

**Fix:** The veto is only surfaced when the latest failed adjudication still applies to active P0/P1 findings.

---

### Reducer is now fail-closed on corrupt JSONL

**Problem:** `reduceReviewState()` silently continued on corrupt JSONL when `lenient` was false, and wrote output files before checking corruption.

**Fix:** Corruption check now runs before any file writes. The function throws a structured error when corruption is detected and `lenient` is false.

---

### ACTIVE RISKS no longer renders `[object Object]`

**Problem:** The blocked-stop dashboard path rendered raw objects instead of formatted strings.

**Fix:** Added `formatBlockedByList()` helper. The ACTIVE RISKS section now reuses normalized entries from `blockedStopHistory`.

---

### STOP_BLOCKED events now include graph blocker detail

**Problem:** Both workflow YAMLs emitted `blocked_stop` events without the structured `graphBlockerDetail` payload.

**Fix:** `spec_kit_deep-review_auto.yaml` and `_confirm.yaml` now include `graphBlockerDetail` in the emitted payload.

---

### reviewDimensions normalized once during initialization

**Problem:** The dimensions array was parsed in two separate places, creating divergent sources of truth.

**Fix:** Normalization happens once during init and the same parsed array feeds both `deep-review-config.json` and the JSONL config row.

## Documentation

### Iteration skeleton collapsed to canonical template

The agent definition (`deep-review.md`) iteration skeleton was internally contradictory — different heading names than the reducer parser expected. Collapsed to one canonical fenced template matching the sections `reduce-state.cjs` actually consumes.

### convergence.md normalized to blocked_stop schema

Remaining `legalStop` references in later sections were updated to the persisted `blocked_stop` event schema throughout.

### loop_protocol.md corrected to full-history replay

Conflicting delta replay text removed. The entire file now consistently describes full-history replay, which is what the reducer actually does.

### review_mode_contract.yaml updated to 3 modes

Removed `fork` and `completed-continue` as live review modes. Only `new`, `resume`, `restart` remain.
