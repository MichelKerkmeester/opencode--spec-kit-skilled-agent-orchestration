## New Features

#### First-write executor provenance on review iterations

The review YAML (`spec_kit_deep-review_auto.yaml` + `spec_kit_deep-review_confirm.yaml`) `step_dispatch_iteration` now runs `writeFirstRecordExecutor` before the dispatch branch for every non-native kind. Review iterations retain the same dimensions, filesReviewed, findingsSummary, newFindingsRatio schema plus the new first-write executor block. Missing provenance on non-native runs fails fast during post-dispatch validation.

&nbsp;

#### Dispatch failure event for review

Symmetric with the research path. A review iteration crash or timeout now emits `{type:'event', event:'dispatch_failure', executor, reason, iteration, timestamp}` so failure attribution survives cases where no valid iteration record exists.

&nbsp;

#### Copilot @path large-prompt fallback for review

The `if_cli_copilot` branch in both review YAMLs now runs a 16 KB size check. Above threshold, the dispatch uses the same `Read the instructions in @<path>` wrapper prompt as the research path. Review prompt paths use `review/prompts/` as expected.

## Bug Fixes

#### description.json repair safety applies to review packets too

Since review packets use the same `generate-description.ts` loader, the new discriminated-result loader plus `mergePreserveRepair` apply to review description.json files as well. The specimen regression suite covers research packet fixtures. Review packets benefit because the helper is packet-agnostic.
