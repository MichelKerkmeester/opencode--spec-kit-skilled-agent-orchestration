This release restructures the cli-opencode provider surface to match the operator's actual `opencode providers list` output: three providers (`github-copilot`, `opencode-go`, `deepseek`) instead of the previous five-way Anthropic/OpenAI/Google/opencode-go/deepseek mix. The canonical default flips from `opencode-go/deepseek-v4-pro` to `github-copilot/gpt-5.4 --variant high` so dispatches resolve cleanly under the Copilot OAuth boundary which is pre-authenticated for users with an active subscription. Copilot models named in the skill are scoped to two (`gpt-5.4` default, `claude-sonnet-4.6` alternative); the DeepSeek direct API is trimmed to `deepseek-v4-pro` (reasoning) and `deepseek-v4-flash` (latency).

> Live verification: `opencode providers list` showed only `OpenAI`, `GitHub Copilot`, `DeepSeek`, `OpenCode Go` registered — confirming `anthropic/`, `google/` were never reachable via the operator's auth set. `opencode models` confirmed `github-copilot/gpt-5.4` and `github-copilot/gpt-5.5` are valid model ids.

## What Changed

#### Default flips → `github-copilot/gpt-5.4 --variant high`

The skill default routes through GitHub Copilot's `gpt-5.4` at high reasoning effort. Copilot ships pre-authenticated for active subscribers and exposes the broadest model surface (Claude / GPT / Gemini variants) under one OAuth token, removing the per-provider login friction that the previous `opencode-go/deepseek-v4-pro` default depended on. `gpt-5.4` accepts the full GPT variant range (`minimal / low / medium / high / xhigh`) so the existing `--variant high` posture carries over without behavioral change. Default switched from `gpt-5.5` to `gpt-5.4` mid-release to match the cli-copilot default and reflect operator preference.

#### Provider surface trimmed to 3

`anthropic/`, `openai/`, `google/` are removed from invocation strings, override tables, model-selection tables, FAQ entries, and the variant matrix throughout `SKILL.md`, `references/cli_reference.md`, and `README.md`. Operators using those providers should override per-invocation if they have separate auth, or migrate to the equivalent `github-copilot/`-prefixed model.

#### Copilot models surfaced by name → `gpt-5.4` + `claude-sonnet-4.6`

The Copilot model surface advertised in the skill is intentionally narrow: `gpt-5.4` (default) and `claude-sonnet-4.6` (Anthropic alternative for code review and balanced reasoning). The full Copilot lineup (opus / haiku / additional GPT / Gemini / Grok variants) remains reachable via `--model github-copilot/<id>` overrides but is not surfaced in the skill's tables to keep the documented surface focused.

#### DeepSeek direct API → `deepseek-v4-pro` + `deepseek-v4-flash`

`deepseek/deepseek-reasoner` and `deepseek/deepseek-chat` are dropped from the model selection tables and override examples. The direct DeepSeek API surface (which bypasses opencode-go) keeps `deepseek-v4-pro` (reasoning effort accepted) and `deepseek-v4-flash` (non-reasoning sibling). Provider-isolation testing in CO-011 still validates that `deepseek/` and `opencode-go/` are independent surfaces, not aliases.

#### Multi-provider playbook restructured

`manual_testing_playbook/03--multi-provider/` test files renamed and rewritten to match the new provider lineup:

- `001-opencode-go-default-deepseek.md` → `001-copilot-default-gpt-5-4.md` — CO-009 now validates `--model github-copilot/gpt-5.4 --variant high` against a multi-dimensional architecture trade-off prompt.
- `002-openai-gpt-5-5.md` → `002-copilot-claude-sonnet-4-6.md` — CO-010 validates the Anthropic alternative under the Copilot OAuth boundary using a TypeScript helper code-generation prompt with `tsc` validation.
- `003-google-gemini-2-5-pro.md` → `003-deepseek-direct-api.md` — CO-011 validates the deepseek direct API surface (bypasses opencode-go gateway) using a small implementation-planning prompt.
- `004-variant-levels-comparison.md` — variant cap table dropped Anthropic/OpenAI/Google rows; now lists `github-copilot/gpt-5.4` (xhigh), `github-copilot/claude-sonnet-4.6` (max), `opencode-go/deepseek-v4-pro` (provider-specific), `deepseek/deepseek-v4-pro` (reasoning effort accepted).

#### Documentation sweep

`SKILL.md` activation triggers, override tables, Essential Commands (7 examples), Rules §3, the `--model` flag default, and the Memory Selection table all updated. `README.md` Default invocation row, Copilot-section quick-start examples, Models table, Authentication table, Model Defaults paragraph (with Copilot / opencode-go / DeepSeek override snippets), and FAQ-Models updated. `references/cli_reference.md` §5 Model Selection and §5 Variant matrix updated. `references/integration_patterns.md`, `opencode_tools.md`, `agent_delegation.md`, `assets/prompt_templates.md` swept. The full `manual_testing_playbook/` (root + 25 nested test files) swept for stale `--model anthropic/` / `--model openai/` / `--model google/` references.

## Upgrade

Callers using the documented default invocation:

```bash
# Old (v1.1):
opencode run --model opencode-go/deepseek-v4-pro --variant high --agent general --format json --dir <repo>

# New (v1.2):
opencode run --model github-copilot/gpt-5.4 --variant high --agent general --format json --dir <repo>
```

Operators previously relying on `--model anthropic/...`, `--model openai/...`, or `--model google/...` should either:

1. Migrate to the equivalent `github-copilot/<id>` if they have an active Copilot subscription (e.g. `github-copilot/claude-sonnet-4.6` for Anthropic; `github-copilot/gpt-5.4` for OpenAI; `github-copilot/gemini-3.1-pro-preview` for Google — Copilot exposes all three families under one OAuth token).
2. Run `opencode providers login <provider>` for the direct provider, then continue using the original prefix per-invocation.

The new default is reachable from any environment where `opencode providers list` reports `github-copilot` as authenticated. Verify with `opencode providers list` before first dispatch.

Multi-provider playbook tests (CO-009 / CO-010 / CO-011) renamed at the file level — any external scripts referencing the old filenames must update to `001-copilot-default-gpt-5-4.md` / `002-copilot-claude-sonnet-4-6.md` / `003-deepseek-direct-api.md`.
