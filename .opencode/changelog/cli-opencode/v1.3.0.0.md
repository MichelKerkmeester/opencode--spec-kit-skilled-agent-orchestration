This release reverses the v1.2 default-provider flip and removes `github-copilot` from the cli-opencode skill entirely. The new canonical default is `--model opencode-go/deepseek-v4-pro --variant high`. The direct DeepSeek API surface stays exactly as before. A new **Provider Auth Pre-Flight (Smart Fallback)** protocol is added so the AI detects missing provider credentials before dispatching, surfaces a 3-state ASK-not-substitute decision tree, and never silently swaps a model the user didn't approve.

> Spec folder: `specs/051-cli-opencode-providers/` (Level 1). Verification: `grep -ri "github-copilot" .opencode/skill/cli-opencode/` returns 0 hits; `grep -ri "deepseek" .opencode/skill/cli-opencode/` returns 150+ preserved hits; `grep -ri "cli-copilot" .opencode/skill/cli-opencode/` returns 10 sibling-skill references intentionally preserved.

## What Changed

#### Default flips back → `opencode-go/deepseek-v4-pro --variant high`

The v1.2 default `github-copilot/gpt-5.4` is removed across SKILL.md (frontmatter Keywords, §3 Default Invocation block, override table, Model Selection table, Agent Delegation invocation patterns, Essential Commands 7 examples, ALWAYS rule 3), README.md (§1 key statistics, §3.2 Models table, §5 authentication + model defaults, §6 examples, §8 FAQ), all four `references/*.md` files, and `assets/prompt_templates.md`. The new default routes through OpenCode Go's gateway with DeepSeek's reasoning model, providing elevated reasoning at low cost without per-provider OAuth friction.

#### Provider surface trimmed to 2

`github-copilot/` is fully removed as a provider. The skill now documents two providers: `opencode-go` (DEFAULT — gateway routing for DeepSeek and other open models) and `deepseek` (direct DeepSeek API — bypasses the opencode-go gateway). The `cli-copilot` *sibling skill* and "Copilot" external-runtime caller mentions in cross-AI handback prose stay — those are caller identities, not the github-copilot provider.

#### Provider Auth Pre-Flight (Smart Fallback) protocol added

New §3 subsection codifies the pre-dispatch auth check. One-shot bash pre-flight (`opencode providers list`) captures `OPENCODE_GO_OK` and `DEEPSEEK_OK` flags. A 3-state decision table (default-available / default-missing-fallback-ready / both-missing) governs routing. Two user-prompt templates (default-missing-with-fallback A/B/C menu; both-missing login-command menu) make the ASK-not-substitute contract explicit. Cache invalidation on auth-error mid-dispatch reruns the pre-flight before retry. ALWAYS rule 11 mandates the pre-flight before first dispatch; the Error Handling table now points `provider/model not found` and `401 Unauthorized` rows back to the pre-flight.

#### Multi-provider playbook trimmed

Two copilot-only test entries deleted: `manual_testing_playbook/03--multi-provider/001-copilot-default-gpt-5-4.md` and `002-copilot-claude-sonnet-4-6.md`. The `03--multi-provider/` section header now reads `CO-011..CO-012` (was `CO-009..CO-012`). The remaining test files (`003-deepseek-direct-api.md`, `004-variant-levels-comparison.md`) keep their existing IDs so cross-references stay valid; their narratives lost the github-copilot mentions. 28 other playbook entries (across `01--cli-invocation`, `02--external-dispatch`, `04--agent-routing`, `05--session-continuity`, `06--integration-patterns`, `07--prompt-templates`, `08--parallel-detached`, `09--cross-repo-cross-server`) were swept via bulk substitution — `--model github-copilot/{gpt-5.4,claude-sonnet-4.6}` → `--model opencode-go/deepseek-v4-pro`.

#### Graph-metadata refresh

`graph-metadata.json` `causal_summary` now reads "Defaults to provider opencode-go (model deepseek-v4-pro, variant high) and additionally surfaces the direct deepseek API." The github-copilot keyword is dropped from the SKILL.md frontmatter HTML comment Keywords list.

## Upgrade

Callers using the v1.2 documented default invocation:

```bash
# Old (v1.2):
opencode run --model github-copilot/gpt-5.4 --variant high --agent general --format json --dir <repo>

# New (v1.3):
opencode run --model opencode-go/deepseek-v4-pro --variant high --agent general --format json --dir <repo>
```

Operators previously relying on `github-copilot/<id>` should either:

1. Migrate to the equivalent `opencode-go/<id>` (e.g. `opencode-go/glm-5.1`, `opencode-go/kimi-k2.6`) if the upstream model is available through the gateway.
2. Run `opencode providers login opencode-go` (or `opencode providers login deepseek`) before first dispatch.

The Provider Auth Pre-Flight ALWAYS rule 11 mandates a one-shot `opencode providers list` check on session start. If the default isn't configured, the AI surfaces a 3-option menu (use configured fallback / log in to default / name a different model) instead of silently swapping. Operators expecting a silent fallback should explicitly approve option A in the prompt.

Multi-provider playbook tests CO-009 (was `001-copilot-default-gpt-5-4.md`) and CO-010 (was `002-copilot-claude-sonnet-4-6.md`) are deleted entirely. External scripts referencing those filenames must drop the references.
