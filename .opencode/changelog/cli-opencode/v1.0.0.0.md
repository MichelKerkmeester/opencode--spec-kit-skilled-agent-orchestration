Initial release of cli-opencode, the fifth sibling under `.opencode/skill/cli-*/`. The skill lets external AI assistants and detached OpenCode sessions invoke `opencode run` for full plugin, skill, and MCP runtime dispatch with three documented use cases plus a layered self-invocation guard that protects against circular dispatch.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/047-cli-opencode-creation/` (Level 3)

---

## What Changed

#### New Features

- **OpenCode CLI orchestrator skill** -- The fifth sibling under `cli-*/` was missing. We added cli-opencode with the canonical 9-file blueprint (3 root, 2 assets, 4 references). External runtimes (Claude Code, Codex, Copilot, Gemini, raw shell) can now dispatch into a fresh OpenCode session that loads every project plugin, skill, MCP tool, and the Spec Kit Memory database in a one-shot call.

- **Three documented use cases** -- A naive "delegate to opencode" skill cannot work in a repo where opencode IS the host runtime. We split the surface into three orthogonal use cases: external runtime to OpenCode, in-OpenCode parallel detached session for ablation and worker farms, and cross-AI handback for Codex / Copilot / Gemini reaching the project's spec-kit workflows. Each case has its own copy-paste prompt template and its own `opencode run` invocation shape.

- **Self-invocation guard with three detection layers** -- Without protection, an in-OpenCode operator asking "delegate this to opencode" would create a circular dispatch. We added a three-layer detection signal (env var lookup for any `OPENCODE_*` variable, process ancestry probe for an `opencode` parent, lock-file probe at `~/.opencode/state/<id>/lock`). The smart router refuses self-dispatch with a documented error message that names the three remediation options.

- **Sibling parity edges** -- Without symmetric edges, the advisor graph would never surface cli-opencode as a sibling of the existing four cli-* skills. We patched the graph-metadata.json files of cli-claude-code, cli-codex, cli-copilot, and cli-gemini to add a sibling edge to cli-opencode at weight 0.5 with context "CLI orchestrator peer". The round-trip is now symmetric across all five cli-* skills.

#### Documentation

- **Skill counts and tables refreshed** -- Both `.opencode/skill/README.md` and `.opencode/README.md` claimed "20 skills" or "four CLI skills" before this release. We applied 8 line-anchored patches to the skill README and 2 to the root README so all counts, tables, tree diagrams, and feature rows reflect the new "21 skills" / "five CLI skills" reality.

## Upgrade

No migration required. Sibling-edge additions are additive. The advisor scoring tables continue to operate at their baseline thresholds. Existing cli-* dispatches are unchanged.

To pick up the new skill, restart the MCP server (so it reloads `skill-graph.sqlite`) and rerun `python3 .opencode/skill/system-spec-kit/mcp_server/skill_advisor/scripts/skill_advisor.py --health` to confirm `skill_graph_skill_count=22` and `inventory_parity.in_sync=true`.
