This release closes 7 playbook authoring bugs surfaced by running the full 25-scenario cli-codex playbook end-to-end against codex 0.125.0. All critical-path scenarios (CX-001 default invocation, CX-005 read-only sandbox, CX-006 workspace-write sandbox) PASS. The remaining FAILs traced to playbook authoring drift, not codex runtime issues — fixes restore them to the working pattern.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/011-manual-testing-playbook-coverage-and-run/` (Level 3)

---

## What Changed

#### Gate-3 preamble added to 5 dispatched-prompt scenarios

CX-008, CX-016, CX-018, CX-021, CX-022 previously omitted the documented `Spec folder: <path> (pre-approved, skip Gate 3)` preamble required by SKILL.md ALWAYS rule 8. Codex inherited the project's CLAUDE.md / AGENTS.md gate definitions and paused non-interactively waiting for spec-folder selection that no human would answer in `exec` mode. Each prompt now leads with the preamble so codex skips Gate 3 and executes the dispatched task.

#### CX-014 broken.ts JS syntax + stdin redirect

The `printf` block intended to write a deliberately-broken JavaScript file with an off-by-one bug had two unrelated typos: `let i = 0.` (period instead of semicolon) and `I <= arr.length. I++` (capital I). The test was failing because codex couldn't parse the input file at all. Fixed to use canonical lowercase JS (`for (let i = 0; i <= arr.length; i++)`) so the off-by-one bug is the only flaw. Also added `< /dev/null` to the codex exec invocation per ALWAYS rule 5.

#### CX-017 session resume — capture UUID from verbose header

The session-resume scenario asked the model to "announce 'SESSION_ID: <id>' on the last line of your output", but the model treated the marker as a literal label and emitted the test ID `cx017` instead of the real UUID. Resume calls then failed against an invalid id. The scenario now extracts the session UUID via codex's verbose-header regex (`session id: [a-f0-9-]+`) with a fallback to `ls -t ~/.codex/state/`. Also dropped `--sandbox` from `codex exec resume` (rejected by clap — not a valid flag for that subcommand) and added `< /dev/null` to all calls.

#### Root playbook filename normalized

Renamed `MANUAL_TESTING_PLAYBOOK.md` → `manual_testing_playbook.md` to match the lowercase canonical convention adopted across all skill playbooks (sk-doc template was patched at the same time so future `/create:testing-playbook` runs produce lowercase by default). All cross-references in scenario files updated.
