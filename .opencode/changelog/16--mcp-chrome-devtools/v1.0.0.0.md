## [**1.0.0.0**] - 2025-12-27

This first release gives the team a shared way to use browser developer tools from AI workflows. It covers everyday browser checks, visual capture, and troubleshooting in one package.

---

### Two ways to work

- Need: Teams needed a fast option for routine browser tasks and a deeper option for bigger multi-tool jobs.
- Change: This release introduced a lightweight command-line path for quick work and a fuller integration path for more complex workflows.
- Why it matters: People can choose the simplest route for the job instead of carrying extra setup every time.

---

### Built-in browser actions

- Need: Common testing and debugging tasks were spread across separate tools and notes.
- Change: The skill now bundles page control, clicks, screenshots, network checks, console viewing, and storage access.
- Why it matters: One skill can now cover most of the browser work operators do day to day.

---

### Starter guides

- Need: New users needed clear setup and usage instructions before they could trust the workflow.
- Change: The release shipped a main guide, install help, and focused reference notes for common browser patterns.
- Why it matters: People can get started faster and look up details without digging through code.
