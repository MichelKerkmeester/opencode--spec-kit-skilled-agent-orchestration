This release gives `sk-deep-research` a fully explicit, executable packet expected behavior. The research loop now has main run history and lifecycle rules, reducer-owned state surfaces, runtime-parity guidance across all mirrors, and focused expected behavior tests that lock the behavior in place. The goal is to make the workflow easier to follow and easier to trust.

---

#### Research Contract Alignment

- Canonical `deep-research-*` packet artifacts are now frozen across the skill docs, references, runtime mirrors, and workflow assets.
- Lineage fields are now explicit and shared: `sessionId`, `parentSessionId`, `run historyMode`, `generation`, and `continuedFromRun`.
- Lifecycle branches are now documented consistently across the skill: `resume`, `restart`, `fork`, and `completed-continue`.
- Reducer-owned outputs are now clearly separated from agent-authored iteration artifacts, removing ambiguity around dashboard, strategy, and registry refreshes.

---

#### Executable Helpers

- Added a machine-readable runtime capability matrix plus a human-readable capability reference.
- Added `runtime-capabilities.cjs` to resolve runtime metadata programmatically from the main matrix.
- Added `reduce-state.cjs` so packet state can be reduced into synchronized strategy, findings-registry, and dashboard outputs.
- Helper packaging now matches the repo's ESM boundary by using `.cjs` entry points instead of incompatible CommonJS-in-ESM paths.

---

#### Runtime and Workflow Parity

- OpenCode, Claude, Gemini, Codex, and the compatibility wrapper now describe the same deep-research packet expected behavior.
- Auto and confirm workflow YAML assets now carry the same lifecycle, migration, registry, and reducer expectations.
- The manual testing playbook was updated so reducer-owned refresh behavior is described consistently in both the root playbook and the concrete iteration scenarios.

---

#### Verification

- Added `deep-research-expected behavior-parity.vitest.ts` to keep the docs, command assets, capability matrix, and runtime mirrors synchronized.
- Added `deep-research-reducer.vitest.ts` to verify reducer idempotency and packet-integrity expectations.
- Phase validation passed under the strict Spec Kit validator for `040-sk-auto-deep-research-review-improvement/001-sk-deep-research-improvements`.
