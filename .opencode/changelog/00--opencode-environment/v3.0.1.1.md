## [**3.0.1.1**] - 2026-03-29

Split the unified `sk-deep-research` skill (which handled both investigation and code auditing) into two independent skills. Each now has its own command, YAML workflows, reference documentation, manual testing playbook, and runtime wrappers.

> Spec folder: `.opencode/specs/03--commands-and-skills/036-sk-deep-research-review-split/`

---

## What Changed

### 1. New skill created: sk-deep-review v1.0.0.0

A standalone skill for iterative code review and quality auditing, extracted from the review-mode half of the unified sk-deep-research.

**New command:** `/spec_kit:deep-review` with `:auto` and `:confirm` modes. This replaces the old `/spec_kit:deep-research:review` composite command that routed through the research command.

**Skill package:** SKILL.md with 8 sections, README.md, 4 reference documents (loop protocol, convergence algorithms, state formats, quick reference), 5 asset files including the review mode contract, and 28 manual testing scenarios across 6 categories.

**YAML workflows:** Renamed from `spec_kit_deep-research_review_auto.yaml` to `spec_kit_deep-review_auto.yaml` (and the confirm variant). Internal references updated to point to the new sk-deep-review skill.

### 2. Existing skill updated: sk-deep-research v1.3.0.0

The original skill, now stripped of all review-mode content and focused purely on iterative investigation.

**Removed:** Review mode triggers, REVIEW_* intent signals, review architecture diagrams, review-specific ALWAYS rules, review success criteria, review references, `:review` command suffixes, review examples, review testing playbook category.

**Added:** Cross-references to sk-deep-review in the "When to Use" and "Related Resources" sections. Numbered H2 sections in the quick reference per template standard.

### 3. Skill advisor routing split

Review keywords (`"deep review"`, `"review mode"`, `"iterative review"`, `"code audit"`, `":review"`, etc.) now route to `sk-deep-review` instead of `sk-deep-research`. Research keywords (`"autoresearch"`, `"deep research"`, `"research loop"`, etc.) unchanged. Updated in both Public and Barter repos.

### 4. Agent definitions updated

All `@deep-review` agent definitions across 4 runtimes (OpenCode, Claude, Codex, Gemini/Agents) updated to reference the new command and skill names.

### 5. Documentation updated

CLAUDE.md, AGENTS.md, AGENTS_example_fs_enterprises.md, root README.md, `.opencode/skill/README.md`, `.opencode/command/README.txt`, `.opencode/README.md`, and system-spec-kit README all updated with separate deep-research and deep-review entries.

### 6. Runtime wrappers

New `.agents/commands/spec_kit/deep-review.toml` created. Existing deep-research TOML updated to remove review references.
