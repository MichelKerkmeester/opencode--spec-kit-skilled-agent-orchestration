## [**1.2.0.0**] - 2026-02-03

This release updates the OpenCode environment with work on memory system overhaul (spec 082), command consolidation (spec 083), bug remediation (specs 083-085). The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Memory System Overhaul (Spec 082)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Causal Memory Graph: 6 relationship types (caused, enabled, supersedes, contradicts, derived_from, supports) with depth-limited traversal. Session Deduplication: Hash-based duplicate prevention achieving 50% token savings on follow-up queries. Intent-Aware Retrieval: 5 intent types (add_feature, fix_bug, refactor, security_audit, understand) with automatic query classification.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Command Consolidation (Spec 083)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: 44% Reduction: 9 commands consolidated to 5. /memory: context - Unified entry with intent-aware routing (absorbed search). /memory: learn - Feedback loop with correct subcommand (absorbed correct).
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Bug Remediation (Specs 083-085)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: 86 Bugs Fixed Total: 21 highest-priority, 32 high-priority, 27 medium-priority, 6 P3. 30-agent Audit: Spec 084 with 77% false positive filtering (8 confirmed fixes). 10-agent Parallel Fixes: Spec 085 with 34 bugs across core infrastructure, scoring, extractors, templates.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Documentation Quality (Spec 004)
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Validate_document.py: New Python validation script with auto-fix capability. Template Rules: 5 document types (readme, skill, reference, asset, agent) with severity levels. Test Suite: 6 test fixtures, 53 files validated, 0 invalid in main codebase.
