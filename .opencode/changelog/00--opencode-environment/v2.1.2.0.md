## [**2.1.2.0**] - 2026-02-17

This release updates the OpenCode environment with work on plan-to-implementation gate enforcement (04--commands/v2.0.4.0), source #6: skill reference indexing (01--system-spec-kit/v2.2.21.0). The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Plan-to-Implementation Gate Enforcement (04--commands/v2.0.4.0)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: AGENTS.md Gate 3 phase boundary -- Gate 3 answers no longer carry over across workflow phases. Plan-to-implement transition re-evaluates Gate 3. Memory Save Rule scoping -- "do NOT re-ask" instruction scoped explicitly to memory saves only, preventing over-generalization. Plan structured config files enforcement -- Both spec_kit_plan_auto.yaml and spec_kit_plan_confirm.yaml termination sections now enforce routing free-text implement requests through /spec_kit:implement.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Source #6: Skill Reference Indexing (01--system-spec-kit/v2.2.21.0)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: New config section -- config.jsonc section 12 skillReferenceIndexing with user-editable indexedSkills[]. 3 new document types -- skill_reference (0.35), skill_checklist (0.35), skill_asset (0.30). Discovery function -- findSkillReferenceFiles() in memory-index.ts follows existing pattern.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
