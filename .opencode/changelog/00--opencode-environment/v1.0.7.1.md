## [**1.0.7.1**] - 2026-01-23

This release updates the OpenCode environment with work on multi-agent dispatch (a/b/c selection), command phase structure, structured config files configuration. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Multi-Agent Dispatch (A/B/C Selection)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Dispatch Mode Phase: New mandatory phase added to 5 commands (/spec_kit:complete, /spec_kit:plan, /spec_kit:implement, /spec_kit:research, /spec_kit:debug). Option A: Single Agent - Execute with one Opus agent (default). Option B: Multi-Agent (1+2) - 1 Opus orchestrator + 2 Sonnet workers.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Command Phase Structure
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Complete.md: Phase 2.5 → Phase 3 (Research), new Phase 4 (Dispatch), Phase 3 → Phase 5 (Memory). Plan.md: New Phase 3 (Dispatch), Phase 3 → Phase 4 (Memory). Implement.md: New Phase 3 (Dispatch), Phase 3 → Phase 4 (Memory).
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Structured Config Files Configuration
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: 12 Structured Config Files Configs Updated With Multi_agent_config Section: spec_kit_research_auto.yaml, spec_kit_research_confirm.yaml, spec_kit_debug_auto.yaml, spec_kit_debug_confirm.yaml, spec_kit_complete_auto.yaml, spec_kit_complete_confirm.yaml, spec_kit_plan_auto.yaml, spec_kit_plan_confirm.yaml, spec_kit_implement_auto.yaml, spec_kit_implement_confirm.yaml. Worker Definitions: Research (codebase_explorer, external_researcher, technical_analyzer), Debug (call_path_tracer, pattern_searcher, edge_case_hunter), Complete/Plan/Implement (architecture_explorer, feature_explorer, dependency_explorer).
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Fixed
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Phase Numbering: All commands now use sequential integers (no 2.5 or 3.5). Structured Config Files Phase References: phase_2_5_research → phase_3_research in complete YAMLs.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
