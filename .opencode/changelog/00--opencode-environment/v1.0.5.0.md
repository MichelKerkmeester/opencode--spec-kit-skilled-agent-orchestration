## [**1.0.5.0**] - 2026-01-17

This release updates the OpenCode environment with work on memory command separation, dynamic complexity-based templates, composite folder ranking. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Memory Command Separation
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: /memory: database Command - New dedicated command for database management with 9 modes: stats, scan, cleanup, tier, triggers, validate, delete, health. Safety Gates For Destructive Operations: Hard block confirmations for cleanup and delete operations. Automatic Checkpoint Creation: Pre-cleanup checkpoint created before bulk deletions.
