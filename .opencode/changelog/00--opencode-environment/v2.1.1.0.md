## [**2.1.1.0**] - 2026-02-17

This release updates the OpenCode environment with work on agent orchestration (03--agent-orchestration/v2.0.4.0 + v2.0.5.0), commands (04--commands/v2.0.2.0 + v2.0.3.0), documentation workflow (06--workflows-documentation/v1.0.7.0 + v1.0.8.0). The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Agent Orchestration (03--agent-orchestration/v2.0.4.0 + V2.0.5.0)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Documentation alignment pass -- Memory/spec_kit command docs and cross-platform agent docs were normalized to current orchestration behavior. Orchestrate restructure (27 -> 10 sections) -- Decision-loop-based layout, Nesting Depth Protocol prominence, budget consolidation, and anti-pattern reduction. Cross-variant parity and sync -- Base/chatgpt parity maintained, copilot aligned, and.claude/agents/ synced for all core agent profiles.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Commands (04--commands/v2.0.2.0 + V2.0.3.0)
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Agent routing compliance baseline -- 33 command files aligned to AGENTS.md routing model across create/spec_kit command stacks. Codex Compatibility Hardening -- Three-part Fix Shipped For Spec_kit: strip trigger-prone markdown routing blocks, add explicit CONSTRAINTS guards, and convert structured config files routing semantics to conditional availability. Dispatch timing protection -- Reduced false early dispatch for @review, @handover, and @debug by moving dispatch intent into safer structured config files semantics.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### Documentation Workflow (06--workflows-documentation/v1.0.7.0 + V1.0.8.0)
- Need: This part of the release makes the written guidance easier to follow and easier to trust.
- What changed: Emoji enforcement removed -- H2/TOC emoji requirements dropped from templates, validators, references, and test fixtures. Routing template modernization -- Smart Routing docs updated to detection-first flow with merged resource domains and a single authoritative pseudocode section.
- Why it matters: That matters because people can understand the system faster and use it more confidently.

### System Spec Kit Series (01--system-spec-kit/v2.2.9.0 -> V2.2.20.0)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Indexable spec documentation introduced (prominent) -- Spec folder documents became a first-class index source (spec, plan, tasks, checklist, decision records, implementation summary, research, handover), enabling find_spec/find_decision workflows and spec-level retrieval. README indexing and anchor coverage strengthened -- README indexing quality and anchor/HVR compliance were expanded to improve retrieval precision and consistency. v2.2.9.0 -- Agent routing doc update in SKILL/README, including dispatch table and mode suffix guidance.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Smart Router And Skill Alignment
- Need: This part of the release makes the written guidance easier to follow and easier to trust.
- What changed: V2 rollout and refinement included -- Weighted routing, adaptive top-N ambiguity handling, synonym expansion, UNKNOWN fallback guidance, and benchmark-backed validation coverage. Skill/changelog consistency updates -- Version normalization and changelog additions for workflows-code--opencode, workflows-code--web-dev, workflows-code--full-stack, and sk-git.
- Why it matters: That matters because people can understand the system faster and use it more confidently.
