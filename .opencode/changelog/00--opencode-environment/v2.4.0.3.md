## [**2.4.0.3**] - 2026-02-26

This release updates the OpenCode environment with work on memory export importance tier conflict resolved, backward-compatible resolution. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Memory Export Importance Tier Conflict Resolved
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Memory Export Importance Tier Conflict Resolved: Template had hardcoded importance_tier: "normal" alongside dynamic importanceTier: "{{IMPORTANCE_TIER}}", producing files with two conflicting values. Now emits a single field.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Backward-compatible Resolution
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Backward-compatible Resolution: Existing memory files with dual fields are handled correctly - prefers the non-default value.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
