## [**1.2.4.0**] - 2026-02-06

Context Window Budget System - Adds CWB system to orchestrate agent, preventing context overflow when dispatching 5-20 sub-agents in parallel. This release focuses on making the environment easier to use and easier to trust.

### 1. New Features
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Budget Formula: available_budget = context_window - system_overhead - conversation_history - agent_definition with conservative defaults (~150K tokens available for results). Scale Thresholds - Four Tiers: 1-4 agents (direct/full), 5-9 (summary-only, 30 lines), 10-15 (file-based + waves of 5), 16-20 (file-based + waves of 5). Pre-dispatch Checklist: Mandatory CWB check for 5+ agent dispatches before any Task tool calls.
- Why it matters: That matters because the environment can do more useful work without extra setup.

### 2. Changed
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Core Workflow (§1): Added step 5 CWB CHECK between DECOMPOSE and DELEGATE; renumbered steps 5→6 through 9→10; Updated mermaid diagram with CWB CHECK node. Sub-Orchestrator Pattern (§5) - Added Context Budget Constraint: sub-orchestrators MUST compress results before returning to parent; Added Return Size Rule subsection with max return sizes by sub-orchestrator scale (2-4K tokens). Resource Budgeting (§9): Added Orchestrator Self-Budget subsection with budget component table (~25K overhead + ~15K agent def + ~10K history = ~150K available).
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### 3. Fixed
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
