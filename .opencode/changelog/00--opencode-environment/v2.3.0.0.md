## [**2.3.0.0**] - 2026-02-20

This release updates the OpenCode environment with work on features, architecture, documentation. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. Features
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: .gemini/ Directory Structure: Complete Gemini command-line interface runtime adapter with agents/, commands/, skills/, specs shortcut link, and settings.json. 8 Gemini-native Agents: orchestrate, context, debug, research, review, speckit, write, handover -- each with Gemini structured config files frontmatter (kind: local, tools: array, max_turns, timeout_mins), routing-optimized descriptions, and 1:1 OpenCode body content. 19 Config File Format Command Wrappers: All OpenCode commands wrapped in Gemini's.toml format using @{file} injection to preserve single source of truth -- spec_kit/ (7), memory/ (5), create/ (6), root (1).
- Why it matters: That matters because the environment can do more useful work without extra setup.

### 2. Architecture
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: 4-provider Runtime Matrix: Framework now supports Copilot, ChatGPT, Claude, and Gemini command-line interface as first-class runtimes with consistent agent/command/skill access. Token Budget Optimization: All agents calibrated for gemini-3.1-pro 400K effective window -- role-tuned max_turns (10-25) and timeout_mins (5-15) based on per-agent token budget analysis. Tool Scoping Per Role: Context/review read-only (read_file, grep_search, list_directory); research/write include web_search; handover excludes run_shell_command; orchestrate minimal (read_file only).
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3. Documentation
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: AGENTS.md §7 Extended: Runtime resolution table now includes Gemini command-line interface row. Path Conventions Aligned: All Gemini agent bodies reference.gemini/agents/*.md. Template Compliance Validated: All 8 agents pass workflows-documentation agent template checklist (CORE WORKFLOW, CAPABILITY SCAN, OUTPUT VERIFICATION, ANTI-PATTERNS, RELATED RESOURCES, SUMMARY).
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

## Upgrade

- Pull latest to get the new.gemini/ directory.
- Install Gemini command-line interface if not already installed.
- Verify with gemini agents list and /mcp inside a Gemini command-line interface session.
- No migration required for existing runtimes -- all changes are additive.
