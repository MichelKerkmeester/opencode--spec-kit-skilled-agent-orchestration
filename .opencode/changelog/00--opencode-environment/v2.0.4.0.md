## [**2.0.4.0**] - 2026-02-16

This release updates the OpenCode environment with work on codex profile system, sub-agent dispatch tool connection layer, cross-platform agent model alignment. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. Codex Profile System
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Fast: gpt-5.3-codex-spark. Balanced: gpt-5.3-codex. Powerful: gpt-5.3-codex.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### 2. Sub-Agent Dispatch Tool Connection Layer
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Added codex-specialized-subagents tool connection layer server enabling the delegate tool. Reads.codex/agents/*.md frontmatter and spawns codex exec --profile <name> "<task>".
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3. Cross-Platform Agent Model Alignment
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Fast: gpt-5.3-codex-spark. Balanced: gpt-5.3-codex. Powerful: gpt-5.3-codex.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### 4. Handover Agent Downgrade
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Handover agent changed from Sonnet to Haiku (Copilot + Claude Code) and fast profile (Codex). Validated by spec 012/013 for structured template-filling tasks.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.
