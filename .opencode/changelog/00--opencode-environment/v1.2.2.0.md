## [**1.2.2.0**] - 2026-02-04

Coding Analysis Lenses - New framework for improved code quality decisions with 6 lenses and anti-pattern detection. This release focuses on making the environment easier to use and easier to trust.

### New Coding Analysis Framework
- Need: This part of the release makes the day-to-day workflow easier to follow and less error-prone.
- What changed: 6 Coding Lenses: CLARITY (simplicity), SYSTEMS (dependencies), BIAS (wrong problem), SUSTAINABILITY (maintainability), VALUE (actual impact), SCOPE (complexity matching). 6 Anti-Patterns: Over-engineering, Premature optimization, Cargo culting, Gold-plating, Wrong abstraction, Scope creep with trigger phrases and responses. Solution Flow Integration: Added SYSTEMS → BIAS → SCOPE lens steps to request analysis workflow.
- Why it matters: That matters because the workflow is easier to use correctly without extra guesswork.

### Threshold Alignment
- Need: This part of the release improves threshold alignment.
- What changed: Threshold Alignment: Renamed "Legacy thresholds" to "Simple thresholds" for confidence-only queries.
- Why it matters: That matters because it makes the environment easier to use and maintain.

### Dual-Threshold Alignment
- Need: This part of the release improves dual-threshold alignment.
- What changed: Dual-Threshold Alignment: Aligned percentages with dual-threshold: 40-69% PROCEED WITH CAUTION, ≥70% PASS.
- Why it matters: That matters because it makes the environment easier to use and maintain.
