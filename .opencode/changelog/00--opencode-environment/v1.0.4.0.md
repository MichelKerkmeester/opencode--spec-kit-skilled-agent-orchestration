## [**1.0.4.0**] - 2026-01-15

This release updates the OpenCode environment with work on new, changed, fixed. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. New
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Anchor System For Targeted Memory Retrieval - The Memory_search Tool Now Accepts An Anchors Parameter Enabling Retrieval Of Specific Memory Sections (e.g., "summary", "decisions") Instead Of Full File Content. Verified Savings: 73% for summary-only, 87% for decisions-only, 61% for summary+decisions. Response metadata includes tokenMetrics with savings calculations. Modular Tool Connection Layer Server Architecture - Decomposed Context-server.js Into 19 Focused Modules Across 5 Directories: core/ (3 files, 507 lines) - Server configuration, database state management - handlers/ (7 files, 1,395 lines) - Tool handlers for search, triggers, CRUD, checkpoints - formatters/ (3 files, 353 lines) - Search results and token metrics formatting - utils/ (4 files, 478 lines) - Validators, JSON helpers, batch processing - hooks/ (2 files, 223 lines) - SK-004 auto memory surfacing. Voyage 4 Embedding Support: Added voyage-4, voyage-4-large, and voyage-4-lite to supported models with automatic database separation per model (existing voyage-3.5 embeddings preserved).
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 2. Changed
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Default Embedding Model: Changed from voyage-3.5 to voyage-4 for Spec Kit Memory tool connection layer. Narsil retains voyage-code-2 until a code-specific Voyage 4 model is released. Entry Point Reduction: context-server.js reduced from 2,703 lines to 319 lines (88% reduction). Documentation Accuracy: ANCHOR system documentation updated from "93% token savings" claim to verified metrics. Debug delegation threshold standardized to "3+ failed attempts" across all documentation.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3. Fixed
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Critical: Missing await in memory_search - Fixed formatSearchResults() calls returning Promise objects instead of resolved results when includeContent=true. Critical: Undefined E429 Error Code - Added definition to errors.js and documented in troubleshooting guide. Critical: Embedding API Rate Limiting - Added BATCH_DELAY_MS (100ms default) to prevent provider throttling.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
