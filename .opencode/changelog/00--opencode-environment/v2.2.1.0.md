## [**2.2.1.0**] - 2026-02-20

Context overload prevention integrated into all 3 orchestrator variants with 5 new sections per runtime, runtime-adapted thresholds, and 1 retroactive spec folder documenting the work. This release focuses on making the environment easier to use and easier to trust.

### Context Pressure Response Protocol (section 7)
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Context Pressure Response Protocol (section 7): 4-step sequence added to all orchestrator variants: pause dispatching, announce to user, wait for confirmation, fallback to handover. Claude uses /compact, Copilot/ChatGPT use "save context".
- Why it matters: That matters because the environment can do more useful work without extra setup.

### Output Discipline (section 7)
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Output Discipline (section 7): Rules for summarizing large outputs, never echoing raw tool results. Threshold is >50 lines for Claude/Copilot, >80 lines for ChatGPT/Codex.
- Why it matters: That matters because the environment can do more useful work without extra setup.

### Orchestrator Self-Protection Rules (section 8)
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Orchestrator Self-Protection Rules (section 8): Four explicit rules prevent the orchestrator from overloading its own context: targeted reads (offset/limit on files >200/300 lines), no accumulation (3/4 file limit), write-don't-hold, batch tool calls.
- Why it matters: That matters because the environment can do more useful work without extra setup.

### Recovery Protocol (section 6)
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Recovery Protocol (section 6): Explicit steps for post-compaction (Claude) and post-session-degradation (Copilot/ChatGPT) recovery. Claude references CLAUDE.md + MEMORY.md, other runtimes reference AGENTS.md + project config.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3 New Anti-patterns (section 9)
- Need: This part of the release fixes rough edges and makes the environment more dependable.
- What changed: 3 New Anti-patterns (section 9): Never bulk-read large files in main context, never echo raw output, never continue after degradation without user confirmation.
- Why it matters: That matters because the environment is easier to maintain and less likely to break in everyday use.

### Runtime-adapted Thresholds
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Runtime-adapted Thresholds: ChatGPT/Codex (~220K context) uses higher limits than Claude/Copilot (~150K): 300 vs 200 line file reads, 4 vs 3 file accumulation, >80 vs >50 line output summarization.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### Spec Folder 022
- Need: This part of the release makes the written guidance easier to follow and easier to trust.
- What changed: Spec Folder 022: Retroactive Level 2 spec folder with spec.md, plan.md, tasks.md, checklist.md, and implementation-summary.md documenting all decisions and verification results.
- Why it matters: That matters because people can understand the system faster and use it more confidently.

- What changed: Changelog V2.0.8.0: Agent-orchestration series changelog documenting the orchestrator improvements.
