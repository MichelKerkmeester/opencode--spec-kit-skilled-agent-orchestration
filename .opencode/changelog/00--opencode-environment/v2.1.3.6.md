## [**2.1.3.6**] - 2026-02-19

This release updates the OpenCode environment with work on spec folder resolution fixed, shortcut link-safe specs discovery, session memory corrected. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. Spec Folder Resolution Fixed
- Need: This part of the release fixes rough edges and makes the environment more dependable.
- What changed: Spec Folder Resolution Fixed: JSON SPEC_FOLDER values now support absolute, specs/..., and.opencode/specs/... forms without fallback misrouting.
- Why it matters: That matters because the environment is easier to maintain and less likely to break in everyday use.

### 2. Shortcut Link-safe Specs Discovery
- Need: This part of the release improves shortcut link-safe specs discovery.
- What changed: Shortcut Link-safe Specs Discovery: Specs roots are de-duplicated by realpath to avoid dual-root warnings and selection ambiguity when specs is symlinked.
- Why it matters: That matters because it makes the environment easier to use and maintain.

### 3. Session Memory Corrected
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Session Memory Corrected: Misrouted memory was moved into.opencode/specs/04--agent-orchestration/021-codex-orchestrate/memory/ and normalized to the correct spec metadata.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
