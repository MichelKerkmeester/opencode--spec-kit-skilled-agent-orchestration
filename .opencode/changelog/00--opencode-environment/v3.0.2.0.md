## What Changed

### Runtime correctness hardened

- **Entrypoint guard** -- importing `@spec-kit/mcp-server` no longer starts the server as a side effect. The `main()` call is now gated behind an entrypoint check.
- **ESM globals fixed** -- two scripts that still used the CommonJS `__dirname` global now use `import.meta.dirname`.
- **Node version aligned** -- all 4 workspace packages now require `>=20.11.0`.
- **Root export corrected** -- `@spec-kit/shared` root export now points at `dist/index.js` instead of `dist/embeddings.js`.

### Security gaps closed

- **Path boundary validation** -- `shared/paths.ts` now validates that resolved paths stay within the workspace root, preventing escape to arbitrary directories.
- **V-rule fail-closed** -- the validation bridge now blocks saves when the quality validation module is missing, instead of silently succeeding.
- **Scope-aware duplicate detection** -- the preflight duplicate check now filters by tenant/user/agent scope and redacts cross-scope metadata.
- **Admin auth warning** -- shared-memory admin operations now log a trusted-transport warning when caller identity cannot be verified.

### Reliability and performance improved

- **Typed warnings** -- file-persistence failures now carry a distinct category instead of being lumped into generic "anchor issues."
- **Import consolidation** -- three different dynamic-import error-handling patterns in `workflow.ts` unified into one helper.
- **Hot-path caching** -- vector-index imports cached at module level instead of re-resolved per call.
- **CLI startup deferred** -- `--help` and lightweight commands no longer load the full database stack.

### Full spec packet closed

The 023-esm-module-compliance spec is now fully complete across all 6 phases. Parent packet docs truth-synced, Phase 4 child packet closed, all P0/P1 checklist items verified with evidence.
