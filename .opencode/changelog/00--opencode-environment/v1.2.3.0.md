## [**1.2.3.0**] - 2026-02-05

This release updates the OpenCode environment with work on critical bug fixes, sqlite schema unification, gate numbering standardization. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. Critical Bug Fixes
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: CREATE_LINKED SQL Constraint Violation: memory_conflicts table CHECK constraint rejected valid CREATE_LINKED PE decisions, silently losing conflict tracking data. Added CREATE_LINKED to both migration v4 schema and create_schema(). Ghost Tool References In Speckit.md: Agent tool layers table referenced 2 non-existent tools (memory_drift_context, memory_drift_learn) and showed wrong 5-layer architecture. Rewritten to correct 7-layer, 22-tool architecture. Stale SKILL.md Cross-references: 5 references to renamed file corrected, plus stale section numbers fixed.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 2. SQLite Schema Unification
