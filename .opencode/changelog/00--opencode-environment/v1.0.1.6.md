## [**1.0.1.6**] - 2025-12-29

Spec Kit Memory System Upgrade - Major release with 7 new tools, checkpoint support, and 30+ metadata fields for advanced filtering. This release focuses on making the environment easier to use and easier to trust.

### 1. New
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: 7 New Tool Connection Layer Tools: memory_health, memory_delete, memory_update, memory_validate, memory_index_scan, checkpoint_create, checkpoint_list, checkpoint_delete. 30+ Metadata Fields: specFolder, level, status, tags[], priority, git_branch, author, related_specs[], dependencies[], blockers[]. Smart Ranking: Time-decay scoring with configurable weights.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 2. Changed
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Database schema updated with 30+ new columns. Memory index now includes heading-based anchors. Constitutional memories separated from specs.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3. Fixed
- Need: This part of the release fixes rough edges and makes the environment more dependable.
- What changed: Migration scripts for schema changes (v15 → v16). Backward compatibility for existing memories. Error handling for missing metadata fields.
- Why it matters: That matters because the environment is easier to maintain and less likely to break in everyday use.
