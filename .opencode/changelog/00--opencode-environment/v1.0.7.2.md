## [**1.0.7.2**] - 2026-01-23

This release updates the OpenCode environment with work on single consolidated prompt, user-facing model selection, dispatch mode descriptions. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### Single Consolidated Prompt
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: All 7 Commands Now Ask All Questions In ONE Prompt Instead Of Multiple Phases: research, plan, implement, complete, debug, resume, handover. Before: 3-5 separate interactions (spec folder → execution mode → dispatch mode → memory context). After: Single consolidated prompt with Q0-Q5, reply with "A, B, A" format.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### User-Facing Model Selection
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Q2 AI Model Options In Debug.md Simplified: B) Gemini - Google models (Pro/Ultra) → B) Gemini - Google, C) Codex - OpenAI models (GPT-4/o1) → C) Codex - OpenAI.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.

### Dispatch Mode Descriptions
- Need: This part of the release makes agent behavior more consistent across the supported runtimes.
- What changed: Mode Descriptions Made Model-agnostic Across All Structured Config Files Configs: 1 Opus orchestrator + 2 Sonnet parallel workers → 1 orchestrator + 2 parallel workers, 1 Opus orchestrator + 3 Sonnet parallel workers → 1 orchestrator + 3 parallel workers.
- Why it matters: That matters because the same instructions now behave more predictably in different runtimes.
