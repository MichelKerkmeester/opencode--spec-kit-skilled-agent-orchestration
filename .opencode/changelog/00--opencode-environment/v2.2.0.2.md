## [**2.2.0.2**] - 2026-02-20

tool connection layer memory_stats enhancement, SQLite write-ahead logging contention fix, and documentation refresh across 3 skills with 3 new API response fields, 1 concurrency fix, and 2 FAQ sections added. This release focuses on making the environment easier to use and easier to trust.

### Memory_stats() Enriched
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Memory_stats() Enriched: Returns tierBreakdown, databaseSizeBytes, and lastIndexedAt - eliminates raw sqlite3 command-line interface fallback for the /memory:manage stats dashboard.
- Why it matters: That matters because the environment can do more useful work without extra setup.

### SQLite SQLite Write-ahead Logging Contention Resolved
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: SQLite SQLite Write-ahead Logging Contention Resolved: Added busy_timeout = 10000 PRAGMA to database initialization, fixing intermittent "no such table: memory_index" errors under concurrent access.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### Package Version Corrected
- Need: This part of the release fixes rough edges and makes the environment more dependable.
- What changed: Package Version Corrected: @spec-kit/mcp-server version reverted from 1.7.3 to 1.7.2 to match actual release state.
- Why it matters: That matters because the environment is easier to maintain and less likely to break in everyday use.
