## [**1.3.0.0**] - 2026-02-10

This release updates the OpenCode environment with work on new features, changed, cleanup. The goal is to make the platform clearer, more reliable, and easier to use day to day.

### 1. New Features
- Need: This part of the release introduces the main capabilities added in this version.
- What changed: Context Retrieval Agent: Created.opencode/agent/context.md (716 lines). Specialized context scout with 3 thoroughness levels (quick/medium/thorough), Memory tool connection layer integration, and structured Context Package output. Two-Tier Dispatch Model - Phase 1: @context gathers context → Phase 2: orchestrator dispatches implementation agents with that Context Package. Eliminates redundant exploration across agents. @explore Prohibition (Rule 4): orchestrate.md §6 now prohibits direct @explore dispatch. Only @context can internally dispatch the built-in @explore subagent type.
- Why it matters: That matters because the environment can do more useful work without extra setup.

### 2. Changed
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Orchestrate.md: 39.7% reduction (1,316 → 793 lines). Aggressive prompt compression across 10 sections; all @explore dispatch references replaced with @context. Agent Selection Matrix and Routing Logic updated. Pre-Delegation Reasoning format now includes complexity line. Research.md: 42.4% reduction (793 → 457 lines). Aggressive compression with low-complexity bypass added. Write.md: 59.2% reduction (847 → 346 lines). Largest reduction; most aggressive trimming of all agents.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.

### 3. Cleanup
- Need: This part of the release improves how the environment saves, finds, and uses context.
- What changed: Dead Coordinator Mode - Removed §9-§10 From Research.md (coordinator/worker Dispatch Code). Agent Has Task: deny so this was unreachable dead code. Dead Worker Mode - Removed §4-§5 From Debug.md (coordinator/worker Dispatch Code). Same Reason: task: deny makes it unreachable.
- Why it matters: That matters because saved context is easier to find, easier to trust, and less likely to get in the way.
