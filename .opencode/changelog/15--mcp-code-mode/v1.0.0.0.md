## [**1.0.0.0**] - 2025-12-28

This first release introduces a shared tool runner written in TypeScript so many connected tools can be used through one workflow. It is designed to reduce repeated setup and make multi-tool work feel more coherent.

### 1. One bridge for many tools

- Need: People had to manage each connected tool separately, which made larger tasks awkward to run.
- Change: The release introduced one shared runner that can call many tools from the same flow.
- Why it matters: Complex tasks are easier to chain together and easier to explain to new users.

### 2. Less repeated context

- Need: Large tool lists made prompts heavy and follow-up steps harder to manage.
- Change: The new system loads tool details when needed and keeps state between steps.
- Why it matters: Everyday use feels lighter, and multi-step work is easier to continue.

### 3. Setup and reference material

- Need: Teams needed install help and a clear map of what the new runner could do.
- Change: The release shipped guides, templates, and reference documents for configuration and tool discovery.
- Why it matters: People can onboard faster and find the right capability without guesswork.
