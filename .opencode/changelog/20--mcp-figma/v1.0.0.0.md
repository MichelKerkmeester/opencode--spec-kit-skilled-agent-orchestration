## [**1.0.0.0**] - 2025-12-30

This first release adds a Figma skill for design-to-code work. It gives teams one place to find setup help, tool coverage, and guidance on when to use the faster or more capable path.

---

### Design access in one package

- Need: Teams needed a clearer way to bring design information into implementation workflows.
- Change: The skill bundled the main Figma capabilities, from file access to image export, comments, and style lookup.
- Why it matters: People can use design data in a more structured way instead of piecing together separate notes.

---

### Two setup paths

- Need: Different teams needed different installation options depending on their environment.
- Change: The release documented both the official hosted option and a local package-based option.
- Why it matters: More teams can adopt the skill without forcing one setup model.

---

### Clear support material

- Need: New users needed more than a short overview to use the skill confidently.
- Change: The release shipped a main guide, quick start, install help, tool reference, and category map.
- Why it matters: People can onboard faster and find the right help when questions come up.
