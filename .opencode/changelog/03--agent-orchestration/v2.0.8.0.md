## [**2.0.8.0**] - 2026-02-20

The orchestrator now has explicit self-protection rules, compaction recovery protocol, output discipline enforcement, and a structured context pressure response sequence to prevent irrecoverable context window failures. The main changes here are orchestrator self-protection rules added and compaction recovery protocol added. The goal is to make the workflow easier to follow and easier to trust.

---

### Orchestrator Self-protection Rules Added

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** Four explicit rules prevent the orchestrator from overloading its own context: targeted reads only (offset/limit on files >200 lines), no accumulation (delegate bulk reads to @context), write-don't-hold (scratch files for intermediate results), and batch tool calls. This makes the workflow easier to discover and use.

---

### Compaction Recovery Protocol Added

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** Explicit 5-step protocol when context compaction occurs: stop, re-read the Claude the related file, summarize state, wait for user confirmation. After 2+ compactions, recommends /clear with @handover continuation document. This makes the workflow easier to discover and use.

---

### Output Discipline Rules Added

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** Enforces concise responses: summarize tool results (never echo raw output), cap >50 line returns to 3-5 bullet points, produce one unified response for multi-agent synthesis. This makes the workflow easier to discover and use.

---

### Context Pressure Response Protocol Added

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** The existing context health monitoring table now has an explicit 4-step response sequence: pause dispatching, announce pressure to user with /compact recommendation, wait for confirmation, fallback to synthesize + suggest /spec_kit:handover. This makes the workflow easier to discover and use.

---

### Three Context-related Anti-patterns Added

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** Never read 3+ large files back-to-back in main context, never echo full tool output (>50 lines), never continue after compaction without user confirmation. This makes the workflow easier to discover and use.
