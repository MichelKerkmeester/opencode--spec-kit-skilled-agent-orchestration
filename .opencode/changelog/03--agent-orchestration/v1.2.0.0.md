## [**1.2.0.0**] - 2026-02-03

Updated handover, speckit, and write agents with new memory integration patterns, ANCHOR format validation, and the related file workflow. The main changes here are the related file and the related file. The goal is to make the workflow easier to follow and easier to trust.

---

### The Related File

**Problem:** The older guidance still pointed to something that was outdated or no longer used.

**Fix:** Simplified Task dispatch syntax. removed model-specific examples (model: "sonnet", model: "opus", model: "gemini"), replaced with generic description: and session_id: approach. Model selection now delegated to orchestration layer. This reduces confusion by keeping the notes focused on supported behavior.

---

### The Related File

**Problem:** People needed clearer support for this new part of the workflow.

**Fix:** Added tool connection Tool Layers table (L1-L5), memory folder updated to "5-state model, ANCHOR format". Added 5 /memory:* commands to commands table. New Section 11.5 "TEMPLATE PATTERNS (Spec 082)" with workstream prefixes [W:XXXX], block-task references [B:T###], evidence log format [E:filename]. Added pre-flight validation gates (4 gates) and ANCHOR format validation. Added response envelope structure template. This makes the workflow easier to discover and use.

---

### The Related File

**Problem:** This part of the documentation needed to match the related files more closely.

**Fix:** Step 3 (LOAD TEMPLATE) made hard gate with explicit HALT. Step 8 changed from "VALIDATE ALIGNMENT" to "VALIDATE FORMAT" using the related file (blocking, exit code based). Step 10 delivery now gated on validation pass AND DQI >= 75. Updated Mermaid flowchart with gate decision nodes. This helps keep different copies of the same guidance from drifting apart.
