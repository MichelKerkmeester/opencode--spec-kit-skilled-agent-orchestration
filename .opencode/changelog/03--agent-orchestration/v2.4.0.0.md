Added guard-safe prompt-quality routing for the four CLI orchestration skills, introduced the shared `@improve-prompt` runtime agent across active runtimes, and landed a drift-check script for main-versus-mirror prompt guidance. The main changes here are mirror-card fast path and structured escalation path. The goal is to make the workflow easier to follow and easier to trust.

---

### Mirror-card Fast Path

**Problem:** Routine CLI delegation prompts were still being built ad hoc from template text with no lightweight quality framework.

**Fix:** Added local `prompt_quality_card.md` mirror assets under the Claude Code, Codex, Copilot, and Gemini CLI skills, then wired each skill to load the mirror card in its `ALWAYS` resource path and enforce the CLEAR pre-dispatch check before prompt construction. This improves prompt quality for ordinary delegations without loading the full `sk-improve-prompt` skill body on every dispatch.

---

### Structured Escalation Path

**Problem:** High-complexity, compliance-sensitive, and multi-stakeholder prompt work needed better isolation than an inline skill load could provide.

**Fix:** Added the main `@improve-prompt` agent expected behavior and runtime mirrors under `.opencode/agent/`, `.claude/agents/`, `.codex/agents/`, and `.gemini/agents/` so escalated CLI work can request a structured `FRAMEWORK / CLEAR_SCORE / ENHANCED_PROMPT` response. This keeps the caller context cleaner while still giving complex prompts access to the deeper framework-selection and CLEAR-scoring workflow.

---

### Sk-improve-prompt Contract Sync

**Problem:** The prompt-engineering skill needed to expose a stable external-consumption surface for both the mirror-card path and the escalation agent.

**Fix:** Added the main `cli_prompt_quality_card.md`, documented the agent invocation expected behavior and fast-path asset in `sk-improve-prompt/SKILL.md`, and bumped the skill version to `1.3.0.0`. This makes the fast path and deep path explicit, traceable, and easier to maintain.

---

### Drift Check

**Problem:** Four short mirror cards create a real drift risk unless maintainers have a quick way to confirm sync with the main card.

**Fix:** Added `.opencode/skill/scripts/check-prompt-quality-card-sync.sh` to hash and compare the framework-selection table across the main card and all four mirrors. This gives maintainers a cheap, repo-local verification step before or after prompt-guidance edits.
