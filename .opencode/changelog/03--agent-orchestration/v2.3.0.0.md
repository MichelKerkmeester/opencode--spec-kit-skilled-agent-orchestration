## [**2.3.0.0**] - 2026-03-06

**Model configuration overhaul: dual-model strategy with GPT-5.4 for high-reasoning agents and Codex 5.3 for all others.** All agent definitions across `.codex/agents/` (9 TOML files), `.codex/config.toml`, and `.opencode/agent/chatgpt/` (7 markdown files) updated to a split model configuration. Orchestrate, ultra-think, and research retain `gpt-5.4` for maximum reasoning quality; all other agents switch to `gpt-5.3-codex` with `xhigh` reasoning effort uniformly applied.

---

## Highlights

### Category: Dual-Model Strategy

- **Three agents retained on GPT-5.4** — orchestrate, ultra-think, and research stay on `gpt-5.4` with `xhigh` reasoning effort for tasks requiring maximum reasoning depth (task decomposition, multi-strategy planning, technical investigation).
- **Six agents migrated to Codex 5.3** — context, debug, review, handover, speckit, and write switched to `gpt-5.3-codex` for efficient execution of structured workflows.

### Category: Reasoning Effort Normalization

- **All agents now use `xhigh` reasoning effort** — previously tiered across `medium`, `high`, and `xhigh` per role. Unified to `xhigh` across all agents in both `.codex/` and `.opencode/agent/chatgpt/` directories.
- **All `.codex/config.toml` profiles updated** — context (was `high`), handover (was `medium`), speckit (was `medium`), and write (was `medium`) profiles all raised to `xhigh`.

### Category: Cross-Runtime Consistency

- **Codex CLI agents (`.codex/agents/*.toml`)** — model and reasoning effort updated in all 9 agent files plus `config.toml` global model.
- **ChatGPT agents (`.opencode/agent/chatgpt/*.md`)** — model and reasoning effort updated in 7 agent frontmatter blocks. Orchestrate and ultra-think have no `model:` field (inherit global `gpt-5.4`).
