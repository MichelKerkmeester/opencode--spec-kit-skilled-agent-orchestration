## [**2.3.0.0**] - 2026-03-06

dual-model strategy with GPT-5.4 for high-reasoning agents and Codex 5.3 for all others.** All agent definitions across.codex/agents/ (9 runtime config files), the related file, and.opencode/agent/chatgpt/ (7 markdown files) updated to a split model configuration. Orchestrate, ultra-think, and research retain gpt-5.4 for maximum reasoning quality; all other agents switch to gpt-5.3-codex with xhigh reasoning effort uniformly applied. The main changes here are three agents retained on gpt-5.4 and six agents migrated to codex 5.3.

### 1. Three Agents Retained on Gpt-5.4

- Need: This part of the agent coordination guidance needed clearer guidance and less room for confusion.
- What changed: orchestrate, ultra-think, and research stay on gpt-5.4 with xhigh reasoning effort for tasks requiring maximum reasoning depth (task decomposition, multi-strategy planning, technical investigation).
- Why it matters: This makes the agent coordination guidance easier to follow.

### 2. Six Agents Migrated to Codex 5.3

- Need: This area needed a clearer and safer way to work.
- What changed: context, debug, review, handover, speckit, and write switched to gpt-5.3-codex for efficient execution of structured workflows.
- Why it matters: This makes the behavior more predictable and lowers the chance of mistakes.

### 3. All Agents Now Use Xhigh Reasoning Effort

- Need: This part of the agent coordination guidance needed clearer guidance and less room for confusion.
- What changed: previously tiered across medium, high, and xhigh per role. Unified to xhigh across all agents in both.codex/ and.opencode/agent/chatgpt/ directories.
- Why it matters: This makes the agent coordination guidance easier to follow.

### 4. All the Related File Profiles Updated

- What changed: context (was high), handover (was medium), speckit (was medium), and write (was medium) profiles all raised to xhigh.
