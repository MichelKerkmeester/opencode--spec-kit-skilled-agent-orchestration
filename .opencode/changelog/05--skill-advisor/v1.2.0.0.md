## [**1.2.0.0**] - 2026-03-27

**Cross-repo routing parity for CocoIndex discovery and autonomous review workflows.** -- Aligned Public and Barter skill advisor behavior for semantic code search and deep research/review prompts, added dual CocoIndex CLI aliases, expanded regression coverage, refreshed script docs, and stabilized benchmark measurements for the larger routing dataset.

---

## Highlights

### New

- **Dual CocoIndex CLI aliases** -- Added equivalent support for `--semantic` / `--cocoindex` and `--semantic-hits` / `--cocoindex-hits` in `skill_advisor.py`.
- **Expanded regression dataset** -- Grew `fixtures/skill_advisor_regression_cases.jsonl` from 34 to 41 cases, covering CocoIndex discovery prompts, `autoresearch`, slash-based autonomous review, plain-language auto-review boundaries, and the responsive CSS/layout routing fix.

### Changed

- **CocoIndex-first discovery routing** -- Boosted high-signal discovery phrases such as `find code that`, `semantic code search`, `code search`, `where is the logic`, and direct `mcp-coco-index` path references so code-discovery intents consistently favor `mcp-coco-index`.
- **Autonomous review boundary** -- Standardized routing so explicit autonomous workflows such as `autoresearch`, `/autoresearch`, `deep review`, `review loop`, `:review:auto`, and `auto review release readiness` go to `sk-deep-research`, while ordinary review phrasing such as `code review`, `review this PR`, and `auto review this PR` stays on `sk-code--review`.
- **Responsive CSS/layout recovery** -- Removed the ambiguous `fix` multi-skill penalty that was causing `implement responsive css layout fix` to abstain in Public; the prompt now routes to the repo’s implementation skill family as intended.
- **Cross-repo parity preserved** -- Applied the same CocoIndex and autonomous-review intent vocabulary to Barter while preserving Barter’s `sk-code` consolidation and removing its unsupported `sk-doc-visual` routing references.
- **Benchmark harness isolation** -- Updated `skill_advisor_bench.py` to disable external built-in semantic lookup during benchmark runs via `SKILL_ADVISOR_DISABLE_BUILTIN_SEMANTIC`, keeping latency measurements focused on advisor overhead instead of `ccc` subprocess time.
- **Cold-start gate tuning** -- Raised the benchmark harness default cold p95 gate from `55.0 ms` to `60.0 ms` so the larger 41-prompt dataset is evaluated against a realistic subprocess envelope.

### Validation Snapshot

- **Regression suite** -- `41 / 41` cases passed, including `12 / 12` P0 cases, `100%` top-1 accuracy on evaluated cases, and `0` command-bridge false positives.
- **Performance snapshot** -- Public benchmark refreshed at `0.6196 ms` warm p95, `48.7295 ms` subprocess one-shot p95, and `33.1276x` batch throughput multiplier.
