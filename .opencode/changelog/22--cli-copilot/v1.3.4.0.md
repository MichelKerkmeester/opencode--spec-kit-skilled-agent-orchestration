## [**1.3.4.0**] - 2026-04-12

This patch release adds a concurrency limit to prevent system memory exhaustion during parallel copilot dispatch. The goal is to make the workflow easier to follow and easier to trust.

---

## New Features

These additions expand what the workflow can do without changing the release facts. Each item explains the user-facing gain before the technical detail.

### Concurrency limit: max 5 concurrent copilot processes

**Problem:** Dispatching 12+ parallel `copilot -p` calls caused system memory exhaustion (36 processes, each loading GPT-5.4 context).

**Fix:** Added `CONCURRENCY LIMIT` section to SKILL.md. Maximum 5 concurrent copilot processes at any time. Callers must check `pgrep -f "copilot" | wc -l` before launching new calls. For large campaigns, sequential dispatch (1 at a time) or small batches (2-3) are preferred.

**NEVER rule #5 updated:** "NEVER run more than 5 concurrent copilot processes - system memory exhaustion risk."
