## [**1.2.0.0**] - 2026-03-21

**Cross-CLI adoption boosting** -- Added direct skill_advisor.py routing for CocoIndex, inserted a Code Search Protocol section into all instruction files (AGENTS.md, GEMINI.md, AGENTS_example_fs_enterprises.md, Barter AGENTS.md), integrated CocoIndex as a first-class tool in all 5 @context agent definitions, replaced generic provider notes with per-CLI activation recipes, and expanded SKILL.md activation triggers.

> Spec folder: `.opencode/specs/03--commands-and-skills/022-mcp-coco-integration/` (Level 2, Phase 7)

---

## Highlights

### New: Skill Advisor Direct Routing

- **8 INTENT_BOOSTERS** added for `mcp-coco-index`: `cocoindex` (2.5), `ccc` (2.0), `semantic` (1.5), `vector search` (2.0), `similar code` (1.8), `concept search` (2.0), `discover` (0.6), `implementation` (0.5).
- **10 PHRASE_INTENT_BOOSTERS** added: `find code that`, `similar code`, `where is the logic`, `search codebase`, `code that handles`, `find implementations`, `find similar`, `semantic code search`, `how is.*implemented`, `how does.*work`.
- **1 MULTI_SKILL_BOOSTER** added: `discover` → mcp-coco-index (0.5).
- **Auto-semantic-search boost** increased from 1.2 to 1.8 for stronger automatic routing.
- Applied to both Public and Barter `skill_advisor.py` (Barter received the delta on top of its existing CocoIndex entries).

### New: Code Search Protocol in Instruction Files

- **AGENTS.md** and **GEMINI.md**: Added a Code Search Protocol section with an ASCII decision tree, activation triggers list, MCP/CLI quick reference table, and `refresh_index=false` guidance.
- **AGENTS_example_fs_enterprises.md**: Same Code Search Protocol added for the enterprise template.
- **Barter AGENTS.md**: Same Code Search Protocol added for the Barter codebase.
- All 4 files now instruct every AI to try CocoIndex BEFORE Grep/Glob for concept-based queries.

### New: @context Agent CocoIndex Integration (All 5 Runtimes)

- **`.opencode/agent/context.md`** (OpenCode/Copilot): Added `cocoindex_code` to mcpServers, CocoIndex row in tools table, CONCEPT/INTENT branch in Tool Selection Guide, CocoIndex as first tool in Layer 2 strategy, updated Tool Sequence, added `mcp-coco-index` to Skills table.
- **`.opencode/agent/chatgpt/context.md`** (ChatGPT): Same 6 changes.
- **`.agents/agents/context.md`** (Copilot agents): Same changes (no mcpServers field in this format).
- **`.claude/agents/context.md`** (Claude Code): Same 6 changes including `cocoindex_code` in mcpServers.
- **`.codex/agents/context.toml`** (Codex CLI): Same changes adapted to TOML format.
- **Key insight**: The @context agent -- the exclusive entry point for ALL codebase exploration -- previously had zero CocoIndex references. This was the single biggest gap in cross-CLI adoption.

### Updated: Cross-CLI Playbook Per-Provider Recipes

- Replaced the generic 3-line Provider Notes with 5 detailed per-CLI activation recipes covering Claude Code, Codex CLI, Gemini CLI, GitHub Copilot, and OpenCode.
- Each recipe documents: MCP tool name/config path, auto-activation mechanism, best practices, and known issues.

### Updated: SKILL.md Activation Triggers

- Added 5 new "Use when" triggers: "how is X implemented", architecture/module relationships, onboarding queries, @context agent exploration, unknown function/class names.
- Added 3 new "Automatic Triggers" groups: intent phrases (`how is`, `what handles`, `where does`), exploration phrases (`explore`, `architecture`), onboarding phrases (`unfamiliar code`, `new to this`).
