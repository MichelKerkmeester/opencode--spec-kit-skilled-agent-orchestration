## [**1.1.0.0**] - 2026-03-18

**Strict readiness and downstream adoption hardening** -- Added a shared readiness contract for CocoIndex helper scripts, introduced strict machine-readable failure modes for automation and CI, documented sibling-repo adoption requirements, and aligned the Phase 3 docs with both the current CocoIndex CLI contract and repo documentation standards.

## Highlights

### New: Strict Readiness Contract

- **`common.sh` shared validation engine**: Centralized readiness status computation, config detection, recommended next-step generation, and strict failure codes `20-25` for binary, payload, index, config, and daemon readiness.
- **`doctor.sh` strict inspection**: Added `--strict`, `--require-config`, `--require-daemon`, and repeatable `--expect-config`, plus stable JSON fields for `status`, `blockingIssues`, `warnings`, `detectedConfigs`, and `expectedConfigs`.
- **`ensure_ready.sh` strict post-bootstrap validation**: Preserved install/init/index ownership while adding optional strict validation and machine-readable output after bootstrap completes.

### New: Downstream Adoption Packaging

- **`downstream_adoption_checklist.md`**: Added a sibling-repo rollout checklist covering the minimum adoption bundle: local skill payload, `cocoindex_code` config wiring, and `.cocoindex_code/` gitignore hygiene.
- **`cross_cli_playbook.md`**: Added stricter readiness usage guidance, clarified `refresh_index=false` follow-up-query defaults, and documented that downstream repos do not inherit CocoIndex from advisor heuristics alone.
- **Runtime-contract drift removed**: Cleaned stale guidance that still implied `ccc daemon start` or `ccc index --refresh` should be used in the current installed CLI flow.

### Standards Alignment

- **Reference docs fixed to pass `sk-doc`**: Added required `OVERVIEW` sections to the new reference docs so they pass reference-mode validation.
- **Script headers now self-document strict exits**: Added explicit exit-code blocks to `doctor.sh` and `ensure_ready.sh` so the strict readiness contract is visible to operators and agents reading the files directly.
- **Manual-testing snippets updated**: Replaced stale remediation text in the incremental-index and daemon-status scenario files to match the current CLI behavior and troubleshooting path.
