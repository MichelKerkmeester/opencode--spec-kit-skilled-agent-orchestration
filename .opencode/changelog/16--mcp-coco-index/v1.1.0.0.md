## [**1.1.0.0**] - 2026-03-18

This release makes the CocoIndex setup easier to check before you trust it in automation. It adds stricter readiness checks, clearer failure messages, and rollout guidance for sibling repos.

### 1. Stricter readiness checks

- Need: The health-check scripts were not strict enough for reliable automation and continuous integration.
- Change: A shared validation engine and stronger readiness commands were added so failure states are clearer and more machine-friendly.
- Why it matters: Automated workflows can fail fast with a concrete reason instead of vague setup noise.

### 2. Better rollout guidance

- Need: Teams copying this skill into another repo could miss required files or configuration.
- Change: The release added an adoption checklist and tightened the cross-repo playbook.
- Why it matters: Rollouts into sibling repos are easier to complete correctly the first time.

### 3. Docs aligned to current behavior

- Need: Some reference notes still hinted at older command flows and stale troubleshooting advice.
- Change: The docs and manual-test scenarios were updated to match the current tool behavior.
- Why it matters: People now see the steps that actually work today.
