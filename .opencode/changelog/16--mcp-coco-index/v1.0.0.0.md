## [**1.0.0.0**] - 2026-03-18

**Initial release providing semantic code search via vector embeddings across 105K+ chunks and 28+ languages, with MCP integration for all 6 CLI configs and cross-CLI auto-usage validation.**

> Spec folder: `02--system-spec-kit/023-coco-index-integration` (Level 2)

---

## Highlights

### New

- **CocoIndex Code installation** -- `cocoindex-code` v0.2.3 installed via pipx with Python 3.11, providing the `ccc` CLI binary and MCP server (`ccc mcp` via stdio)
- **Code index** -- 6,792 files indexed into 105,965 chunks across 14 languages using all-MiniLM-L6-v2 local embedding model (384 dimensions, no API key required)
- **6-CLI MCP registration** -- `cocoindex_code` server entry added to `.mcp.json`, `opencode.json`, `.agents/settings.json`, `.gemini/settings.json`, `.claude/mcp.json`, and `.codex/config.toml` with consistent snake_case naming
- **SKILL.md** -- Complete 8-section skill document with activation triggers, smart routing, CLI/MCP usage, rules, references, success criteria, and integration points

### Category: Cross-CLI Validation

- **Auto-discovery confirmed** -- Claude Code, Gemini, and Copilot all spontaneously used CocoIndex from the MCP tool description alone, without agent routing changes
- **Phase 2 agent routing deprioritized** -- Cross-CLI test proved auto-discovery works; @context agent routing adds maintenance without benefit
- **Token efficiency** -- CocoIndex saves 10-50x tokens vs Grep-Read-Filter cascades (Gemini P2: 75K tokens vs Copilot P1 fallback: 3.6M tokens)

### Category: Daemon Bug Discovery

- **Root cause identified** -- Copilot MCP failures caused by upstream CocoIndex daemon concurrency bug, not a Copilot issue
- **Reproduction** -- 5 concurrent `refresh_index=true` queries generate +165 daemon errors (`ComponentContext` crash in `cocoindex_core::engine::component`); `refresh_index=false` generates 0 errors
- **Query optimization guidance** -- Added to SKILL.md: short natural-language queries (2-5 words) outperform keyword-stuffed queries due to embedding dilution
- **Concurrent session guidance** -- Added to SKILL.md: use `refresh_index=false` after first query in multi-query sessions
