This update requires the calling AI to pass along the active spec folder when it delegates work to Copilot. It avoids a common stop point where the delegated agent would otherwise pause for missing project context. The goal is to make the workflow easier to follow and easier to trust.

---

#### Pass project context forward

Delegated Copilot runs could stop early because they did not know which spec folder had already been approved. The delegation rules now require the calling AI to include that spec-folder context in the handoff when it exists. Non-interactive delegation is more likely to keep moving instead of pausing for avoidable questions.
