## [**1.0.0.0**] - 2026-03-02

This first release lets other AI assistants delegate focused work to OpenAI's Codex command-line tool. It packages safe execution rules, delegation patterns, and reference docs so teams can use Codex more deliberately.

### 1. Structured delegation into Codex

- Need: Other assistants needed a clear way to decide when Codex was the right helper for a task.
- Change: The skill introduced routing patterns for generation, review, research, architecture work, and agent-based delegation.
- Why it matters: Requests can be handed off more intentionally instead of treating Codex as a generic fallback.

### 2. Safety and specialist support

- Need: Delegated work needed guardrails around file access and a clearer map of specialist roles.
- Change: The package documents sandbox levels and the specialist agent set that Codex can use.
- Why it matters: Teams can choose the right balance of safety and capability for each job.

### 3. Reusable support material

- Need: People needed guides and templates they could reuse instead of rebuilding delegation prompts from scratch.
- Change: The release shipped a main guide, references, comparisons, and prompt templates.
- Why it matters: Codex becomes easier to adopt and easier to use consistently.
