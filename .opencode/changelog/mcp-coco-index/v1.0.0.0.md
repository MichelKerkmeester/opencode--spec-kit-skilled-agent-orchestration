## [**1.0.0.0**] - 2026-03-18

This first release adds semantic code search so people can find code by meaning, not only by exact text. It also wires the tool into the main AI runtimes so the same search experience works across them.

#### Search by intent

- Need: Exact-text search often fails when people do not know the right file names or code terms.
- Change: The repo now includes a meaning-based code search tool and a project index that can answer broader questions.
- Why it matters: People can find relevant code faster, even when their starting question is vague.

&nbsp;

#### Shared setup across AI runtimes

- Need: Teams wanted the same search tool to be available from different assistants.
- Change: The tool was registered across the main runtime configuration files and paired with a dedicated skill guide.
- Why it matters: The search workflow is more consistent no matter which assistant starts the task.

&nbsp;

#### Early stability guidance

- Need: Concurrent refresh requests exposed an upstream failure pattern that users could easily hit.
- Change: The release documented the issue and added safer follow-up-query guidance.
- Why it matters: Teams can avoid a known failure pattern while still getting the benefits of the new search tool.
