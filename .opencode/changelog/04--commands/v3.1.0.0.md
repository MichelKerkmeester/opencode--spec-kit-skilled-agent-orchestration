The `/improve:prompt` command can now route through a fresh-context `@improve-prompt` agent instead of always relying on inline skill loading. Previously, prompt improvement stayed entirely in the caller context even for crowded, complex, or compliance-heavy cases. Now the command supports both paths and documents the same escalation surface used by the CLI prompt-quality mirror-card pipeline.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/043-cli-skill-improved-prompting/` (Level 3)

---

## What Changed

---

### Dispatch Modes

- **Inline and agent modes now coexist in one command** -- `/improve:prompt` now supports `dispatch_mode = INLINE` for ordinary interactive work and `dispatch_mode = AGENT` when the user wants fresh-context isolation or a structured escalation result.

- **The setup flow can auto-select agent mode** -- When `complexity_hint >= 7` is already present, or when the caller explicitly asks for isolation, fresh context, or parallelism, the command skips the manual dispatch-mode question and routes directly to the agent branch.

---

### Shared Escalation Surface

- **The command now shares one escalation surface with the CLI mirror-card flow** -- In agent mode, `/improve:prompt` dispatches `@improve-prompt` and expects the same structured return shape used by the CLI orchestrator skills: framework selection, CLEAR score, rationale, enhanced prompt, and escalation notes.

- **Phase guidance now explains the branch clearly** -- The command docs spell out when `@general` stays inline and when it hands off to `@improve-prompt`, so the command no longer reads like an inline-only wrapper around `sk-improve-prompt`.

---

### Documentation Parity

- **The improve-command README now matches reality** -- `README.txt` no longer describes prompt improvement as inline-only. It now reflects the optional fresh-context agent branch and the shared escalation model with the CLI prompt-quality pipeline.
