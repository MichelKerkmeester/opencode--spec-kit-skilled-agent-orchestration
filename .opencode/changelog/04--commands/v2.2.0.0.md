## Highlights

### Category: New Command

- **`/create:prompt` command** — New 530-line command at `.opencode/command/create/prompt.md` that wraps the sk-prompt-improver skill for prompt creation and improvement
- **Phase 0 + Unified Setup Phase** — Full create namespace gate pattern: @general agent verification, consolidated Q0-Q3 prompt (prompt input, enhancement mode, save location, execution mode) in a single user interaction
- **7 mode prefixes** — `$text`, `$improve`, `$refine`, `$short`, `$json`, `$yaml`, `$raw` with aliases, mapping to sk-prompt-improver operating modes and energy levels (Raw/Quick/Standard/Deep)
- **DEPTH processing pipeline** — 5-phase enhancement (Discover-Engineer-Prototype-Test-Harmonize) with configurable round counts per mode
- **CLEAR scoring** — 50-point quality scoring (Correctness, Logic, Expression, Arrangement, Reusability) with dimension floors and 40+/50 threshold enforcement
- **Prompt saving to spec folders** — Optional save to `specs/[folder]/prompts/[descriptive-name].md` with frontmatter metadata (title, framework, mode, CLEAR score, created date)
- **Dual execution modes** — `:auto` (autonomous, default) and `:confirm` (interactive with approval gates at each DEPTH phase)

## Design Decisions

- **Self-contained command** (no YAML dispatch files): Unlike other create commands that produce file outputs via YAML workflows, this command invokes a skill for conversation-based enhancement. YAML dispatch would add complexity without value.
- **Mode-Based + Argument Dispatch hybrid**: Combines command_template.md §11 and §12 for dual parsing of mode prefixes and execution modes.
- **Default autonomous execution**: Prompt enhancement is inherently low-risk (conversation output), so `:auto` is the default.
- **Save gate in Setup Phase**: Save location (Q2) is collected upfront in the consolidated prompt rather than after delivery, following the single-interaction pattern.
