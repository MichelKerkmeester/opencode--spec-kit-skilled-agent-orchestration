## [**2.6.0.0**] - 2026-03-25

Added mcp__cocoindex_code__search to allowed-tools in all spec_kit and create command files, enhanced parallel explorer agent prompts with CocoIndex-first guidance, added Code Search Routing protocol blocks, and updated all.agents/commands/ matching runtime config files. Agents now prefer search by meaning instead of exact words for concept-based discovery over exact text search pattern guessing. The main changes here are explorer agent prompts enhanced and code search routing block.

### 1. Explorer Agent Prompts Enhanced

- Need: People needed a better way to find the right information without guessing exact words.
- What changed: All 8 parallel explorer agents (4 in plan, 4 in complete) now explicitly instruct: "Use CocoIndex first for concept-based discovery, exact text search only for exact known tokens".
- Why it matters: This helps people find the right place faster, even when they do not know the exact term to search for.

### 2. Code Search Routing Block

- Need: People needed clearer support for this new part of the workflow.
- What changed: Reusable routing table added to the related file and the related file showing when to use CocoIndex vs exact text search vs file-pattern search vs memory_search.
- Why it matters: This makes the workflow easier to discover and use.

### 3. Deep Research Agents Upgraded

- Need: This part of the command documentation needed clearer guidance and less room for confusion.
- What changed: tools: and mcp_servers: arrays updated in all 4 deep-research workflow config variants to include CocoIndex.
- Why it matters: This makes the command documentation easier to follow.

### 4. Debug Agent Prompt Enhanced

- Need: This area needed a clearer and safer way to work.
- What changed: Debug sub-agent dispatch template now includes CocoIndex guidance for finding similar working patterns.
- Why it matters: This makes the behavior more predictable and lowers the chance of mistakes.

### 5. Allowed-tools Frontmatter

- Need: People needed clearer support for this new part of the workflow.
- What changed: Added mcp__cocoindex_code__search to 7 spec_kit.md files (plan, complete, implement, deep-research, debug, phase, resume).
- Why it matters: This makes the workflow easier to discover and use.

### 6. Allowed-tools Frontmatter

- Need: People needed clearer support for this new part of the workflow.
- What changed: Added mcp__cocoindex_code__search to all 7 create.md files (sk-skill, feature-catalog, testing-playbook, agent, changelog, folder_readme, prompt).
- Why it matters: This makes the workflow easier to discover and use.

### 7. YAML Code_search_note

- Need: People needed clearer support for this new part of the workflow.
- What changed: Added CocoIndex discovery guidance to request_analysis_framework in all 12 create workflow config files (6 auto + 6 confirm).
- Why it matters: This makes the workflow easier to discover and use.

### 8. .agents/commands/ Tomls Synced

- Need: This part of the documentation needed to match the related files more closely.
- What changed: Updated allowed-tools in all 14 applicable runtime config command mirrors (6 spec_kit + 7 create + 1 deep-research) to include mcp__cocoindex_code__search.
- Why it matters: This helps keep different copies of the same guidance from drifting apart.
