## [**2.4.0.0**] - 2026-03-16

**Merge `/memory:context` into `/memory:analyze` -- unified 7-to-6 command suite** -- Absorbed the retrieval functionality of `/memory:context` (L1/L2: `memory_context`, `memory_search`, `memory_match_triggers`) into `/memory:analyze`, creating a single unified command that handles both intent-aware retrieval (default mode) and analysis subcommands (preflight, postflight, causal, ablation, dashboard, etc.). Command count reduced from 7 to 6.

---

## Highlights

### Category: Command Consolidation

- **`context.md` merged into `analyze.md`** -- The standalone `/memory:context` command (438 lines, 3 L1-L2 tools) was merged into `/memory:analyze` (now 926 lines, 12 tools across L1/L2/L6/L7). Retrieval is the default mode; analysis subcommands route via keyword detection.
- **Routing model** -- Flat subcommand routing: if the first argument matches a known analysis keyword (`preflight`, `postflight`, `causal`, `link`, `unlink`, `causal-stats`, `ablation`, `dashboard`, `history`), route to analysis mode. Otherwise, treat the entire input as a retrieval query with optional `--intent:` flag.
- **Tool ownership consolidated** -- `/memory:analyze` now owns 12 MCP tools (was 9 for analyze + 3 for context). Coverage matrix updated across all documentation.
- **Command count: 7 to 6** -- `save`, `continue`, `analyze`, `learn`, `manage`, `shared`.
- **Cross-runtime alignment** -- Updated `.opencode/command/`, `.opencode/agent/`, `.claude/agents/`, `.agents/agents/`, `.codex/agents/`, `.agents/commands/`, `.gemini/commands/` (TOML command files regenerated, `context.toml` deleted).
