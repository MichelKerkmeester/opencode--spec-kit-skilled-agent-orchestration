## 1. OVERVIEW

This changelog captures one connected commands pass: two new create commands were shipped for feature catalogs and testing playbooks, then the surrounding create-command YAML suite was refined so the workflows present one more consistent contract. It also records the follow-up mirror audit that confirmed the `.gemini` create TOMLs remain aligned with the canonical `.agents` mirrors.

---

## 2. HIGHLIGHTS

### Category: New Create Commands

- **`/create:feature-catalog` added**: Scaffolds the shipped `feature_catalog/` contract using the `sk-doc` feature-catalog creation reference plus both root and snippet templates.
- **`/create:testing-playbook` added**: Scaffolds the shipped integrated `manual_testing_playbook/` contract with root-owned review/orchestration guidance, root-level category folders, and no legacy sidecar files or `snippets/` subtree.
- **Auto/confirm workflow pairs shipped**: Both commands include dedicated `:auto` and `:confirm` YAML assets under `.opencode/command/create/assets/`.

### Category: Runtime Alignment

- **`.agents` mirrors added**: New runtime mirrors were added at `.agents/commands/create/feature-catalog.toml` and `.agents/commands/create/testing-playbook.toml`.
- **Discovery surfaces synchronized**: Create-command inventories and write-agent docs were updated across `.opencode`, `.agents`, and `.codex`.
- **Gemini mirrors verified aligned**: Follow-up exact-match checks confirmed the `.gemini/commands/create/` TOMLs match the canonical `.agents` mirrors for the create-command family.

### Category: YAML Workflow Refinement

- **Feature/testing YAMLs strengthened**: The new feature-catalog and testing-playbook YAML pairs were rewritten to keep richer request analysis, gate logic, workflow overview, recovery, termination, and rules sections.
- **Broader create suite normalized**: `create_agent_*`, `create_changelog_*`, and `create_folder_readme_*` were aligned to the same shared top-level contract so the suite reads as one coherent family.
