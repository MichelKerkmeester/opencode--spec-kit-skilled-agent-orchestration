### Category: Command Consolidation

- **Canonical command entrypoint** -- Standardized all skill command flows on `/create:sk-skill` with explicit operation routing for `full-create`, `full-update`, `reference-only`, and `asset-only`.
- **Unified mode workflows** -- Kept one workflow per mode with `.opencode/command/create/assets/create_sk_skill_auto.yaml` (`:auto`) and `.opencode/command/create/assets/create_sk_skill_confirm.yaml` (`:confirm`) while preserving shared operation contracts.
- **Legacy surface removed** -- Deleted deprecated command files (`skill.md`, `skill_reference.md`, `skill_asset.md`) and deprecated workflow YAML variants (`create_skill*.yaml`) to prevent routing drift.
- **Prompt routing updated** -- Updated `.opencode/command/create/prompt.md` references to canonical `/create:sk-skill` variants.
- **Cross-runtime sync completed** -- Aligned active write-agent/runtime docs across `.opencode`, `.agents`, `.codex`, `.claude` references and install/readme surfaces to canonical command naming.
- **Spec + memory completion** -- Completed Level 2 spec artifacts and verified memory save/indexing for `016-create-skill-merger`.
