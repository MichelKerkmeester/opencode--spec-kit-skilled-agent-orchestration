## [**2.7.0.0**] - 2026-03-29

Merged the standalone `/spec_kit:phase` command into `/spec_kit:plan` and `/spec_kit:complete` as an optional `:with-phases` flag. The dedicated phase command, its two YAML assets, and its TOML variant were removed. All 23 files referencing the old command were updated across agents, skills, templates, and install guides.

> Spec folder: `specs/03--commands-and-skills/037-cmd-merge-spec-kit-phase`

### 1. Phase Decomposition as `:with-phases` Flag

- Need: Phase decomposition required a separate three-command chain which added friction.
- What changed: Added `:with-phases` optional workflow flag to both plan and complete commands. When present, a 4-step phase decomposition pre-workflow runs before the normal steps: analyze scope, define decomposition, create folders, and populate templates.
- Why it matters: Users can now create phased spec structures and begin planning in one command instead of three.

### 2. Unified Setup Phase with Phase Question

- Need: Users needed a way to opt into phase decomposition during setup, not just via flags.
- What changed: Added Q6 (Phase Decomposition), Q7 (Phase Count), and Q8 (Phase Names) to the consolidated setup prompt. The complete command also received a full Unified Setup Phase section.
- Why it matters: Phase decomposition is now discoverable during interactive setup.

### 3. Phase YAML Workflow Configuration

- Need: The YAML workflow assets needed phase decomposition config to match the command changes.
- What changed: Added `phase_decomposition` block to `optional_workflows` in all 4 YAML assets (plan auto/confirm, complete auto/confirm) with trigger, insert point, 4 pre-workflow steps, templates, checkpoint, and defaults.
- Why it matters: The YAML layer handles phase creation identically across all 4 workflow variants.

### 4. Standalone Phase Command Removed

- Need: The dedicated phase command was redundant after integration.
- What changed: Deleted `phase.md`, `spec_kit_phase_auto.yaml`, `spec_kit_phase_confirm.yaml`, and `phase.toml`. Updated 23 files across READMEs, agents (5 runtimes), SKILL.md, templates, nodes, feature catalog, testing playbook, AGENTS.md, and install guides.
- Why it matters: Cleaner command surface with no orphaned references.
