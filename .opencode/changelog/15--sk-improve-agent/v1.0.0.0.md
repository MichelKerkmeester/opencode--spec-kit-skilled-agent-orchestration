Initial release of `sk-improve-agent` -- an evaluator-first skill for bounded agent improvement with 5-dimension integration-aware scoring, dynamic profiling, deterministic benchmarks, and guarded promotion. The skill evaluates any agent across its full system integration surface rather than just checking keyword presence in the markdown file. Developed across 8 phases under spec folder `041-sk-recursive-agent-loop`, originally as `sk-recursive-agent` and renamed in Phase 008. The goal is to make the workflow easier to follow and easier to trust.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/041-sk-recursive-agent-loop` (Level 2, 8 phases)

---

## Evaluator-First Architecture (Phase 001-002)

---

### Proposal-Only Improvement Loop

The skill writes candidates to packet-local runtime areas instead of editing main agent files directly. Each candidate is scored independently by a deterministic evaluator, benchmarked against target-specific fixture sets, and tracked in an append-only JSONL ledger. Promotion to the main target requires passing all evidence gates plus explicit operator approval.

---

### Target Profiles and Benchmarks

Two target agents are supported with static profiles: `handover` (promotion-eligible) and `context-prime` (candidate-only). Each profile defines its own scoring rubric, fixture directory, aggregate score threshold, and repeatability tolerance. The benchmark runner (`run-benchmark.cjs`) scores fixture outputs against required headings, required patterns, and forbidden patterns.

---

### Guarded Promotion and Rollback

Canonical mutation requires a `candidate-better` score, passing benchmark report, repeatability evidence, manifest boundary compliance, and explicit approval. Rollback restores the archived pre-promotion backup and records a post-rollback comparison for audit continuity.

---

## 5-Dimension Holistic Evaluation (Phase 008)

---

### Integration Scanner

The `scan-integration.cjs` script discovers every surface an agent touches across the repo: main definition, 3 runtime mirrors (Claude, Codex, .agents), command dispatch references, YAML workflow references, skill mentions, global doc references, and skill advisor routing entries. Mirror sync is detected by signal-string matching against the main body.

---

### Dynamic Profile Generator

The `generate-profile.cjs` script derives a scoring profile from any agent's own structure -- parsing YAML frontmatter, ALWAYS/NEVER/ESCALATE IF rules, output verification checklist, capability scan, anti-patterns, and related resources. This eliminates the need for manually created profiles and makes any agent in `.opencode/agent/` a valid evaluation target.

---

### 5-Dimension Scoring Framework

The refactored `score-candidate.cjs` scores candidates across 5 weighted dimensions when invoked with `--dynamic`:

| Dimension | Weight | What It Measures |
| --- | --- | --- |
| Structural Integrity | 0.20 | Agent template compliance (required sections) |
| Rule Coherence | 0.25 | ALWAYS/NEVER rules align with workflow steps |
| Integration Consistency | 0.25 | Mirror sync, command coverage, skill references |
| Output Quality | 0.15 | Output verification items present, no placeholders |
| System Fitness | 0.15 | Permission-capability alignment, resource references valid |

Legacy `--profile` mode with fixed keyword checks remains available and backward compatible.

---

### Dimensional Progress Tracking

The refactored `reduce-state.cjs` tracks per-dimension scores across iterations, renders a Dimensional Progress table in the dashboard, and detects when all dimensions plateau (3+ identical scores) to trigger an automatic stop condition.

---

## Documentation and Packaging (Phases 003-007)

---

### sk-doc Alignment

All skill documentation surfaces (SKILL.md, README.md, 11 references, 4 markdown assets, command, agent) follow sk-doc template standards with retrieval anchors, frontmatter, section dividers, and HVR-compliant language.

---

### Runtime Parity

The `@agent-improver` agent and `/improve:agent` command are mirrored across all 5 runtimes (OpenCode, Claude, Codex, .agents, Gemini). The integration scanner checks mirror parity automatically.

---

### Manual Testing Playbook

A 21-scenario testing playbook covers 6 categories: integration scanner, profile generator, 5D scorer, benchmark integration, reducer dimensions, and end-to-end loop. Each scenario has copy-pasteable commands and verification one-liners.
