## Bug Fixes

### Lineage schema aligned between producer and reducer

**Problem:** `candidate-lineage.cjs` wrote `candidateId`/`parentCandidateId` but the reducer expected `id`/`parentId`. The reducer silently failed to read lineage files.

**Fix:** Schema aligned on one canonical shape so the reducer can consume lineage artifacts.

---

### Mutation-coverage metrics now derivable by reducer

**Problem:** The reducer expected a `metrics` block that `mutation-coverage.cjs` never wrote.

**Fix:** The reducer now derives coverage ratio and uncovered counts from the raw arrays when the computed metrics block is missing.

---

### Dashboard now surfaces journal, lineage, and coverage

**Problem:** The reducer read replay artifacts into the registry but dropped them from the dashboard. Operators had no visibility.

**Fix:** Added dedicated dashboard sections for journal summary, candidate lineage, and mutation coverage.

---

### Legal-stop events now emitted from improve-agent workflows

**Problem:** The improve-agent loop never emitted `legal_stop_evaluated` or `blocked_stop` events that the other reducers depend on.

**Fix:** Both auto and confirm workflow YAMLs now emit structured legal-stop events with gate bundles.

---

### Trade-off detector trajectory extraction fixed

**Problem:** Journal-based trajectory extraction was incompatible with emitted `candidate_scored` payloads — the scored dimension array was missing from journal rows.

**Fix:** `trade-off-detector.cjs` now emits the scored dimension array into the journal row.

---

### Plateau stop reason added to taxonomy

**Problem:** The stop-reason taxonomy could not represent plateau exits. Plateau stops were falsified as `converged`.

**Fix:** Added `plateau` to the `STOP_REASONS` enum in `improvement-journal.cjs`.

---

### Phase 005 helpers now called in live workflows

**Problem:** `mutation-coverage.cjs`, `candidate-lineage.cjs`, `trade-off-detector.cjs`, and `benchmark-stability.cjs` were never called from the auto/confirm workflow steps — they were dead code.

**Fix:** Added explicit loop steps calling all 4 helpers at appropriate workflow points.

---

### Integration scan now targets actual agent, not profile label

**Problem:** `scan-integration.cjs` received the profile label instead of the real target agent identifier, so it scanned the wrong file.

**Fix:** Both workflow YAMLs now pass the real target agent identifier.

---

### Session-boundary gate enforced in confirm mode

**Problem:** The confirm-mode session gate was advisory only — no `verify`/`on_fail` enforcement.

**Fix:** Confirm mode now mirrors the auto workflow's structured halt pattern.
