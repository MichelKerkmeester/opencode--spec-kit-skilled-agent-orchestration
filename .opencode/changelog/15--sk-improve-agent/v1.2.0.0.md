This release lands Phase 008 of the 042 Deep Research & Review Runtime Improvement Bundle for `sk-improve-agent`. The skill now (a) actually calls `improvement-journal.cjs` from the visible `/improve:agent` YAML workflow at every iteration boundary, (b) enforces minimum sample sizes before trade-off and stability verdicts, and (c) implements ADR-002 Option A by reading `improvement-journal.jsonl`, `candidate-run history.json`, and `mutation-coverage.json` during reducer refresh. The visible operator path now emits the richer runtime-truth state the v1.1.0.0 expected behavior promised. The goal is to make the workflow easier to follow and easier to trust.

---

### Journal Wiring (REQ-004, REQ-005)

- `improve_agent-improver_{auto,confirm}.yaml` now invoke `improvement-journal.cjs` at three boundary points: session start, every iteration boundary (candidate generated to scored to gate evaluation), and session end. The orchestrator-only constraint is preserved - the proposal-only agent still never writes journal rows itself.
- The malformed `--event=session_initialized` CLI example in `.opencode/command/improve/agent.md` is fixed. The working form is `--emit session_start --payload '<json>'` matching the helper's actual interface.
- `SKILL.md` adds a new "Journal Wiring Contract" section documenting the three boundary points, the frozen `STOP_REASONS` / `SESSION_OUTCOMES` enums (aligned to the helper's actual validator: `converged`, `maxIterationsReached`, `blockedStop`, `manualStop`, `error`, `stuckRecovery` / `keptBaseline`, `promoted`, `rolledBack`, `advisoryOnly`), and the orchestrator-only emission rule.

---

### Sample-Size Enforcement (REQ-006, REQ-007)

- `trade-off-detector.cjs` now requires `minDataPoints` (default: 3, env-overridable) before reporting a trade-off. Below threshold, returns `{state: "insufficientData", dataPoints: n, minRequired: 3}` instead of fabricating confidence from 2 points. Exports `MIN_DATA_POINTS_DEFAULT = 3`.
- `benchmark-stability.cjs` now requires `minReplayCount` (default: 3). Below threshold, returns `{state: "insufficientSample", replayCount: n, minRequired: 3}` and treats that state as "not stable" in the is stable helper. Exports `MIN_REPLAY_COUNT_DEFAULT = 3`.
- Corresponding vitest updates: `tests/trade-off-detector.vitest.ts` (17 passing) and `tests/benchmark-stability.vitest.ts` (23 passing) now assert the new insufficient states AND validate configurable overrides.

---

### ADR-002 Replay Consumer Implementation

- **ADR-002 Option A accepted: implement replay consumers** (rather than downgrading docs to ledger-only reality).
- `scripts/reduce-state.cjs` now reads three additional artifacts during each refresh pass:
- `improvement-journal.jsonl` to extracts `lastSessionStart`, `lastSessionEnd`, `totalEvents`, `eventTypeCounts`, and terminal `stopReason` + `sessionOutcome` from the latest session_end event.
- `candidate-run history.json` to extracts `run historyDepth` (max depth from root), `totalCandidates`, `currentLeaf`.
- `mutation-coverage.json` to extracts `coverageRatio` and `uncoveredMutations`.
- Each artifact gracefully defaults to `null` when absent - no throw. Registry exposes them as top-level fields: `journalSummary`, `candidateLineage`, `mutationCoverage`.
- The reducer also surfaces `insufficientDataIterations` and `insufficientSampleIterations` as new registry fields, propagated from the Part A sample-size gates.

---

### Sample Quality Dashboard Section (REQ-019)

- Reducer dashboard gains a new **Sample Quality** section that renders `replayCount`, `stabilityCoefficient`, `insufficientSampleIterations` count, and `insufficientDataIterations` count as distinct fields, separate from generic benchmark failure counters. Operators can now distinguish "real regression" from "not enough samples to decide".
- When either insufficient count is > 0, the section includes an explicit operator guidance note.

## Upgrade

**Backward-compatible.** Existing sessions without the three new replay artifacts (journal, lineage, mutation-coverage) still work - the reducer defaults each consumer field to `null` when the artifact is absent. The `trade-off-detector` and `benchmark-stability` default thresholds (3 data points / 3 replays) are enforced on new sessions but can be overridden via `minDataPoints` / `minReplayCount` constructor options or env variables for legacy replay flows.

Operators invoking `/improve:agent` now see a richer dashboard with the Sample Quality section and the full journal/lineage/coverage summary. The visible workflow emits the journal events the v1.1.0.0 contract already promised - the contract break that Phase 008 Part A surfaced is now closed.
