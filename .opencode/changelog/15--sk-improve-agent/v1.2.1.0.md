## [**1.2.1.0**] - 2026-04-11

This patch release retracts a multi-session lifecycle that `sk-improve-agent` had documented but never actually shipped. It is a documentation-only correction that brings the written guidance back in line with the real runtime.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/042-sk-deep-research-review-improvement-2/` (Level 3)

## What Changed

This release touched several parts of the workflow. The notes below explain the main changes in plain language.

### Bug Fixes

- **Resume/continuation expected behavior retracted to match reality** -- The skill's runtime-truth section and the `/improve:agent` command definition both listed `resume`, `restart`, `fork`, and `completed-continue` as supported lifecycle modes for evaluating an agent across multiple sessions. The shipped workflow has no inputs for any of them, the reducer does not produce ancestry output, and the journal helper has no boundary event for a multi-session run history. The only thing the runtime actually supports is `new` - every `/improve:agent` invocation starts a fresh session with generation 1. The documentation now says so explicitly. Operators who want to continue an evaluation should archive the prior `improve/` folder under a timestamped path and re-invoke the command. A future release that ships real multi-session run history will arrive with first-class event emission, reducer ancestry output, and replay coverage; until then, treat every invocation as a standalone evaluation.
