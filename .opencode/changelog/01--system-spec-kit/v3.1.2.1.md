## What Changed

### Context-server startup tests fixed (T000k, T000l, T000m)

- **T000k, T000l, T000m** -- the Phase 6 entrypoint guard that prevents `main()` from running on import also prevented three startup tests from exercising dimension validation, dynamic instructions, and per-init regeneration. Exported `main()` via `__testables` and updated the test harness to call it explicitly. Context-server tests now pass 359/359.

### All 10 lint errors resolved

- **Unused imports removed** -- `embeddings`, `createRequire`, `SPEC_DOCUMENT_FILENAMES`, `CheckpointScopeOptions`, `fileName` across 5 files.
- **let-to-const** -- `dbConsumersRef` in `db-state.ts`, `pending` in `tool-cache.ts`.
- **Type casts cleaned up** -- `typedWarnings` assignment in `response-builder.ts` uses `Record<string, unknown>` instead of `any`. Two SDK-required `any` types in `context-server.ts` get eslint-disable comments.

### Ground-truth evaluation IDs re-mapped

- **290 relevance entries** across 99 queries re-mapped from stale memory IDs to current database rows. Zero unmapped entries. Ablation studies can now run without the E030 alignment error.

### Dead code removed

- **Unused `createRequire` import** and its call in `file-watcher.ts` removed entirely (was a leftover from CJS-to-ESM migration).
