# v3.4.0.1 — Silent Failures Made Loud

This patch closes 63 distinct findings surfaced by a 50-iteration deep-research pass on the MCP runtime. The previous version shipped a lot of new surface; this one hardens the seams that were already there. Nothing in the runtime was crashing — every failure mode was silent, success-shaped, and layered. The fix pattern throughout is the same: replace booleans that collapsed success, skip, fail, and defer into one value with typed results that callers can actually read.

Four attack scenarios that combined layered findings into systemic data-integrity risks are now blocked outright. The save path gained predecessor CAS. Hook state gained schema validation with quarantine. Graph metadata can no longer be "laundered" through a legacy fallback. TOCTOU-style cleanup races can no longer delete fresh state. Skill routing now disambiguates deep-research vs deep-review, and the command-bridge layer no longer collapses six subcommands into one generic label.

> Scope: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/016-foundational-runtime-deep-review/` — charter-driven remediation. 27 commits to main across ~60 files + test files.

## The P0 Composites

No single finding reached P0 on its own. Four combinations did. Each has a dedicated regression test that reconstructs the full attack chain.

### Cross-runtime tempdir poisoning (P0-A)

One corrupt `HookState` file in the shared tempdir used to poison five hook entrypoints at once, because every runtime called `JSON.parse(raw) as HookState` with no validation. Now every load runs through a Zod schema. Invalid files get renamed to `.bad` siblings. Other entrypoints keep working.

HookState gained a `schemaVersion` field. Missing-session payloads no longer collapse onto a shared "unknown" state key. `updateState()` returns `{ok, merged, persisted}` so callers can abort autosave when the underlying write failed. `clearCompactPrime()` uses payload identity instead of session ID to avoid erasing newer payloads on overlap.

&nbsp;

### Reconsolidation corruption window (P0-B)

Two concurrent saves could both supersede the same predecessor, permanently forking lineage. `executeConflict()` now does CAS: `UPDATE ... WHERE content_hash = ? AND is_deprecated = FALSE`. The second writer gets `conflict_stale_predecessor` instead of silent success.

Complement decisions moved inside the writer transaction to close the stale-search duplicate window. Scope filtering uses batched snapshot reads instead of per-candidate queries. Assistive recommendations flag `advisory_stale: true` when the snapshot predates commit.

&nbsp;

### Graph-metadata laundering (P0-C)

Malformed modern `graph-metadata.json` used to be accepted as "legacy", stamped with fresh timestamps, rewritten as canonical JSON, assigned `qualityScore: 1`, and boosted +0.12 in packet search — erasing the corruption signal along the way.

`validateGraphMetadataContent()` now returns `{ok, metadata, migrated, migrationSource?}`. The `migrated` marker propagates through refresh. Stage-1 indexing penalizes migrated rows instead of boosting them. `readDoc()` distinguishes "I/O failure" from "file does not exist" via a three-state return.

&nbsp;

### TOCTOU cleanup race (P0-D)

A routine `--finalize` cleanup sweep overlapping with live writers could delete the newest session state. Both `loadMostRecentState()` and `cleanStaleStates()` had all-or-nothing try blocks — one bad sibling file aborted the whole scan.

Both functions now do per-file error isolation and return `{removed, skipped, errors}`. `cleanStaleStates()` re-stats before `unlinkSync()` and skips files that changed since the initial scan. `session-stop.ts` got a monotonicity guard on `lastTranscriptOffset` so overlapping stop hooks cannot regress it backwards.

## Truthful State Contracts

### Post-insert enrichment status

`enrichmentStatus = true` used to collapse four different outcomes (success, feature-disabled skip, nothing-to-do skip, full deferral) into one boolean. The response envelope had no way to tell which happened. Now every step returns an `OperationResult<T>`:

```ts
type OperationStatus = 'ran' | 'skipped' | 'failed' | 'deferred' | 'partial';
type OperationResult<T = void> = {
  status: OperationStatus;
  reason?: string;
  warnings?: StructuredWarning[];
  payload?: T;
};
```

Entity linking gates on `extraction.status === 'ran'` instead of just the feature flag. Partial causal-link failures surface as `partial` with a reason. Density-guard skips carry their reason through to the response. The response builder aggregates step-level status into an envelope-level summary and propagates warnings.

&nbsp;

### Trust-state vocabulary

The old vocabulary collapsed `missing` and `empty` into `stale`. Consumers couldn't tell whether to retry, wait, or bootstrap. The new type has four distinct states:

| State | Meaning |
|---|---|
| `live` | fresh, validated |
| `stale` | exists but may be outdated; query still valid |
| `absent` | does not exist for this scope |
| `unavailable` | should exist but inaccessible (I/O error, lock held) |

Every producer updated. Every consumer updated. TypeScript catches non-exhaustive switches. Routing no longer recommends `code_graph_query` when readiness is `unavailable`.

&nbsp;

### Session-stop atomicity

Stop-hook state used to get written three separate times (metrics, retarget, summary) with a disk reload in between. Two concurrent stop events could race during the gap and produce a torn state. All three writes now accumulate into a single `Partial<HookState>` patch applied via one `updateState()` call. Autosave consumes the merged state directly instead of re-reading from disk.

## Skill Routing

### Per-subcommand command bridges

`COMMAND_BRIDGES` used to collapse every `/spec_kit:*` subcommand into one generic `command-spec-kit` label at `kind_priority=2`. Six canonical subcommands now each have their own bridge entry:

- `command-spec-kit-plan`
- `command-spec-kit-complete`
- `command-spec-kit-implement`
- `command-spec-kit-deep-research`
- `command-spec-kit-deep-review`
- `command-spec-kit-resume`

Each carries `owning_skill` metadata. The deprecated generic bridge stays for one-release compat; insertion order is dict-order so specific markers match first. `/spec_kit:deep-research investigate auth bug` now routes to `command-spec-kit-deep-research` at 0.95 confidence instead of the generic label.

&nbsp;

### Disambiguation and parity

Deep-research versus deep-review used to collide with `sk-code-review` on borderline prompts. A disambiguation tier now enforces a ≥0.10 confidence gap between the iterative-loop skills and the single-pass review skill on unambiguous phrases.

`conflicts_with` edges require mutual declaration before the routing penalty applies. `health_check()` reports `status: degraded` when the inventory of discovered `SKILL.md` files and the compiled skill graph drift apart. Topology warnings (weight-band, weight-parity, orphans, asymmetric edges) are threaded through the compiled graph and surfaced in health output.

&nbsp;

### Keyword coverage

The frontmatter parser used to strip HTML comment blocks before routing — any skill that declared extra keywords via `<!-- Keywords: ... -->` had those keywords dropped on the floor. The parser now captures them. `intent_signals` wired from per-skill graph metadata into the runtime scoring path, so signal-only matches can win routing when keywords aren't sufficient.

## Gate 3 Typed Classifier

The Gate 3 trigger list used to be prose words scattered across AGENTS.md, CLAUDE.md, the command entry docs, and a few YAML assets. Different runtimes could classify the same request differently. Tokens like `analyze`, `decompose`, and `phase` were positive triggers — which meant read-only review prompts ("please analyze this PR") falsely triggered spec-folder questions.

Gate 3 now runs through a shared classifier module at `.opencode/skill/system-spec-kit/shared/gate-3-classifier.ts`. The module exports positive triggers, memory-save triggers, resume triggers, and read-only disqualifiers. Memory-save and resume triggers beat read-only disqualifiers (so `save context` still fires even when the prompt also says "review"). All four runtime root docs point at the shared module instead of maintaining their own prose lists.

## YAML Predicate Grammar

Workflow YAML files used expressions like `intake_only == TRUE` and `folder_state == populated` — quoted strings with no mechanical grammar contract. Two incompatible argument dialects lived side by side in the corpus. Same-named keys meant different things in different files.

A `BooleanExpr` schema now lives in `.opencode/skill/system-spec-kit/shared/predicates/boolean-expr.ts`. Supported operators: `==`, `!=`, `in`, `not_in`. Expressions can be authored in typed form `{field, op, value}` or as string literals the parser validates against the grammar. The `when:` field is now a predicate only. Prose timing notes moved to a new `after:` field. Non-predicate overloads of `when:` in `spec_kit_complete_auto.yaml` got renamed to `trigger_reason:` and `applies_when:` so the grammar contract is unambiguous.

Deep-research `resume` is now explicitly tagged `gate3_classification: write-path` in `spec_kit_deep-research_auto.yaml`, matching the new Gate 3 classifier additions for save/resume/continue triggers.

## Playbook Runner Isolation

`manual-playbook-runner.ts` used to evaluate object-literal arguments through `Function(...)()`. The runtime state it consumed included `lastJobId` fed from prior handler return values — a documentation-to-code trust boundary crossing. The runner also accepted a shorthand dialect (`{jobId}` with no value) that depended on undefined JS scoping semantics.

The `Function()` eval is gone, replaced by a strict object-literal parser. `lastJobId` substitution is sanitized before reaching the parser; adversarial payloads can no longer escape the literal slot. The shorthand dialect is rejected; the typed `{key: value}` form is required and validated against a per-tool schema. Scenario `automatable: boolean` metadata now lives in typed frontmatter instead of being inferred from filename substrings.

Playbook parser failures (10 out of 291 scenario files were silently dropped at coverage-count time) now trigger a named warning when `parsedCount` diverges from `filteredCount`.

## Runtime Safety Nets

- `graph-metadata-parser.ts` temp-path construction: `process.pid + Date.now()` collides at millisecond precision. Replaced with `.tmp-<pid>-<counter>-<random>`.
- `hook-state.ts` writer: same pattern as above, prevents two same-session writers from swapping bytes before rename.
- `ensure-ready.ts` now stages a placeholder mtime, persists nodes and edges, then finalizes the real mtime. Partial persistence failures no longer look like successful refreshes.
- `ensureCodeGraphReady()` exceptions surface as `status: "error"` instead of fall-open.
- `blast_radius` rejects unresolved subjects instead of treating the raw input as a seed file path.
- `edgeType` and outline subject paths validated up front.
- Dangling edges flagged as corruption instead of returned as successful relationships.
- Ambiguous subject resolution returns `ambiguous_subject` with candidate list instead of arbitrary first match.
- Response-level edge trust aggregates across all edges, not just `edges[0]`.

## Shared-Path Elimination

`/tmp/save-context-data.json` was the documented handoff path for `/memory:save`. Parallel runtime sessions (Claude + Gemini + OpenCode) overwrote each other. The shared path is gone. Every caller now uses a per-session path: `/tmp/save-context-data-<session-id>.json`.

The change touched `data-loader.ts`, `generate-context.ts`, three `/memory:save` / `/spec_kit:*` command docs, four runtime root docs (AGENTS.md, CLAUDE.md, CODEX.md, GEMINI.md), and the four Gemini CLI command TOML mirrors — around 50 files in total.
