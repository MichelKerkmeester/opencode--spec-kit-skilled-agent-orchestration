This release lays down some of the core rules the spec kit uses to judge whether work is ready to move forward. It also reorganizes a few internal scripts so the project is easier to maintain.

---

## What Changed

---

### Clearer confidence checks

**Problem:** The project needed a simple way to decide when the assistant had enough confidence to continue.

**Fix:** This release introduced a two-part check that looks at both confidence and uncertainty before work moves ahead. This makes decisions more consistent and reduces the chance of pushing ahead on weak assumptions.

---

### A stronger review checklist for complex work

**Problem:** Larger specs needed more structure than a quick yes-or-no check.

**Fix:** This release added the Five Checks Framework so bigger changes are reviewed for fit, quality, and maintainability before they move forward. This helps the team catch risky or overcomplicated work earlier.

---

### Session learning support

**Problem:** The system needed a better way to remember what happened during a working session.

**Fix:** This release added a session learning handler for the memory workflow. This gives later steps more context and makes follow-up work easier.

---

### Script organization cleanup

**Problem:** Some helper scripts were sitting in the wrong folders, which made the codebase harder to navigate.

**Fix:** This release moved the decision tree generator and the capture script into more appropriate locations. This makes the internal structure easier to understand and maintain.
