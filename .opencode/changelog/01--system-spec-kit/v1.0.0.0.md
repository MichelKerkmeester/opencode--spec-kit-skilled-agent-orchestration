## [**1.0.0.0**] - 2025-01-23

This release lays down some of the core rules the spec kit uses to judge whether work is ready to move forward. It also reorganizes a few internal scripts so the project is easier to maintain.

---

## What Changed

### 1. Clearer confidence checks

- Need: The project needed a simple way to decide when the assistant had enough confidence to continue.
- Change: This release introduced a two-part check that looks at both confidence and uncertainty before work moves ahead.
- Why it matters: This makes decisions more consistent and reduces the chance of pushing ahead on weak assumptions.

### 2. A stronger review checklist for complex work

- Need: Larger specs needed more structure than a quick yes-or-no check.
- Change: This release added the Five Checks Framework so bigger changes are reviewed for fit, quality, and maintainability before they move forward.
- Why it matters: This helps the team catch risky or overcomplicated work earlier.

### 3. Session learning support

- Need: The system needed a better way to remember what happened during a working session.
- Change: This release added a session learning handler for the memory workflow.
- Why it matters: This gives later steps more context and makes follow-up work easier.

### 4. Script organization cleanup

- Need: Some helper scripts were sitting in the wrong folders, which made the codebase harder to navigate.
- Change: This release moved the decision tree generator and the capture script into more appropriate locations.
- Why it matters: This makes the internal structure easier to understand and maintain.
