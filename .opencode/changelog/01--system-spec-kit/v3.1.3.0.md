## [**3.1.3.0**] - 2026-03-30

The memory search pipeline was completely broken -- every query returned zero results. Two independent filtering bugs were silently discarding all candidates at different stages of the pipeline, while the health check still reported "healthy." The trigger system (phrase matching) and memory listing continued to work, masking the severity. A 12-step diagnostic investigation using Sequential Thinking traced the failure through 6 pipeline files to identify both root causes. After the fix, all three test queries return 4-5 results with both vector search and full-text search contributing.

> Spec folder: `.opencode/specs/02--system-spec-kit/023-esm-module-compliance/007-hybrid-search-null-db-fix/` (Level 2)

---

## What Changed

### Bug Fixes

- **Scope enforcement silently rejected every search result** -- A security feature ("scope enforcement") was turned on by default. When active, it requires every search query to include identity parameters (tenant ID, user ID, agent ID). Queries without these parameters -- which is every normal search in a single-user system -- were treated as "no access" and returned nothing. The feature is now opt-in: set `SPECKIT_MEMORY_SCOPE_ENFORCEMENT=true` to enable it, otherwise searches work without identity parameters. (`scope-governance.ts`: `isScopeEnforcementEnabled()` changed from default-ON to explicit opt-in)

- **State filter removed all candidates because the state column doesn't exist yet** -- The search handler required every memory to have a "warmth state" (HOT, WARM, COLD, etc.) of at least WARM. This state system was designed but the database column was never created -- every memory resolved to UNKNOWN (priority zero), failing the threshold check. All 15-18 candidates found by vector and text search were discarded at the final filtering stage. The WARM default has been removed; state filtering is now inactive until the state column is implemented. (`memory-search.ts`: removed `minState = 'WARM'` default; `memory-context.ts`: removed two hardcoded `minState: 'WARM'` values)
