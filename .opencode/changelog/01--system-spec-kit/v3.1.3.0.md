## [**3.1.3.0**] - 2026-03-30

The memory search pipeline was completely broken -- every query returned zero results. Five independent bugs were silently discarding all candidates at different stages of the pipeline, while the health check still reported "healthy." A 12-step diagnostic investigation using Sequential Thinking traced the failures through 8 pipeline files, followed by a 10-agent deep research + review wave that uncovered three additional issues. After all fixes, search returns 8+ results with vector, FTS5, and graph channels all contributing.

> Spec folder: `.opencode/specs/system-spec-kit/023-hybrid-rag-fusion-refinement/007-hybrid-search-null-db-fix/` (Level 2)

---

## What Changed

---

### Bug Fixes

- **Scope enforcement silently rejected every search result** -- A security feature ("scope enforcement") was turned on by default. When active, it requires every search query to include identity parameters (tenant ID, user ID, agent ID). Queries without these parameters -- which is every normal search in a single-user system -- were treated as "no access" and returned nothing. The feature is now opt-in: set `SPECKIT_MEMORY_SCOPE_ENFORCEMENT=true` to enable it, otherwise searches work without identity parameters. (`scope-governance.ts`: `isScopeEnforcementEnabled()` changed from default-ON to explicit opt-in)

- **State filter removed all candidates because the state column doesn't exist yet** -- The search handler required every memory to have a "warmth state" (HOT, WARM, COLD, etc.) of at least WARM. This state system was designed but the database column was never created -- every memory resolved to UNKNOWN (priority zero), failing the threshold check. Removed `minState = 'WARM'` defaults from `memory-search.ts` and `minState: 'WARM'`/`'COLD'` hardcodes from `memory-context.ts`.

- **UNKNOWN state priority was below all valid states** -- Even after removing the WARM default, callers passing any `minState` value (including ARCHIVED, the lowest) still filtered out all UNKNOWN memories because `UNKNOWN_STATE_PRIORITY = 0` was below `ARCHIVED = 1`. Changed to `6` (above `HOT = 5`) so unclassified memories always pass state filters until the memoryState column is implemented. (`stage4-filter.ts`)

- **Governance guardrails defaulted ON like scope enforcement** -- `isGovernanceGuardrailsEnabled()` used the same `isDefaultOnFlagEnabled()` helper that caused the scope enforcement bug. Changed to opt-in to prevent similar silent failures for memory save/ingest operations. (`scope-governance.ts`)

- **Feature flags silently disabled when rollout < 100% without identity** -- `isFeatureEnabled()` returned `false` when `SPECKIT_ROLLOUT_PERCENT < 100` and no identity parameter was provided. Since most search flags omit identity, setting rollout below 100 would silently disable search fallback, embedding expansion, cross-encoder reranking, and dozens of other features. Changed to `return true` when no identity is provided. (`rollout-policy.ts`)

---

### Observability

- **Silent catch blocks in hybrid search now log warnings** -- The vector, graph, and degree search channels had `catch (_err) {}` blocks that swallowed all errors silently. Operators had no way to distinguish "no results found" from "search channel crashed." All three now emit `console.warn` with the error message. (`hybrid-search.ts`)

---

### Schema Compatibility

- **Removed oneOf/not from MCP tool schemas** -- Claude Code's MCP client rejects schemas with `oneOf`/`anyOf`/`allOf`/`not` at the top level. Removed from `shared_space_upsert`, `shared_space_membership_set`, and `shared_memory_status`. Simplified nested `oneOf` in `memory_drift_why` and `memory_causal_link`. (`tool-schemas.ts`)
