## [**2.4.0.0**] - 2026-03-02

Opus review remediation: 35 fixes across 5 sprints resolving all 4 P0 critical bugs and 31 P1 findings in the spec-kit-memory MCP server. Removed the legacy V1 pipeline (~550 LOC), unified scoring across pipeline stages, hardened delete cleanup, and improved graph/cognitive memory integrity.

## Highlights

### Removed

- **Legacy V1 pipeline** -- Deleted ~550 lines of dead code including `STATE_PRIORITY`, `postSearchPipeline()`, `buildDeepQueryVariants()`, `strengthenOnAccess()`, `applyTestingEffect()`, `filterByMemoryState()`, `applyCrossEncoderReranking()`, and `applyIntentWeightsToResults()`. Resolves P0-1 through P0-3.
- **`isPipelineV2Enabled()`** -- Deprecated to always return `true`. The `SPECKIT_PIPELINE_V2` env var is accepted but ignored.

### Fixed

- **Orphaned chunk detection** -- `verify_integrity()` now detects and optionally auto-cleans chunk records whose parent was deleted (P0-4).
- **Intent weight recency** -- `applyIntentWeights` now includes timestamp-based recency scoring instead of ignoring the recency component (#5).
- **Stack overflow prevention** -- `normalizeCompositeScores` uses loop-based min/max instead of `Math.max(...scores)` which fails on arrays >100K elements (#7).
- **BM25 specFolder filter** -- Replaced no-op string ID comparison with DB lookup to resolve spec_folder per result (#8).
- **RRF convergence double-count** -- Cross-variant fusion now subtracts per-variant bonus before applying cross-variant bonus (#9).
- **Stemmer double-consonant** -- `simpleStem()` now handles "running"->"run" and "stopped"->"stop" instead of producing "runn"/"stopp" (#18).
- **Delete cleanup** -- Memory deletion now cleans degree_snapshots, community_assignments, memory_summaries, memory_entities, causal_edges, and BM25 index (#20, #21).
- **Self-loop prevention** -- `insertEdge()` rejects `sourceId === targetId` (#24).
- **WM score clamping** -- Working memory scores clamped to [0, 1] to prevent mention boost overflow (#29).
- **Trigger double-decay** -- Removed extra `* turnDecayFactor` on working memory attention scores (#30).
- **Dedup hash collision** -- Session dedup hash extended from 64-bit to 128-bit (#37).

### Changed

- **Five-factor weight normalization** -- Composite scoring weights auto-normalize to sum 1.0 after partial overrides (#6).
- **Adaptive fusion normalization** -- Core weights (semantic + keyword + recency) normalize to 1.0 after doc-type adjustments (#10).
- **Shared `resolveEffectiveScore()`** -- Single function in `pipeline/types.ts` replaces divergent `resolveBaseScore()` (Stage 2) and `effectiveScore()` (Stage 3) with consistent `intentAdjustedScore -> rrfScore -> score -> similarity/100` fallback chain, all clamped [0,1] (#11).
- **Community debounce** -- Replaced edge-count-only debounce with `count:maxId` hash to detect delete+insert sequences (#27).
- **Postflight re-correction** -- `task_postflight` now allows UPDATE of existing complete records instead of rejecting (#35).
- **Exit handler cleanup** -- `cleanupExitHandlers()` now calls `process.removeListener()` using stored handler refs (#38).

### New

- **Schema params exposed** -- `memorySearch` tool schema now includes `trackAccess`, `includeArchived`, and `mode` parameters (#13).
- **Constitutional count passthrough** -- Stage 1's constitutional injection count flows through the orchestrator to Stage 4 output (#15).
- **Embedding caching** -- Stage 1 caches the query embedding for reuse in the constitutional injection path, saving one API call per search (#16).
- **Full-content embedding on update** -- `memory_update` now embeds `title + content_text` instead of title alone (#19).
- **Configurable interference threshold** -- `computeInterferenceScoresBatch()` accepts optional `threshold` parameter (#12).
- **`cleanupOrphanedEdges()`** -- New export from `causal-edges.ts` for automated orphan edge cleanup (#28).
- **maxDepth clamping** -- `handleMemoryDriftWhy` clamps maxDepth to [1, 10] server-side (#25).
- **Ablation recallK** -- Ablation framework uses `recallK` for search limit instead of hardcoded 20 (#33).
- **evalRunId persistence** -- Counter initializes from DB MAX on first call, surviving server restarts (#34).
- **parseArgs guard** -- `parseArgs<T>()` guards against null/undefined/non-object input (#36).
- **Co-activation cache clear** -- `clearRelatedCache()` called after bulk delete operations (#32).
