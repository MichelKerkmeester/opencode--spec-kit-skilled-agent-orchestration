## Security Fixes

These changes close safety gaps where the system could cross boundaries it should respect. They make day-to-day use safer without changing the release scope.

#### Coverage-graph namespace isolation hardened

Coverage-gap, unresolved-finding, and contradiction queries used bare IDs in subqueries, allowing foreign session scopes to satisfy them. Read tools permitted cross-session aggregation without explicit admin scope. Full identity predicates (`spec_folder`, `loop_type`, `session_id`) were added to all `NOT EXISTS` subqueries and contradiction joins in `coverage-graph-query.ts`. A shared query helper now enforces that same session scope everywhere, `sessionId` is required for non-admin reads, and `coverage-graph-signals.ts` carries the full scope through nested edge lookups.

&nbsp;

#### Optimizer path traversal and boundary enforcement

The promotion-report save path could write to arbitrary repo locations. Config search also accepted out-of-bounds and locked-field changes, while corpus helpers hid integrity failures. Promotion output is now restricted to a dedicated audit directory with symlink rejection. The evaluator requires a manifest, search space comes from that manifest with unknown and locked fields rejected, and corpus helpers now fail loudly when integrity checks break.

## Bug Fixes

These fixes correct behavior that could confuse operators or produce the wrong result. The goal is to make the workflow more predictable in normal use.

#### Wave hotspot detection uses composite scoring

The hotspot spread calculation treated every median-complexity directory as a hotspot, which triggered false waves on otherwise uniform inventories. Hotspots now require a strict-above-baseline composite score including churn and issues, not just `complexity >= median`.

&nbsp;
#### Domain clustering uses registrable-domain heuristic

The cluster-assignment helper used the TLD, collapsing unrelated research domains into the same cluster. Fallback clusters derived from registered-domain or topic-based rule.

&nbsp;

#### Wave lifecycle enforces adjacent phase transitions

The phase-advance helper enforced forward progress but not the required phase order, allowing illegal jumps straight to merge. Added an allowed-transition matrix. Non-adjacent jumps require explicit audited API.

&nbsp;

#### Board merge uses full composite key

Merge deduplicated findings by `findingId` alone, silently collapsing distinct segment findings. Storage and replay now use the full 5-key composite: `(sessionId, generation, segment, wave, findingId)`.

&nbsp;

#### Unified merge contract across wave modules

Merge and provenance rules were implemented three different ways across lifecycle, board, and segment-state. The board module now owns one shared merge path. Lifecycle and segment-state only call into that shared logic instead of each carrying their own version.

&nbsp;

#### Board transition API wired to lifecycle

The board transition API existed but the main board never advanced its own status. Lifecycle updates now flow through `transitionBoardStatus()` inside `updateBoard()`, so the board actually advances its own status.

&nbsp;

#### Replay schema normalized

Graph-aware replay scoring was out of sync between corpus extraction and runner. The corpus builder now emits the exact metric shape the replay runner expects.

## Maintainability

Some changes were mainly about keeping the system easier to understand and maintain over time. They reduce hidden complexity without changing the core purpose of the release.

#### Session normalization centralized

`coverage-graph-core.cjs` now uses a shared session normalization helper. Auto-created nodes propagate `metadata.sessionId`.

&nbsp;

#### computeMomentum renamed and alias removed

`computeMomentum` renamed to `computeRecentEdgeActivity` in `coverage-graph-signals.cjs`. The compatibility alias was removed. The TS export in `coverage-graph-signals.ts` adds `sessionId` parameter.

&nbsp;

#### Cycle handling matches documented contract

`coverage-graph-signals.cjs` cycle-handling now implements the documented fallback for cyclic components.

&nbsp;

#### Mixed-session edge precedence documented

`coverage-graph-session.cjs` now rejects mixed-session edges without explicit `edge.sessionId` or documents and tests the precedence rule.
