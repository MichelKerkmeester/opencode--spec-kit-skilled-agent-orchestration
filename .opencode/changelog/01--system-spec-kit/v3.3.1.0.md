## Security Fixes

### Coverage-graph namespace isolation hardened

**Problem:** Coverage-gap, unresolved-finding, and contradiction queries used bare IDs in subqueries, allowing foreign namespaces to satisfy them. Read tools permitted cross-session aggregation without explicit admin scope.

**Fix:** Full composite-key predicates (`spec_folder`, `loop_type`, `session_id`) added to ALL `NOT EXISTS` subqueries and contradiction joins in `coverage-graph-query.ts`. A namespace-safe query helper centralizes composite-key enforcement. `sessionId` is now required for all non-admin reads. `coverage-graph-signals.ts` threads full namespace through every nested edge lookup.

---

### Optimizer path traversal and boundary enforcement

**Problem:** `savePromotionReport()` could write to arbitrary repo paths. Config search accepted out-of-bounds and locked-field mutations. Corpus helpers masked integrity failures.

**Fix:** Promotion output restricted to dedicated audit directory with symlink rejection. Manifest is now mandatory for `evaluateCandidate()`. Search space derived from canonical manifest with unknown/locked field rejection. Corpus helpers constrained to approved roots with explicit integrity errors.

## Bug Fixes

### Wave hotspot detection uses composite scoring

**Problem:** `computeHotspotSpread()` treated every median-complexity directory as a hotspot, causing false wave activation on uniform inventories.

**Fix:** Hotspots now require a strict-above-baseline composite score including churn and issues, not just `complexity >= median`.

---

### Domain clustering uses registrable-domain heuristic

**Problem:** `assignCluster()` used the TLD, collapsing unrelated research domains into the same cluster.

**Fix:** Fallback clusters derived from registrable-domain or topic-based heuristic.

---

### Wave lifecycle enforces adjacent phase transitions

**Problem:** `advancePhase()` enforced forward monotonicity but not the required phase order, allowing illegal skips straight to merge.

**Fix:** Added an allowed-transition matrix. Non-adjacent jumps require explicit audited API.

---

### Board merge uses full composite key

**Problem:** Merge deduplicated findings by `findingId` alone, silently collapsing distinct segment findings.

**Fix:** Storage and replay now use the full 5-key composite: `(sessionId, generation, segment, wave, findingId)`.

---

### Unified merge contract across wave modules

**Problem:** Merge and provenance rules were implemented three different ways across lifecycle, board, and segment-state.

**Fix:** One canonical merge contract in the board module. Lifecycle and segment-state are now thin adapters.

---

### Board transition API wired to lifecycle

**Problem:** The board transition API existed but the canonical board never advanced its own status.

**Fix:** Lifecycle updates routed through `transitionBoardStatus()` inside `updateBoard()`.

---

### Replay schema normalized

**Problem:** Graph-aware replay scoring was schema-disconnected between corpus extraction and runner.

**Fix:** `buildCorpus()` now emits the exact metric shape that `replayRun()` consumes.

## Maintainability

### Session normalization centralized

`coverage-graph-core.cjs` now uses a shared session normalization helper. Auto-created nodes propagate `metadata.sessionId`.

### computeMomentum renamed and alias removed

`computeMomentum` renamed to `computeRecentEdgeActivity` in `coverage-graph-signals.cjs`. The compatibility alias was removed. The TS export in `coverage-graph-signals.ts` adds `sessionId` parameter.

### Cycle handling matches documented contract

`coverage-graph-signals.cjs` cycle-handling now implements the documented fallback for cyclic components.

### Mixed-session edge precedence documented

`coverage-graph-session.cjs` now rejects mixed-session edges without explicit `edge.sessionId` or documents and tests the precedence rule.
