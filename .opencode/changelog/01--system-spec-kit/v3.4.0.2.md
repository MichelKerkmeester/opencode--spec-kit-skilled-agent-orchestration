# v3.4.0.2 ŌĆö Canonical Save, Actually

This patch closes the 28 findings surfaced by the deep-review of v3.4.0.1 plus 14 more exposed by a follow-up 7-iteration meta-research. The previous version made failure modes loud; this one fixes the largest one of all ŌĆö it turned out the canonical save path wasn't writing any metadata at all. Not "eventually stale." Zero. Every default `/memory:save` since the guard shipped had been a no-op on `description.json.lastUpdated` and `graph-metadata.json.derived.*`.

The fix is two lines in `scripts/core/workflow.ts`. A dead-code guard at `:1259` hardcoded `ctxFileWritten = false`, making the per-folder description block unreachable. A plan-only gate at `:1333` kept `refreshGraphMetadata` disabled on the default code path. Once both came out, every save since has written fresh timestamps across 38 folders in the 026 tree. The "0 of 16 sibling folders have fresh `lastUpdated`" symptom that the deep-review flagged as drift was actually the deterministic output of code that never wrote.

Three follow-on surfaces: code-graph sibling handlers now emit the same readiness vocabulary as `query.ts` instead of just 1 of 7 being hardened; Copilot finally has compact-cache parity with Claude and Gemini; and the MCP transport layer now carries caller identity through `AsyncLocalStorage`, closing a session-resume cross-UID leakage path without a 30-file signature refactor.

A 10-iteration deep-review of Phase 017 itself surfaced one more round ŌĆö 5 P1 + 15 P2 findings, verdict CONDITIONAL at iter 6. A Phase 018-lite sweep of 11 fix commits closed all 20, flipping the verdict to PASS: the Unicode fold was Greek-only (homoglyph bypass via Cyrillic `ąĪ/ąĢ/ą×/ąÉ`, Latin-accented `├ē/├ü`, and mixed variants), session-resume caller propagation dropped the context on the `args.sessionId`-absent path, and 79+26+61+36 task ledger boxes had stayed `[ ]` after the work landed. ## Canonical Save, Actually

The symptom that the deep-review flagged was "0 of 16 sibling folders have fresh `description.json.lastUpdated`" and labeled it drift. It wasn't drift. Three conditions composed into a deterministic no-op:

```ts
// scripts/core/workflow.ts:1259
const ctxFileWritten = false;  // stub

// scripts/core/workflow.ts:1261-1331
if (ctxFileWritten) {
  // ... 70 LOC of description.json + memorySequence tracking ...
  // never reached.
}

// scripts/core/workflow.ts:1333
const shouldRunExplicitSaveFollowUps = options.plannerMode === 'full-auto';
if (shouldRunExplicitSaveFollowUps) {
  await refreshGraphMetadata(/* ... */);
  // ...
}

// scripts/memory/generate-context.ts:415
plannerMode: 'plan-only',  // the default
```

Three code paths that individually looked defensive. Together, they meant the default `/memory:save` path wrote zero metadata. Since the guards shipped.

The fix removes both. `ctxFileWritten` is now `true`; the block runs on every save. `shouldRunExplicitSaveFollowUps` is unconditional; `refreshGraphMetadata` runs on every save. A new `sequenceSnapshot.lastUpdated = new Date().toISOString()` line inside the retry loop actually writes the timestamp that the block was always supposed to. [EVIDENCE: aaf0f49a8]

The fix cascaded immediately. During implementation of the rest of Phase 017, `/memory:save` invocations refreshed 38 folder metadata files automatically ŌĆö the visible proof that canonical save now writes what it says it writes.

&nbsp;

### The meta-irony

Fixing the canonical save path meant we couldn't verify the fix by running `/memory:save` on the 017 spec folder itself ŌĆö until the fix landed, 017's own `description.json.lastUpdated` wouldn't update. The verification test had to prove the logic worked independent of any live folder's state.

`scripts/tests/workflow-canonical-save-metadata.vitest.ts` builds a synthetic spec-folder fixture inside a temp dir with `spec.md` + `description.json` stubs. It invokes the canonical save directly with `plannerMode: 'plan-only'` (the default), asserts `lastUpdated` gets written, asserts successive saves produce monotonically advancing timestamps, asserts `graph-metadata.json.derived.last_save_at` advances in plan-only mode. Only once the fixture test passed did we run the live `/memory:save`.

## MCP Caller Context

Session-resume auth needed caller identity. The MCP transport layer didn't expose any. Adding it via explicit argument threading would have touched ~30 handler signatures across the server surface.

`lib/context/caller-context.ts` adds `AsyncLocalStorage`-based propagation:

```ts
export interface MCPCallerContext {
  readonly sessionId: string | null;
  readonly transport: 'stdio' | 'sse' | 'ws' | 'unknown';
  readonly connectedAt: string;
  readonly callerPid?: number;
  readonly metadata: Record<string, unknown>;
}

export function runWithCallerContext<T>(ctx: MCPCallerContext, fn: () => T): T;
export function getCallerContext(): MCPCallerContext | null;
export function requireCallerContext(): MCPCallerContext;
```

`context-server.ts` wraps each `CallToolRequestSchema` dispatch with `runWithCallerContext`. Handlers that need caller identity call `getCallerContext()` at their boundary. Everything else stays untouched. [EVIDENCE: debb5d7a8]

The test suite covers 12 cases ŌĆö propagation across `await`, `setTimeout`, `Promise.all`, nested runs, concurrent requests. The nested case is important: if a handler internally runs another operation inside `runWithCallerContext`, the inner context correctly shadows the outer for the duration of the inner run.

## Session-Resume Auth

`handleSessionResume` used to accept `args.sessionId` as trusted input. A same-UID attacker with access to the process environment could pass a different session's ID and receive that session's cached summary ŌĆö the classic same-machine cross-session leak.

The fix reads `getCallerContext()?.sessionId` at handler entry. If `args.sessionId` is provided and doesn't match, the handler throws. If it matches, or if either is absent, the original flow runs.

For production deployments that already had workflows with mismatched session IDs, the rollout is staged:

```bash
export MCP_SESSION_RESUME_AUTH_MODE=permissive
```

Permissive mode logs the mismatch and continues. Default is strict. The flag is intended for canary verification before flipping the default elsewhere. [EVIDENCE: 87636d923]

## Unicode Sanitization

`shared/gate-3-classifier.ts:normalizePrompt` used to lowercase and collapse whitespace. That's enough for ASCII but not for homoglyph attacks. `"fileąĄ write"` (with Cyrillic `ąĄ` U+0435 instead of Latin `e`) would pass the Gate 3 trigger check ŌĆö the classifier was looking for `"file write"` literally. `"f\u00ADile write"` (with a soft hyphen between `f` and `i`) would also pass.

`normalizePrompt` now runs `.normalize('NFKC')` first, then strips `[\u00AD\u200B-\u200F\uFEFF]`. `sanitizeRecoveredPayload` applies the same normalization to each line before the system-role-prefix regex runs, so `"SYST\u0395M:"` (Greek epsilon U+0395) gets caught by the same pattern as `"SYSTEM:"`. [EVIDENCE: 1bd7856a9]

New test file at `scripts/tests/gate-3-classifier.vitest.ts` ŌĆö 5 unicode cases: Cyrillic `ąĄ`, soft hyphen, zero-width space, Greek `╬Ģ`, combined homoglyph + zero-width.

## Retry Budget

`partial_causal_link_unresolved` outcomes used to schedule `runEnrichmentBackfill` with no attempt counter. If the underlying condition was structurally non-retryable (malformed causal-link reference, missing predecessor), the backfill would loop indefinitely, pinning CPU on the same failure mode.

`lib/enrichment/retry-budget.ts` tracks attempts keyed on `(memoryId, step, reason)`:

```ts
export function shouldRetry(memoryId: number, step: string, reason: string): boolean;
export function recordFailure(memoryId: number, step: string, reason: string): RetryBudgetEntry;
export function clearBudget(memoryId?: number): void;
```

`post-insert.ts` calls `shouldRetry` before scheduling the backfill and `recordFailure` on each failure. After 3 attempts on the same `(memoryId, step, reason)` tuple, the budget denies further retries and emits a structured warning. `clearBudget(memoryId)` fires on successful enrichment completion. [EVIDENCE: 61f93c9bf]

The budget is in-memory per process. If durability across restarts matters, that's a follow-up.

## Phase 018-lite ŌĆö Closing the Review Findings

After Phase 017 landed, a 10-iteration `/spec_kit:deep-review` of the 32-commit scope converged at iteration 6 with 5 P1 + 15 P2 findings and verdict CONDITIONAL. ### Unicode fold, for real this time

`normalizeRecoveredPayloadLineForMatching` had been Greek-only ŌĆö `╬Ģ/╬Ą ŌåÆ E/e` was the entire fold. The review surfaced a 19-case adversarial matrix: `SYSTąĢM:` with Cyrillic `ąĢ` (U+0415), `SYSTąüM:` with `ąü`, `ASSISTąÉNT:` with `ąÉ` (U+0410), `DąĢVELOPER:` compound, `SYST├ēM:` with Latin-accented `├ē`, Cyrillic `č¢` inside `č¢gnore previous`, full-width `’╝│’╝╣’╝│’╝┤’╝ź’╝Ł:` for the NFKC regression guard. `foldUnicodeConfusablesToAscii(value)` now exports from `hooks/shared-provenance.ts` with NFKC + NFD + combining-mark strip + an explicit confusables table covering Cyrillic uppercase and lowercase (ąĪŌåÆC, ąĢŌåÆE, ą×ŌåÆO, ąÉŌåÆA, ąÜŌåÆK, ąźŌåÆX, ą£ŌåÆM, ąĀŌåÆP, ąóŌåÆT, ąÆŌåÆB, ąØŌåÆH, ąģŌåÆS, ąåŌåÆI, ąŻŌåÆY) and the lowercase mirror. `normalizeRecoveredPayloadLineForMatching` delegates to it. `normalizePrompt` in `shared/gate-3-classifier.ts` imports the same helper so Gate 3 trigger classification sees the same ASCII-folded string as the payload sanitizer ŌĆö no two normalization code paths, no drift. [EVIDENCE: a131c2193]

### Session-resume caller-context propagation

`handleSessionResume` had been reading `args.sessionId` from the request and falling back to `undefined` when absent ŌĆö even if the trusted transport context had one. A local-UID attacker couldn't forge a mismatch (the auth binding from v3.4.0.2's Wave B rejects that), but a legitimate caller omitting `args.sessionId` would silently lose their session identity and the cached-summary lookup would come back empty. The deep-review flagged this as a positive-binding gap: the handler had the identity, and it just dropped it.

The fix is one line. `claudeSessionId: typeof args.sessionId === 'string' ? args.sessionId : undefined` became `claudeSessionId: requestedSessionId ?? callerCtx?.sessionId ?? undefined`. The locked-in contract test at `session-resume-auth.vitest.ts:212-220` inverted: it used to assert no propagation; it now asserts `expectLatestScopeClauses('transport-session')` when `args.sessionId` is absent but the transport context has one. [EVIDENCE: 6238fdc3a]

The same commit adds a 3-5 line doc block at the guard site and at `buildCallerContext` in `context-server.ts`, explaining the stdio-vacuous-guard semantics ŌĆö why the same code is load-bearing over HTTP/WS (server-generated and validated sessionId) and a correctness-irrelevant no-op over stdio (single-UID subprocess with no cross-trust boundary). The SDK source path at `@modelcontextprotocol/sdk/protocol.js:280-316` is cited inline so future readers don't have to re-derive the asymmetry.

### Retry-budget API split + compact-cache contract lock

`clearBudget(memoryId?: number)` accepted an optional argument where undefined meant "clear everything" ŌĆö the kind of API shape that silently deletes all budgets when a caller forgets the argument. ### Reconsolidation discriminated union

`executeAtomicReconsolidationTxn` took a `mode: 'deprecate' | 'content_update'` string plus a grab-bag of optional fields (`deprecateStatus?`, `contentUpdateFields?`).
