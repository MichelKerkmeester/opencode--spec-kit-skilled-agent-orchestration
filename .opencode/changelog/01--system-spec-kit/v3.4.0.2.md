# v3.4.0.2 — Canonical Save, Actually

This patch closes the 28 findings that the deep-review of v3.4.0.1 surfaced, plus 14 more that a follow-up 7-iteration meta-research exposed. The headline discovery was subtler than v3.4.0.1's "silent failures" theme: the canonical save path had a dead-code guard that made every default `/memory:save` invocation write zero metadata. Not "eventually stale" — always stale, by design, since the guard was committed. The fix is two lines in `workflow.ts`. The cascade of everything else that depended on canonical save working — 17 folders with stale timestamps, research folders invisible to memory search, continuity validation with nothing to validate against — was infrastructure built on a broken foundation.

Three clusters dominate the remediation: **Cluster B** (canonical save surface hygiene, 7 findings), **Cluster D** (code-graph sibling readiness asymmetry, 1 → 6 handlers hardened), **Cluster E** (Copilot runtime observability gap, net-new). A fourth compound hypothesis (H-56-4: Copilot silent-failure cluster) was confirmed and resolved by bringing Copilot to compact-cache parity with Claude and Gemini.

> Scope: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/017-review-findings-remediation/` — 27 tasks across 4 waves, ~105h effort budget compressed to ~6h wall-clock via parallel cli-copilot gpt-5.4 high dispatches capped at 3 concurrent. 25 commits to main across ~45 files.

## The Headline: H-56-1

A 7-iteration deep-research pass meta-audited the review findings and confirmed what segment-2 research iter 56 proposed: the canonical save path was structurally a no-op on metadata freshness. The evidence:

1. **`scripts/core/workflow.ts:1259`** — `const ctxFileWritten = false;` hardcoded as a stub. The entire `description.json` memory-tracking block (lines 1261-1331, ~70 LOC) was wrapped in `if (ctxFileWritten)` — unreachable dead code.
2. **`scripts/core/workflow.ts:1333`** — `const shouldRunExplicitSaveFollowUps = options.plannerMode === 'full-auto';` gated `refreshGraphMetadata` (and two other follow-ups) on explicit full-auto mode.
3. **`scripts/memory/generate-context.ts:415`** — default `plannerMode: 'plan-only'`.

Net effect: every default `/memory:save` wrote zero metadata. The review-report claim "0 of 16 sibling folders have fresh `description.json.lastUpdated`" (R5-P1-001) was not drift. It was the **deterministic output** of a code path that never wrote anything since this guard shipped.

The fix (`aaf0f49a8`) was two lines: `ctxFileWritten = true` + remove the plan-only gate. Everything else in Phase 017 built on top of that first line.

H-56-1 was the root; the symptoms spanned the whole save pipeline. Fixed:

- **T-CNS-02** (`88063287b`): research folder backfill for `research/NNN-*/iterations/` metadata
- **T-W1-CNS-05** (`32a180bba`): continuity freshness validator wired into validate.sh --strict
- **T-CGC-02** (`db36c3194`): context.ts explicit `readiness_check_crashed` error branch (no more silent catch)
- **T-RBD-03** (`f42c5d3b6`): design-intent comments documenting the intentional rollup divergence between `post-insert.ts` executionStatus + `response-builder.ts` postInsertEnrichment
- **T-EVD-01** (`e40dff0bb`): evidence-marker bracket-depth parser + strict lint
- **T-CNS-03** (`176bad2b2`): one-time 17-folder `lastUpdated` sweep to clear the historical staleness

The EVIDENCE audit in Wave A prep (`7d85861a0`) scanned 1962 markers across 16 folders, distinguishing real unclosed markers from paren-in-content false positives, and rewrapped the genuinely malformed ones (210 across 016's checklist + plan + tasks).

### Cluster D — Code-Graph Sibling Asymmetry (NEW — 1 → 6 handlers)

v3.4.0.1 had hardened `handlers/code-graph/query.ts` with `canonicalReadiness + trustState + lastPersistedAt`. The deep-review found this was asymmetric: 6 sibling handlers in the same directory emitted zero such tokens. Phase 017:

- **T-CGC-01** (`4a154c555`): extracted 4 readiness helpers from `query.ts:225-300` into `lib/code-graph/readiness-contract.ts` — single canonical source for all 7 handlers
- **T-W1-CGC-03** (`f253194bf`): propagated the contract to `scan`, `status`, `context`, `ccc-status`, `ccc-reindex`, `ccc-feedback`. CCC trio ships stub rollout (`trustState: 'unavailable'` with `reason: 'readiness_not_applicable'`) where full-readiness semantics don't apply

Aligned to the existing 8-state `SharedPayloadTrustState` (`lib/context/shared-payload.ts:36`) rather than defining a new 4-state enum. Zero type proliferation.

### Cluster E — Copilot Runtime Observability Gap (NEW)

Compound-hypothesis H-56-4 confirmed: Copilot hook runtime was the only one of the three missing compact-cache + compact-inject. Claude and Gemini both read `payloadContract?.provenance.trustState` at session-prime entry; Copilot didn't.

- **T-W1-HOK-02** (`77da3013a`): extracted `hooks/shared-provenance.ts` from `hooks/claude/shared.ts` so all three runtimes could import from one canonical source. Also broke Gemini's transitive `../claude/shared.js` coupling.
- **T-W1-HOK-01** (`5923737c7`): new `hooks/copilot/compact-cache.ts` + extended `session-prime.ts`. Copilot now writes and reads `trustState: 'cached'` in the same shape as Claude/Gemini.

Required before autonomous Copilot iteration launches — per `feedback_copilot_concurrency_override.md`, Copilot is the Codex-fallback runtime capped at 3 concurrent dispatches, and H-56-4 observability blind spot would have accumulated per-session failures.

## MCP Caller Context + Session-Resume Auth

The review's R2-P1-001 flagged that `handleSessionResume` trusted `args.sessionId` without authentication — enabling cross-session cached-summary leakage for same-UID callers. Fix required adding MCP transport-layer caller identity, which didn't exist.

Rather than a 30-file handler signature refactor, Phase 017 went with AsyncLocalStorage:

- **Lane B4a** (`debb5d7a8`): new `lib/context/caller-context.ts` with `runWithCallerContext` + `getCallerContext`. Transport wraps each request in the context; handlers access via `getCallerContext()` at the boundary — zero signature churn. 12 tests cover propagation across `await`, `setTimeout`, `Promise.all`, nested runs, readonly immutability.
- **Lane B4b** (`87636d923`): `handleSessionResume` reads `callerCtx.sessionId`, rejects mismatched `args.sessionId`. `MCP_SESSION_RESUME_AUTH_MODE=permissive` env flag logs warnings instead of rejecting — enables canary rollout. Default = strict. 8 test cases.

## Unicode Hardening (Cluster C)

Three sanitization surfaces were ASCII-only:

- **T-SAN-01** (`1bd7856a9`): `shared/gate-3-classifier.ts:normalizePrompt` — `.normalize('NFKC')` + strip `[\u00AD\u200B-\u200F\uFEFF]` zero-width/soft-hyphen characters before `.toLowerCase()`. Cyrillic `е` (U+0435) homoglyph no longer hides `file_write` triggers.
- **T-SAN-02** (same commit): `hooks/claude/shared.ts:sanitizeRecoveredPayload` — mirror NFKC pass so `"SYST\u0395M:"` (Greek epsilon) gets stripped by the system-role-prefix regex.
- **T-SAN-03** (same commit): new `scripts/tests/gate-3-classifier.vitest.ts` — 5+ unicode test cases (Cyrillic `е`, soft hyphen, zero-width space, Greek `Ε`, combined homoglyph+zero-width).
