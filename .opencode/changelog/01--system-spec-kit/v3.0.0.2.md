## Highlights

### Category: Bug Fix

- **Gates 1-3 bypass in processPreparedMemory** -- Added `qualityGateMode === 'warn-only'` conditional checks to the V-rule rejection (line 333), sufficiency rejection (line 350), and template contract rejection (line 354) in `memory-save.ts`. When processing spec documents, these gates now log warnings and continue instead of returning `status: 'rejected'`. Gate 4 bypass (v3.0.0.1) was already in place.
- **Verified: 0 rejected spec docs** -- Full force re-index (`memory_index_scan({ force: true, incremental: false })`) processed 1,929 files: 1,670 indexed, 185 updated, 0 rejected spec docs (was 1,807). Memory files still subject to all 4 quality gates (24 rejected, confirming no regression).

### Category: Command Consolidation (spec 012-command-alignment)

The memory command surface was restructured from 5 commands to 4, aligning the live 33-tool MCP surface under cleaner command homes. This work was tracked in spec `012-command-alignment`.

- **`memory:analyze` renamed to `memory:search`** -- The former `analyze` command became `/memory:search`, now the unified home for retrieval, `memory_quick_search`, analysis/eval tooling (`task_preflight`, `task_postflight`, `eval_run_ablation`, `eval_reporting_dashboard`), causal graph tools (`memory_causal_link`, `memory_causal_stats`, `memory_causal_unlink`, `memory_drift_why`), and `memory_get_learning_history`. The `analyze.toml` and `analyze.md` files deleted across all runtimes (.opencode, .claude, .agents, .codex).
- **`memory:shared` merged into `memory:manage shared`** -- Shared-memory lifecycle operations (`shared_space_upsert`, `shared_space_membership_set`, `shared_memory_status`, `shared_memory_enable`) moved under `/memory:manage shared` as a sub-surface. The standalone `shared.toml` and `shared.md` files deleted across all runtimes.
- **`memory:manage` expanded** -- Now owns 19 tools across maintenance, mutation, checkpoint, ingest, and shared-memory lifecycle layers (L3-L5, L7).
- **`/spec_kit:resume` session recovery** -- Session recovery and crash/interrupted-session continuation formally documented as owned by `/spec_kit:resume`, not the memory command suite. Relies on `/memory:search` retrieval surfaces and `/memory:manage` shared inventory.
- **33-tool coverage matrix** -- The `README.txt` in `.opencode/command/memory/` now maps all 33 MCP tools to their command homes with 0 uncovered tools.
