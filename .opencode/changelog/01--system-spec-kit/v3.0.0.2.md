## [**3.0.0.2**] - 2026-03-28

Completes the v3.0.0.1 fix for spec document indexing and simplifies the memory command structure from 5 commands down to 4.

> Spec folder: `specs/system-spec-kit/022-hybrid-rag-fusion/022-spec-doc-indexing-bypass/`

---

## What Changed

### Spec document indexing: the rest of the fix

The v3.0.0.1 release fixed one of four quality gates that was rejecting spec documents during indexing. But three earlier gates were still rejecting them before they even reached the fixed gate. The result: 1,807 spec documents were still being rejected.

The quality gate for saving memories has 4 checkpoints, in order:

1. **Validation rules check** -- Does the content follow required formatting rules?
2. **Sufficiency check** -- Is there enough meaningful content to be worth storing?
3. **Template contract check** -- Does the content match its expected template structure?
4. **Metadata check** -- Does the file have required fields like trigger phrases? (Fixed in v3.0.0.1)

All four gates now switch to "warn-only" mode for spec documents, meaning they log concerns but do not block indexing. Memory files are still fully validated through all 4 gates (24 memory files were correctly rejected during the verification re-index, confirming no regression).

**After the fix:** A full re-index processed 1,929 files with 0 spec documents rejected (previously 1,807 were rejected).

### Memory commands simplified

The system had 5 separate commands for working with memories. Two of them overlapped significantly with others, so they were merged:

- **`/memory:analyze` merged into `/memory:search`** -- Search is now the single command for finding memories, running evaluations, exploring causal graphs, and viewing learning history. The separate "analyze" command is gone.
- **`/memory:shared` merged into `/memory:manage shared`** -- Shared-memory operations (creating shared spaces, managing memberships, checking status) now live under the "manage" command as a sub-section. The separate "shared" command is gone.

The remaining 4 commands are:
- `/memory:search` -- Find and analyze memories (now covers 33 tools)
- `/memory:manage` -- Database maintenance, shared-memory lifecycle, checkpoints, and bulk operations
- `/memory:save` -- Save conversation context to memory
- `/memory:learn` -- Create and manage constitutional memories (always-present rules)

### Documentation updates

- Updated spec folders, feature catalog entries, and manual testing scenarios to reflect the command consolidation.
- Updated agent definitions across all 4 runtime directories to reference the new command names.
- Remapped evaluation test IDs for consistency.
