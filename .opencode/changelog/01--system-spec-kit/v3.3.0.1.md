This patch tightens the System Spec Kit memory runtime around one of its most expensive graph-maintenance paths. Save-time entity linking now scopes itself to the memory that was just written instead of rescanning the full cross-document entity graph on every save, and the scheduled graph refresh path is now wired to a real runtime callback instead of falling through to a warning-only no-op. The goal is to make the workflow easier to follow and easier to trust.

> Scope: direct runtime patch in `system-spec-kit` memory/causal-graph code paths (no spec folder used for this change)

---

## Memory Graph Runtime

#### Incremental linking

Every memory save still called the full `runEntityLinking(...)` pipeline. That meant one new memory could trigger a corpus-wide entity-linking scan even when only the saved memory's new entity set needed cross-folder linking. Save-time linking now uses a memory-scoped incremental path. The linker resolves entities for the saved memory, expands them through `entity_catalog` aliases, finds only matching memories in other spec folders, and creates just those cross-document edges. The full linker still exists as the broad fallback path, but normal save-time work is now bounded to the changed memory.

&nbsp;

#### Scheduled refresh

The graph lifecycle already supported scheduled refreshes, but the runtime bootstrap had not registered a real refresh callback. In practice, scheduled refresh mode could queue work and then log that no refresh function was registered. The MCP server now registers a global refresh function during startup. When scheduled refresh runs, it recomputes the dirty node set against the live database, clears the dirty markers after success, and logs the applied refresh with dirty-node and updated-edge counts.

&nbsp;

#### Coverage

The new incremental linker path needed proof that it stayed narrowly scoped and still handled alias-normalized entity matches like `TF-IDF` versus `tf idf`. Added focused regression tests covering memory-scoped match discovery, alias-aware incremental linking, and save-scope edge creation so the optimized path is guarded independently of the older full-linking pipeline.

## Upgrade

No migration is required. This patch is runtime-only and backward compatible. The main effect is lower save-time graph work for memory writes plus a scheduled refresh path that now executes real graph maintenance instead of warning-only fallback behavior.
