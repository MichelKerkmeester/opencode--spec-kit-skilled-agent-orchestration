## Concurrency Guards (4)

### 1. Checkpoint restore now blocks mutations during rebuild

**Problem:** restoreCheckpoint() performs a large restore transaction then rebuilds derived tables, but nothing prevented ordinary writes (memory_save, index_scan, bulk_delete) from entering during or after the restore. Live writes during rebuild could create inconsistent derived state.

**Fix:** A module-level restore barrier is set before the restore transaction and cleared after all post-restore rebuilds complete. Mutation handlers fail-fast with E_RESTORE_IN_PROGRESS if the barrier is held.

### 2. Shared-space creation race eliminated

**Problem:** Two concurrent create requests for the same space could both see no existing space, both report created=true, and both bootstrap owner membership -- upgrading a second caller to owner incorrectly.

**Fix:** Creation is now detected from the INSERT result (INSERT ... ON CONFLICT DO NOTHING RETURNING), not from a pre-read SELECT. Owner bootstrap only runs when the insert actually created the row.

### 3. Reconsolidation stale-merge guard

**Problem:** executeMerge() reads a predecessor row, then awaits embedding generation (slow). If another writer modifies or deletes that memory during the gap, the merge proceeds with stale data.

**Fix:** Before the merge transaction commits, the predecessor row is reloaded and compared via content_hash/updated_at. If changed: merge aborts with predecessor_changed. If deleted/archived: predecessor_gone.

### 4. Scan cooldown converted to atomic lease

**Problem:** The index scan cooldown was a TOCTOU check -- read timestamp, check if enough time passed, then set new timestamp. Concurrent scans could both pass the check.

**Fix:** acquireIndexScanLease() reserves the scan slot atomically in a transaction. completeIndexScanLease() converts to last_index_scan on completion. Crashed scans auto-expire.

## Search Performance (7)

### 5. Fallback pipeline runs enrichment once

Fallback search no longer reruns the entire pipeline (reranking, co-activation, token truncation) for each fallback tier. Candidates are collected and fused per-tier, but expensive enrichment runs only once on the final merged set.

### 6. Token estimation cached per request

truncateToBudget() caches token estimates in a Map keyed by result.id. Each result is sized once, reused for both total-budget and greedy admission passes. Field-based estimator replaces JSON.stringify.

### 7. BM25 demoted to opt-in; FTS5 is default

In-memory BM25 is now behind an explicit ENABLE_BM25 flag. FTS5 is the default lexical engine. Synchronous full rebuild replaced with incremental row-ID-based maintenance.

### 8. Graph degree scoring batched

N+1 per-candidate degree queries replaced with a single SQL (WHERE source_id IN / target_id IN GROUP BY). Global max cached alongside per-node cache.

### 9. Graph FTS query rewritten as CTE

OR-based join shape (causing duplicates + JS dedup) replaced with a CTE that materializes matched rowids once, then UNION ALL for source-side and target-side edge lookups. FTS-table availability cached per DB instance.

### 10. Adaptive fusion single-pass weights

New getAdaptiveWeights(intent, documentType) helper avoids running standard fuse when only weights are needed. hybridSearchEnhanced() builds fusionLists once.

### 11. MMR embedding cache

Vector channel returns embedding buffers for top-K hits. MMR reranking reuses cached embeddings instead of re-fetching from DB. reranked.find() replaced with Map lookup.

## SQLite Query Optimization (6)

### 12. Save-path dedup queries rewritten

Nullable OR predicates that defeated indexes replaced with dynamically assembled exact-match queries. Two new composite partial indexes: idx_save_canonical_path and idx_save_parent_content_hash_scope.

### 13. Trigger cache uses partial index

New idx_trigger_cache_source partial index. Prepared loader statement cached per DB connection.

### 14. Co-activation lookups batched

N+1 per-related-ID SELECT replaced with single WHERE id IN batch. getCausalNeighbors() rewritten as single SQL + JOIN. Stage-2 fusion precomputes neighbor counts.

### 15. Temporal-contiguity bounded range query

Function-wrapped created_at (defeating indexes) replaced with bounded range query on created_at first, then time_delta_seconds on narrowed set. New idx_spec_folder_created_at index.

### 16. Causal-link resolution index-first

Leading-wildcard LIKE replaced with exact canonical_file_path/file_path equality first. Fuzzy matching reserved for fallback. Batch reference resolution.

### 17. Working-memory indexes added

New idx_wm_session_focus_lru and idx_wm_session_attention_focus. SELECT COUNT(*) existence probe removed from upsertExtractedEntry(); relies on ON CONFLICT.

## Error Recovery Hardening (4)

### 18. Chunked save PE finalize is transactional

PE supersede logic now runs in a single finalize transaction. On failure, compensating cleanup deletes the newly created parent and child rows before returning error. No more partial commits where caller sees failure but data is saved.

### 19. Safe-swap old-child deletion in transaction

Old-child deletion moved into finalization transaction. Parent_id nulled before bulk delete, making it rollback-safe.

### 20. BM25 rollback on all-chunks-failed

Parent BM25 mutation delayed until at least one chunk succeeds. On all-chunks-failed rollback, old parent BM25 state preserved.

### 21. BM25 repair flag for durable retry

When BM25 repair fails after DB merge commit, a bm25_repair_needed flag is persisted on the merged row so a future reconciler can retry.
