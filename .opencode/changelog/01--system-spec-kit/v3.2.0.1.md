## [**3.2.0.1**] - 2026-04-02

Post-release hardening pass on v3.2.0.0. Closes a path traversal gap in graph enrichment, hardens the compact-recovery payload sanitizer against prompt injection patterns, stops query-time graph reads from triggering write-path indexing, and gives session bootstrap the full resume payload instead of a minimal stub. Also trims duplicate detection I/O by pre-filtering candidates on file size and fixes a broken symlink that prevented the shared package from resolving.

> Spec folders: `.opencode/specs/system-spec-kit/023-hybrid-rag-fusion-refinement/` and `.opencode/specs/system-spec-kit/024-compact-code-graph/`

---

## What Changed

---

### Security

- **Graph enrichment path traversal blocked** -- `normalizeGraphFilePath` now checks that the resolved path stays inside the workspace root. Paths that escape via `../` or absolute jumps are rejected before reaching any file-system operation.

- **Compact-recovery payload sanitizer hardened** -- The sanitizer that strips system-instruction lines from recovered transcripts now catches broader injection patterns: `[developer]:`, `Role:`, `Ignore previous instructions`, `## Instructions`, and markdown headings that name system roles. Previously only a single regex covered these cases.

- **V-rule optional bypass removed** -- The `SPECKIT_VRULE_OPTIONAL` environment variable could skip memory quality validation entirely. That escape hatch is gone — validation now always runs or reports unavailable honestly.

---

### Code Graph

- **Query reads no longer trigger auto-indexing** -- `code_graph_query` now calls `ensureCodeGraphReady({ allowInlineIndex: false })`. Read-path callers get a freshness report without accidentally triggering a write-heavy reindex mid-query. Scan and explicit reindex paths still auto-index normally.

- **Edge type resolution extracted and included in results** -- A new `resolveRequestedEdgeType()` helper resolves the edge filter from the explicit `edgeType` argument or falls back to the operation prefix (`calls_*` → CALLS, `imports_*` → IMPORTS). Query responses now include the resolved `edgeType` in output so callers can see which filter was applied.

---

### Session Recovery

- **Bootstrap returns full resume instead of minimal stub** -- `session_bootstrap` was passing `minimal: true` to session_resume, which stripped useful context. It now returns the full resume payload and reports completeness as `'full'` instead of `'minimal'`.

- **Server instructions recommend `session_bootstrap()`** -- The recovery hint in server instructions now points to the canonical composite tool instead of `session_resume()`.

- **Compact-recovery description corrected** -- The recovery instruction text now says "cached compact brief" instead of the inaccurate "3-source merge (Memory + Code Graph + CocoIndex)".

---

### Saving Memories

- **Duplicate detection pre-filters by file size** -- Overlap detection was reading up to 20 prior context files and hashing each one. It now checks the 12 most recent candidates and skips any whose file size doesn't match the new content's byte length before doing the full hash comparison. Same correctness, less I/O.

- **Session ID passthrough** -- When the caller provides an explicit session ID, it now propagates through the workflow into the session-data collector instead of being silently replaced by a generated one.
