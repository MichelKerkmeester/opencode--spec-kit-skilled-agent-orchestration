## Security

#### Reconsolidation scope-boundary fix (P0)

The save-time reconsolidation path could merge records across tenant, user, agent, and session boundaries. The merged survivor lost governance metadata. The fix adds scope-field validation before any merge decision, aborts on mismatch, and preserves both records. Regression tests cover the cross-scope rejection path.

---

## Bug Fixes

#### Path-boundary hardening

Validators now reject absolute paths that try to escape the project folder. `graph-metadata-parser`, `config.ts`, `skill-graph/scan.ts`, and `resume-ladder.ts` all got `path.resolve()` + `startsWith(allowedRoot)` guards. Symlink resolution in database paths was tightened.

&nbsp;

#### Workflow quality score override

`workflow.ts` was reporting a hardcoded quality score of 100 instead of the pipeline's computed value. Fixed to use the actual `filterStats.qualityScore`.

&nbsp;

#### Context-server timeout

The 250ms dispatch graph-context timeout was not actually aborting the query. Added an `aborted` flag so the synchronous build call is properly abandoned when the timeout fires.

&nbsp;

#### Frontmatter YAML comment stripping

List-style frontmatter values kept trailing YAML comments as data. `parseSectionValue` now strips inline comments from both scalars and list items. Quoted trigger phrases that generated invalid YAML are now properly escaped.

&nbsp;

#### 14 YAML tool references

10 command YAML files referenced `spec_kit_memory_memory_save(...)` instead of the live `memory_save` tool name. All replaced.

&nbsp;

#### Trigger phrase sanitizer order-sensitivity

`sanitizeTriggerPhrases()` produced different results depending on input order. Fixed by sorting candidates by comparison-key length before the shadow-removal loop.

&nbsp;

#### Error handling: silent failures surfaced

`session_bootstrap`, `extractSpecTitle`, `session-prime`, `skill_graph_compiler`, and `skill_advisor_runtime` all had silent failure paths that reported "healthy" while swallowing errors. Each now propagates or logs the failure.

&nbsp;

#### Checkpoint scoped restore DB handle

`checkpoints.ts` used the wrong database handle for causal edges during scoped restores. Fixed to use the passed `database` parameter.

## Features

#### Root-level review/research folder convention

A shared `resolveArtifactRoot(specFolder, mode)` function walks up the spec tree to find the root parent. Both reducers, all 4 command YAMLs, and all 8 agent definitions (4 runtimes x 2 skills) now use it. Review and research artifacts always land at the spec root with phase subfolders for child targets.

&nbsp;

#### Post-save reviewer wired into pipeline

`post-save-review.ts` was disconnected from the production workflow. Now called as Step 11.75 in full-auto mode, non-blocking.

#### Deep-review iteration schema updated
