## [**2.2.31.0**] - 2026-02-21

This release fix Constitutional Tier Mis-assignment in Memory Parser -- Fixed extractImportanceTier() matching importanceTier: 'constitutional' from HTML template comments instead of the actual YAML metadata block, causing 595 SESSION SUMMARY memories to be incorrectly classified as constitutional tier. It focuses on making the instructions and release notes easier for non-developers to follow.

---

## What Changed

---

### HTML comment stripping

- Need: People needed clearer guidance about HTML comment stripping.
- Change: HTML comment stripping: Added content.replace to extractImportanceTier() before regex matching. The function previously scanned the entire file content including HTML comments, where the context template instructional comment on line 77 (importanceTier: 'constitutional') was matched before the actual YAML metadata block.
- Why it matters: This matters because the release notes and instructions are easier to read and trust.

---

### Bulk re-tier

- Need: There was a reliability problem around Bulk re-tier.
- Change: Bulk re-tier: Updated 595 SESSION SUMMARY entries from constitutional to normal tier in the SQLite database. Only 2 legitimate constitutional memories (IDs 567, 568) were preserved.
- Why it matters: This matters because it reduces confusing failures and makes the system more dependable.

---

### Parsing README

- Need: People needed clearer guidance about Parsing README.
- Change: Parsing README: Updated extractImportanceTier function description to document the HTML comment stripping behavior.
- Why it matters: This matters because the release notes and instructions are easier to read and trust.
