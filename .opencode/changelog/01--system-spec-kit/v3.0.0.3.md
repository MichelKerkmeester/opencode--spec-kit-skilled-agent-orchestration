## [**3.0.0.3**] - 2026-03-28

The deep-research system stored its working files (iteration notes, strategies, state) in inconsistent locations across the codebase. Some ended up at the root of spec folders, some inside `scratch/`, and review files had no standard home. This release moves everything into a clean, predictable structure -- `research/` for research artifacts and `review/iterations/` for review artifacts -- across 790 files, removing 47,315 lines of legacy layout. The goal is to make the workflow easier to follow and easier to trust.

> Spec folder: `.opencode/specs/03--commands-and-skills/035-sk-deep-research-path-migration/` (Level 3)

---

## What Changed

---

### New file organization for research and review

**Problem:** Research files (iteration logs, config, strategy, state) were scattered between the spec folder root and a `scratch/` folder with no clear convention. Review iteration files sat alongside the final review report with no separation.

**Fix:** All research artifacts now live under `research/` (e.g., `research/research.md`, `research/iterations/iteration-001.md`). Review iterations live under `review/iterations/`, separate from the final `review/review-report.md`. The system's indexing logic now knows to treat these iteration folders as working artifacts, not as formal spec documents.

---

### Commands and workflows updated

All deep-research commands and workflow files now use the new paths. This includes the main command file, the 4 workflow variations (auto, confirm, review:auto, review:confirm), and the workflow wrapper files used by the `.agents` system.

---

### Documentation synchronized across all 5 runtimes

The deep-research and deep-review agent definitions are now consistent across OpenCode, Claude, Codex, Gemini, and Agents runtimes -- previously they could reference different file paths depending on which system you were using.

---

### Migration bug found and fixed

The migration script initially had a bug where references pointing to files within the same spec folder would get rewritten into an incorrectly nested form (e.g., doubling up the path). This was caught and fixed, with a second pass cleaning up the 300 files affected.

## Upgrade

No manual migration required. The migration utility has already moved all tracked files to the new layout. If you maintain custom spec folders outside this repo that use the old layout (root-level `research.md`, `scratch/iteration-*`, or direct `review/iteration-*` paths), move them to `research/` and `review/iterations/` before relying on the updated system.
