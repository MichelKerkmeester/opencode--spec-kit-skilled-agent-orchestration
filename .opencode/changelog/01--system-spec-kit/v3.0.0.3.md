## [**3.0.0.3**] - 2026-03-28

**Deep-research path migration** -- Canonicalizes the research packet contract across **790 files**, moving research-mode state to a dedicated `research/` packet root, normalizing review iterations under `review/iterations/`, and retiring the legacy root-plus-scratch layout with **47,315 lines removed**.

> Spec folder: `.opencode/specs/03--commands-and-skills/035-sk-deep-research-path-migration/` (Level 3)

---

## Highlights

### Category: Architecture

- **Canonical `research/` packet root** -- Research mode now writes `research/research.md`, `research/iterations/iteration-NNN.md`, and related config/state/strategy files under a single canonical packet instead of mixing root files with `scratch/`.
- **Review iteration normalization** -- Review mode keeps the final verdict at `review/review-report.md` while moving iteration artifacts into `review/iterations/`.
- **Iteration folders excluded from spec indexing** -- Helper and routing logic now treat `research/iterations/` and `review/iterations/` as working artifacts rather than canonical spec documents.

### Category: Commands

- **Deep-research command updated** -- `.opencode/command/spec_kit/deep-research.md` and all 4 YAML workflows (`auto`, `confirm`, `review:auto`, `review:confirm`) now use the canonical packet paths.
- **`.agents` wrappers aligned** -- `plan.toml`, `complete.toml`, `phase.toml` no longer refer to stale `/spec_kit:research` or `@research` surfaces.
- **Workflow shim retargeted** -- `.agents/workflows/spec_kit_research.md` now points at the renamed command entrypoint.

### Category: Documentation

- **sk-deep-research v1.2.2.0** -- Skill docs, references, assets, and manual testing playbook updated to the canonical packet contract.
- **Runtime agent parity** -- Deep-research and deep-review guidance synchronized across all 5 runtimes (`.opencode`, `.claude`, `.codex`, `.gemini`, `.agents`).
- **system-spec-kit helper logic** -- Packet discovery, parsing, frontmatter migration, folder scoring, and the new `spec-doc-paths.ts` module aligned with the new layout.

### Category: Fixes

- **Migration utility regression fixed** -- Same-packet research references were being rewritten into duplicated nested form; utility now collapses same-packet duplicate references safely.
- **Corpus fully migrated** -- 347 files rewritten (initial pass), 300 files cleaned (post-fix pass), zero residual legacy paths confirmed via sweeps.

## Upgrade

No manual migration required. The in-repo migration utility has already moved all tracked packets to the new layout. If you maintain custom external spec folders using root `research.md`, `scratch/iteration-*`, or direct `review/iteration-*` paths, migrate them to `research/` and `review/iterations/` before relying on the updated contract.
