This release spec folder level upgrade script - New 1,490+ LOC bash script that upgrades existing spec folders between documentation levels (L1toL2, L2toL3, L3toL3+) with full user content preservation, dry-run preview, template-based generation, and idempotent operation. It focuses on making the instructions and release notes easier for non-developers to follow. The goal is to make the workflow easier to follow and easier to trust.

## What Changed

#### Multi-level upgrade support

This release updates Multi-level upgrade support. L1toL2, L2toL3, L3toL3+, and skip-level chains (L1toL3 via sequential L1toL2 then L2toL3). This matters because the release is clearer and easier to work with.

&nbsp;

#### Dry-run mode

This release updates Dry-run mode. Safe preview of all planned changes without modifying files (--dry-run). This matters because the release is clearer and easier to work with.

&nbsp;

#### Template-based file generation

People needed clearer guidance about Template-based file generation. New files created from SpecKit addendum templates. This matters because the release notes and instructions are easier to read and trust.

&nbsp;

#### Metadata and marker updates

This release updates Metadata and marker updates. SPECKIT_LEVEL markers and metadata table level fields updated across all modified files. This matters because the release is clearer and easier to work with.

&nbsp;

#### Idempotent operation

This release updates Idempotent operation. Dual-pattern detection prevents duplicate sections on re-run. This matters because the release is clearer and easier to work with.

&nbsp;

#### JSON output mode

This release updates JSON output mode. Machine-parseable output for toolchain integration (--json). This matters because the release is clearer and easier to work with.

&nbsp;

#### 1,490+ LOC, 22 functions

The team needed stronger proof that 1,490+ LOC, 22 functions worked as expected. 6-phase architecture: input validation, level detection, backup creation, file upgrade, marker updates, output generation. This matters because the team has clearer proof that the change works.

&nbsp;

#### Bash 3.2+ compatible

The project needed a cleaner setup for Bash 3.2+ compatible. Works on stock macOS without requiring brew-installed bash 5; no associative arrays or bashisms. This matters because setup and maintenance are easier to manage.

&nbsp;

#### Portable sed handling

This release updates Portable sed handling. _sed_inplace() wrapper abstracts macOS vs GNU sed -i differences. This matters because the release is clearer and easier to work with.

&nbsp;

#### Fuzzy heading matching

There was a reliability problem around Fuzzy heading matching. Finds approximate insertion points when users have modified standard headings, with end-of-file fallback. This matters because it reduces confusing failures and makes the system more dependable.

&nbsp;

#### Auto-cleanup

Auto-cleanup was no longer needed in its old form. Keeps 3 most recent backups, removes older ones automatically. This matters because the project becomes smaller and easier to maintain.
