## [**2.2.29.0**] - 2026-02-20

This release records the first real-world validation pass for the older SGQS navigation system. It explains where that navigation approach still works, where it falls short, and why future improvements were needed.

---

## What Changed

### 1. A lightweight test harness was created

- Need: The team needed a simple way to run navigation tests without pulling in extra dependencies.
- Change: This release added a small test harness that runs SGQS queries and saves the results in structured JSON.
- Why it matters: This makes repeat testing easier and gives the team a reliable baseline for future comparisons.

### 2. The system was tested through multiple user perspectives

- Need: A navigation tool can look fine in theory but still fail real users with different goals.
- Change: This release ran the legacy navigation flow across several developer personas and a broad set of scenarios.
- Why it matters: This shows how the system performs in real use instead of only in narrow technical checks.

### 3. Basic stability held up, but quality was not strong enough

- Need: The team needed to know whether the old system was merely imperfect or fundamentally unstable.
- Change: The tests showed that malformed queries were handled safely, but overall usefulness still landed below the project’s expected quality bar.
- Why it matters: This makes the outcome clearer: the system was stable enough to inspect, but not strong enough to keep as-is.

### 4. Several core gaps were identified

- Need: The team needed a concrete list of the main reasons navigation quality was weak.
- Change: This release documented the biggest problems, including missing links between nodes, case-sensitive matching that hid valid results, weak coverage for common domain terms, and poor skill-advisor accuracy.
- Why it matters: This turns a vague “it feels weak” result into a practical improvement list.

### 5. Cross-cutting results were captured for follow-up work

- Need: A one-off test report is hard to act on unless the biggest patterns are summarized clearly.
- Change: This release recorded the main navigation, discovery, vocabulary, and resilience outcomes so the next phase could focus on the right fixes.
- Why it matters: This gives future work a clearer starting point instead of forcing the next team to re-run the investigation from scratch.
