## [**2.3.0.1**] - 2026-02-20

This release unified Memory Dashboard Visual Design System across all 5 memory commands with 10 shared output components: MEMORY:<COMMAND> headers, → Title Case section labels, █░ bar charts, PASS/WARN/FAIL indicators, and consistent key-value formatting. Approximately 29 output templates standardized. It focuses on making the instructions and release notes easier for non-developers to follow.

---

## What Changed

### 1. Unified command headers

- Need: There was a reliability problem around Unified command headers.
- Change: Unified command headers: All memory commands now use MEMORY:<COMMAND> ALL CAPS format with thick ━━━ separator line, replacing 5 different header styles (box-drawing, emoji-prefixed, mixed case).
- Why it matters: This matters because it reduces confusing failures and makes the system more dependable.

### 2. Section labels

- Need: There was a reliability problem around Section labels.
- Change: Section labels: Introduced → Title Case ──────────── count arrow-prefixed section labels across all commands, replacing ad-hoc dividers and mixed formatting.
- Why it matters: This matters because it reduces confusing failures and makes the system more dependable.

### 3. Bar charts

- Need: This release updates Bar charts.
- Change: Bar charts: Added █░ block character visualizations for scores, progress, tier distribution, confidence, and stability across all numeric displays.
- Why it matters: This matters because the release is clearer and easier to work with.

### 4. Action menus

- Need: This release updates Action menus.
- Change: Action menus: Standardized interactive prompts to [key] verb format after thin ─── separator, replacing inconsistent [a]ction | [b]ction pipe-separated styles.
- Why it matters: This matters because the release is clearer and easier to work with.

### 5. ASCII indicators

- Need: This release updates ASCII indicators.
- Change: ASCII indicators: Replaced all emoji status indicators (✓, ⚠, ❌, 🔄, ✅) with plain text PASS / WARN / FAIL words.
- Why it matters: This matters because the release is clearer and easier to work with.

### 6. 10-component design system

- Need: People needed clearer guidance about 10-component design system.
- Change: 10-component design system: Defined shared output vocabulary (command header, section label, key-value pair, bar chart, result item, status line, action menu, indicators, inline lists, empty state).
