## [**2.3.0.1**] - 2026-02-20

This release unified Memory Dashboard Visual Design System across all 5 memory commands with 10 shared output components: MEMORY:<COMMAND> headers, to Title Case section labels, █░ bar charts, PASS/WARN/FAIL indicators, and consistent key-value formatting. Approximately 29 output templates standardized. It focuses on making the instructions and release notes easier for non-developers to follow. The goal is to make the workflow easier to follow and easier to trust.

---

## What Changed

---

### Unified command headers

**Problem:** There was a reliability problem around Unified command headers.

**Fix:** Unified command headers: All memory commands now use MEMORY:<COMMAND> ALL CAPS format with thick ━━━ separator line, replacing 5 different header styles (box-drawing, emoji-prefixed, mixed case). This matters because it reduces confusing failures and makes the system more dependable.

---

### Section labels

**Problem:** There was a reliability problem around Section labels.

**Fix:** Section labels: Introduced to Title Case ──────────── count arrow-prefixed section labels across all commands, replacing ad-hoc dividers and mixed formatting. This matters because it reduces confusing failures and makes the system more dependable.

---

### Bar charts

**Problem:** This release updates Bar charts.

**Fix:** Bar charts: Added █░ block character visualizations for scores, progress, tier distribution, confidence, and stability across all numeric displays. This matters because the release is clearer and easier to work with.

---

### Action menus

**Problem:** This release updates Action menus.

**Fix:** Action menus: Standardized interactive prompts to [key] verb format after thin ─── separator, replacing inconsistent [a]ction | [b]ction pipe-separated styles. This matters because the release is clearer and easier to work with.

---

### ASCII indicators

**Problem:** This release updates ASCII indicators.

**Fix:** ASCII indicators: Replaced all emoji status indicators (✓, ⚠, ❌, 🔄, ✅) with plain text PASS / WARN / FAIL words. This matters because the release is clearer and easier to work with.

---

### 10-component design system

**Problem:** People needed clearer guidance about 10-component design system.

**Fix:** 10-component design system: Defined shared output vocabulary (command header, section label, key-value pair, bar chart, result item, status line, action menu, indicators, inline lists, empty state). This matters because the release notes and instructions are easier to read and trust.
