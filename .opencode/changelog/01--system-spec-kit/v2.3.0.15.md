## Highlights

### Category: Database and Schema Safety (B1-B4)

- **B1:** Fixed `reconsolidation.ts` referencing non-existent `frequency_counter` column — replaced with `importance_weight` merge logic.
- **B2:** Moved DDL statements before `BEGIN` in `checkpoints.ts` — SQLite silently auto-commits on DDL, corrupting transaction boundaries during restore.
- **B3:** Fixed SQL operator precedence in `causal-edges.ts` — `WHERE a AND b OR c` → `WHERE a AND (b OR c)`.
- **B4:** Added `.changes > 0` guard on UPDATE statements in `memory-save.ts` — previously reported success on zero-row updates.

### Category: Scoring and Ranking Corrections (C1-C4)

- **C1:** Clamped composite score to [0,1] with `Math.max(0, Math.min(1, composite))` in `composite-scoring.ts`.
- **C2:** Removed citation fallback chain (was conflating recency with citation authority); returns 0 when no citation data.
- **C3:** Changed `UNION ALL` to `UNION` in recursive CTE in `causal-boost.ts` — prevents cycle amplification.
- **C4:** Replaced naive binomial coefficient with log-space computation in `ablation-framework.ts` — prevents overflow for n>50.

### Category: Search Pipeline Safety (D1-D3)

- **D1:** Summary candidates in Stage 1 now pass through `minQualityScore` filter in `stage1-candidate-gen.ts`.
- **D2:** Unified FTS5 tokenization via shared `sanitizeQueryTokens()` array across `bm25-index.ts` and `sqlite-fts.ts`.
- **D3:** Lowered `QUALITY_FLOOR` from 0.2 to 0.005 in `channel-representation.ts` — RRF scores (~0.01-0.03) were being filtered out.

### Category: Guards and Edge Cases (E1-E2)

- **E1:** Fixed inner loop `j = i + 1` in `temporal-contiguity.ts` — eliminated double-counting of pair boosts.
- **E2:** Removed wrong-memory fallback in `extraction-adapter.ts` — returns `null` on entity resolution failure.

### Category: Entity Normalization (A1-A2)

- **A1:** Consolidated divergent `normalizeEntityName` functions — single Unicode-aware version in `entity-linker.ts`, imported by `entity-extractor.ts`.
- **A2:** Consolidated duplicate `computeEdgeDensity` — single version in `entity-linker.ts`, imported by `entity-extractor.ts`.

### Category: Dead Code Removal (~360 lines)

- Removed dead RSF branch and shadow-scoring branch from `hybrid-search.ts` (~80 lines).
- Removed dead feature flag functions: `isShadowScoringEnabled()`, `isRsfEnabled()`, `isInShadowPeriod()`.
- Removed dead module-level state: `stmtCache`, `lastComputedAt`, `activeProvider`, `flushCount`, 3 dead config fields.
- Removed dead functions/exports: `computeCausalDepth` single-node, `getSubgraphWeights`, `RECOVERY_HALF_LIFE_DAYS`, `logCoActivationEvent`.
- Preserved `computeStructuralFreshness`/`computeGraphCentrality` in `fsrs.ts` (planned features, not dead).

### Category: Performance Improvements (P1-P4)

- **P1a:** `Math.max(...spread)` → `reduce`-based max in `tfidf-summarizer.ts` (RangeError prevention).
- **P1b:** Added `LIMIT` clause to unbounded query in `memory-summaries.ts`.
- **P1c:** SQL `COUNT(*)` with `json_extract` in `mutation-ledger.ts` (replaced O(n) JS scan).
- **P3a:** `Set` for O(1) lookups in `entity-linker.ts` (was O(n) `.includes()`).
- **P3b:** Batch edge count query in `entity-linker.ts` (was N+1).
- **P4a:** Eliminated post-upsert SELECT via `lastInsertRowid` in `causal-edges.ts` (3→2 round-trips).
- **P4b:** WeakMap TTL cache (60s) for `buildHierarchyTree()` in `spec-folder-hierarchy.ts`.

### Category: Test Quality (P2a-P2d)

- Increased timeout in `memory-save-extended.vitest.ts` (5000→15000ms).
- Added `db.close()` in `entity-linker.vitest.ts` afterEach.
- Rewrote 4 tautological tests in `integration-search-pipeline.vitest.ts`.
- Added `/z_archive/` exclusion to `isMemoryFile()` in `memory-parser.ts`.

### Category: Verification

- TypeScript compilation: `npx tsc --noEmit` — 0 errors.
- Full test suite: 226 files, 7003/7003 tests, 0 failures.
- Dead code grep: 0 hits for all removed functions.
- Spot-check: all 8 critical fix verifications pass.
