## [**3.1.0.0**] - 2026-03-28

This release closes out the full 121-finding code audit by fixing the final 15 issues that were previously deferred, and updating all documentation to reflect the complete set of changes. ## What Changed: Final 15 Fixes

### Graph and Traversal (3 fixes)

**Graph reads could see inconsistent data during concurrent writes.** The causal graph (which tracks relationships between memories -- "this memory was caused by that one") could return partial results if another operation was adding or removing links at the same time. Fixed by wrapping reads in a single database transaction so they see a consistent snapshot.

**Highly connected nodes silently dropped edges.** When a memory had more than 100 relationships, excess edges were silently truncated with no indication to the caller. Fixed by returning a `truncated` flag and the applied limit so callers know data was cut off.

**Connection scores survived database swaps.** The cache tracking how well-connected each memory is (its "degree") wasn't cleared when the database was swapped (e.g., during a checkpoint restore). This meant connection scores from the old database influenced search results in the new one. Fixed by clearing the cache on any database change.

### Identity and Shared Memory (4 fixes)

**Parallel version assignments could collide.** When two saves happened simultaneously for the same memory, both could read the same "latest version number" and try to create the same next version, causing a database constraint error. Fixed by retrying up to 3 times with a fresh version-number read on each attempt.

**Documentation referenced a feature mode that doesn't exist.** The environment variable docs listed a `full` mode for graph traversal that was never implemented. Fixed by removing it and listing only the supported modes: `off`, `trace_only`, and `bounded_runtime`.

**Documentation showed wrong default for shared memory.** The docs said shared memory was enabled by default (`true`), but the actual runtime default is `false`. Fixed the documentation.

**Shared memory could be enabled/disabled without admin credentials.** The enable/disable endpoint didn't check who was calling it. Fixed by requiring admin caller identity, matching the security model used by all other shared-memory management operations.

### Query Understanding and Text Search (3 fixes)

**Complex queries with multiple intents only used the first one.** If someone asked "find the search pipeline spec and show recent changes," the system would only process the first intent ("find spec") and ignore the second ("show changes"). Fixed by returning a ranked list of detected intents so downstream processing can address multiple facets.

**Failed text-search updates during memory merges were silently ignored.** When the word-matching index (BM25) failed to update during a merge operation, the error was caught and logged but the merge continued, leaving the merged memory partially invisible to text search. Fixed by triggering an immediate repair attempt, and surfacing a structured warning if repair also fails.

**Auto-discovered spec folders were found too late in the process.** When the system automatically discovered which spec folder a session was working in, this discovery happened after the initial processing pass, meaning the first set of results didn't benefit from folder-scoped context. Fixed by including discovered folders in the initial processing call.

### Trigger Matching (2 fixes)

**Trigger matching broke on non-Latin text.** The trigger matcher (which surfaces memories based on keyword patterns) used Latin-only word boundaries. Chinese, Japanese, Korean, and other non-Latin scripts either weren't split into words at all or were split incorrectly. Fixed by using Unicode-aware word boundary detection with special handling for scripts that don't use spaces between words.

**Trigger matching slowed down as the database grew.** Every trigger check scanned all stored triggers linearly. With 500+ triggers, this became noticeably slow. Fixed by building a token-based candidate index that narrows the search space before running the full match, reducing the scan from "check everything" to "check likely matches."

### Embedding, Response Format, and Startup (3 fixes)

**Embedding generation could hang indefinitely.** When using the local embedding model or cloud providers, there was no timeout -- a stalled network request or frozen model would block the save pipeline forever. Fixed by adding timeouts to both local inference and cloud provider calls.

**Auto-surfaced context was outside the standard response envelope.** One response field (`autoSurfacedContext`, which contains proactively surfaced related memories) was placed at the top level of the response instead of inside the standard `meta` wrapper. This inconsistency made it harder for clients to parse responses uniformly. Fixed by moving it to `meta.autoSurfacedContext`.

**All tool handlers loaded at startup, even unused ones.** Every handler module was imported eagerly when the server started, triggering initialization code for all tools even if most would never be called in a session. Fixed by switching to lazy loading -- handlers are loaded on first use, reducing startup time and memory usage.

## Upgrade

No migration required. All changes are backward compatible except:

- `autoSurfacedContext` moved from top-level response to `meta.autoSurfacedContext` -- clients reading this field need to update their parsing
- The shared-memory enable/disable endpoint now requires admin caller identity -- unauthenticated calls will be rejected
- Tool handlers now load lazily -- any code that relied on side effects at import time should move those effects to the handler's first invocation
