**Ablation benchmark integrity + MCP DB config unification** -- Fixes the ablation benchmark to fail closed on DB/ground-truth mismatches, adds evaluation-only search mode, refreshes the ground-truth dataset, and repoints all **7 MCP runtime configs** from an external `~/.codex/memories/` DB (44 records) to the repo DB (2,417 records).

## Highlights

### Category: Fixes

- **Ablation fail-fast guard** -- `assertGroundTruthAlignment()` rejects benchmark runs when the active DB and `ground-truth.json` do not share the same parent-memory ID universe.
- **Evaluation-only search mode** -- `hybridSearchEnhanced()` accepts `evaluationMode: true` to bypass confidence and token-budget truncation for benchmark callers only.
- **Parent-ID normalization** -- Both CLI and MCP ablation adapters now score `parentMemoryId ?? row.id` consistently.
- **Ground-truth dataset refreshed** -- 294 parent-backed relevances across 126 unique memory IDs, validated with zero chunk-backed or missing IDs.
- **FTS5 verdict reversed** -- Clean reruns show baseline Recall@20 = 0.323, FTS5 ablated = 0.0. FTS5 weight 0.3 is validated; the earlier "FTS5 is harmful" signal was a benchmark artifact.

### Category: Architecture

- **MCP eval DB override** -- `eval-reporting.ts` supports `SPECKIT_EVAL_DB_PATH` via `withAblationDb()` for eval-only DB swapping with automatic restore.
- **MCP DB config unification** -- All 7 runtime configs (`opencode.json`, `.claude/mcp.json`, `.mcp.json`, `.vscode/mcp.json`, `.codex/config.toml`, `.agents/settings.json`, `.gemini/settings.json`) repointed `MEMORY_DB_PATH` from `~/.codex/memories/spec_kit_memory/` to `.opencode/skill/system-spec-kit/mcp_server/database/context-index.sqlite`.
- **Backward-compat symlink** -- `~/.codex/memories/spec_kit_memory/context-index.sqlite` is now a symlink to the repo DB so Codex CLI still resolves its expected path.
