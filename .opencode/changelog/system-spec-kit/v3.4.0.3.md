This release hardens the core spec-kit infrastructure as part of the Phase 020 research-findings remediation. `generate-description.js` no longer silently destroys authored rich content when a `description.json` file has a schema violation. `graph-metadata.json` gains a `save_lineage` tag so freshness claims no longer depend on timestamp subtraction alone. A new continuity timestamp precision helper prevents date-only values from producing false-stale warnings. The retry-budget module emits structured telemetry events. The evidence-marker audit parser fixes indented and nested triple-backtick fence handling.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/018-cli-executor-remediation` (Level 3)

---

## Bug Fixes

#### description.json merge-preserving repair

The loader inside `scripts/spec-folder/generate-description.ts` previously collapsed parse errors and schema violations into the same `null` branch, then rebuilt a minimal replacement that dropped authored rich content. The 30-iteration research quantified the blast radius: 29 of 86 packet-level `description.json` files in the `026` tree were one invalid edit away from lossy repair. The loader now returns a discriminated result `{ok, reason: 'file_missing' | 'parse_error' | 'schema_error', partial?}`. Schema violations on parseable files route through the new `mergePreserveRepair` helper, which preserves authored narrative plus extension keys while canonical derived fields win on conflict. Parse errors still route to minimal replacement because unparseable content cannot be preserved.

Rollback flag: `SPECKIT_DESCRIPTION_REPAIR_MERGE_SAFE=false` reverts to the prior minimal-replacement path for schema errors.

&nbsp;

#### Evidence-marker fence parser

Indented triple-backtick fences (four-space-leading) no longer produce false-positive evidence-marker warnings. Nested line-start triple-backtick fences inside an already-open fence are tracked as content rather than fence closers. Four-backtick fences in isolation remain safe. Mismatched or unclosed fences stay separate false-negative cases with their own regression fixtures.

## New Features

#### graph-metadata save-lineage tag

`graph-metadata.json derived.save_lineage` is a new optional enum field: `description_only`, `graph_only`, `same_pass`. The canonical save path in `workflow.ts` now stamps the appropriate lineage value per branch. The graph-metadata backfill script stamps `graph_only` on existing files during migration. Freshness claims can now distinguish same-pass refresh from later one-sided rewrites without relying on timestamp subtraction alone.

&nbsp;

#### Continuity timestamp precision helper

A new `lib/continuity/timestamp-normalize.ts` module exposes `isLowPrecision(raw)`, `normalizeTimestampPrecision(raw)` and `comparePrecisionAware(a, b, thresholdMs)`. Date-only values and midnight-UTC values are treated as low-precision and bypass strict staleness thresholds. The 10-minute continuity threshold value is unchanged.

&nbsp;

#### Retry-budget telemetry

The `lib/enrichment/retry-budget.ts` module now emits `{type:'event', event:'retry_attempt', memoryId, step, reason, attempt, outcome, timestamp}` records on each attempt decision. `MAX_RETRIES = 3` is unchanged. Future calibration packets can use the emitted data to tune the budget against real attempt histograms.

## Documentation

#### Heuristic language for continuity threshold and retry budget

The 10-minute continuity threshold and `N=3` retry budget are both labeled as bounded policy heuristics in `SKILL.md` and supporting documentation. Both are unchanged in this release. The updated language prevents future readers from assuming they were empirically calibrated.

## Upgrade

No migration required. `save_lineage` is optional in this release. A follow-up packet will tighten to required once backfill runs across production metadata. `SPECKIT_DESCRIPTION_REPAIR_MERGE_SAFE=false` fully reverts description repair to the prior behavior.

The rollback path for each module is a single-commit revert of the Phase 020 atomic-ship PR. Feature flags isolate the merge-preserving repair. The other changes (telemetry emit, save-lineage tag, timestamp helper, fence parser fix) are additive and backward-compatible.
