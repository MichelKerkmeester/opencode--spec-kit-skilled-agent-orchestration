## [**1.0.1.0**] - 2026-01-14

This update makes the lightweight route the default for everyday browser work. It also adds several missing actions so the skill is more useful without extra steps.

### Faster everyday routing

- Need: Simple browser tasks did not need the overhead of the more complex integration path.
- Change: The routing rules now prefer the lightweight command-line path first and keep the fuller path as a fallback when needed.
- Why it matters: Routine work feels quicker and more predictable for operators.

### More built-in browser checks

- Need: Users still had gaps when they wanted to capture visuals, inspect traffic, or inspect browser state.
- Change: The skill now documents screenshot capture, network monitoring, console access, cookie handling, and better terminal composability.
- Why it matters: More common browser tasks can stay inside one workflow instead of branching out to extra tools.

### Reference cleanup

- Need: A few documentation links and section pointers were wrong.
- Change: The release corrected the broken references in the support material.
- Why it matters: People can follow the documentation without hitting dead ends.
