This release ports the cli-opencode v1.3 Provider Auth Pre-Flight (Smart Fallback) pattern into cli-gemini's `SKILL.md` §3 and adds a Default Invocation block. Part of the 052 cli-skill-patterns harmonization that brings all 5 cli-* siblings to a shared structural surface. cli-gemini's auth surface is the most diverse (Google OAuth free-tier, `GEMINI_API_KEY` paid plan, Vertex AI enterprise) — the pre-flight decision table accounts for all three.

## What Changed

#### Provider Auth Pre-Flight (Smart Fallback) subsection added

New §3 subsection codifies the pre-dispatch auth check for Google credentials across the three documented auth surfaces. One-shot bash pre-flight detects `$GEMINI_API_KEY` env var, `gemini config list` OAuth output, and `$GOOGLE_GENAI_USE_VERTEXAI` env var, capturing `GEMINI_KEY_OK`, `GOOGLE_OAUTH_OK`, and `VERTEX_OK` flags. A 4-state decision table (default-available OAuth / OAuth-missing-API-key-ready / only-Vertex-configured / all-missing) governs routing — one extra state vs siblings to handle the enterprise Vertex AI billing surface separately.

Two user-prompt templates (OAuth-missing-with-API-key-fallback A/B/C menu; all-missing login-command menu) make the ASK-not-substitute contract explicit. Notable nuance: free-tier rate-limit errors (HTTP 429) are explicitly called out as *not* auth errors — surfacing them as quota guidance prevents triggering an unnecessary fallback flow. Cache invalidation on real auth errors mid-dispatch reruns the pre-flight before retry.

#### Default Invocation (Skill Default) block added

New §3 subsection pins the canonical default invocation: `gemini "<prompt>" -o text`. The override table covers 5 user-phrasing patterns: "JSON output" → `-o json`; "Auto-approve all tools" → append `--yolo` (or `-y`); "Use Google Search" → add `google_web_search` tool invocation in prompt content; "Investigate codebase" → add `codebase_investigator` tool invocation in prompt content. The single-model nature of cli-gemini (only `gemini-3.1-pro-preview` surfaced) means model overrides aren't expected.

#### Position in §3

Both new subsections inserted between Prerequisites and Core Invocation Pattern, matching the cli-opencode donor ordering. No existing subsections renamed or reordered; the `.geminiignore` mention, free-tier callout, and unique-tool inventory (google_web_search, codebase_investigator, save_memory) all preserved.

## Upgrade

No invocation breaking changes. Existing `gemini "<prompt>"` dispatches keep working. AI consumers of the skill should now run the Provider Auth Pre-Flight on session start (a one-time `gemini config list 2>&1; env | grep -E '^GEMINI_API_KEY|^GOOGLE_GENAI_USE_VERTEXAI'` call) before the first dispatch, cache the result for the session, and apply the 4-state decision tree if `GOOGLE_OAUTH_OK=0`.

The pre-flight matches the cli-opencode v1.3 pattern with one substantive extension: the 4th decision state explicitly handles the enterprise Vertex AI surface (which has different billing and rate-limit semantics). Operators using only Vertex AI will see an explicit confirmation prompt before dispatch — this prevents accidental enterprise-billing usage when free-tier OAuth is the expected path.

Free-tier rate-limit errors (60 req/min, 1000 req/day) remain non-auth errors and don't trigger the fallback flow.
