## [**1.2.0.0**] - 2026-03-21

This release makes semantic code search much easier for AI assistants to discover and use automatically. It updates the router, the instructions, and the context agents so the search tool becomes a first-choice option instead of a hidden extra.

---

### Direct routing into semantic search

- Need: The advisor was not steering enough concept-based search requests toward the meaning-based search tool.
- Change: The routing rules were expanded so more search and exploration requests land in CocoIndex automatically.
- Why it matters: Users are more likely to get the right search behavior without asking in a very specific way.

---

### Clearer instructions for all assistants

- Need: The repo guidance still leaned too heavily on exact-text search tools for broad code questions.
- Change: The instruction files now tell assistants when to try meaning-based search first, and the context agents were updated to treat it as a first-class option.
- Why it matters: More assistants will reach for the better search method earlier in the task.

---

### Provider-specific guidance

- Need: Different assistant runtimes needed clearer, more concrete activation guidance.
- Change: The playbook and skill guide now explain how the search tool shows up and behaves in each runtime.
- Why it matters: Cross-runtime adoption becomes more predictable and easier to troubleshoot.
