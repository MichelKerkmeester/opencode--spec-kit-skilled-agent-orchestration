## [**3.0.1.4**] - 2026-03-29

The git-finish workflow now cross-references the create:changelog command for a guided release experience. Previously, Step 6 (Create Release) was the only documented way to publish a release, and it required manual changelog composition, tag creation, and `gh release create` invocation. Now the workflow points users to `/create:changelog --release :auto` as a guided alternative that handles changelog creation, tag management, and release publishing in one flow.
