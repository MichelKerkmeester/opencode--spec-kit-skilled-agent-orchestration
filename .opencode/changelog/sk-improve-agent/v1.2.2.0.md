## Bug Fixes

These fixes correct behavior that could confuse operators or produce the wrong result. The goal is to make the workflow more predictable in normal use.

#### Lineage schema aligned between producer and reducer

`candidate-run history.cjs` wrote `candidateId`/`parentCandidateId` but the reducer expected `id`/`parentId`. The reducer silently failed to read run history files. Schema aligned on one main shape so the reducer can consume run history artifacts.

&nbsp;

#### Mutation-coverage metrics now derivable by reducer

The reducer expected a `metrics` block that `mutation-coverage.cjs` never wrote. The reducer now derives coverage ratio and uncovered counts from the raw arrays when the computed metrics block is missing.

&nbsp;

#### Dashboard now surfaces journal, lineage, and coverage

The reducer read replay artifacts into the registry but dropped them from the dashboard. Operators had no visibility. Added dedicated dashboard sections for journal summary, candidate run history, and mutation coverage.

&nbsp;

#### Legal-stop events now emitted from improve-agent workflows

The improve-agent loop never emitted `legal_stop_evaluated` or `blocked_stop` events that the other reducers depend on. Both auto and confirm workflow YAMLs now emit structured legal-stop events with gate bundles.

&nbsp;

#### Trade-off detector trajectory extraction fixed

Journal-based trajectory extraction was incompatible with emitted `candidate_scored` payloads - the scored dimension array was missing from journal rows. `trade-off-detector.cjs` now emits the scored dimension array into the journal row.

&nbsp;

#### Plateau stop reason added to taxonomy

The stop-reason taxonomy could not represent plateau exits. Plateau stops were falsified as `converged`. Added `plateau` to the `STOP_REASONS` enum in `improvement-journal.cjs`.

&nbsp;

#### Phase 005 helpers now called in live workflows

`mutation-coverage.cjs`, `candidate-run history.cjs`, `trade-off-detector.cjs`, and `benchmark-stability.cjs` were never called from the auto/confirm workflow steps - they were dead code. Added explicit loop steps calling all 4 helpers at appropriate workflow points.

&nbsp;

#### Integration scan now targets actual agent, not profile label

`scan-integration.cjs` received the profile label instead of the real target agent identifier, so it scanned the wrong file. Both workflow YAMLs now pass the real target agent identifier.

&nbsp;

#### Session-boundary gate enforced in confirm mode

The confirm-mode session gate was advisory only - no `verify`/`on_fail` enforcement. Confirm mode now mirrors the auto workflow's structured halt pattern.
