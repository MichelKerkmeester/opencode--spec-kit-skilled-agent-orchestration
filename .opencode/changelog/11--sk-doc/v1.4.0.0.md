## 1. OVERVIEW

This changelog records the `sk-doc` evolution that made feature catalogs and manual testing playbooks first-class documentation families. It covers the new template bundles, the new creation references, the reference-tree regrouping, the install-guide naming cleanup, and the downstream path updates that let the create-command ecosystem consume the current `sk-doc` contract cleanly.

---

## 2. HIGHLIGHTS

### Category: New Documentation Assets

- **Feature-catalog templates added**: Introduced dedicated root and per-feature templates under `.opencode/skill/sk-doc/assets/documentation/feature_catalog/`.
- **Testing-playbook templates added**: Introduced dedicated root and per-feature templates under `.opencode/skill/sk-doc/assets/documentation/testing_playbook/`.
- **Creation references added**: Added `feature_catalog_creation.md` and `manual_testing_playbook_creation.md` as standards-and-workflow references, not scaffold files.

### Category: Contract Alignment

- **Integrated playbook model documented**: `sk-doc` now reflects the live manual-testing contract with root-owned review/orchestration guidance, root-level numbered category folders, and no canonical `review_protocol.md`, `subagent_utilization_ledger.md`, or `snippets/` subtree.
- **Feature-catalog style standardized**: Feature catalogs and playbooks now share frontmatter, catalog-style intros, unnumbered `TABLE OF CONTENTS`, and numbered all-caps section headers.
- **Snippet/frontmatter conventions aligned**: Per-feature files in both families now carry frontmatter and mirror the shipped feature-catalog snippet style more closely.

### Category: Reference System Cleanup

- **References regrouped**: The `sk-doc/references/` tree is now split into `global/` and `specific/` so cross-cutting standards and component-specific creation guides are easier to navigate.
- **`agent_creation.md` added**: Filled the previously missing agent-creation reference that downstream create-agent docs already expected.
- **Install-guide reference renamed**: The old `install_guide_standards.md` naming was retired in favor of `install_guide_creation.md`.

### Category: Downstream Sync

- **`SKILL.md`, `README.md`, and references updated**: Discovery and workflow docs now point to the new subfoldered template bundles and the regrouped reference paths.
- **Create-command ecosystem aligned**: The create-command docs, assets, and runtime write-agent guidance now point to the current `sk-doc` template/reference paths instead of the older flat layout.
- **System examples refreshed**: The live `system-spec-kit` and `mcp-cocoindex-code` feature-catalog and testing-playbook packages were restructured to match the current contract and serve as working examples.
