## Highlights

### New: Manual Testing Playbook Template (Mode 5)

- **`manual_testing_playbook_template.md`**: New template in `assets/documentation/` with 10 anchored sections covering the full 3-file playbook package (main playbook, review protocol, sub-agent utilization ledger)
- **SKILL.md**: Added Mode 5 (Playbook Creation) across all sections: use case (S1), PLAYBOOK intent with weight 4 (S2), workflow (S3), rules with ALWAYS/NEVER/ESCALATE (S4), template in references (S5), completion checklist (S6), five-mode footer
- **README.md**: Updated description, overview, features (Mode 5 block), and file structure tree
- **quick_reference.md**: Added playbook creation quick steps (Section 12), template link, renumbered related resources to Section 13
- **workflows.md**: Added Mode 5 link in scope note
- **template_rules.json**: Added `playbook` document type with 4 required sections, TOC + H2 uppercase enforcement

### Breaking: camelCase Key Alignment

- **`template_rules.json`**: Converted all property keys from snake_case to camelCase per sk-code--opencode JSON standards (e.g., `document_types` to `documentTypes`, `required_sections` to `requiredSections`, `toc_required` to `tocRequired`). Section identifier strings (values inside sectionEmojis/sectionAliases) remain snake_case as they are functional identifiers derived from heading text parsing.
- **`validate_document.py`**: Updated all JSON key references to match the new camelCase property keys (8 key lookups changed)
- **`template_rules.json` version**: Bumped from 1.1.0 to 1.2.0
