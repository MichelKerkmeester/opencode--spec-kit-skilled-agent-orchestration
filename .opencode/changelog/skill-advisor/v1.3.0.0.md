This release moves the skill graph from a single JSON file to a SQLite database with MCP query tools, adds an init script that sets up the store on first run and ships a drift check for the CLI prompt-quality cards.

## What Changed

#### Skill graph now lives in SQLite

The graph used to ship as one JSON file. Every query re-parsed the whole file. Outside tools had no way to traverse it.

The graph now lives in a SQLite database loaded by a startup scan. Four MCP tools give you programmatic access. An auto-reindex path keeps the store in sync when skill metadata changes. The JSON file stays on disk as a fallback for hosts without SQLite, so skill routing still works if the database is missing.

#### First-run auto-setup

Before, you had to init the graph yourself. Hosts without SQLite just silently degraded with no clear error.

`scripts/init-skill-graph.sh` now builds the SQLite store on first run. If SQLite is not available, the runtime falls back to the JSON corpus on its own. You get a working skill router either way.

#### CLI prompt-quality drift check

Phase 043 shipped a source-of-truth prompt-quality card in `sk-improve-prompt` and four mirror copies under `cli-claude-code`, `cli-codex`, `cli-copilot` and `cli-gemini`. Drift between the copies would quietly weaken the pre-dispatch check every CLI skill runs.

`scripts/check-prompt-quality-card-sync.sh` hashes the framework-selection table in all five files. It exits clean only when every copy matches. Run it locally to catch drift before review does.

#### Feature catalog picked up the SQLite section

The catalog now has a `05--sqlite-graph/` section with four files covering the SQLite startup scan, MCP query tools, auto-reindex and JSON fallback. The `02--graph-system/` section got two new entries for the graph store and auto-indexing.

#### Gen2 review fixes landed, gen1 archive moved

A second review pass found a few issues the first review missed. Those fixes landed in place. The gen1 review folder was moved under `review/archive-gen1/` so the active review directory shows only the current state.
