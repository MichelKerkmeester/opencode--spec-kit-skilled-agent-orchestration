# SpecKit Skill Advisor

Interactively analyze all skills in the repo, propose optimized scoring (TOKEN_BOOSTS, PHRASE_BOOSTS, graph-metadata triggers/keywords, CATEGORY_HINTS), apply approved changes, re-index the skill graph, and validate with the advisor test suite.

```yaml
role: Expert Operator running interactive skill-advisor optimization
purpose: Analyze skill coverage, optimize scoring tables, re-index graph, validate
action: Run 5-phase workflow from discovery through verification

operating_mode:
  workflow: sequential_5_phase
  workflow_compliance: MANDATORY
  workflow_execution: autonomous_or_interactive
  approvals: per_phase_in_confirm_mode
  tracking: phase_by_phase
  validation: tests_pass_after_changes
```

---

## 1. PURPOSE

Deliver a guided optimization pass over the skill advisor scoring system without requiring users to know the internal scoring architecture. The command reads all skills from `.opencode/skill/*/SKILL.md`, current scoring source (`explicit.ts`, `lexical.ts`, `weights-config.ts`, `graph-metadata.json`), and detects coverage gaps. It proposes optimized trigger phrases, tokens, and phrase boosts; in confirm mode the user approves each mutation; in auto mode it applies all proposals; then re-indexes via `skill_graph_scan` and validates with the advisor test suite.

## 4. WORKFLOW OVERVIEW

| Phase | Name      | Purpose                                          | Outputs                       |
| ----- | --------- | ------------------------------------------------ | ----------------------------- |
| 0     | Discovery | List skills, check graph health, detect repo context | discovery_report          |
| 1     | Analysis  | Read SKILL.md + graph-metadata.json + scoring tables | analysis_report           |
| 2     | Proposal  | Generate optimized triggers, tokens, phrases     | proposal_diff                 |
| 3     | Apply     | Write graph-metadata.json, edit scoring files, rebuild | files_updated           |
| 4     | Verify    | Run skill_graph_scan, run advisor tests          | verification_report           |

---

## 5. KEY BEHAVIORS

### Autonomous Mode (`:auto`)

- Executes all 5 phases without user approval gates
- Self-validates at each phase boundary (Discovery → Analysis → Proposal → Apply → Verify)
- Applies all proposals that pass quality gates and lack collision warnings
- Skips skills with low-confidence proposals (marks as needs-review)
- Stops on phase failure with rollback hint, no retry

### Interactive Mode (`:confirm`)

- Pauses at four approval gates: pre-Phase 2, pre-Phase 3, pre-Phase 4, post-Phase 4
- Phase 3 supports per-skill or per-lane review loops (`A` apply all / `B` per-skill / `C` per-lane / `D` reject / `E` dry-run)
- Presents diff before any mutation; rebuild + re-index happen only after explicit approval
- Offers rollback option at the post-Phase 4 gate when tests fail
- Documents user decisions at each checkpoint

### Dry-Run Mode (`--dry-run`)

- Available in both autonomous and interactive modes
- Generates the full proposal diff and writes it to a packet-local scratch path: `<spec-folder>/scratch/skill-advisor-proposal-<timestamp>.md` (or `.opencode/scratch/skill-advisor-proposal-<timestamp>.md` when invoked outside a spec folder)
- Skips Phase 3 entirely; no source files or `graph-metadata.json` are modified
- Phase 4 is also skipped (nothing to verify); status returns `STATUS=OK` with the diff path

### Five Scoring Lanes

| Lane | Source | Weight | Tunable Field |
| ---- | ------ | ------ | ------------- |
| explicit | `explicit.ts` (TOKEN_BOOSTS, PHRASE_BOOSTS) | configured | tokens, phrases |
| derived | `graph-metadata.json` per skill | 0.15 | derived.trigger_phrases, derived.key_topics |
| lexical | `lexical.ts` (CATEGORY_HINTS) | configured | category hints |
| semantic | embedding similarity | configured | (read-only) |
| graph | skill graph relationships | configured | (read-only) |

### Mutation Boundaries

The command modifies **only** these files:
- `lib/scorer/lanes/explicit.ts` — TOKEN_BOOSTS / PHRASE_BOOSTS dictionaries
- `lib/scorer/lanes/lexical.ts` — CATEGORY_HINTS dictionary
- `.opencode/skill/*/graph-metadata.json` — derived.trigger_phrases / derived.key_topics arrays

The command **never** modifies: SKILL.md content, weights-config.ts (lane weights), fusion scorer logic, daemon code.
