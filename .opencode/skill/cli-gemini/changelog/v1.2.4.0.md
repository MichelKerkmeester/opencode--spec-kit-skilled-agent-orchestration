This release closes 2 playbook authoring bugs surfaced by running the full 18-scenario cli-gemini playbook end-to-end against Gemini CLI 0.37.1. The fixes flip CG-005 from FAIL to PASS and CG-008 from SKIP to runnable.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/011-manual-testing-playbook-coverage-and-run/` (Level 3)

---

## What Changed

#### CG-005 — `--approval-mode never` (deprecated) → `--approval-mode plan`

Gemini CLI 0.37.1 dropped the `never` approval mode. Valid choices are now `default | auto_edit | yolo | plan`. The CG-005 scenario was authored against an older CLI version and failed with `Invalid values` at the second call's argument parser before any API request fired. Updated to use `plan` (the modern read-only equivalent).

#### CG-008 — sandbox HOME pre-stages auth credentials via rsync

The save-memory persistence scenario sandboxed HOME to `/tmp/cg-008-sandbox` so `save_memory` writes to a disposable `.gemini/memory.json` instead of the operator's production memory. But Gemini CLI looks up auth credentials in real `~/.gemini/`, which the sandbox HOME bypassed, so calls exited with `Please set an Auth method...` before reaching the model. Step 1 now pre-stages the sandbox `.gemini/` with `rsync -a --exclude="memory.json"` from the real `~/.gemini/`, mirroring auth credentials while keeping `memory.json` fresh in the sandbox.

#### Root playbook filename normalized

Renamed `MANUAL_TESTING_PLAYBOOK.md` → `manual_testing_playbook.md` to match the lowercase canonical convention. All cross-references in scenario files updated.
