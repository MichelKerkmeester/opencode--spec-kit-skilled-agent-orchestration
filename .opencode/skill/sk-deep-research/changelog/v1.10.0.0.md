This release hardens the shipped cli executor matrix against the seven P1 findings surfaced by the 30-iteration deep-research pass on Phase 017 refinements. Executor identity is now written on the first JSONL record of every non-native iteration. A new typed `dispatch_failure` event preserves executor attribution when a crash or timeout leaves no valid iteration record. Copilot large-prompt handling gains an `@path` fallback in every dispatch branch so oversized prompts no longer risk shell argument-length limits. `graph-metadata.json` records a `save_lineage` tag so freshness claims stop depending on timestamp subtraction alone. The readiness contract docs narrow to the four trust-state values the seven code-graph handlers actually emit. The evidence-marker parser now handles indented and nested fences correctly. Retry-budget telemetry emits structured `retry_attempt` records without changing the `N=3` heuristic. Caller-context coverage extends to `setImmediate`, `queueMicrotask` and `timers/promises` boundaries.

## New Features

#### First-write executor provenance

Before this release, the `executor` audit block was appended AFTER validation as the final merge on the last JSONL record. If the CLI executor crashed or produced no output, the failure was anonymous. `writeFirstRecordExecutor` now runs before dispatch for every non-native kind and stamps executor identity on the first iteration record. Post-dispatch validation now REQUIRES the executor field for non-native runs, so missing provenance fails fast with a typed `executor_missing` reason. Native runs are unchanged.

&nbsp;

#### Dispatch failure event

A new typed `{type:'event', event:'dispatch_failure', executor, reason, iteration, timestamp}` record lets the workflow preserve executor identity when no valid iteration record exists. Crashes, timeouts and missing-output cases now emit this event with the executor block attached. The reducer tolerates the event without emitting a duplicate `schema_mismatch`. Three consecutive dispatch_failure events trigger the existing stuck-recovery path.

&nbsp;

#### Copilot @path large-prompt fallback

Phase 019 documented an `@path` fallback for oversized Copilot prompts. This release actually implements it. The `if_cli_copilot` branch in all four YAMLs (research auto + confirm, review auto + confirm) now runs a size check on the rendered prompt file. If the prompt is at or below 16 KB, the dispatch uses the existing positional form. Above 16 KB the dispatch uses a wrapper prompt: `copilot -p "Read the instructions in @<path> and follow them exactly. Do not deviate." --model X --allow-all-tools --no-ask-user`. The `cli-matrix.vitest.ts` suite gained two cases covering both sides of the threshold.

&nbsp;

#### Metadata save-lineage tag

`graph-metadata.json derived.save_lineage` is a new optional enum field (`description_only`, `graph_only`, `same_pass`). The canonical save path in `workflow.ts` stamps `same_pass` when both description and graph metadata refresh together. The graph-metadata backfill script stamps `graph_only` on existing files during migration. Freshness claims can now distinguish same-pass refresh from later one-sided rewrites without relying on timestamp subtraction alone.

&nbsp;

#### Continuity timestamp precision helper

A new `lib/continuity/timestamp-normalize.ts` module exposes `isLowPrecision(raw)`, `normalizeTimestampPrecision(raw)` and `comparePrecisionAware(a, b, thresholdMs)`. Date-only values and midnight-UTC values are treated as low-precision and bypass strict staleness thresholds so they stop producing false-stale warnings. The 10-minute continuity threshold is unchanged. The helper is additive for future validator wiring. Existing validator callers are untouched this release.

&nbsp;

#### Retry-budget structured telemetry

The retry-budget module now emits `{type:'event', event:'retry_attempt', memoryId, step, reason, attempt, outcome, timestamp}` records on each attempt decision. `MAX_RETRIES = 3` is unchanged. Telemetry lays the groundwork for a future calibration packet that can tune the budget against real attempt histograms.

&nbsp;

#### Caller-context async boundary coverage

Three new committed vitest cases cover `setImmediate`, `queueMicrotask` and `timers/promises` async boundaries for the AsyncLocalStorage caller-context path. The 30-iteration research verified these boundaries with local probes. This release promotes the proofs into permanent regression coverage so future consumers can trust the contract holds.

## Bug Fixes

#### Evidence-marker fence parser

Indented code blocks (four-space-leading triple-backtick fences) no longer produce false-positive evidence-marker warnings. Nested line-start triple-backtick fences inside an already-open fence are tracked as content rather than fence closers. Four-backtick fences in isolation remain safe. Mismatched or unclosed fences stay separate false-negative cases with their own regression fixtures documenting expected behavior.

&nbsp;

#### description.json merge-preserving repair

Phase 017's `generate-description.js` collapsed parse failures and schema violations into the same `null` branch, which then rebuilt a minimal replacement that dropped authored rich content. The 30-iteration research quantified the blast radius: 29 of 86 packet-level `description.json` files in the `026` tree were one invalid edit away from lossy repair. The loader now returns a discriminated result `{ok, reason: 'file_missing' | 'parse_error' | 'schema_error', partial?}`. The new `mergePreserveRepair` helper preserves authored narrative plus extension keys when the file is parseable but schema-invalid. Canonical derived fields always win on conflict. A new specimen regression suite uses real rich 017 packet files. The `SPECKIT_DESCRIPTION_REPAIR_MERGE_SAFE` feature flag defaults to on. Setting it to `false` reverts to Phase 017 behavior for rollback.

## Documentation

#### Readiness contract narrowed to reachable vocabulary

The seven code-graph handlers only emit four trust-state values in practice: `live`, `stale`, `absent`, `unavailable`. The shared type still declares eight. Readiness-contract documentation in `SKILL.md` now marks `cached`, `imported`, `rebuilt` and `rehomed` as compatibility vocabulary not emitted by the audited handler set. This closes the docs-vs-code parity gap surfaced by research question Q3.

&nbsp;

#### Copilot concurrency and reasoning-effort clarification

`cli-copilot/SKILL.md` now explicitly states the repo-safe concurrency cap is 3 (per the Phase 018 `feedback_copilot_concurrency_override` memory rule) while the upstream API cap is 5. Reasoning effort is clarified as config-driven (`~/.copilot/config.json`) rather than available as a CLI flag.

&nbsp;

#### Continuity threshold heuristic language

Docs that previously suggested the 10-minute continuity threshold was empirically calibrated now state it is a one-sided policy budget pending telemetry. The numeric value is unchanged.

&nbsp;

#### Retry-budget heuristic language

`N=3` retry budget is labeled a bounded-hot-loop heuristic in the relevant SKILL.md sections. Calibration will follow once the new `retry_attempt` telemetry produces real attempt histograms.
