This release adds a feature catalog for `sk-deep-review` so you can see what the review loop actually does without reading source code. The catalog lists 19 features grouped into 4 areas and points to the code file for each one.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/011-skill-advisor-graph/008-deep-skill-feature-catalogs/` (Level 3)

---

## What Changed

#### New feature catalog

Before this release, the only way to see what the review loop did was to read `SKILL.md` or the source files. No map, no per-feature breakdown, no pointer from a review behavior back to the code that implements it.

The new catalog at `.opencode/skill/sk-deep-review/feature_catalog/feature_catalog.md` groups the loop into 4 areas:

- Loop lifecycle: init, iteration dispatch, convergence check, synthesis, memory save
- State management: JSONL log, strategy tracking, config, findings registry, dashboard
- Review dimensions: correctness, security, traceability, maintainability
- Severity system: P0 / P1 / P2 rules, verdict logic, claim adjudication, blocked-stop events, remediation routing

Each feature has its own file with an overview, what the code actually does today and a pointer to the source.

#### README link

The skill README now points to the catalog as the first place to look when you want to understand the loop.
