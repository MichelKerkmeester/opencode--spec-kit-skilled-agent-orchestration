## Bug Fixes

These fixes correct behavior that could confuse operators or produce the wrong result. The goal is to make the workflow more predictable in normal use.

#### Reducer now consumes terminal stop metadata

The reducer ignored `synthesis_complete` events and derived dashboard status from stale config, so the dashboard never showed the actual stop reason or legal-stop outcome after a loop finished. `reduce-state.cjs` now parses the latest `synthesis_complete` JSONL record and derives `stopReason` and terminal legal-stop bundle from it.

&nbsp;

#### Resume/restart lineage now persists to reducer-visible state

Lifecycle events (`resumed`, `restarted`) were emitted to JSONL but never propagated to config-visible fields. The dashboard always showed run history mode as `new` even after a resume. The reducer detects lifecycle events and updates `parentSessionId`, `continuedFromRun`, and `run historyMode` in the registry and dashboard.

&nbsp;

#### Claim-adjudication finalSeverity is now canonical

The reducer ignored `finalSeverity` from claim-adjudication packets, so the findings registry used the original severity even after adjudication changed it. The reducer now parses `claim_adjudication` events by `findingId`, uses `finalSeverity` as the main registry value, and seeds transitions from the packet.

&nbsp;

#### Stale claim-adjudication STOP veto cleared when findings resolve

The dashboard could show a STOP veto from a failed adjudication even after all active P0/P1 findings were resolved. The veto is only surfaced when the latest failed adjudication still applies to active P0/P1 findings.

&nbsp;

#### Reducer is now fail-closed on corrupt JSONL

the reduce review state helper silently continued on corrupt JSONL when `lenient` was false, and wrote output files before checking corruption. Corruption check now runs before any file writes. The function throws a structured error when corruption is detected and `lenient` is false.

&nbsp;

#### ACTIVE RISKS no longer renders `[object Object]`

The blocked-stop dashboard path rendered raw objects instead of formatted strings. Added the format blocked by list helper helper. The ACTIVE RISKS section now reuses normalized entries from `blockedStopHistory`.

&nbsp;

#### STOP_BLOCKED events now include graph blocker detail

Both workflow YAMLs emitted `blocked_stop` events without the structured `graphBlockerDetail` payload. `spec_kit_deep-review_auto.yaml` and `_confirm.yaml` now include `graphBlockerDetail` in the emitted payload.

&nbsp;

#### reviewDimensions normalized once during initialization

The dimensions array was parsed in two separate places, creating divergent sources of truth. Normalization happens once during init and the same parsed array feeds both `deep-review-config.json` and the JSONL config row.

## Documentation

The supporting guidance was updated so the written instructions match the shipped behavior. That makes the workflow easier to follow without reading the code first.

#### Iteration skeleton collapsed to canonical template

The agent definition (`deep-review.md`) iteration skeleton was internally contradictory - different heading names than the reducer parser expected. Collapsed to one main fenced template matching the sections `reduce-state.cjs` actually consumes.

#### convergence.md normalized to blocked_stop schema

Remaining `legalStop` references in later sections were updated to the persisted `blocked_stop` event schema throughout.

#### loop_protocol.md corrected to full-history replay

Conflicting delta replay text removed. The entire file now consistently describes full-history replay, which is what the reducer actually does.

#### review_mode_contract.yaml updated to 3 modes

Removed `fork` and `completed-continue` as live review modes. Only `new`, `resume`, `restart` remain.
