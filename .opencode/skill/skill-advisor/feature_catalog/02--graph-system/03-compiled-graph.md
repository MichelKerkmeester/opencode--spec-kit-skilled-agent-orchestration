# Compiled graph

## 1. OVERVIEW

Describes the structure and current contents of the runtime `skill-graph.json` artifact.

The advisor does not read every metadata file during prompt analysis. It loads one compiled JSON artifact that already contains the graph relationships, families, signals, and conflict pairs needed for runtime boosting.

---

## 2. CURRENT REALITY

`compile_graph()` writes a compact structure with `schema_version`, `generated_at`, `skill_count`, `families`, `adjacency`, `signals`, `conflicts`, and `hub_skills`. Family membership is sorted, adjacency is sparse, and only the routable edge groups (`depends_on`, `enhances`, `siblings`, `prerequisite_for`) are materialized into the runtime map. Conflicts are deduplicated into sorted pairs instead of being stored inline in adjacency.

The current `skill-graph.json` generated by this pipeline contains 20 skills across 6 families. It includes sparse adjacency for routing overlays, per-skill trigger signals, an empty `conflicts` list, and a `hub_skills` list containing the highest inbound-degree nodes such as the CLI family and several code-review surfaces. `skill_advisor.py` loads this file lazily through `_load_skill_graph()` and keeps it cached in memory for later requests.

## 3. SOURCE FILES

### Implementation

| File | Layer | Role |
|---|---|---|
| `skill_graph_compiler.py` | Build | Assembles the compact runtime object with families, adjacency, signals, conflicts, and hub skills. |
| `skill_advisor.py` | Runtime | Loads the compiled graph lazily and reuses it for relationship-aware scoring. |

### Validation And Tests

| File | Type | Role |
|---|---|---|
| `skill-graph.json` | Compiled artifact | Stores the live graph payload currently used by the advisor. |
| `skill_graph_compiler.py` | Validation CLI | Rebuilds or revalidates the artifact before it is consumed by runtime routing. |
