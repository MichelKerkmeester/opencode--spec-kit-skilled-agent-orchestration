## What Changed

#### Provider Auth Pre-Flight (Smart Fallback) subsection added

New §3 subsection codifies the pre-dispatch auth check for Anthropic credentials. One-shot bash pre-flight detects `$ANTHROPIC_API_KEY` env var and `claude config list` OAuth/setup-token output, capturing `ANTHROPIC_KEY_OK` and `CLAUDE_OAUTH_OK` flags. A 3-state decision table (default-available / API-key-missing-OAuth-ready / both-missing) governs routing. Two user-prompt templates (key-missing-with-OAuth-fallback A/B/C menu; both-missing login-command menu including `claude auth login` and `claude setup-token`) make the ASK-not-substitute contract explicit. Cache invalidation on auth-error mid-dispatch reruns the pre-flight before retry.

#### Default Invocation (Skill Default) block added

New §3 subsection pins the canonical default invocation: `claude -p "<prompt>" --model claude-sonnet-4-6 --output-format text`. The override table covers 5 user-phrasing patterns: "Use Opus extended thinking" → `--model claude-opus-4-6 --effort high`; "JSON schema output" → append `--json-schema '<schema>' --output-format json`; "Cost-capped" → append `--max-budget-usd 1.00`; "Plan mode" → append `--permission-mode plan`.

#### Position in §3

Both new subsections inserted between Prerequisites and Core Invocation Pattern, matching the cli-opencode donor ordering. No existing subsections renamed or reordered; all do-not-collapse terms preserved at counts ≥ baseline.

## Upgrade

No invocation breaking changes. Existing dispatches keep working as before. AI consumers of the skill should now run the Provider Auth Pre-Flight on session start (a one-time `claude config list 2>&1; env | grep -E '^ANTHROPIC_API_KEY'` call) before the first dispatch, cache the result for the session, and apply the 3-state decision tree if `ANTHROPIC_KEY_OK=0`.

The pre-flight matches the cli-opencode v1.3 pattern exactly — same ASK-not-substitute language, same cache-invalidation contract, same 3-state column structure. Substitutions are limited to the detection command, var names, model invocations, and login commands per the substitution table in `specs/052-cli-skill-patterns/scratch/provider-auth-preflight-template.md`.
