## [**3.1.0.0**] - 2026-04-30

This release reconciles the multi-stack scaffolding left in a partial state by packet 056 and replaces it with honest, project-agnostic stubs across the NEXTJS and GO stacks. The Webflow stack remains live and untouched.

#### Folder rename — nextjs → react

- `git mv references/nextjs references/nextjs` to align the disk path with the `STACK_FOLDERS["NEXTJS"]` contract used by SKILL.md, `references/router/stack_detection.md`, `references/router/resource_loading.md`, `description.json`, and `graph-metadata.json`. Every `resource_map_for(NEXTJS)` path now resolves.
- Created `assets/nextjs/{checklists,patterns,integrations}/` and `assets/go/{checklists,patterns}/` directory trees (previously absent).

#### Project-agnostic placeholder fill

- 43 stub files written across `references/{react,go}/` and `assets/{react,go}/`: 23 NEXTJS (12 references + 3 checklists + 5 patterns + 3 integrations) and 20 GO (12 references + 3 checklists + 5 patterns).
- Every stub uses the canonical placeholder template: YAML frontmatter (`status: stub`, `populated: false`, `last_synced_at: 2026-04-30`), short Intended-scope paragraph, Outline (TODO) bullets, See-also references. No fenced code blocks in markdown stubs. No library version pins. No project-specific names.
- Code stubs (`.tsx`, `.ts`, `.go`, `.css.ts`, `_test.go`) are comment-only, with no function bodies (Go files include the trivial `package patterns` declaration; the test stub uses an empty `TestStub` placeholder).

#### Carry-over rewrites + cross-stack scrub

- Rewrote `references/{react,go}/README.md` and `references/{react,go}/implementation/implementation_workflows.md` (4 carry-over files from packet 056) as project-agnostic stubs. Removed project-specific phrasing; kept the structural sections (subfolder map, asset map, verification commands, see-also).

All structural sections (topology, error envelope, JWT handoff, CORS, pagination, deploy table, drift detection) are intact.
