### T-PPT-03 â€” Create prompt-pack renderer module

**File to create**: `.opencode/skill/system-spec-kit/mcp_server/lib/deep-loop/prompt-pack.ts`

Write a TypeScript module that:

1. Imports `readFileSync` from `'node:fs'` and `z` from `'zod'`.
2. Exports type `PromptPackVariables = Record<string, string | number>`.
3. Exports custom error class `PromptPackError` extending `Error` with field `missingVariables: string[]`.
4. Exports function `renderPromptPack(templatePath: string, variables: PromptPackVariables): string`:
   - Reads `templatePath` via readFileSync as UTF-8.
   - Finds all `{variable_name}` tokens (regex: `/\{([a-zA-Z_][a-zA-Z0-9_]*)\}/g`).
   - For each token, substitutes `String(variables[token_name])`.
   - If any token has no corresponding key in `variables`, collects missing names and throws `PromptPackError` with the list.
   - Variable name regex: must match `[a-zA-Z_][a-zA-Z0-9_]*` (no dots, no brackets â€” flat names only).
   - Does not re-scan for tokens introduced by substitution (no recursive expansion).
5. Exports function `validatePromptPackTemplate(templatePath: string, expectedVariables: string[]): { present: string[]; missing: string[]; extra: string[] }`:
   - Reads template, extracts all tokens.
   - Returns three arrays: tokens present in both the template and `expectedVariables`; expected tokens NOT found in template; tokens in template NOT in `expectedVariables`.
   - Does NOT throw â€” this is a validator for template-update workflows.
6. Module-level header `// MODULE: Deep-Loop Prompt Pack Renderer` (match Phase A pattern).

Reject: no `any` casts, no `unknown as X` assertions, no recursive rendering, no async.
