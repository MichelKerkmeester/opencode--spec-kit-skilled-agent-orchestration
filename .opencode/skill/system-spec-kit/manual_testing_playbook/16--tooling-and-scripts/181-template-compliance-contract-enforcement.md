# 181 -- Template Compliance Contract Enforcement

## 1. OVERVIEW

This scenario validates the end-to-end template compliance enforcement pipeline. It tests Layer 1 (agent contract awareness) and Layer 2 (post-write validation) working together to ensure that spec documents generated by the @speckit agent pass `validate.sh --strict` on first generation without requiring post-hoc fixes.

---

## 2. CURRENT REALITY

Operators create a Level 2 spec folder via the @speckit agent and verify that all generated documents comply with template contracts on first pass.

- Objective: Confirm that the 3-layer template compliance system produces compliant spec documents on first generation
- Prompt: `Create a Level 2 spec folder for a test feature. Use the template compliance contract to ensure correct headers and anchors. Then run validate.sh --strict and report exit code and any violations.`
- Expected signals: All 5 Level 2 files (spec.md, plan.md, tasks.md, checklist.md, implementation-summary.md) pass validate.sh --strict with exit code 0; no post-hoc fixes needed
- Pass/fail: PASS if 0 errors on first generation across all 5 document types
