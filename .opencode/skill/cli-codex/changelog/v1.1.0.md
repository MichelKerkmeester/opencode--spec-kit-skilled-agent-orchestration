# cli-codex v1.1.0

**Date:** 2026-03-02
**Type:** Fix — CLI flag correction

---

## Summary

Replaced non-existent `--agent <name>` flag with correct `--profile` (`-p`) flag and `codex exec review` subcommand across all skill documentation. Added `[profiles.*]` sections to `.codex/config.toml` to enable profile-based delegation.

## Breaking Change

All `codex exec --agent <name>` invocation patterns are replaced. Users must use `-p <name>` instead.

## Changes

### Fixed: `--agent` flag does not exist on `codex exec` (v0.106.0)

**Root cause:** The skill was authored with an assumed `--agent` flag that Codex CLI does not implement. Testing the skill immediately failed with `error: unexpected argument '--agent' found`.

**Correct mechanisms discovered:**

| Mechanism | Purpose |
|-----------|---------|
| `-p <profile>` / `--profile` | Load named profile from `[profiles.<name>]` in config.toml |
| `codex exec review` | Built-in subcommand for git diff-aware code review |
| `codex review` | Top-level shorthand for diff review |
| `.codex/agents/*.toml` | Multi-agent TUI feature only (not `codex exec`) |
