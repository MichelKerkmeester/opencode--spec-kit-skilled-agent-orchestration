This first release lets other AI assistants delegate focused work to OpenAI's Codex command-line tool. It packages safe execution rules, delegation patterns, and reference docs so teams can use Codex more deliberately. The goal is to make the workflow easier to follow and easier to trust.

---

#### Structured delegation into Codex

Other assistants needed a clear way to decide when Codex was the right helper for a task. The skill introduced routing patterns for generation, review, research, architecture work, and agent-based delegation. Requests can be handed off more intentionally instead of treating Codex as a generic fallback.

---

#### Safety and specialist support

Delegated work needed guardrails around file access and a clearer map of specialist roles. The package documents sandbox levels and the specialist agent set that Codex can use. Teams can choose the right balance of safety and capability for each job.

---

#### Reusable support material

People needed guides and templates they could reuse instead of rebuilding delegation prompts from scratch. The release shipped a main guide, references, comparisons, and prompt templates. Codex becomes easier to adopt and easier to use consistently.
