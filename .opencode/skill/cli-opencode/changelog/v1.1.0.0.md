This release flips the cli-opencode canonical default model from `anthropic/claude-opus-4-7` (which was not registered in the operator's opencode auth) to `opencode-go/deepseek-v4-pro --variant high`, fixes the long-standing primary-vs-subagent confusion in the agent docs, and validates the full 31-scenario playbook end-to-end with a 77.4% PASS rate. The 18 ProviderModelNotFoundError FAILs from the prior run all flipped to PASS as expected.

## What Changed

#### Canonical default ŌåÆ `opencode-go/deepseek-v4-pro --variant high`

The skill defaulted to `anthropic/claude-opus-4-7` but the operator's environment registers only copilot, opencode-go, deepseek, and openai. Every cli-opencode dispatch was failing at provider lookup before any tool work could happen. The new canonical default routes through the opencode-go subscription against DeepSeek V4 Pro at high reasoning effort, matching the actual provider auth set. Verified via opencode-go docs (provider/model id format `opencode-go/<model-id>`).

#### Agent surface: primary vs subagent

cli-opencode previously documented 9 agents as `--agent`-invokable, but only 4 are actually primary (directly callable). The `general` and `plan` agents are OpenCode built-ins; `orchestrate` is mode=primary; `multi-ai-council` is mode=all. The other 8 (`context`, `debug`, `review`, `write`, `deep-research`, `deep-review`, `improve-agent`, `improve-prompt`) are mode=subagent and dispatch via Task tool from a primary. SKILL.md ┬¦3 now splits into two tables ŌĆö primary agents (directly invokable) and subagents (routed via `--agent orchestrate "Use the X subagent to ..."`).

#### Playbook validated end-to-end (24 PASS / 2 FAIL / 2 SKIP across 31 scenarios)

Critical-path scenarios all PASS: CO-008 self-invocation refusal, CO-009 opencode-go default, CO-012 variant levels (`--variant high` vs `minimal` ratio Ōēź 2.0 ŌĆö variant flag IS honored by opencode-go's deepseek route), CO-028 parallel detached ablation, CO-031 cross-repo nested guard. Remaining 2 FAILs are real opencode runtime hangs on `--agent write` (separate runtime issue, not skill-level). 2 SKIPs are expected environmental gaps (`openai/gpt-5.5` not in registry; google provider not authenticated).

#### Subagent scenarios re-routed via orchestrate

Scenarios CO-014 (context), CO-015 (review), CO-016 (write), CO-032 (deep-research), CO-033 (deep-review) previously dispatched `--agent <subagent>` directly, which opencode rejects with a "agent X is a subagent, not a primary agent. Falling back to default agent" warning. Each scenario's TEST EXECUTION bash command now dispatches via `--agent orchestrate` with a `"Use the <subagent> subagent to ..."` prompt prefix. CO-013 (general), CO-017 (multi-ai-council), CO-034 (orchestrate) were already correctly using primaries.

#### Root playbook filename normalized

Renamed `MANUAL_TESTING_PLAYBOOK.md` ŌåÆ `manual_testing_playbook.md` to match the convention used by mcp-* and other skill playbooks. 120 cross-references in scenario files swapped to the lowercase canonical via `git mv` (2-step for macOS case-insensitive FS). The sk-doc template `manual_testing_playbook_creation.md` was the source of the all-caps prescription ŌĆö also patched to use lowercase canonical so future `/create:testing-playbook` runs produce lowercase by default.

## Upgrade

No script changes required for callers using the documented default invocation `opencode run --model opencode-go/deepseek-v4-pro --variant high --agent general --format json`. Operators previously relying on `anthropic/claude-opus-4-7` as the default should either:

1. Run `opencode auth login anthropic` to re-enable that path explicitly, OR
2. Override per-invocation with `--model anthropic/claude-opus-4-7` when needed.

The opencode-go subscription is now the assumed default auth path; verify with `opencode auth list` showing `opencode-go` registered.

Subagent dispatchers using `opencode run --agent <subagent>` directly (write/review/context/debug/deep-research/deep-review/improve-agent/improve-prompt) must reroute to `opencode run --agent orchestrate "Use the <subagent> subagent to ..."` per the corrected primary-vs-subagent surface in SKILL.md ┬¦3.
