## What Changed

#### Provider Auth Pre-Flight (Smart Fallback) subsection added

New §3 subsection codifies the pre-dispatch auth check for GitHub Copilot credentials. One-shot bash pre-flight runs `gh auth status` and detects `$GH_TOKEN` env var, capturing `GH_OAUTH_OK` and `GH_TOKEN_OK` flags. A 3-state decision table (default-available / OAuth-missing-PAT-ready / both-missing) governs routing. Two user-prompt templates make the ASK-not-substitute contract explicit, including the option to use an existing `$GH_TOKEN` PAT for non-interactive `copilot -p` calls when interactive `gh auth login` OAuth isn't available. Cache invalidation on auth-error mid-dispatch reruns the pre-flight before retry.

#### Default Invocation (Skill Default) block added

New §3 subsection pins the canonical default invocation: `copilot -p "<prompt>" --model gpt-5.4 --allow-all-tools`. The override table covers 5 user-phrasing patterns: "Use copilot claude sonnet" → `--model claude-sonnet-4.6 --allow-all-tools`; "Use gpt-5.5" → `--model gpt-5.5 --allow-all-tools`; "Read-only exploration" → `--agent explore` (no `--allow-all-tools`); "Cloud delegation" → append `/delegate` to the prompt body.

#### Error Handling table added

New §3 table at the end of HOW IT WORKS (just before §4 RULES) covers 9 common failure modes and their recovery paths: command-not-found, self-invocation refused, `not authenticated` / `401 Unauthorized` (initial + mid-dispatch — both route back to the Provider Auth Pre-Flight), `model not found`, reasoning-effort flag rejected on older `copilot` (< 1.0.36), cloud delegation hangs, concurrency throttling at the GitHub-API max-3 ceiling, and Spec Kit context missing (with pointer to `assets/shell_wrapper.md` for scripted `-p` calls). Brings cli-copilot to feature parity with the other 4 cli-* skills which already had Error Handling tables.

#### Position in §3

All three new subsections inserted in the cli-opencode donor ordering: Provider Auth Pre-Flight + Default Invocation between Prerequisites and Core Invocation Pattern; Error Handling table at the end of §3 before §4 RULES. No existing subsections (Spec Kit Context Parity, Model Selection, Reasoning Effort, Copilot CLI Agent Delegation, Unique Copilot Capabilities) renamed or reordered.

## Upgrade

No invocation breaking changes. Existing `copilot -p` dispatches keep working. AI consumers of the skill should now run the Provider Auth Pre-Flight on session start (a one-time `gh auth status 2>&1` + `[ -n "$GH_TOKEN" ]` check) before the first dispatch, cache the result for the session, and apply the 3-state decision tree if `GH_OAUTH_OK=0`.

The pre-flight matches the cli-opencode v1.3 pattern exactly. Error Handling table rows give actionable recovery paths for the 9 most common Copilot CLI failure modes; operators previously hitting silent auth failures now see explicit references back to the pre-flight.

The max-3-concurrent constraint (existing rule from `references/concurrency.md`) is now also surfaced inline in the Error Handling table for visibility — same constraint, no behavior change.
