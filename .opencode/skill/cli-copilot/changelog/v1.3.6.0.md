This release switches the cli-copilot canonical default model from `claude-sonnet-4.6` to `gpt-5.4` and adds `gpt-5.5` to the surfaced model selection table. Live verification against the standalone Copilot CLI (`copilot -p`) confirmed `gpt-5.5` is a valid model id alongside the existing `gpt-5.4 / gpt-5.3-codex / claude-opus-4.7 / claude-sonnet-4.6 / gemini-3.1-pro-preview` lineup.

---

## What Changed

#### Default flips → `gpt-5.4`

The skill default routes through `gpt-5.4` (frontier reasoning with configurable effort levels — `minimal / low / medium / high / xhigh`) at the user's preferred reasoning posture. Sonnet 4.6 stays in the model table as the Anthropic alternative for code review and balanced coding tasks. The reasoning-effort matrix gets `gpt-5.4 (skill default)` annotation, and the `~/.copilot/config.json` example in `README.md` flips to `"model": "gpt-5.4"` so persistent operator state matches the skill default.

#### `gpt-5.5` added to model selection

Copilot 1.0.36+ exposes `gpt-5.5` as a newer GPT for complex implementation work. It is added to the SKILL.md `### Model Selection` table (now 6 surfaced models across 3 providers, up from 5) and to the reasoning-effort table with the `low / medium / high / xhigh` range, default `high`. The model is positioned as an alternative to `gpt-5.4` rather than replacing it; operators choose per-invocation via `--model gpt-5.5`.

#### `claude-opus-4.6` → `claude-opus-4.7` in surfaced lineup

Carried forward from v1.3.5: the surfaced opus model id is `claude-opus-4.7`, matching the live Copilot CLI lineup. v1.3.6 reaffirms this in the `README.md` features matrix.

## Upgrade

Operators with `~/.copilot/config.json` using `claude-sonnet-4.6` as the persistent default should update to `gpt-5.4` if they want the new skill default to apply consistently across interactive and `-p` modes:

```bash
# Update persistent default
python3 -c "
import json
cfg_path = '$HOME/.copilot/config.json'
with open(cfg_path) as f: cfg = json.load(f)
cfg['model'] = 'gpt-5.4'
with open(cfg_path, 'w') as f: json.dump(cfg, f, indent=2)
"
```

No changes required for callers passing `--model <id>` explicitly. Operators who want the Anthropic default behavior preserved can pin `--model claude-sonnet-4.6` per invocation.
