This release closes 6 playbook authoring bugs surfaced by running the full 21-scenario cli-copilot playbook end-to-end against copilot 1.0.36. The biggest finding: copilot 1.0.36 added an explicit `--effort` / `--reasoning-effort` CLI flag that supersedes the legacy `~/.copilot/config.json` JSON path the skill previously documented as the only non-interactive mechanism. Skill knowledge updated.

> Spec folder: `.opencode/specs/system-spec-kit/026-graph-and-context-optimization/011-manual-testing-playbook-coverage-and-run/` (Level 3)

## What Changed

#### `--effort` / `--reasoning-effort` CLI flag (copilot 1.0.36+)

cli-copilot SKILL.md §3 previously claimed "there is no `--reasoning-effort` flag — `~/.copilot/config.json` is the only non-interactive mechanism". Copilot 1.0.36 added the flag explicitly: `--effort <level>` and `--reasoning-effort <level>` are now exposed in `copilot -p --help`. The skill section now documents the CLI flag as preferred for scripted `-p` usage; the legacy JSON config path stays as a persistent default fallback.

CP-007 scenario rewritten to dispatch with `--effort xhigh` / `--effort low` directly instead of writing JSON config to a sandboxed HOME. The destructive HOME-sandboxing (and tripwire SHA256 of the operator's real `~/.copilot/config.json`) is removed since no config file is touched.

#### Retired model `claude-opus-4.6` → `claude-opus-4.7` (4 scenarios)

Copilot 1.0.36 only exposes `claude-opus-4.7`; the previously-documented `claude-opus-4.6` was retired. Scenarios CP-006 (architectural reasoning), CP-012 (mid-session model switch), CP-014 (session continuity model switch), CP-015 (cross-AI generate-review-fix) all updated to the current model id.

#### CP-017 capitalized shell tokens

The shell-wrapper context-injection scenario had 8 capitalized shell keywords (`Then`, `Else`, `Fi`, `Echo`, `Shasum`, `Diff`, `Export`, `If`) that bash interprets as command names rather than reserved words. Pattern was likely from a markdown auto-capitalize linter that rewrote `; then` as `. Then`. All instances corrected to lowercase with proper semicolon delimiters.

#### CP-018 — Gemini family removed from copilot 1.0.36

`gemini-3.1-pro-preview` (and all Gemini variants) no longer exposed by current copilot CLI. Scenario substituted with `gpt-5.4` for the security audit; same OWASP-style criteria preserved. The asset `prompt_templates.md` Security Audit template also patched to drop the Gemini reference.

#### Root playbook filename normalized

Renamed `MANUAL_TESTING_PLAYBOOK.md` → `manual_testing_playbook.md` to match the lowercase canonical convention. All cross-references in scenario files updated.

## Upgrade

Operators previously tuning reasoning effort via JSON config:

```bash
# Old (still works, but deprecated):
echo '{"reasoning_effort":"xhigh"}' > ~/.copilot/config.json
copilot -p "..." --model gpt-5.4

# New (preferred for scripted use):
copilot -p "..." --model gpt-5.4 --effort xhigh
```

Operators with `claude-opus-4.6` references in their scripts should update to `claude-opus-4.7`. Operators with `gemini-3.1-pro-preview` references should substitute with the closest available model (e.g. `gpt-5.4`) until / unless GitHub re-adds the Gemini family to copilot.
